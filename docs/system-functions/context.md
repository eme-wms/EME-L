# Работа с контекстом

в языке EME-L контекст используется для передачи данных между различными частями приложения, хранения состояния и проверки текущего контекста выполнения (диалог, запрос, отчет).

---

### is_put_context — сохранение значения в контекст

в языке EME-L функция `is_put_context` сохраняет строковое значение в глобальном контексте под указанным именем.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя ячейки контекста для сохранения |
| args[1] | String | Да | Строковое значение для запоминания в контексте |

**Возвращает**: пустое значение.

```EME-L
' Сохранить значение в контексте '
is_put_context("MyContext", "Hello")
```

---

### is_get_context_as_string — получение контекста как строки

в системе EME.WMS функция `is_get_context_as_string` получает сохраненное строковое значение из глобального контекста по указанному имени без преобразования типа.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя ячейки контекста для чтения |

**Возвращает**: `String` — сохраненное строковое значение из контекста. Если ячейка не найдена, возвращает пустое значение.

```EME-L
' Прочитать строковое значение из контекста '
Value = is_get_context_as_string("MyContext")
```

---

### is_get_context_as_ref — получение контекста как числа

в языке EME-L функция `is_get_context_as_ref` получает сохраненное значение из глобального контекста по указанному имени и преобразует его в целое число.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя ячейки контекста для чтения |

**Возвращает**: `Integer` — значение из контекста, преобразованное в целое число. Если ячейка не найдена, возвращает `NULL_REF` (0).

```EME-L
' Прочитать ссылку из контекста '
Ref = is_get_context_as_ref("MyRef")
```

### is_del_context — удаление контекста

в EME БД функция `is_del_context` удаляет ячейку глобального контекста по указанному имени.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя ячейки контекста для удаления |

**Возвращает**: пустое значение.

```EME-L
' Удалить ячейку контекста '
is_del_context("MyContext")
```

### is_office_doc_save — проверка сохранения документа офиса

в языке EME-L функция `is_office_doc_save` проверяет, был ли сохранен документ офисного приложения в базу данных. Функция формирует имя контекста на основе номера записи, поля и строки, и проверяет наличие соответствующего маркера в глобальном контексте. При необходимости удаляет контекст после проверки.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер записи в базе данных |
| args[1] | Integer | Да | Номер поля в записи |
| args[2] | Integer | Да | Номер строки в базе данных |
| args[3] | Boolean | Нет | Признак удаления контекста после проверки. По умолчанию `FALSE` |

**Возвращает**: `Boolean` — `TRUE`, если документ был сохранен (в контексте установлено значение "1"); `FALSE` — в противном случае.

```EME-L
' Проверить, сохранен ли документ '
Saved = is_office_doc_save(100, 5, 1)

' Проверить и удалить контекст '
Saved = is_office_doc_save(100, 5, 1, TRUE)
```

### is_office_doc_already_open — проверка открытия документа офиса

в системе EME.WMS функция `is_office_doc_already_open` проверяет, был ли открыт документ офисного приложения для указанной записи, поля и строки. Функция формирует имя контекста на основе переданных параметров и проверяет его наличие в глобальном контексте.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер записи в базе данных |
| args[1] | Integer | Да | Номер поля в записи |
| args[2] | Integer | Да | Номер строки в базе данных |

**Возвращает**: `Boolean` — `TRUE`, если документ уже открыт (контекст существует); `FALSE` — в противном случае.

```EME-L
' Проверить, открыт ли документ '
Open = is_office_doc_already_open(100, 5, 1)
```

### is_check_context — проверка текущего контекста

в языке EME-L функция `is_check_context` проверяет текущий контекст выполнения на соответствие указанному типу. При передаче имени дополнительно проверяет совпадение имени диалога или запроса.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Тип проверяемого контекста: `"Dialog"` или `"Query"` |
| args[1] | String | Нет | Имя диалога или запроса для проверки. По умолчанию не проверяется |

**Возвращает**: `Integer` — 1, если текущий контекст соответствует указанному типу (и имени, если задано); 0 — в противном случае. При неподдерживаемом типе контекста генерирует ошибку.

```EME-L
' Проверить, является ли текущий контекст диалогом '
Result = is_check_context("Dialog")

' Проверить конкретный диалог '
Result = is_check_context("Dialog", "Clients.Dialog")

' Проверить, является ли текущий контекст запросом '
Result = is_check_context("Query")
```

### is_get_context — получение имени текущего контекста

в системе EME.WMS функция `is_get_context` возвращает имя текущего контекста выполнения указанного типа (диалога, запроса или отчета).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Тип запрашиваемого контекста: `"Dialog"`, `"Query"` или `"Report"` |

**Возвращает**: `String` — имя текущего диалога, запроса или отчета. Если текущий контекст не является указанным типом, возвращает пустую строку. При неподдерживаемом типе генерирует ошибку.

```EME-L
' Получить имя текущего диалога '
DialogName = is_get_context("Dialog")

' Получить имя текущего запроса '
QueryName = is_get_context("Query")

' Получить имя текущего отчета '
ReportName = is_get_context("Report")
```

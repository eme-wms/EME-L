# Товары, организации и документы

> Synonyms for search: товары, организации, документы, is_vendor, is_rounded_week, is_rounded_date, is_production_date, is_default_vendor, is_force_document_export, is_base_curr, is_column, is_enter_value_dialog, is_convert_string_to_date, is_EAN13_by_order, is_BARCODE_by_order, is_BARCODE_by_document, is_ccd, is_document_line_pallet_ms_up, is_document_pallet_ms_up, is_document_change_mu, поставщик, штрих-код, дата производства

в языке EME-L функции группы WMS для работы с товарами, организациями и документами обеспечивают проверку организаций-продавцов, работу с базовой валютой, вычисление дат производства, формирование штрих-кодов и управление операциями документов. в системе EME.WMS эти функции используются для складского учёта, маркировки товаров и обработки документооборота.

## Организации и поставщики

### is_vendor — проверка организации-продавца

проверяет, является ли указанный клиент организацией-продавцом.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на клиента (целое число) |

**Возвращает**: `Boolean` — `TRUE` (1), если клиент является организацией-продавцом; `FALSE` (0), если не является. При невалидной строке возвращает пустое значение.

```EME-L
' Проверить, является ли клиент продавцом (TRUE — продавец) '
result = is_vendor(clientRef)
' result = TRUE — клиент является организацией-продавцом '
```

## Даты и календарь

### is_production_date — дата производства

вычисляет дату производства на основе даты срока годности и ссылки на товарную единицу.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата срока годности `BestBeforeDate` |
| args[1] | Integer | Да | Ссылка на товарную единицу (номер строки в записи «Товарная единица») |

**Возвращает**: `Date` — дата производства.

```EME-L
' Вычислить дату производства по сроку годности '
prodDate = is_production_date(bestBeforeDate, wareUnitRef)
' prodDate = 15.01.2024 — дата производства '
```

---

### is_rounded_week — номер недели по Rounded-календарю

возвращает номер недели по Rounded-календарю для указанной даты.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата, для которой определяется номер недели |

**Возвращает**: `Integer` — номер недели по Rounded-календарю.

```EME-L
' Получить номер недели по Rounded-календарю '
weekNum = is_rounded_week(Date(2024, 1, 15))
' weekNum = 3 — номер недели '
```

---

### is_rounded_date — дата начала недели по Rounded-календарю

возвращает дату начала указанной недели по Rounded-календарю.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Год |
| args[1] | Integer | Да | Номер недели |

**Возвращает**: `Date` — дата начала недели.

```EME-L
' Получить дату начала 3-й недели 2024 года '
weekStart = is_rounded_date(2024, 3)
' weekStart = 15.01.2024 — дата начала недели '
```

## Штрих-коды

### is_EAN13_by_order — код EAN-13 по заказу

возвращает код EAN-13 для указанного заказа.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Номер заказа (строка) или ссылка на заказ (целое число) |
| args[1] | String | Нет | Территория |
| args[2] | Integer | Нет | Количество строк |

**Возвращает**: `String` — код EAN-13. При невалидном заказе возвращает пустую строку.

```EME-L
' Получить EAN-13 по номеру заказа '
ean = is_EAN13_by_order("ORD-2024-001")
' ean = "4601234567890" — код EAN-13 '

' Получить EAN-13 по ссылке на заказ '
ean = is_EAN13_by_order(orderRef, "RU", 10)
```

### is_ccd — формирование строки штрих-кода

формирует строку для штрих-кода в формате `(AI)ЗНАЧЕНИЕ` на основе данных из базы.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер записи: `DOC_LINES_RECORD`, `CELLS_RECORD`, `REGISTERS_RECORD`, `QTY_BY_ORDERS_RECORD` или `SERIAL_NUMBER_RECORD` |
| args[1] | Integer | Да | Номер строки записи |
| args[2] | Integer | Да | Тип данных для штрих-кода: `0` — SSCC, `2` — EAN номер товара, `3` — сгенерированный EAN-13, `10` — номер партии, `11` — дата производства, `15` — срок годности, `37` — количество, `89` — SCD, `90` — артикул товара, `92` — полный сокращенный адрес ячейки, `93` — код склада, `94` — номер сектора ячейки, `95` — номер колонны ячейки, `96` — номер ряда ячейки, `97` — номер уровня ячейки, `21` — серийный номер |
| args[3] | Boolean | Нет | Выводить скобки в начале. По умолчанию `FALSE` |

**Возвращает**: `String` — строка для штрих-кода вида `(AI)ЗНАЧЕНИЕ`.

```EME-L
' Сформировать строку EAN-13 для строки документа '
eanStr = is_ccd(DOC_LINES_RECORD, lineNum, 3)
' eanStr = "(01)4601234567890" — строка штрих-кода '

' Сформировать строку SSCC без скобок '
ssccStr = is_ccd(DOC_LINES_RECORD, lineNum, 0, FALSE)
' ssccStr = "003761234567890123" — строка SSCC '
```

## Документы и операции

### is_document_line_pallet_ms_up — проверка пополнения пикинга строки

проверяет, является ли строка документа операцией пополнения пикинга (pallet MS up).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Номер строки документа. По умолчанию используется текущая строка `param->line` |

**Возвращает**: `Boolean` — `TRUE`, если строка является пополнением пикинга; `FALSE` — в противном случае.

```EME-L
' Проверить, является ли текущая строка пополнением пикинга (TRUE — пополнение) '
result = is_document_line_pallet_ms_up()
' result = TRUE — строка является пополнением пикинга '

' Проверить конкретную строку документа '
result = is_document_line_pallet_ms_up(5)
```

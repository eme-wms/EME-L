# Управление продуктивной и тестовой средой (УПиТ)

> Synonyms for search: УПиТ, upit, is_copy_from_prod, is_copy_prod_files, is_get_log, is_create_process, is_compile_expr, продуктивная среда, тестовая среда, копирование базы данных, лог базы данных, создание процесса, компиляция выражения

### is_copy_from_prod — копирование данных из продуктивной среды в тестовую

запускает копирование данных из продуктивной среды в тестовую. Определяет ссылку на базу данных по её номеру в записи «Базы данных», получает список запрещённых записей проекта, корректирует переданный список записей с учётом запрещённых и инициирует процесс копирования через сетевое подключение. В системе EME.WMS функция используется в метапроекте УПиТ для создания тестовых копий продуктивных баз данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | IP-адрес продуктивного сервера |
| args[1] | Integer | Да | Порт прослушивания продуктивного сервера |
| args[2] | String | Да | Путь к папке dat тестовой базы данных |
| args[3] | Integer | Да | Номер базы данных в записи баз данных |
| args[4] | Reference | Нет | Массив номеров записей для копирования. Если не указан, копируются все записи с учётом фильтра запрещённых |
| args[5] | Boolean | Нет | Режим фильтрации записей из args[4]. TRUE — копировать только записи из списка args[4] (по умолчанию). FALSE — копировать все записи, кроме записей из списка args[4] |
| args[6] | String | Нет | Дополнительная тестовая команда. Передаётся в процесс копирования. По умолчанию пустая строка |
| args[7] | String | Нет | Код корректировки. Передаётся в процесс копирования. По умолчанию пустая строка |
| args[8] | Boolean | Нет | Флаг запуска сервера после копирования. TRUE — запустить сервер. По умолчанию FALSE |

**Возвращает**: `Boolean` — результат запуска операции копирования:
- TRUE — копирование успешно запущено
- FALSE — ошибка при запуске копирования или база данных не найдена

```EME-L
' Копирование всех записей из продуктивной среды '
result = is_copy_from_prod("192.168.1.100", 8080, "D:\\EME\\TestDB", 1)
' result = TRUE — копирование запущено '

' Копирование только указанных записей '
records = Array()
records[0] = 100
records[1] = 200
records[2] = 300
result = is_copy_from_prod("192.168.1.100", 8080, "D:\\EME\\TestDB", 1, records, TRUE)
' result = TRUE — копирование запущено для записей 100, 200, 300 '

' Копирование всех записей кроме указанных с запуском сервера '
result = is_copy_from_prod("192.168.1.100", 8080, "D:\\EME\\TestDB", 1, records, FALSE, "", "", TRUE)
' result = TRUE — копирование запущено, сервер будет запущен после завершения '
```

### is_copy_prod_files — копирование файлов архива из продуктивной среды в тестовую

запускает копирование файлов архива из продуктивной среды в тестовую. Определяет ссылку на базу данных по её номеру в записи «Базы данных», получает список запрещённых записей проекта, корректирует переданный список записей с учётом запрещённых и инициирует копирование файлов архива. В системе EME.WMS функция используется в метапроекте УПиТ для копирования архивных файлов между средами.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Путь к архиву продуктивной среды |
| args[1] | String | Да | Путь к папке тестовой базы данных |
| args[2] | Integer | Да | Номер базы данных в записи баз данных |
| args[3] | Reference | Да | Массив номеров записей для обработки. Может быть пустым, но параметр обязателен. Если массив пуст, обрабатываются все записи с учётом фильтра запрещённых |
| args[4] | Boolean | Нет | Режим фильтрации записей из args[3]. TRUE — обрабатывать только записи из списка args[3] (по умолчанию). FALSE — обрабатывать все записи, кроме записей из списка args[3] |
| args[5] | String | Нет | Дополнительная тестовая команда. Передаётся в процесс копирования. По умолчанию пустая строка |
| args[6] | String | Нет | Код корректировки. Передаётся в процесс копирования. По умолчанию пустая строка |
| args[7] | Boolean | Нет | Флаг запуска сервера после копирования. TRUE — запустить сервер. По умолчанию FALSE |

**Возвращает**: `Boolean` — результат запуска операции копирования:
- TRUE — копирование успешно запущено
- FALSE — ошибка при запуске копирования или база данных не найдена

```EME-L
' Копирование архивных файлов для всех записей '
empty_array = Array()
result = is_copy_prod_files("D:\\EME\\Prod\\archives", "D:\\EME\\TestDB", 1, empty_array)
' result = TRUE — копирование архивов запущено '

' Копирование архивов для конкретных записей '
records = Array()
records[0] = 50
records[1] = 75
result = is_copy_prod_files("D:\\EME\\Prod\\archives", "D:\\EME\\TestDB", 1, records, TRUE)
' result = TRUE — копирование архивов для записей 50 и 75 запущено '
```

### is_get_log — получение строк из лога базы данных

получает заданное количество последних строк из лога базы данных. Выполняет чтение лог-файла из указанной директории данных БД и возвращает запрошенные строки в виде текста. В системе EME.WMS функция используется для удалённого мониторинга состояния базы данных и диагностики проблем.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Путь к папке данных базы данных |
| args[1] | Integer | Да | Количество строк лога для получения |

**Возвращает**: `String` — текст запрошенных строк лога. При ошибке чтения или отсутствии данных может возвращаться пустая строка.

```EME-L
' Получение последних 100 строк лога '
log_text = is_get_log("D:\\EME\\Data\\DB001", 100)
' log_text содержит последние 100 строк лога '

' Получение последних 10 строк для быстрой диагностики '
log_tail = is_get_log("D:\\EME\\Data\\DB001", 10)
' log_tail содержит 10 последних строк '

' Проверка на ошибки в логе '
log_text = is_get_log("D:\\EME\\Data\\DB001", 50)
If is_in_string(log_text, "ERROR")
    is_message("Обнаружены ошибки в логе", "Внимание", 1)
End If
```

### is_create_process — создание процесса операционной системы

создаёт и запускает новый процесс операционной системы Windows. Выполняет запуск исполняемого файла с указанными параметрами командной строки. Процесс создаётся без видимого консольного окна (флаг CREATE_NO_WINDOW). В языке EME-L функция используется для запуска внешних приложений и утилит из кода.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Путь к исполняемому файлу или полная командная строка. Если arg1 не указан, интерпретируется как полная командная строка |
| args[1] | String | Нет | Параметры командной строки. Если указаны, передаются отдельно от имени исполняемого файла. По умолчанию пустая строка |

**Возвращает**: `String` — результат операции создания процесса:
- "" (пустая строка) — процесс успешно создан и запущен
- `<описание_ошибки>` — текстовое описание системной ошибки Windows при неудачном создании процесса

```EME-L
' Запуск блокнота с файлом '
error = is_create_process("notepad.exe", "D:\\temp\\notes.txt")
' error = "" — блокнот запущен успешно '

' Запуск утилиты командной строки '
error = is_create_process("cmd.exe", "/c dir D:\\EME > D:\\temp\\dir.txt")
If error = ""
    is_message("Команда выполнена", "Успех", 1)
Else
    is_message("Ошибка: " + error, "Ошибка", 1)
End If

' Запуск с полной командной строкой (без отдельных параметров) '
error = is_create_process("D:\\Tools\\backup.exe")
' error = "" — процесс запущен '
```

### is_compile_expr — компиляция выражения EME-L

выполняет компиляцию выражения языка EME-L без его исполнения. Создаёт компилятор выражений, включает режим компиляции и проверяет корректность синтаксиса и семантики выражения. В языке EME-L функция используется для проверки корректности динамически формируемых выражений перед их выполнением.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Выражение EME-L для компиляции |

**Возвращает**: `String` — результат компиляции:
- "" (пустая строка) — выражение скомпилировано успешно, ошибок не обнаружено
- `<текст_ошибки>` — сообщение об ошибке компиляции (например, синтаксическая ошибка или неизвестный идентификатор)

```EME-L
' Проверка корректности простого выражения '
expr = "x = 10 + 20"
error = is_compile_expr(expr)
' error = "" — выражение корректно '

' Проверка выражения с ошибкой '
expr = "x = 10 +"
error = is_compile_expr(expr)
' error содержит сообщение о синтаксической ошибке '

' Проверка динамически сформированного выражения перед выполнением '
user_expr = is_input("Введите выражение для проверки:", "", "Проверка")
compile_error = is_compile_expr(user_expr)
If compile_error = ""
    is_message("Выражение корректно", "Проверка", 1)
Else
    is_message("Ошибка компиляции: " + compile_error, "Ошибка", 1)
End If
```

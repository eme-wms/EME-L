> Synonyms for search: профилирование, отладка, is_prf, is_profile_point, is_profile_name, is_breakpoint, is_error, is_catastrophic_log, is_log, is_post_console_out, is_debug, is_debug_mode, is_ui_test_mode, is_eximport, is_eximport2, is_eximport3, is_merge_EMEL, is_add_string_to_output, is_clear_output_console, is_show_output_console, is_project_list, is_get_type, is_help_file, is_object_ptr, is_mem_dump, is_SYS_mem_dump, is_memory_leak, is_mem_usagestat, is_lost_fields, is_switch_workstation_mode, is_run_robot, is_server_log, is_vfs_test, is_OLE_error_trace, is_trace_exist, is_resolve_double_keys, is_OLEAUT32_error, is_eventer_tick, is_force_standard_export, is_generate_NSI_script, is_OLE_serverbusydelay, is_get_data_path, robot, is_enable_dataslice, is_restart_as_server_request, is_sys_import, дамп памяти, планировщик, робот, тестирование, трассировка, консоль

is-функции группы Debug отвечают за отладку, профилирование, диагностику, экспорт/импорт данных, управление консолью вывода и вспомогательные операции разработки. в системе EME.WMS эти функции используются при разработке метапроектов, тестировании и диагностике проблем.

## Профилирование

### is_prf — профилирование

функция для профилирования кода. Аналог ядерных функций профилирования.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Тип: 0 — reset_prf, 1 — start_prf, 2 — stop_prf, 3 — get_prf |
| args[1] | Integer | Нет | Номер таймера (0-31). Для args[0]=0 не нужен |

**Возвращает**: при args[0]=3 — время в мс. В остальных случаях — ничего.

```EME-L
' Запуск таймера 0 '
is_prf(1, 0)

' Остановка '
is_prf(2, 0)

' Получить время '
time = is_prf(3, 0)
```

### is_profile_point — профилируемая точка

в языке EME-L функция `is_profile_point` регистрирует профилируемую точку в метапроектном профайлере. Измеряет время от начала текущего метода или от предыдущей точки до себя. Для учёта точек в диалоге EMEProfiler необходимо установить флаг Point.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя профилируемой точки |

**Возвращает**: пустое значение.

```EME-L
' Отметить точку входа в метод '
is_profile_point("Вход_в_Обработку")

' ... код обработки ... '

' Отметить точку выхода '
is_profile_point("Выход_из_Обработки")
```

## Отладка и ошибки

### is_breakpoint — точка останова

в языке EME-L функция `is_breakpoint` выводит диагностическое сообщение «Точка останова» через механизм smart_error. При каждом вызове переключает внутренний флаг, позволяя пропускать чередующиеся вызовы.

**Параметры**: нет.

**Возвращает**: пустое значение.

```EME-L
' Установить точку останова для отладки '
is_breakpoint()
' Выведено сообщение "Точка останова" '
```

### is_error — вывод ошибки

в EME БД функция `is_error` выводит сообщение об ошибке указанного типа.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Тип ошибки: 0 — stealth_error (скрытая), 1 — smart_error (с сообщением), 2 — user_error (пользовательская), 3 — soft_error (программная), 4 — developer_error (ошибка разработчика) |
| args[1] | String | Да | Текст сообщения об ошибке |

**Возвращает**: пустое значение.

```EME-L
' Вывести сообщение об ошибке с диалогом '
is_error(1, "Неверное значение поля")

' Скрытая ошибка — без диалога '
is_error(0, "Ошибка в фоновой операции")

' Ошибка разработчика с трассировкой '
is_error(4, "Нулевой указатель в методе X")
```

### is_ui_test_mode — режим тестирования UI

возвращает TRUE, если программа в режиме тестирования интерфейса. В этом режиме база последовательно запускает все пункты меню. Прикладной код может использовать для отключения браузеров и MessageBox.

**Параметры**: нет.

**Возвращает**: `Boolean` — `TRUE` если включён режим тестирования.


## Экспорт и импорт данных

### is_eximport — экспорт/импорт массива

в языке EME-L функция `is_eximport` выполняет экспорт или импорт одномерного массива в бинарный файл.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Каталог файла |
| args[1] | String | Да | Имя файла |
| args[2] | Reference | Да | Массив данных (CIArray) |
| args[3] | Integer | Да | Направление: 1 — экспорт, -1 — импорт |

**Возвращает**: пустое значение.

```EME-L

Arr.Add("Элемент1")
Arr.Add("Элемент2")
is_eximport("C:\\Temp", "data.bin", Arr, 1)

' Импорт массива из файла '
Arr2 = Object("Array")
is_eximport("C:\\Temp", "data.bin", Arr2, -1)
```

| args[2] | Reference | Нет | Массив для дубликатов методов (CIArray). Формат: имя_класса\|имя_метода |

**Возвращает**: пустое значение.

```EME-L

TypeErrors = Object("Array")
Dups = Object("Array")
is_lost_fields(Missing, TypeErrors, Dups)

If (Missing.GetSize() > 0)

```

# Инвентаризация и склады

> Synonyms for search: инвентаризация, склад, is_inventarisation, is_inventarisation_by_vendor, is_vendor_inventarisation, is_user_vendor, is_inventarisation_off, is_save_ware_timer_param, is_transit_ware, is_test_block_write, is_check_online_terminals, is_serial_generate_on, WMS, терминалы

в языке EME-L функции инвентаризации и складов обеспечивают управление режимом инвентаризации, работу с транзитным складом, проверку очереди приказов на предмет инвентаризации и контроль состояния терминалов. в системе EME.WMS эти функции используются для организации складских операций, учёта товарных запасов и управления терминалами сбора данных.

## Управление режимом инвентаризации

### is_inventarisation — проверка и управление режимом инвентаризации

проверяет или изменяет состояние режима инвентаризации.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Код операции: `1` — включить, если возможно; `2` — включить без проверки (системный режим, только на системном филиале); `-1` — выключить, закрыв все документы инвентаризации; `-2` — выключить без закрытия документов (системный режим, только на системном филиале); `0` — проверить текущее состояние |
| args[1] | Boolean / Reference | Нет | Флаг подавления диалогов (`FALSE` — выводить), либо массив `CArrayOfLong` для выходных данных |

**Возвращает**: `Integer` — состояние режима инвентаризации после операции: `0` — выключен, `1` — включен.

```EME-L
' Проверить текущее состояние инвентаризации '
state = is_inventarisation(0)
' state = 0 — режим выключен, 1 — включен '

' Включить режим инвентаризации '
result = is_inventarisation(1)
' result = 1 — режим включен '

' Выключить режим, закрыв все документы инвентаризации '
result = is_inventarisation(-1)
' result = 0 — режим выключен '
```

### is_inventarisation_by_vendor — инвентаризация с привязкой к организации

проверяет или изменяет состояние режима инвентаризации с привязкой к организации.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Код операции: `1` — включить, если возможно; `2` — включить без проверки (системный режим); `-1` — выключить, закрыв все документы; `-2` — выключить без закрытия; `0` — проверить состояние |
| args[1] | Reference | Нет | Массив для выходных данных или флаг подавления диалогов |
| args[2] | Integer | Нет | Ссылка на организацию |

**Возвращает**: `Integer` — состояние режима инвентаризации после операции: `0` — выключен, `1` — включен.

```EME-L
' Проверить состояние инвентаризации по организации '
state = is_inventarisation_by_vendor(0, NULL, vendorRef)
' state = 0 — выключен, 1 — включен '

' Включить инвентаризацию для конкретной организации '
result = is_inventarisation_by_vendor(1, NULL, vendorRef)
' result = 1 — режим включен '
```

---

### is_inventarisation_off — выключение инвентаризации и снятие флажков

выключает режим инвентаризации и снимает флажок инвентаризации у ячеек склада.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на организацию для фильтрации ячеек. `NULL_REF` — обрабатывать все ячейки |
| args[1] | Integer | Да | Ссылка на склад. Если валидная строка — обходит ячейки этого склада, иначе — все ячейки |

**Возвращает**: `Integer` — `0`.

```EME-L
' Выключить инвентаризацию для всех ячеек склада '
is_inventarisation_off(NULL_REF, warehouseRef)
' Режим инвентаризации выключен, флажки сняты '

' Выключить инвентаризацию для конкретной организации '
is_inventarisation_off(vendorRef, warehouseRef)
```

### is_vendor_inventarisation — организация инвентаризации

возвращает ссылку на организацию, по которой выполняется инвентаризация.

**Параметры**: нет.

**Возвращает**: `Integer` — ссылка на организацию инвентаризации. Если инвентаризация не включена, возвращает `NULL_REF`.

```EME-L
' Получить организацию, по которой выполняется инвентаризация '
vendorRef = is_vendor_inventarisation()
' vendorRef = 12345 — ссылка на организацию, или NULL_REF '
```

## Очередь приказов и инвентаризация

### is_once_queue_orders_inventarisation — проверка единственности приказа инвентаризации

проверяет, что в массиве объектов присутствует не более одного приказа инвентаризации.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference | Да | Массив объектов для проверки |
| args[1] | Reference | Да | Интерфейс интегратора |

**Возвращает**: `Integer` — `1`, если в массиве не более одного приказа инвентаризации и нет других документов при наличии инвентаризации; `0`, если условие нарушено.

```EME-L
' Проверить массив приказов на наличие не более одного приказа инвентаризации '
result = is_once_queue_orders_inventarisation(ordersArray, integrator)
' result = 1 — условие выполнено, 0 — нарушено '
```

---

### is_queue_orders_inventarisation — проверка приказа на инвентаризацию

проверяет, является ли объект приказом инвентаризации, и возвращает ссылку на документ инвентаризации.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Идентификатор объекта в интеграторе |
| args[1] | Reference | Да | Интерфейс интегратора |

**Возвращает**: `Integer` — ссылка на документ инвентаризации, или `NULL_REF`, если объект не является приказом инвентаризации.

```EME-L
' Проверить, является ли объект приказом инвентаризации '
docRef = is_queue_orders_inventarisation(objId, integrator)
' docRef = ссылка на документ инвентаризации, или NULL_REF '
```

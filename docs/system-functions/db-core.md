# База данных: ядро

is-функции группы DB отвечают за работу с данными базы данных в языке EME-L. в системе EME.WMS эти функции обеспечивают чтение и запись полей, навигацию по записям и строкам, поиск данных, получение метаданных структуры БД и проверку состояния строк.

### is_data — чтение данных по реляционному пути

в языке EME-L функция `is_data` читает содержимое поля базы данных по реляционному пути через одну или несколько записей. Поддерживает прохождение через ссылочные поля, мягкие ссылки и массивы ссылок.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя записи, физический номер (>= 1000), заголовок структуры БД (>= 2000) или положение объекта |
| args[1] | Integer | Да | Номер строки в записи |
| args[2] | String / Integer | Да | Имя или номер первого поля реляционного пути |
| args[3+] | String / Integer / Integer | Нет | Дополнительные поля пути или индекс массива ссылок |

**Возвращает**: значение последнего поля в пути — `String`, `Integer`, `Boolean`, `Date`, `Time`, `Reference` или `Array` в зависимости от атрибута поля. При пустой ссылке в середине пути возвращает `NULL_REF` для ссылочных типов или пустое значение.

```EME-L
' Прочитать название клиента из записи Клиенты, строка 5, поле Название '
Name = is_data("Клиенты", 5, "Название")
' Name = "ООО Ромашка" '

' Прочитать через ссылочное поле: заказ -> клиент -> название '
ClientName = is_data("Заказы", 10, "Клиент", "Название")
' ClientName = "ООО Ромашка" '

' Прочитать элемент массива ссылок: строка 3, поле Товары, индекс 0 '
ProductRef = is_data("Заказы", 3, "Товары", 0)
' ProductRef = 15 - ссылка на строку 15 записи Товары '

' Чтение из текущего объекта (положение 0) '
Value = is_data(0, 0, "Название")
' Value = значение поля Название текущей строки '
```

### is_get_data — чтение поля по номерам

в системе EME.WMS функция `is_get_data` читает содержимое поля записи по номеру записи, поля и строки. Использует прямые номера в отличие от `is_data`, которая работает с именами и реляционными путями.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер записи |
| args[1] | Integer | Да | Номер поля |
| args[2] | Integer | Да | Номер строки |
| args[3] | Integer | Нет | Код атрибута для интерпретации данных. По умолчанию используется атрибут поля из БД |

**Возвращает**: содержимое поля — `String`, `Integer`, `Boolean`, `Date`, `Time`, `Reference` или `Array` в зависимости от атрибута.

```EME-L

' Value = содержимое поля '

' Прочитать с явным указанием атрибута (строка) '
```

### is_equ_data — поиск строки по значению

в системе EME.WMS функция `is_equ_data` ищет строку в записи, в которой содержимое указанного поля равно заданному значению.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Нет | Имя записи, физический номер или положение объекта. По умолчанию базовая запись |
| args[1] | String / Integer | Да | Имя или номер поля |
| args[2] | Any | Да | Искомое значение |
| args[3] | String | Нет | Битовая маска для побитового сравнения |
| args[4] | String | Нет | Текстовое выражение для дополнительной проверки |

**Возвращает**: `Integer` — номер найденной строки. Если не найдена, возвращает `NULL_REF`.

```EME-L

' Line = 5 '

' Найти с битовой маской '
```

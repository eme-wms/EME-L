# База данных: файлы и хранилище — is_load_file_to_db, is_db_watch, is_sql_client_execute и другие функции

> Synonyms for search: база данных файлы, хранилище, is_load_file_to_db, is_open_file_from_db, is_print_file_from_db, is_reduce_storage_to_date, is_sql_client_execute, is_db_watch, is_get_modified_lines, is_run_export_ex, загрузка файлов, экспорт, наблюдение за БД

## База данных: файлы и хранилище

is-функции группы DB (файлы и хранилище) отвечают за загрузку файлов в базу данных, извлечение и печать документов, управление хранилищем данных, наблюдение за изменениями и экспорт. в системе EME.WMS эти функции обеспечивают интеграцию с файловой системой и внешними приложениями.

---

### is_load_file_to_db — загрузка файла в БД

в языке EME-L функция `is_load_file_to_db` загружает файл с диска в поле записи базы данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Полный путь к файлу |
| args[1] | String / Integer | Да | Имя или номер записи |
| args[2] | String / Integer | Да | Имя или номер поля |
| args[3] | Integer | Да | Номер строки |

**Возвращает**: `Integer` — ненулевое значение в случае успеха.

```EME-L
' Загрузить PDF-документ в поле Документ записи Клиенты, строка 5 '
Result = is_load_file_to_db("C:\\Documents\\contract.pdf", "Клиенты", "Документ", 5)
' Result = 1 - файл загружен '
```

### is_open_file_from_db — открытие файла из БД

в системе EME.WMS функция `is_open_file_from_db` извлекает файл из поля записи базы данных во временную директорию и открывает его с помощью ассоциированного приложения Windows.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Заголовок окна при открытии |
| args[1] | String | Да | Расширение файла для выбора программы |
| args[2] | String / Integer | Да | Имя или номер записи |
| args[3] | String / Integer | Да | Имя или номер поля |
| args[4] | Integer | Да | Номер строки |

**Возвращает**: пустое значение.

```EME-L
' Открыть загруженный договор в приложении по умолчанию '
is_open_file_from_db("Договор", ".pdf", "Клиенты", "Документ", 5)
' Файл извлечен и открыт в Acrobat Reader '
```

### is_print_file_from_db — печать файла из БД

в языке EME-L функция `is_print_file_from_db` печатает файл, загруженный в поле записи базы данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Расширение файла для выбора программы печати |
| args[1] | String / Integer | Да | Имя или номер записи |
| args[2] | String / Integer | Да | Имя или номер поля |
| args[3] | Integer | Да | Номер строки |
| args[4] | Integer | Нет | Таймаут ожидания завершения печати в миллисекундах. По умолчанию 0 |

**Возвращает**: `Boolean` — `TRUE` если печать завершена, `FALSE` если не завершилась в пределах таймаута.

```EME-L
' Напечатать договор '
Result = is_print_file_from_db(".pdf", "Клиенты", "Документ", 5)
' Result = TRUE '

' Печать с таймаутом 60 секунд '
Result = is_print_file_from_db(".pdf", "Клиенты", "Документ", 5, 60000)
' Result = TRUE '
```

### is_reduce_storage_to_date — очистка хранилища

в EME БД функция `is_reduce_storage_to_date` удаляет файлы из хранилища данных, добавленные до указанной даты включительно.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя или номер записи хранилища |
| args[1] | String / Integer | Да | Имя или номер поля хранилища |
| args[2] | Date | Да | Дата, по которую включительно производится очистка |

**Возвращает**: `Integer` — 0 при успехе, иначе код ошибки.

```EME-L
' Удалить файлы, добавленные до 01.01.2024 '
Result = is_reduce_storage_to_date("Хранилище", "Файлы", Date(2024, 1, 1))
' Result = 0 - очистка выполнена '
```

### is_db_watch — наблюдение за изменениями в БД

в EME БД функция `is_db_watch` управляет наблюдением за изменениями в базе данных в разрезе записи, поля и номера строки.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Команда: 1 — открыть, -1 — закрыть, 0 — отключить. По умолчанию возвращает текущий статус |

**Возвращает**: `Integer` — текущий статус: 0 — отключено, 1 — открыто, 2 — закрыто.

```EME-L
' Открыть наблюдение (вне транзакции) '
Status = is_db_watch(1)
' Status = 1 '

' Закрыть наблюдение (внутри транзакции) '
is_transaction(1, "Закрытие наблюдения")
Status = is_db_watch(-1)
' Status = 2 '
is_transaction(-1)

' Получить текущий статус '
Status = is_db_watch()
' Status = 0 '
```

### is_check_db_watch — проверка изменений при наблюдении

в языке EME-L функция `is_check_db_watch` проверяет сохраненные данные об изменениях в базе данных при включенном наблюдении.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя или номер записи |
| args[1] | String / Integer | Нет | Имя или номер поля. Если не указано, проверяется вся запись |
| args[2] | Integer | Нет | Номер строки. Если не указана, проверяется все поле |

**Возвращает**: `Boolean` — `TRUE` если изменений не обнаружено, `FALSE` если данные изменены.

```EME-L
' Проверить изменения в записи Клиенты '
NoChanges = is_check_db_watch("Клиенты")
' NoChanges = TRUE '

' Проверить конкретное поле '
NoChanges = is_check_db_watch("Клиенты", "Название")
' NoChanges = FALSE - поле было изменено '
```

### is_get_modified_lines — получение измененных строк

в системе EME.WMS функция `is_get_modified_lines` заполняет массив номерами строк, которые были изменены в базе данных при наблюдении.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference | Да | Объект массива для записи результатов |
| args[1] | String / Integer | Да | Имя или номер записи |
| args[2] | String / Integer | Нет | Имя или номер поля. Если не указано, возвращаются изменения по всей записи |

**Возвращает**: пустое значение.

```EME-L
' Получить список измененных строк '
ModArray = Object("Array")
is_get_modified_lines(ModArray, "Клиенты")
' ModArray содержит номера измененных строк '
```

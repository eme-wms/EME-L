# Устаревшие функции — синонимы: is_VT_DATE, is_db_data, is_rel_db_data, is_state, is_substring, is_int_to_char, is_record_count, is_empty_value

Следующие функции объявлены устаревшими. Они сохранены для обратной совместимости,
но в новом коде на языке EME-L следует использовать рекомендуемые альтернативы.

### is_VT_DATE — объединение даты и времени

Устаревшая функция объединения EME-L даты и времени в полноценный VT_DATE.
Рекомендуется использовать обычное сложение даты и времени.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата в формате EME-L, например результат `is_dos_date()` |
| args[1] | Time | Нет | Время в формате EME-L, например результат `is_dos_time()` |

**Возвращает**: `Date` — значение типа VT_DATE, объединяющее дату и время.

```EME-L
' Рекомендуемый способ (не используйте is_VT_DATE): '
dt = is_dos_date() + is_dos_time()
' dt = полная дата-время типа VT_DATE '
```

**Замена**: используйте выражение `is_dos_date() + is_dos_time()`.

### is_db_data — чтение данных из поля БД

Устаревшая функция чтения данных из поля базы данных. Перенаправляет вызов
функции `is_data` с преобразованием аргументов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя поля в базе данных |
| args[1] | Integer | Нет | Расположение объекта в генеалогическом дереве |
| args[2] | String | Нет | Софтовое имя записи (только с args[3]) |
| args[3] | Integer | Нет | Номер строки в записи (только с args[2]) |

**Возвращает**: `Variant` — содержимое поля в соответствии с его атрибутами.

```EME-L
' Рекомендуемый способ (не используйте is_db_data): '
val = is_data("имя_поля")
' val = содержимое поля текущей записи '
```

**Замена**: используйте функцию `is_data`.

### is_rel_db_data — чтение по реляционной цепочке

Устаревшая функция чтения данных из поля базы данных по реляционной цепочке.
Перенаправляет вызов функции `is_data`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Первый элемент реляционной цепочки |
| args[1] | String | Нет | Второй элемент реляционной цепочки |
| args[2] | String | Нет | Третий элемент реляционной цепочки |
| args[3] | String | Нет | Четвертый элемент реляционной цепочки |
| args[4] | String | Нет | Пятый элемент реляционной цепочки |
| args[5] | Integer | Нет | Расположение объекта в генеалогическом дереве |

**Возвращает**: `Variant` — содержимое поля в соответствии с его атрибутами.

```EME-L
' Рекомендуемый способ (не используйте is_rel_db_data): '
val = is_data("Контрагент", "ИНН")
' val = значение поля ИНН через реляционную цепочку '
```

**Замена**: используйте функцию `is_data`.

### is_state — состояние конечного автомата

Устаревшая функция. С появлением EME-L не требуется для использования.
Возвращает состояние конечного автомата на основе матрицы состояний и входных сигналов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Array | Да | Матрица состояний конечного автомата (массив байтов) |
| args[1] | Integer | Да | Входной сигнал 0 (ненулевое добавляет 1 к индексу) |
| args[2] | Integer | Нет | Входной сигнал 1 (ненулевое добавляет 2 к индексу) |
| args[N] | Integer | Нет | Входной сигнал N-1 (ненулевое добавляет N к индексу) |

**Возвращает**: `Boolean` — состояние конечного автомата из матрицы по вычисленному индексу.

```EME-L
' Эта функция устарела и не имеет прямой замены в EME-L. '
' Используйте встроенные конструкции языка для логики состояний. '
```

**Замена**: нет, функция не требуется в EME-L.

### is_substring — извлечение подстроки

Устаревшая функция извлечения подстроки.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Исходная строка |
| args[1] | Integer | Да | Начальная позиция подстроки, отсчет с 0 |
| args[2] | Integer | Да | Количество символов для извлечения |

**Возвращает**: `String` — подстрока указанной длины, начиная с заданной позиции.

```EME-L
' Рекомендуемый способ (не используйте is_substring): '
part = is_mid("Hello World", 6, 5)
' part = "World" '
```

**Замена**: используйте функцию `is_mid`.

### is_int_to_char — преобразование числа в символ

Устаревшая функция преобразования целого числа в символ.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Целое число для преобразования в символ |

**Возвращает**: `String` — символ, соответствующий указанному целочисленному коду.

```EME-L
' Рекомендуемый способ (не используйте is_int_to_char): '
ch = is_char(65)
' ch = "A" '
```

**Замена**: используйте функцию `is_char`.

# Диспетчер терминалов

> Synonyms for search: диспетчер терминалов, is_drag_drop_terminals_dispatcher, is_once_queue_orders_inventarisation, is_queue_orders_inventarisation, is_inventarisation_warehouse, is_get_first_queue_priority, is_set_valid_terminal_ref, is_get_valid_terminal_ref, is_set_first_queue_priority, is_set_orders_type, is_delete_queue_order, is_get_text_info_integrator, is_get_base_icon_color_integrator, is_get_parent_text_info_integrator, is_get_parent_base_icon_color_integrator, is_get_parent_record_for_integrator, is_get_parent_line_for_integrator, is_fill_refs_for_integrator, is_exist_queue_order_with_terminal_for_delete, is_check_both_double_stack_queue, is_undefine_array_to_queue_oders_array, is_add_del_terminals_dispatcher, очередь, терминал, интегратор, приоритет

в языке EME-L функции диспетчера терминалов обеспечивают управление очередью приказов, привязку приказов к терминалам сбора данных, установку приоритетов очереди и интеграцию с внешними системами. в системе EME.WMS эти функции используются для распределения задач между терминалами, управления складскими операциями и визуализации состояния очереди.

## Перетаскивание и привязка приказов

### is_drag_drop_terminals_dispatcher — перетаскивание объектов в диспетчере терминалов

Реализует перетаскивание объектов в диспетчере терминалов. Привязывает или отвязывает приказы от терминалов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Объект-источник перетаскивания (ссылка) |
| args[1] | Integer | Нет | Объект-приемник перетаскивания (ссылка) |
| args[2] | Integer | Нет | Карта для записи ошибок (ссылка) |

**Возвращает**: `String` — пустая строка при успешном выполнении, или строка с описанием ошибки.

```EME-L
' Перетащить приказ на терминал '
result = is_drag_drop_terminals_dispatcher(orderRef, terminalRef)
' result = "" при успехе, или текст ошибки '

' Перетащить с записью ошибок в карту '
errors = Object("Map")
result = is_drag_drop_terminals_dispatcher(orderRef, terminalRef, errors)
```

---

### is_add_del_terminals_dispatcher — привязка или отвязка приказа к терминалу

Реализует привязку или отвязку приказа к терминалу в диспетчере.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на приказ в очереди (целое число) |
| args[1] | Integer | Нет | Ссылка на терминал. NULL_REF для отвязки (целое число) |
| args[2] | Integer | Нет | Карта для записи ошибок (ссылка) |

**Возвращает**: `String` — пустая строка при успехе, или строка с описанием ошибки.

```EME-L
' Привязать приказ к терминалу '
result = is_add_del_terminals_dispatcher(queueOrderRef, terminalRef)
' result = "" при успешной привязке '

' Отвязать приказ от терминала '
result = is_add_del_terminals_dispatcher(queueOrderRef, NULL_REF)
' result = "" при успешной отвязке '
```

## Работа с терминалами

### is_get_valid_terminal_ref — получить ссылку на терминал

Возвращает ссылку на терминал для указанного объекта.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Объект диспетчера терминалов (ссылка) |

**Возвращает**: `Integer` — ссылка на терминал, или NULL_REF если терминал не назначен.

```EME-L

' terminalRef = ссылка на терминал, или NULL_REF '
```

## Заполнение ссылок и массивов

### is_fill_refs_for_integrator — заполнить карты ссылками

Заполняет переданные карты ссылками на документы, пачки и заказы из очереди приказов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Карта для ссылок на документы (ссылка) |
| args[1] | Integer | Да | Резервный параметр (ссылка) |
| args[2] | Integer | Да | Карта для ссылок на пачки заказов (ссылка) |
| args[3] | Integer | Да | Карта для ссылок на заказы на отгрузку (ссылка) |
| args[4] | Integer | Да | Карта для ссылок на пачки заказов типа 2 (ссылка) |
| args[5] | Integer | Да | Карта для ссылок на пачки заказов типа 3 (ссылка) |
| args[6] | Integer | Да | Карта для ссылок на пачки заказов типа 4 (ссылка) |
| args[7] | Integer | Да | Опция фильтрации типов приказов (целое число) |

**Возвращает**: пустое значение.

```EME-L
' Заполнить карты ссылками из очереди '
docRefs = Object("Map")
reserved = Object("Map")
batchRefs = Object("Map")
shipRefs = Object("Map")
batch2 = Object("Map")
batch3 = Object("Map")
batch4 = Object("Map")
is_fill_refs_for_integrator(docRefs, reserved, batchRefs, shipRefs, batch2, batch3, batch4, 0)
```

### is_check_both_double_stack_queue — проверить оба double-stack приказа

Проверяет, что оба приказа double-stack присутствуют в переданном массиве.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Массив объектов для проверки (ссылка) |

**Возвращает**: `Boolean` — TRUE если оба приказа double-stack найдены, FALSE в противном случае.

```EME-L

' bothPresent = TRUE если оба приказа найдены '
```

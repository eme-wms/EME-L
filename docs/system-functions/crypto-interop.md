# Криптография, реестр и взаимодействие

> Synonyms for search: криптография, реестр, is_qhash, is_reg_query_value, is_send_message, is_add_history_line, хеш, реестр Windows, сообщения между компонентами, история операций


### is_qhash — хеш-сумма строки

Вычисляет хеш-сумму строки с использованием функции Qt qHash. Для воспроизводимости результата на разных платформах строка предварительно перекодируется в Unicode через QTextCodec. Доступна только в версии ядра 30.0 и выше.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Строка, хеш которой необходимо вычислить |
| args[1] | String | Нет | Имя кодировки для декодирования строки (по умолчанию "Windows-1251") |

**Возвращает**: `Integer` — хеш-сумму переданной строки, вычисленную функцией qHash. В версиях ядра ниже 30.0 возвращает пустое значение.

```EME-L
h = is_qhash("test string")
h = is_qhash("test string", "UTF-8")
```

## Реестр Windows

### is_reg_query_value — чтение из реестра

читает значение из реестра Windows по указанному пути. Поддерживает типы REG_SZ и REG_DWORD.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Корневой раздел реестра: "HKCR"/"HKEY_CLASSES_ROOT", "HKLM"/"HKEY_LOCAL_MACHINE", "HKCU"/"HKEY_CURRENT_USER", "HKU"/"HKEY_USERS", "HKCC"/"HKEY_CURRENT_CONFIG", "HKDD"/"HKEY_DYN_DATA", "HKPD"/"HKEY_PERFORMANCE_DATA" |
| args[1] | String | Да | Путь к подразделу, например "Software\MyCompany\MyApp" |
| args[2] | String | Да | Имя читаемого параметра |
| args[3] | Any | Нет | Значение по умолчанию, если параметр не найден. При отсутствии — вызывает исключение |

**Возвращает**: значение параметра (строка или целое число в зависимости от типа в реестре).

```EME-L
val = is_reg_query_value("HKLM", "SOFTWARE\\MyApp", "Setting", "default")
```

## Сообщения между компонентами

### is_send_message — отправка сообщения

отправляет сообщение указанному органу управления диалога или всем органам. При отправке в главный диалог (ГФД) требуются дополнительные параметры record и line.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String/Integer | Да | Адресат сообщения: строка — имя органа управления, 0 — все органы текущего диалога, 1 — главный диалог (ГФД) |
| args[1] | Integer | Да | Номер сообщения |
| args[2] | Integer | Нет | Уровень диалога в иерархии (по умолчанию 0 — текущий). 1 — родительский, 2 — родитель родителя и т.д. |
| args[3] | Integer | Нет | Номер записи для вызова метафункции ГФД |
| args[4] | Integer | Нет | Номер строки записи для вызова метафункции ГФД |

**Возвращает**: пустое значение.

```EME-L
is_send_message("Button1", 100)
' Все органы '
is_send_message(0, 200)
```

### is_message_to_workstations — сообщение на рабочие станции

отправляет всплывающее сообщение на рабочие станции указанному пользователю. Вызов EMEShowPopupWindowToWorkstations.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на оператора |
| args[1] | String | Да | Текст сообщения |
| args[2] | String | Да | Имя запускаемого диалога при нажатии на окно |
| args[3] | Integer | Да | Номер строки диалога |
| args[4] | Integer | Да | Код запуска (OVL-код) |

**Возвращает**: пустое значение.

```EME-L
is_message_to_workstations(user_ref, "Внимание! Система перезагружается")
```

## История операций

### is_add_history_line — запись в историю

добавляет строку в историю операций. При двух параметрах используется запись OperationHistory с автоматическим определением текущего оператора. При четырёх параметрах вызывает устаревшую функцию add_history_line. args[2] и args[3] передаются только вместе (оба или ни одного).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Код тома |
| args[1] | String | Да | Текст записи для истории операций |
| args[2] | Integer | Нет | Номер записи |
| args[3] | Integer | Нет | Номер строки записи |

**Возвращает**: пустое значение.

```EME-L
is_add_history_line(1, "Операция выполнена")
is_add_history_line(1, "Операция с записью", rec_no, line_no)
```

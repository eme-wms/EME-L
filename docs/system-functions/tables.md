# Работа с таблицами

> Synonyms for search: таблицы, tables, is_exists, is_all_equal, is_in_rows, is_table_data, is_No_of_rows, is_row, is_view_row, is_db_line, is_table, строки таблицы, колонки таблицы, ячейка таблицы

### is_exists — проверка условия по строкам таблицы

в языке EME-L функция `is_exists` проверяет, удовлетворяет ли хотя бы одна строка таблицы указанному условию. Функция перебирает все таблицы текущего диалога объекта и для каждой строки вычисляет логическое выражение. Если условие истинно хотя бы для одной строки, функция немедленно возвращает признак успеха. в системе EME.WMS функция используется для поиска записей по условию в табличных данных диалога.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Логическое условие для проверки строк таблицы. Выражение вычисляется в контексте таблицы для каждой строки. В условии можно использовать значения текущей строки таблицы |
| args[1] | Integer | Нет | Положение объекта в генеалогии. >0 — дочерние объекты, <0 — родительские объекты, 0 или отсутствует — текущий объект |

**Вызов без аргументов**: невозможен (требуется хотя бы условие).

**Возвращает**: `Boolean` — TRUE, если хотя бы одна строка таблицы удовлетворяет условию. FALSE, если ни одна строка не удовлетворяет условию или таблица пуста.

```EME-L
' Проверить, есть ли в таблице строки с количеством больше 100 '
HasLargeQty = is_exists("qty > 100")
' HasLargeQty = TRUE, если хотя бы одна строка имеет qty > 100 '

' Проверить условие в дочернем объекте '
Found = is_exists("status = 1", 1)
```

### is_all_equal — проверка совпадения значений в колонке

в языке EME-L функция `is_all_equal` проверяет, совпадают ли значения во всех строках указанной колонки таблицы. Если таблица не содержит строк, считается что все значения совпадают (вакуумная истина) и возвращается 1. Если указанный орган не является колонкой таблицы, генерируется ошибка. в системе EME.WMS функция используется для валидации однородности данных в колонке перед групповыми операциями.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Положение объекта в генеалогии. >0 — дочерние объекты, <0 — родительские объекты, 0 — текущий объект (без навигации) |
| args[1] | String | Да | Софтовое имя столбца (колонки) таблицы. Должно соответствовать имени органа управления, являющегося колонкой таблицы |

**Вызов без аргументов или с одним аргументом**: невозможен. Требуется указать оба аргумента.

**Возвращает**: `Integer` — 1, если все значения в колонке совпадают (или таблица пуста). 0, если найдено хотя бы одно отличающееся значение в колонке.

```EME-L
' Проверить, что все строки имеют одинаковый статус '
AllSame = is_all_equal(0, "status")
' AllSame = 1, если во всех строках колонки status одинаковые значения '

' Проверить в дочернем объекте '
AllSameChild = is_all_equal(1, "type_id")
```

### is_in_rows — проверка присутствия строки броузера в таблице

в языке EME-L функция `is_in_rows` проверяет, присутствует ли значение ссылки текущей строки броузера (или отчёта) среди значений ссылок указанной колонки таблицы. Функция навигирует вверх по контексту до броузера/отчёта, запоминает текущую строку (LINE), затем перебирает все строки таблицы и сравнивает значение ссылки колонки с LINE. в системе EME.WMS функция используется для проверки принадлежности текущей записи к набору строк таблицы.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Софтовое имя колонки таблицы. Должно быть именем органа управления, являющегося колонкой таблицы (идентификатор органа должен быть меньше START_IDC) |

**Вызов без аргументов**: невозможен. Требуется указать имя колонки.

**Возвращает**: `Integer` — 1, если найдена строка таблицы, значение ссылки которой совпадает с текущей строкой броузера/отчёта. 0, если совпадение не найдено.

```EME-L
' Проверить, есть ли текущая строка броузера в колонке таблицы '
Found = is_in_rows("order_id")
' Found = 1, если ссылка текущей строки броузера найдена в колонке order_id '
```

### is_table_data — содержимое ячейки таблицы

в языке EME-L функция `is_table_data` возвращает содержимое указанного органа управления (колонки таблицы) в заданной строке таблицы с учётом его атрибутов (формат, тип данных, маски и т.д.). Является обёрткой для функции `is_edit_data` с переключением контекста на табличный. в системе EME.WMS функция используется для чтения данных из конкретной ячейки таблицы по номеру строки.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Софтовое имя органа управления (колонки таблицы). Пустое значение приводит к использованию текущего органа управления (требуется корректный табличный контекст) |
| args[1] | Integer | Да | Номер строки таблицы. Нумерация строк начинается с 0 |
| args[2] | Integer | Нет | Положение объекта в генеалогии. >0 — дочерние объекты, <0 — родительские объекты, 0 или отсутствует — текущий объект |
| args[3] | String | Нет | Реляционный путь для навигации. Используется для дополнительной навигации внутри объекта |

**Вызов без аргументов или с одним аргументом**: невозможен. Требуется минимум два аргумента: имя органа и номер строки.

**Возвращает**: содержимое ячейки — `String`, `Integer`, `Real`, `Date`, `Time`, `Boolean` или `Reference` в зависимости от типа данных органа управления и его атрибутов отображения.

```EME-L
' Получить значение колонки "name" в строке 0 '
NameValue = is_table_data("name", 0)
' NameValue = содержимое ячейки колонки name в первой строке '

' Получить значение из дочернего объекта '
ChildValue = is_table_data("qty", 2, 1)
```

### is_No_of_rows — количество строк в таблице

в языке EME-L функция `is_No_of_rows` возвращает общее количество строк в указанной таблице. в системе EME.WMS функция используется для определения размера таблицы перед итерацией по строкам или проверкой наличия данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Софтовое имя таблицы. Имя должно соответствовать зарегистрированному в диалоге органу управления типа CT_TABLE |

**Вызов без аргументов**: невозможен. Требуется указать имя таблицы.

**Возвращает**: `Integer` — количество строк в таблице (>=0). Если таблица пуста, возвращается 0.

```EME-L
' Получить количество строк в таблице '
RowCount = is_No_of_rows("OrderLines")
' RowCount = общее число строк в таблице OrderLines '

' Проверить, что таблица не пуста '
If (is_No_of_rows("OrderLines") > 0)
    Msg = "Таблица содержит данные"
End If
```

### is_row — номер текущей строки таблицы

в языке EME-L функция `is_row` возвращает номер текущей строки таблицы. Если указана колонка таблицы (arg1), возвращает номер текущей строки именно той таблицы, к которой относится данная колонка. Если колонка не указана, возвращает номер текущей строки из контекста параметра TS_PARAM. в системе EME.WMS функция используется для определения позиции курсора в таблице.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Положение объекта в генеалогии. По умолчанию 0 (текущий объект). >0 — дочерние объекты, <0 — родительские объекты |
| args[1] | String | Нет | Софтовое имя органа управления — колонки таблицы. Если указано, функция определяет родительскую таблицу по данной колонке и возвращает current_grid_y (номер текущей строки с фокусом ввода) |

**Вызов без аргументов**: возвращает номер текущей строки таблицы для текущего объекта (p.row).

**Возвращает**: `Integer` — номер текущей строки таблицы (0-based). Если указана колонка — номер строки таблицы, содержащей данную колонку (current_grid_y). Если колонка не указана — значение p.row из контекста.

```EME-L
' Получить номер текущей строки таблицы '
CurrentRow = is_row()
' CurrentRow = номер текущей строки (0-based) '

' Получить номер строки по колонке '
GridRow = is_row(0, "product_id")
```

### is_view_row — номер верхней видимой строки таблицы

в языке EME-L функция `is_view_row` возвращает номер текущей верхней видимой строки таблицы (индекс первой отображаемой на экране строки). Используется преимущественно в привязках, связанных с виртуальными таблицами, для синхронизации отображаемой области. в системе EME.WMS функция используется для управления прокруткой и позиционированием таблицы в интерфейсе.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Нет | Софтовое имя таблицы. Если указано, функция ищет таблицу с данным именем в текущем диалоге. Если не указано, используется активная таблица диалога (dialog_table) |

**Вызов без аргументов**: возвращает верхнюю видимую строку активной таблицы текущего диалога.

**Возвращает**: `Integer` — номер текущей верхней видимой строки таблицы (0-based). Соответствует свойству view_grid_y объекта CTable.

```EME-L
' Получить верхнюю видимую строку активной таблицы '
TopRow = is_view_row()
' TopRow = индекс первой видимой строки '

' Получить верхнюю видимую строку конкретной таблицы '
ViewRow = is_view_row("OrderLines")
```

### is_db_line — номер строки базы данных по строке таблицы

в языке EME-L функция `is_db_line` возвращает номер строки базы данных (DB line), соответствующий указанной строке таблицы. Используется для получения физического идентификатора записи базы данных по индексу строки в табличном отображении. в системе EME.WMS функция используется для связи между табличным представлением и записями базы данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Положение объекта в генеалогии. По умолчанию 0 (текущий объект). >0 — дочерние объекты, <0 — родительские объекты |
| args[1] | String | Да | Софтовое имя колонки таблицы. По колонке определяется родительская таблица |
| args[2] | Integer | Нет | Номер строки таблицы. По умолчанию — текущая строка (-1 передаётся в GetDBLine для получения текущей строки) |

**Вызов без аргументов или с одним аргументом**: невозможен. При вызове с одним аргументом он интерпретируется как положение в генеалогии (arg0), а имя колонки (arg1) оказывается пустым, что приводит к ошибке. Для корректного вызова с текущим объектом передайте явно 0 как первый аргумент: `is_db_line(0, "имя_колонки")`.

**Возвращает**: `Integer` — номер строки базы данных (DB line), соответствующий указанной строке таблицы. Значение >= 0.

```EME-L
' Получить DB line текущей строки по колонке '
DbLine = is_db_line(0, "order_id")
' DbLine = номер строки базы данных для текущей строки таблицы '

' Получить DB line для конкретной строки '
DbLine2 = is_db_line(0, "order_id", 5)
```

### is_table — имя активной таблицы

в языке EME-L функция `is_table` возвращает софтовое имя активной таблицы, полученное из параметров сообщения диалога. Если активная таблица отсутствует или не зарегистрирована в диалоге, возвращается пустая строка. в системе EME.WMS функция используется для определения имени таблицы, с которой работает текущий диалог или привязка.

**Параметры**: нет.

**Вызов без аргументов**: возвращает имя активной таблицы.

**Возвращает**: `String` — софтовое имя активной таблицы (органа управления типа CTable). Если активная таблица не определена — пустая строка.

```EME-L
' Получить имя активной таблицы '
TableName = is_table()
' TableName = софтовое имя активной таблицы из параметров диалога '

' Проверить активную таблицу '
If (is_table() = "OrderLines")
    Msg = "Работаем с таблицей OrderLines"
End If
```

# База данных: ссылки — is_ref_to, is_vvr, is_check_chain и другие функции работы со ссылками

> Synonyms for search: база данных ссылки, мягкие ссылки, is_ref_to, is_exist_ref_to, is_vvr, is_vvr_rec, is_vvr_line, is_in_ref_array, is_linked, is_check_chain, is_restore_refs, is_relink_ref, is_redirect_referencies, is_clean_refs, is_root, is_parent, ссылочная целостность, классификаторы, цепочки записей

## База данных: ссылки

is-функции группы DB (ссылки) отвечают за работу со ссылочными полями, мягкими ссылками, массивами ссылок, классификаторами и цепочками записей в языке EME-L. в системе EME.WMS эти функции используются для подсчета ссылок, навигации по иерархии, поиска связанных строк и управления ссылочной целостностью.

### is_ref_to — количество ссылок на строку

в языке EME-L функция `is_ref_to` возвращает общее количество ссылок на указанную строку записи.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Строка с номерами записей и полей для пропуска или учета при подсчете |
| args[1] | Integer | Нет | Если 0 — пропускать указанные поля; если не 0 — учитывать только указанные |
| args[2] | String / Integer | Нет | Имя или номер записи для поиска ссылок. По умолчанию базовая запись |
| args[3] | Integer | Нет | Номер строки для поиска ссылок. По умолчанию текущая строка |

**Возвращает**: `Integer` — общее количество ссылок.

```EME-L
' Подсчитать все ссылки на текущую строку '
Count = is_ref_to()
' Count = 5 '

' Пропустить поля (5,1) и (5,2) при подсчете '
Count = is_ref_to("(5,1),(5,2)", 0)
' Count = 3 '

' Учитывать только указанные поля '
Count = is_ref_to("(10,3)", 1, "Заказы", 5)
' Count = 2 '
```

### is_exist_ref_to — проверка наличия ссылок

в системе EME.WMS функция `is_exist_ref_to` проверяет наличие хотя бы одной ссылки на указанную строку записи.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Строка с номерами записей и полей для пропуска или учета |
| args[1] | Integer | Нет | Если 0 — пропускать; если не 0 — учитывать только указанные |
| args[2] | String / Integer | Нет | Имя или номер записи. По умолчанию базовая запись |
| args[3] | Integer | Нет | Номер строки. По умолчанию текущая строка |

**Возвращает**: `Boolean` — `TRUE` если ссылка существует, иначе `FALSE`.

```EME-L
' Проверить наличие ссылок на текущую строку '
HasRefs = is_exist_ref_to()
' HasRefs = TRUE '

' Проверить с пропуском определенных полей '
HasRefs = is_exist_ref_to("(5,1)", 0)
' HasRefs = FALSE - после исключения ссылок нет '
```

### is_referenceed_line — поиск строки по ссылке

в языке EME-L функция `is_referenceed_line` ищет строку в указанной записи, которая ссылается на заданную строку.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя или номер записи-источника ссылки |
| args[1] | String / Integer | Да | Имя или номер поля ссылки |
| args[2] | Integer | Нет | Номер строки, с которой начинать поиск. По умолчанию 0 |
| args[3] | Integer | Нет | Номер строки, на которую должна указывать ссылка. По умолчанию текущая строка |

**Возвращает**: `Integer` — номер найденной строки или `NULL_REF` если не найдена.

```EME-L
' Найти строку в записи Заказы, поле Клиент, ссылающуюся на текущую строку '
Line = is_referenceed_line("Заказы", "Клиент")
' Line = 10 '

' Найти со строки 5, ссылающуюся на строку 20 '
Line = is_referenceed_line("Заказы", "Клиент", 5, 20)
' Line = 15 '
```

### is_ref_to_child — ссылки на дочерние строки

| args[2] | String | Нет | Имя поля сортировки в дочерней записи. По умолчанию "Сортировка" |
| args[3] | String | Да | Строка с номерами записей и полей для пропуска или учета |

**Возвращает**: `Integer` — количество ссылок на дочерние строки.

### is_check_chain — проверка цепочки

в системе EME.WMS функция `is_check_chain` проверяет все строки цепочки на соответствие заданному условию.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя или номер записи |
| args[1] | String / Integer | Да | Имя или номер поля-ссылки на начало цепочки |
| args[2] | Integer | Да | Номер строки-заголовка цепочки |
| args[3] | String | Да | Текстовое условие для проверки каждой строки |
| args[4] | Integer | Да | Режим: 0 — все удовлетворяют, 1 — хотя бы один, 2 — хотя бы один не удовлетворяет, 3 — все не удовлетворяют |

**Возвращает**: `Boolean` — результат проверки в зависимости от режима.

```EME-L
' Проверить, есть ли хотя бы одна строка со статусом 2 '
HasPending = is_check_chain("Документы", "Следующий", 0, "is_data(\"Документы\", is_arg(1), \"Статус\") = 2", 1)

```

### is_root — корневой элемент дерева или классификатора

в EME БД функция `is_root` возвращает номер строки корневого элемента дерева или классификатора. Работает в двух режимах в зависимости от количества аргументов.

| Параметр (режим 1 - дерево) | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя органа управления с деревом |

| Параметр (режим 2 - классификатор) | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String / Integer | Да | Имя или номер записи классификатора |
| args[1] | String / Integer | Да | Имя или номер рекурсивного поля |
| args[2] | Integer | Да | Номер строки для начала обхода |

**Возвращает**: `Integer` — номер строки корневого элемента, -1 если текущий элемент уже корневой, `NULL_REF` при ошибке.

```EME-L
' Получить корневой элемент дерева '
RootLine = is_root("TreeControl")
' RootLine = 0 '

' Получить корень классификатора для строки 20 '
RootLine = is_root("Классификатор", "Родитель", 20)
' RootLine = 0 - корень дерева '
```

### is_parent — родительский элемент дерева

в языке EME-L функция `is_parent` возвращает номер строки родительского элемента дерева для текущего элемента.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя органа с деревом |

**Возвращает**: `Integer` — номер строки родителя или -1 если текущий элемент корневой.

```EME-L
' Получить родителя текущего элемента в дереве '
ParentLine = is_parent("TreeControl")
' ParentLine = 5 '

' Корневой элемент '
ParentLine = is_parent("TreeControl")
' ParentLine = -1 '
```

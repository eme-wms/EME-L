### is_beep — звуковой сигнал

Воспроизводит звуковой сигнал заданной частоты и длительности через API Beep.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Частота звука в герцах |
| args[1] | Integer | Да | Длительность звука в миллисекундах |

**Возвращает**: `пустое значение`.

```EME-L
' Звук 1000 Гц, 500 мс '
is_beep(1000, 500)
```

---

### is_sleep — приостановка потока

Приостанавливает выполнение текущего потока на указанное время. Вызов API Sleep.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Время приостановки в миллисекундах |

**Возвращает**: пустое значение.

```EME-L
' Пауза 1 секунда '
is_sleep(1000)
```

---

### is_exit — завершение приложения

Завершает работу приложения с опциональным сообщением об ошибке. Вызов exit_with_message.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Номер первого сообщения из словаря |
| args[1] | Integer | Нет | Номер второго сообщения из словаря |
| args[2] | String | Нет | Произвольный текст сообщения |

**Возвращает**: `пустое значение` (приложение завершается).

```EME-L
is_exit()
is_exit(7, 366, "Ошибка!")
```

### is_do_wait_cursor — песочные часы

Управляет отображением песочных часов (курсора ожидания). Вызов do_wait_cursor.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Направление действия: 1 — включить (увеличить счётчик), -1 — выключить (уменьшить счётчик), 0 — сбросить счётчик. При отсутствии параметра ничего не делает |

**Возвращает**: `Integer` — 0 (целое число).

```EME-L
' Включить песочные часы '
is_do_wait_cursor(1)

' Выключить песочные часы '
is_do_wait_cursor(-1)

' Сбросить счётчик '
is_do_wait_cursor(0)
```

### is_main_thread — проверка главного потока

Проверяет, является ли текущий поток главным (основным) потоком приложения.

**Параметры**: нет.

**Возвращает**: `Boolean` — TRUE если текущий поток — главный, иначе FALSE.

```EME-L
' Проверить, является ли поток главным (TRUE — главный поток) '
result = is_main_thread()
```

---

### is_managed_thread — управляемый поток

Управляет режимом управляемого потока или проверяет, является ли текущий поток управляемым. Управляемый поток должен периодически сообщать о своей активности.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Интервал проверки потока в миллисекундах. 0 — отключить управление потоком. При отсутствии параметра только возвращает статус |

**Возвращает**: без параметров — `Boolean` — TRUE если текущий поток управляемый, иначе FALSE. С параметром — `пустое значение`.

```EME-L
' Проверить, является ли поток управляемым (TRUE — управляемый) '
result = is_managed_thread()

' Включить управление потоком с интервалом 1 секунда '
is_managed_thread(1000)
' ... выполнение работы ... '
' Отключить управление потоком '
is_managed_thread(0)
```

## Системные настройки

### is_system — системная настройка

Возвращает системную настройку по указанному индексу.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Индекс системной настройки (целое число) |

Индексы:
- 0 — Ссылка на текущего оператора (целое число)
- 1 — Флаги доступа текущего оператора (целое число, битовая маска 32 бита)
- 2 — Идентификатор (имя) текущего оператора (строка)
- 3 — Код запуска модуля (целое число)
- 4 — Код текущего филиала (целое число)
- 5 — Ссылка на текущий филиал (целое число)
- 6 — Код (имя файла) проекта (строка)
- 7 — Признак рабочей станции (boolean)
- 8 — Признак мозгового компьютера (boolean)

**Возвращает**: значение системной настройки в соответствии с индексом. Тип зависит от индекса (см. выше).

```EME-L
' Ссылка на оператора '
ref = is_system(0)
' Флаги доступа (32 бита) '
flags = is_system(1)
' Имя оператора '
name = is_system(2)
' Код запуска модуля '
code = is_system(3)
' Код филиала '
fcode = is_system(4)
' Ссылка на филиал '
fref = is_system(5)
' Код (имя файла) проекта '
pname = is_system(6)
' Признак рабочей станции '
ws = is_system(7)
' Признак мозгового компьютера '
bc = is_system(8)
```

### is_set_system — установка кода запуска (устаревшая)

> **Устаревшая**: функция доступна только в версиях ядра до 52.0 (`#if LAST_VERSION < MAKE_VERSION(52,0)`).

Устанавливает код запуска модуля (OVL-код) для текущего диалога. Вызов `SetCurrentOvlCode`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Новый код запуска модуля (целое число) |

**Возвращает**: пустое значение.

```EME-L
' Установить OVL-код = 10 '
is_set_system(10)
```

### is_sys_param — параметр настройки

Возвращает значение системной настройки по имени. Имя настройки должно начинаться с префикса `W://` (настройки рабочей станции) или `M://` (настройки модулей).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя настройки с префиксом W:// или M:// |
| args[1] | Any | Нет | Тип возвращаемого значения: `VT_BSTR` — строка, `VT_I2`/`VT_I4` — целое число, `VT_R8` — вещественное число. По умолчанию определяется автоматически по типу variant |
| args[2] | Boolean | Нет | Режим обработки ошибок (по умолчанию `FALSE`): `FALSE` — EXCEPT_MODE (при отсутствии настройки вызывает ошибку), `TRUE` — SOFT_MODE (при отсутствии настройки возвращает пустое значение) |

**Возвращает**: значение настройки (тип зависит от args[1]: строка, целое число или вещественное число).

```EME-L
val = is_sys_param("M://MySetting", "")
val = is_sys_param("W://WsSetting", 0)
' SOFT_MODE '
val = is_sys_param("M://SoftSetting", "", TRUE)
```

### is_add_sys_param — добавление настройки

Создаёт или обновляет системную настройку. Имя настройки должно начинаться с префикса `W://` (настройки рабочей станции) или `M://` (настройки модулей). Для `W://` настройка сохраняется в базе данных через `StoreWsSetting`, для `M://` — через `create_module_param`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя настройки с префиксом W:// или M:// |
| args[1] | String | Да | Значение настройки |
| args[2] | String | Нет | Текст подсказки (HelpText) |
| args[3] | String | Нет | Список допустимых значений в виде строки с разделителями `\r\n` |
| args[4] | Boolean | Нет | `TRUE` — перезагрузить все модули после изменения (по умолчанию `TRUE`) |

**Возвращает**: пустое значение.

```EME-L
is_add_sys_param("M://NewSetting", "value", "Описание", "val1\r\nval2")
```

### is_check_text_key — проверка ключа командной строки

Проверяет наличие прикладного ключа командной строки при запуске программы. Прикладные ключи имеют префикс `/d_`, например `/d_emel` передаётся как `"EMEL"`. Вызов `CheckTextKey`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Наименование ключа без префикса `/d_` |

**Возвращает**: `Boolean` — `TRUE`, если ключ установлен, иначе `FALSE`.

```EME-L
' Проверить наличие ключа командной строки /d_emel (TRUE — ключ установлен) '
result = is_check_text_key("EMEL")
```

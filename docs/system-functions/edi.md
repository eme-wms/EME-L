# EANCOM (EDI-интеграция)

> Synonyms for search: EANCOM, EDI, is_import_EANCOM_messages, is_EANCOM_name, is_EANCOM_segment, is_goods_item_code_to_EME, is_goods_item_code_to_SAP, is_turn_EANCOM_messages, импорт EANCOM, сегменты, SAP


### is_import_EANCOM_messages — импорт/экспорт EANCOM-сообщений

выполняет импорт входящих и экспорт исходящих сообщений EANCOM. Читает сообщения из внешних каталогов и внутренней файловой системы EME, передаёт их парсеру EANCOM и архивирует обработанные файлы.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference | Нет | Ссылка на объект EME-L класса-обработчика. Если передан, используется для определения целевой папки во внутренней файловой системе через метод GetInternalFolderName |

**Возвращает**: пустое значение.

```EME-L
' Импорт-экспорт без обработчика '
is_import_EANCOM_messages()

' С объектом-обработчиком для определения папки '
is_import_EANCOM_messages(myHandler)
```

### is_EANCOM_name — код EANCOM-сообщения

определяет код EANCOM-сообщения из переданного текста или файла. Функция последовательно ищет служебные сегменты UNA (опционально), UNB и UNH, извлекая из UNH код сообщения (значение до символа ':').

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Путь к файлу с сообщением EANCOM или непосредственно текст сообщения. Если строка короче 512 символов и соответствует существующему файлу, содержимое считывается из файла |

**Возвращает**: `String` — код EANCOM-сообщения. Если сегменты UNB или UNH отсутствуют, возвращается пустая строка.

```EME-L
code = is_EANCOM_name("UNA:+.? 'UNB+UNOC:2...")
' code = "DESADV" '

code = is_EANCOM_name("C:\Messages\incoming.edi")
```

### is_EANCOM_segment — извлечение сегмента EANCOM

извлекает указанный сегмент из EANCOM-сообщения (файла или строки) и сохраняет его элементы данных (DE) в переданную коллекцию CICollection. Если элемент данных состоит из компонентов (CE, разделённые ':'), он сохраняется как массив CIArray с ключом «Array». в системе EME.WMS используется для парсинга входящих EDI-сообщений.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference | Да | Объект CICollection для сохранения распарсенных элементов сегмента |
| args[1] | String | Да | Путь к файлу с сообщением EANCOM или текст сообщения. Если строка короче 512 символов и соответствует существующему файлу, данные читаются из файла |
| args[2] | String | Да | Идентификатор искомого сегмента, например «UNH» или «LIN» |
| args[3] | Integer | Нет | Порядковый номер вхождения сегмента с указанным именем, начиная с 0. По умолчанию 0 |

**Возвращает**: `Boolean` — `TRUE` если сегмент найден и успешно разобран, `FALSE` если сегмент не найден.

```EME-L
' Извлечь первый сегмент LIN из файла '
Dim coll = New CICollection()
is_EANCOM_segment(coll, "C:\Messages\order.edi", "LIN")
' coll содержит элементы сегмента LIN '

' Извлечь второй сегмент LIN (индекс 1) '
is_EANCOM_segment(coll, messageText, "LIN", 1)
' Второй LIN-сегмент обработан '
```

### is_goods_item_code_to_EME — форматирование кода товара для EME

форматирует код товара из входящего EANCOM-сообщения для использования в EME: удаляет ведущие нули слева до достижения минимальной длины кода (задаётся настройкой EANCOM.Минимальная длина кода, по умолчанию 8 символов). Если код состоял только из нулей, возвращает пустую строку.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Исходный код товара из входящего сообщения |

**Возвращает**: `String` — отформатированный код товара. Если исходный код содержал только нули, возвращается пустая строка.

```EME-L
code = is_goods_item_code_to_EME("0000123456")
' code = "123456" (при минимальной длине 8) '

code = is_goods_item_code_to_EME("00000000")
' code = "" (все нули) '
```

### is_goods_item_code_to_SAP — форматирование кода товара для SAP

форматирует код товара для исходящего EANCOM-сообщения (SAP): дополняет код ведущими нулями слева до минимальной длины, заданной в настройке EANCOM.Минимальная длина кода (по умолчанию 8 символов).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Код товара для отправки в исходящем сообщении |

**Возвращает**: `String` — отформатированный код товара, выровненный нулями слева до минимальной длины.

```EME-L
code = is_goods_item_code_to_SAP("123456")
' code = "00123456" (при минимальной длине 8) '

code = is_goods_item_code_to_SAP("ABC")
' code = "00000ABC" '
```

# База данных: структура, архивация и удаление

> Synonyms for search: база данных, структура БД, архивация, удаление документов, is_export_db_structure, is_import_db_structure, is_archive_DB, is_eme_is_arc, is_eme_arc, is_eme_unarc, is_eme_arc_deleted, is_eme_arc_all, is_eme_unarc_all_records, is_delete_documents, BBD, экспорт структуры, импорт структуры


### is_export_db_structure — экспорт структуры БД

экспортирует структуру БД в файл.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя файла |

**Возвращает**: `Integer` — 1 при успехе, 0 при пустом имени файла.

```EME-L
result = is_export_db_structure("structure.dbh")
```

---

### is_import_db_structure — импорт структуры БД

импортирует структуру БД из файла.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя файла |
| args[1] | String (ByRef) | Нет | Переменная по ссылке для возврата номера версии файла структуры |

**Возвращает**: `Boolean` — TRUE при успехе.

```EME-L
' Импортировать структуру БД из файла (TRUE — импорт успешен) '
result = is_import_db_structure("structure.dbh")
```

## Архивация БД

### is_archive_DB — архивация базы данных

выполняет архивацию базы данных: помечает устаревшие строки записей к удалению и создаёт архивный экспорт обрезанной копии базы. в языке EME-L функция должна вызываться в монопольном (MONO) режиме на мозговом компьютере вне транзакции. Если первый параметр пустой, пометка строк к архивации не выполняется — только архивный экспорт ранее помеченных строк. Вызов без аргументов или с одним аргументом недопустим, вызывает ошибку «Слишком мало параметров».

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата, до которой удалять документы и операции |
| args[1] | Date | Да | Срок годности (BBD), до которого удалять документы |
| args[2] | Date | Нет | Значение некорректного BBD, которое нужно игнорировать при фильтрации. По умолчанию максимально возможная дата (0x7FFF год) |
| args[3] | String | Нет | Директория для сохранения файлов архивного экспорта. По умолчанию используется стандартная директория архивного раздела |

**Возвращает**: пустое значение.

```EME-L
' Архивация: удалить документы до 01.01.2024, BBD до 01.01.2023 '
is_archive_DB(Date(2024, 1, 1), Date(2023, 1, 1))

' С указанием некорректного BBD и пути экспорта '
is_archive_DB(Date(2024, 1, 1), Date(2023, 1, 1), Date(2099, 12, 31), "C:\Archive\export")
```

### is_eme_is_arc — проверка флага архивации строки

проверяет, установлен ли у строки записи флаг архивации (бит `TO_ARCHIVE_EXPORT_LINE_BIT` в ядерном поле флагов). в системе EME.WMS используется для выборочной пометки строк перед процедурой архивации.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference / Integer | Да | Объект записи CEMERec (ссылка) или номер записи в базе данных (целое число) |
| args[1] | Integer | Условно | Номер строки записи. Обязателен, если args[0] задан как номер записи. Не используется, если args[0] — объект CEMERec |

**Возвращает**: `Boolean` — `TRUE` если строка помечена к архивации, `FALSE` если не помечена.

```EME-L
' Проверить флаг архивации строки 5 записи 100 (TRUE — помечена) '
arc = is_eme_is_arc(100, 5)

' Проверить флаг архивации по объекту CEMERec (TRUE — помечена) '
arc = is_eme_is_arc(rec)
```

### is_eme_arc — установка флага архивации

устанавливает флаг архивации (бит `TO_ARCHIVE_EXPORT_LINE_BIT`) для строки записи или набора строк, заданных битовым буфером. Должна вызываться внутри транзакции.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference / Integer | Да | Объект записи CEMERec (ссылка) или номер записи в базе данных (целое число) |
| args[1] | Integer / Reference | Условно | Номер строки записи (целое число) или битовый буфер CIBitBuffer (ссылка). Обязателен, если args[0] задан как номер записи. Если передан битовый буфер, флаг устанавливается для всех строк, присутствующих в буфере |

**Возвращает**: пустое значение.

```EME-L
' Пометить строку 5 записи 100 к архивации '
is_eme_arc(100, 5)

' Пометить все строки из битового буфера '
is_eme_arc(rec, bitBuffer)
```

### is_eme_unarc — снятие флага архивации

снимает флаг архивации (бит `TO_ARCHIVE_EXPORT_LINE_BIT`) со строки записи. Должна вызываться внутри транзакции.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference / Integer | Да | Объект записи CEMERec (ссылка) или номер записи в базе данных (целое число) |
| args[1] | Integer | Условно | Номер строки записи. Обязателен, если args[0] задан как номер записи. Не используется, если args[0] — объект CEMERec |

**Возвращает**: пустое значение.

```EME-L
' Снять пометку архивации со строки 5 записи 100 '
is_eme_unarc(100, 5)

' Снять пометку по объекту CEMERec '
is_eme_unarc(rec)
```

### is_eme_arc_deleted — архивация удалённых строк

помечает к архивации все удалённые (невалидные) строки указанной записи, устанавливая бит `TO_ARCHIVE_EXPORT_LINE_BIT` в ядерном поле флагов. Должна вызываться внутри транзакции.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference / Integer | Да | Объект записи CEMERec (ссылка) или номер записи в базе данных (целое число) |

**Возвращает**: пустое значение.

```EME-L
' Пометить все удалённые строки записи 100 к архивации '
is_eme_arc_deleted(100)
```

---

### is_eme_arc_all — архивация всех строк записи

помечает к архивации все строки указанной записи, устанавливая бит `TO_ARCHIVE_EXPORT_LINE_BIT` в ядерном поле флагов для каждой строки. Должна вызываться внутри транзакции.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Reference / Integer | Да | Объект записи CEMERec (ссылка) или номер записи в базе данных (целое число) |

**Возвращает**: пустое значение.

```EME-L
' Пометить все строки записи 100 к архивации '
is_eme_arc_all(100)
```

---

### is_eme_unarc_all_records — сброс флагов архивации

снимает флаг архивации (бит `TO_ARCHIVE_EXPORT_LINE_BIT`) со всех строк всех записей базы данных. в системе EME.WMS используется для сброса пометок перед началом процедуры пометки строк к архивации. Должна вызываться внутри транзакции.

**Параметры**: нет.

**Возвращает**: пустое значение.

```EME-L
' Сбросить все флаги архивации по всей базе данных '
is_eme_unarc_all_records()
```

## Удаление документов

### is_delete_documents — удаление закрытых документов

удаляет все закрытые документы WMS, строки документов, пачки заказов, заказы, партии, ГТД и связанные подчинённые записи, у которых дата закрытия меньше или равна указанной дате. в языке EME-L функция работает только в MONO-режиме на мозговом компьютере. Перед удалением отключает отложенное обнуление ссылок, выполняет удаление внутри серверной транзакции, а затем восстанавливает отложенное обнуление ссылок. После удаления выполняет архивную выгрузку базы данных.

**Ограничения:**
- Функция доступна только в MONO-режиме работы станции. В противном случае генерируется ошибка «Архивация БД возможна только в МОНО режиме»
- Функция доступна только на мозговом компьютере (canal brain computer). В противном случае генерируется ошибка «Архивация БД возможна только на мозговом компьютере»

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата, до которой (включительно) удаляются закрытые документы и связанные объекты |
| args[1] | String | Нет | Путь для сохранения архивной выгрузки базы данных после удаления. Если не указан или пустая строка, используется стандартный путь архивной выгрузки |

**Возвращает**: пустое значение.

```EME-L
' Удалить все документы, закрытые до 01.01.2024 '
is_delete_documents(Date(2024, 1, 1))

' Удалить с указанием пути архивной выгрузки '
is_delete_documents(Date(2024, 1, 1), "C:\Archive\export")
```

# База данных: транзакции и модификация — is_transaction, is_modify_record, is_delete_records и другие функции

> Synonyms for search: база данных транзакции, модификация, is_transaction, is_open_transaction, is_local_transaction, is_modify_record, is_delete_record_line, is_delete_records, is_compress_records, is_ws_align_data, серверная транзакция, локальная транзакция, сжатие записей, VLF

## База данных: транзакции и модификация

is-функции группы DB (транзакции и модификация) отвечают за управление серверными и локальными транзакциями, модификацию строк и записей, сжатие файлов базы данных и служебные операции. в системе EME.WMS эти функции обеспечивают атомарность операций, контроль целостности данных и оптимизацию хранения.

---

### is_transaction — управление серверной транзакцией

в языке EME-L функция `is_transaction` открывает, закрывает или отменяет серверную транзакцию. Является основной функцией для управления транзакциями в многопользовательском режиме.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Направление: 1 — открыть, -1 — закрыть, 0 — отменить. Если не указан, только проверяет состояние |
| args[1] | String / Boolean | Нет | При открытии: название транзакции (6-63 символа). При закрытии: `FALSE` — не бросать исключение при ошибке |
| args[2] | Integer / String | Нет | При открытии: приоритет транзакции: 0/"LOW"/"НИЗ" — низкий, 1/"NORMAL"/"НОРМ" — нормальный (по умолчанию), 2/"HIGH"/"ВЫС" — высокий |
| args[3] | Integer | Нет | При открытии: таймаут транзакции в миллисекундах. По умолчанию не ограничен |
| args[4] | Boolean | Нет | При открытии: `TRUE` — бросать исключение при таймауте на рабочей станции. По умолчанию `FALSE` |

**Возвращает**: `Boolean` — `TRUE` если транзакция открыта после выполнения операции, иначе `FALSE`.

```EME-L
' Открыть транзакцию с названием и нормальным приоритетом '
is_transaction(1, "Создание заказа")

' Открыть с высоким приоритетом и таймаутом 30 секунд '
is_transaction(1, "Срочное обновление", "HIGH", 30000)

' Закрыть транзакцию '
is_transaction(-1)

' Отменить транзакцию '
is_transaction(0)

' Проверить состояние транзакции '
IsOpen = is_transaction()
' IsOpen = TRUE '
```

### is_open_transaction — условное открытие транзакции

в системе EME.WMS функция `is_open_transaction` открывает серверную транзакцию, если нет другой открытой.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Название транзакции |

**Возвращает**: `Boolean` — `TRUE` если транзакция открыта, иначе `FALSE`.

```EME-L
' Открыть транзакцию, если нет другой '
Opened = is_open_transaction("Обновление данных")
' Opened = TRUE '

' Повторный вызов — транзакция уже открыта '
Opened = is_open_transaction("Другая транзакция")
' Opened = FALSE '
```

### is_modify_record — пометка всех строк записи

в EME БД функция `is_modify_record` помечает все строки указанных записей как модифицированные. Автоматически открывает и закрывает серверную транзакцию.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0+] | String / Integer | Да | Номер или имя записи. Можно указать несколько записей |

**Возвращает**: `Boolean` — `TRUE`.

```EME-L
' Пометить все строки записи Клиенты как модифицированные '
is_modify_record("Клиенты")

' Пометить несколько записей '
is_modify_record("Клиенты", "Заказы", "Товары")
```

### is_unmodify_record — снятие пометок модификации

в языке EME-L функция `is_unmodify_record` снимает пометки модификации со всех строк указанных записей.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0+] | String / Integer / Reference | Да | Номер, имя записи или объект массива с номерами/именами |

**Возвращает**: `Boolean` — `TRUE`.

```EME-L
' Снять пометки с записи Клиенты '

' Снять пометки через массив '
RecArray = Object("Array")

RecArray.Add("Заказы")
is_unmodify_record(RecArray)

**Возвращает**: `Boolean` — `TRUE` в случае успеха.

' Result = TRUE '
```

| args[2] | Integer | Нет | Конечная запись. По умолчанию последняя |

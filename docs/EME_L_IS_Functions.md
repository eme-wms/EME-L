## Обзор is-функций в языке EME-L

is-функции в EME-L являются внешними встроенными функциями, которые не могут быть реализованы на EME-L. Вызов is-функции имеет следующий формат:

```EME-L
{is-функция}([{Аргумент1}, [{Аргумент2} ...]])
```

is-функции языка EME-L подразделяются на функциональные группы: Bit, Browser, Context и другие.

---

## is-функции группы Bit — битовые операции

is-функции группы Bit отвечают за работу с битовыми признаками в языке EME-L. в системе EME.WMS битовые операции используются для манипуляции флагами, проверки прав доступа, формирования масок и низкоуровневой обработки данных. Все функции поддерживают 32-битные целые числа; некоторые также принимают массивы байтов.

---

### is_bit — проверка установленного бита

в языке EME-L функция `is_bit` проверяет состояние указанного бита в битовом поле. Поддерживает целые числа (32-битный `long`, `VT_I4`) и массивы байтов (`VT_ARRAY|VT_UI1`). Для целых чисел проверяется бит в 32-битном значении; для массивов байтов — бит в соответствующем байте массива.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer / Array | Да | Битовое поле для проверки. Целое число (4 байта, VT_I4) или массив байтов (VT_ARRAY\|VT_UI1) |
| args[1] | Integer | Да | Номер проверяемого бита, начиная с младшего бита 0 |

**Возвращает**: `Boolean` — `TRUE`, если указанный бит установлен (равен 1); `FALSE`, если бит сброшен (равен 0).

```EME-L
' Проверка бита 3 в флагах '
Result = is_bit(Flags, 3)

' Проверка бита 5 в массиве байтов '
Result = is_bit(ByteArray, 5)
```

### is_clear_bit — сброс бита

в системе EME.WMS функция `is_clear_bit` сбрасывает указанный бит в битовом поле в значение 0. Для целых чисел возвращает модифицированное число; для массивов байтов — модифицированный массив. Тип возвращаемого значения соответствует типу входного параметра.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer / Array | Да | Битовое поле для модификации. Целое число (VT_I4) или массив байтов (VT_ARRAY\|VT_UI1) |
| args[1] | Integer | Да | Номер сбрасываемого бита, начиная с 0 |

**Возвращает**: `Integer` или `Array` — модифицированное битовое поле со сброшенным указанным битом. Тип соответствует типу входного параметра.

```EME-L
' Снять бит 5, результат присваиваем обратно в ту же переменную '
Flags = is_clear_bit(Flags, 5)

' Снять бит 2 в массиве, результат присваиваем обратно '
ByteArray = is_clear_bit(ByteArray, 2)
```

**Возвращает**: `Integer` или `Array` — модифицированное битовое поле с установленным указанным битом. Тип соответствует типу входного параметра.

### is_reverse_bit — инверсия бита

в EME БД функция `is_reverse_bit` инвертирует значение указанного бита в битовом поле: 0 становится 1, 1 становится 0. Входное значение безусловно преобразуется в массив байтов перед операцией, поэтому при передаче целого числа возвращается массив байтов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer / Array | Да | Битовое поле для модификации. Преобразуется в массив байтов (VT_ARRAY\|VT_UI1) |
| args[1] | Integer | Да | Номер инвертируемого бита, начиная с 0 |

**Возвращает**: `Array` — модифицированное битовое поле в виде массива байтов с инвертированным указанным битом.

```EME-L
' Инвертировать бит 3, результат присваиваем обратно '
Flags = is_reverse_bit(Flags, 3)
```

### is_and — побитовое И

в языке EME-L функция `is_and` выполняет побитовую операцию И (AND) над всеми переданными аргументами. Поддерживает произвольное количество параметров. Каждый параметр может быть целым числом, 8-байтовым числом (при условии значения в диапазоне 0..0xFFFFFFFF) или массивом байтов. Операнды приводятся к 4-байтовым целым числам перед выполнением операции.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[N] | Integer / Array | Нет | Операнды для побитового умножения. Целое число, 8-байтовое число в диапазоне 0..0xFFFFFFFF или массив байтов |

**Вызов без аргументов**: возвращает -1 (все 32 бита установлены в 1).

**Возвращает**: `Integer` — результат побитовой операции И всех переданных операндов.

```EME-L
' Побитовое И двух флагов '
Result = is_and(Flags1, Flags2)

' Побитовое И трёх операндов '
Result = is_and(0xFF, 0x0F, 0xF0)

' Без аргументов - возвращает -1 '
Result = is_and()
```

### is_or — побитовое ИЛИ

в системе EME.WMS функция `is_or` выполняет побитовую операцию ИЛИ (OR) над всеми переданными аргументами. Поддерживает произвольное количество параметров. Каждый параметр может быть целым числом, 8-байтовым числом (при условии значения в диапазоне 0..0xFFFFFFFF) или массивом байтов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[N] | Integer / Array | Нет | Операнды для побитового сложения. Целое число, 8-байтовое число в диапазоне 0..0xFFFFFFFF или массив байтов |

**Вызов без аргументов**: возвращает 0 (все 32 бита сброшены в 0).

**Возвращает**: `Integer` — результат побитовой операции ИЛИ всех переданных операндов.

```EME-L
' Объединение двух флагов '
Result = is_or(Flags1, Flags2)

' Объединение нескольких масок '

' Без аргументов - возвращает 0 '

```

### is_xor — побитовое исключающее ИЛИ

в языке EME-L функция `is_xor` выполняет побитовую операцию исключающее ИЛИ (XOR) между двумя аргументами. Первый аргумент может быть целым числом, 8-байтовым числом (при условии значения в диапазоне 0..0xFFFFFFFF) или массивом байтов. Второй аргумент преобразуется в целое число.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer / Array | Да | Первый операнд. Целое число, 8-байтовое число (0..0xFFFFFFFF) или массив байтов |
| args[1] | Integer | Да | Второй операнд (целое число) |

**Возвращает**: `Integer` — результат побитовой операции исключающее ИЛИ двух операндов.

```EME-L
' XOR двух флагов '
Result = is_xor(Flags1, Flags2)

' Переключение битов по маске '
Result = is_xor(Value, 0xFF)
```

### is_not — побитовое НЕ

в EME БД функция `is_not` выполняет побитовую операцию НЕ (NOT) — инвертирует все биты аргумента. Аргумент может быть целым числом, 8-байтовым числом (при условии значения в диапазоне 0..0xFFFFFFFF) или массивом байтов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer / Array | Да | Операнд для побитовой инверсии. Целое число, 8-байтовое число (0..0xFFFFFFFF) или массив байтов |

**Возвращает**: `Integer` — результат побитовой инверсии всех битов операнда.

```EME-L
' Инвертировать все биты '
Result = is_not(Flags)

' Инвертировать 0x00FF - результат 0xFF00 '
Result = is_not(0x00FF)
```

### is_mask — формирование битовой маски

в языке EME-L функция `is_mask` формирует битовую маску, устанавливая в 1 биты с указанными номерами. Принимает произвольное количество аргументов — номеров битов, которые необходимо установить. Нумерация битов начинается с 0 (младший бит).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[N] | Integer | Нет | Номера битов для установки в маске, начиная с 0 |

**Вызов без аргументов**: вызывает ошибку исполнения (требуется минимум один аргумент).

**Возвращает**: `Integer` — сформированная битовая маска с установленными битами по указанным позициям.

```EME-L
' Маска с битами 1, 3 и 5: 0b101010 = 0x2A '
Mask = is_mask(1, 3, 5)

' Маска с одним битом 4: 0b10000 = 0x10 '
Mask = is_mask(4)
```

### is_shift_left — побитовый сдвиг влево

в системе EME.WMS функция `is_shift_left` выполняет побитовый сдвиг значения влево на заданное количество бит. Операнды преобразуются в беззнаковые 32-битные целые числа перед выполнением сдвига, что предотвращает размножение знакового бита.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Исходное значение для сдвига. Преобразуется в беззнаковое 32-битное целое число |
| args[1] | Integer | Да | Количество бит для сдвига влево |

**Возвращает**: `Integer` — результат побитового сдвига влево. При сдвиге на 32 и более бит результат зависит от архитектуры процессора.

```EME-L
' Сдвиг влево на 2 бита: 0b0001 - 0b0100 '
Result = is_shift_left(1, 2)

' Сдвиг байта влево '
Result = is_shift_left(0x01, 8)
```

в языке EME-L функция `is_shift_right` выполняет побитовый сдвиг значения вправо на заданное количество бит. Операнды преобразуются в беззнаковые 32-битные целые числа перед выполнением сдвига, что предотвращает размножение знакового бита при сдвиге вправо.

| Параметр | Тип | Обязательный | Описание |

**Возвращает**: `Integer` — результат побитового сдвига вправо. При сдвиге на 32 и более бит результат зависит от архитектуры процессора.


## is-функции группы Browser — работа с браузерами

is-функции группы Browser отвечают за работу с браузерами в EME БД. в системе EME.WMS браузеры используются для отображения и выбора данных из записей базы данных. Группа включает функции для управления высечением строк, чтения данных браузера и обработки пользовательского ввода.

---

### is_hewed — проверка высечения строки

в языке EME-L функция `is_hewed` проверяет, высечена ли указанная строка в переменной высечения. Если переменная высечения пуста (ни одна строка не выбрана), все строки считаются высеченными.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя переменной высечения |
| args[1] | Integer | Да | Номер строки для проверки |
| args[2] | Integer | Нет | Уровень навигации по объектам. 0 — текущий объект (по умолчанию) |

**Вызов с двумя аргументами**: используется текущий объект (level = 0).

**Возвращает**: `Boolean` — `VARIANT_TRUE`, если строка высечена или переменная пуста; `VARIANT_FALSE`, если строка не высечена или пропущена.

```EME-L
' Проверить, высечена ли строка 5 '
Result = is_hewed("MyHew", 5)

' Проверка с указанием уровня объекта '
Result = is_hewed("MyHew", 5, 1)
```

---

### is_hew_item — высечение строки

в системе EME.WMS функция `is_hew_item` высекает (выбирает) указанную строку в переменной высечения. Если строка уже пропущена, переменная пуста или строка вне диапазона, операция не выполняется.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя переменной высечения |
| args[1] | Integer | Да | Номер строки для высечения |
| args[2] | Integer | Нет | Уровень навигации по объектам. 0 — текущий объект (по умолчанию) |

**Вызов с двумя аргументами**: используется текущий объект (level = 0).

**Возвращает**: `Boolean` — `VARIANT_TRUE`, если строка успешно высечена; `VARIANT_FALSE`, если строка пропущена, переменная пуста или строка вне диапазона.

```EME-L
' Высечь строку 5 в переменной высечения '
Result = is_hew_item("MyHew", 5)
```

### is_empty_hew — проверка пустоты переменной высечения

в языке EME-L функция `is_empty_hew` проверяет, пуста ли переменная высечения (не выбрана ни одна строка). При установке `clear_flag` предварительно очищает переменную.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя переменной высечения |
| args[1] | Integer | Нет | Уровень навигации по объектам. 0 — текущий объект (по умолчанию) |
| args[2] | Boolean | Нет | Флаг очистки переменной. `TRUE` — очистить перед проверкой (по умолчанию `FALSE`) |

**Вызов с одним аргументом**: текущий объект, очистка не производится.

**Возвращает**: `Boolean` — `VARIANT_TRUE`, если в переменной не выбрана ни одна строка или переменная не существует; `VARIANT_FALSE`, если выбрана хотя бы одна строка.

```EME-L
' Проверить, пуста ли переменная '
Result = is_empty_hew("MyHew")

' Очистить переменную и проверить '
Result = is_empty_hew("MyHew", 0, TRUE)
```

### is_browse — запуск браузера

в EME БД функция `is_browse` последовательно запускает указанные браузеры в каскадном режиме. Каждый следующий браузер фильтруется по выбранной строке предыдущего. Если в промежуточном браузере не выбрана ни одна строка, автоматически выбираются все строки.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[N] | String | Нет | Имена браузеров для последовательного запуска |

**Возвращает**: `Integer` — номер строки, выбранной в последнем браузере.

```EME-L
' Запуск одного браузера '
Line = is_browse("BrowserOrders")

' Каскадный запуск двух браузеров '
Line = is_browse("BrowserClients", "BrowserOrders")
```

### is_browse_data — содержимое колонки браузера

в языке EME-L функция `is_browse_data` возвращает содержимое ячейки указанной колонки браузера. Если колонка с указанным именем не найдена, генерирует ошибку.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя колонки браузера |
| args[1] | Integer | Нет | Номер строки в браузере. По умолчанию используется текущая строка |

**Вызов с одним аргументом**: данные читаются из текущей строки браузера.

**Возвращает**: значение ячейки (тип зависит от колонки: String, Integer, Real, DateTime или Reference).

```EME-L
' Прочитать колонку из текущей строки '
Value = is_browse_data("ClientName")

' Прочитать колонку из указанной строки '
Value = is_browse_data("ClientName", Line)
```

### is_browser_base_record — базовая запись браузера

в языке EME-L функция `is_browser_base_record` возвращает номер базовой записи браузера по номеру строки записи шаблонов браузеров.

| Параметр | Тип | Обязательный | Описание |

```EME-L
' Запустить браузер записи 100 '
```

### is_keydown_flag — флаги состояния клавиатуры

в языке EME-L функция `is_keydown_flag` возвращает флаги состояния клавиатуры. В контексте браузера возвращает его внутренний `keydown_flag`. Вне контекста браузера возвращает текущее состояние модификаторов клавиатуры (Ctrl, Alt, Shift).

**Параметры**: нет.

**Возвращает**: `Integer` — битовая маска состояния клавиатуры:
- 1 — нажат Ctrl (`KF_CONTROL_PRESS`)
- 2 — нажат Alt (`KF_ALT_PRESS`)
- 4 — нажат Shift (`KF_SHIFT_PRESS`)

Значения могут комбинироваться (например, 3 — Ctrl+Alt).

```EME-L
' Прочитать флаги клавиатуры '
Flag = is_keydown_flag()

' Проверить нажатие Ctrl (бит 0) '
CtrlPressed = (Flag & 1)

' Проверить комбинацию Ctrl+Shift (биты 0 и 2) '
CtrlShift = (Flag & 5)
```

## is-функции группы Context — работа с контекстом

is-функции группы Context отвечают за работу с текущим контекстом в EME БД (открытые ресурсы: диалоги, отчёты, классы и т.д.).

### is_put_context — формирование контекста

в языке EME-L функция `is_put_context` отвечает за формирование текущего контекста программы. На вход принимает имя контекста и параметр для его запоминания.

```EME-L
is_put_context("MyContext", Value);
```

### is_get_context_as_string — получение контекста как строки

в системе EME.WMS функция `is_get_context_as_string` позволяет получить параметр контекста в формате строки. На вход принимает имя контекста.

```EME-L
Value = is_get_context_as_string("MyContext");
```

### is_get_context_as_ref — получение контекста как числа

в языке EME-L функция `is_get_context_as_ref` позволяет получить параметр контекста в формате целого числа. На вход принимает имя контекста.

```EME-L
Ref = is_get_context_as_ref("MyContext");
```

### is_del_context — удаление контекста

в EME БД функция `is_del_context` удаляет ячейку контекста по его имени. На вход принимает имя контекста, ничего не возвращает.

```EME-L
is_del_context("MyContext");
```

### is_check_context — проверка контекста

в языке EME-L функция `is_check_context` проверяет текущий контекст программы. Поведение зависит от количества параметров:

- **1 параметр** — проверка, является ли контекст диалогом (`"Dialog"`) или запросом (`"Query"`).
- **2 параметра** — дополнительно проверяется имя диалога/запроса.

Возвращает `TRUE` или `FALSE`.

```EME-L
' Проверить, является ли текущий контекст диалогом '
Result = is_check_context("Dialog")

' Проверить конкретный диалог '
Result = is_check_context("Dialog", "Clients.Dialog")
```

### is_get_context — получение текущего контекста

в системе EME.WMS функция `is_get_context` получает текущий контекст программы. Допустимые параметры: `"Dialog"`, `"Query"` или `"Report"`. Возвращает имя ресурса, считанного из текущего контекста. Может вернуть пустую строку, если контекст не является диалогом, запросом или отчетом.

```EME-L
DialogName = is_get_context("Dialog");
QueryName = is_get_context("Query");
ReportName = is_get_context("Report");
```

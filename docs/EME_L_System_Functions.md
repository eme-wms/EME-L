## Сообщения и диалоги

### is_msg — сообщение проекта

возвращает сообщение проекта по имени.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя сообщения проекта |

**Возвращает**: `String` — текст сообщения проекта.

```EME-L

```

### is_message — вывод MessageBox

выводит диалоговое окно MessageBox с кнопками и иконкой. в языке EME-L является не GUI функцией — в Linux выводит в консоль.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Заголовок окна |
| args[1] | String | Да | Текст сообщения |
| args[2] | Integer/String | Нет | Количество кнопок (по умолчанию 2). Строковые значения: "ABORTRETRYIGNORE", "OK", "OKCANCEL", "RETRYCANCEL", "YESNOCANCEL", "YESNO". 4 = "Да"/"Нет", 5 = диалог из outera |
| args[3] | Integer/String | Нет | Тип иконки: 0/"INFORMATION" — 'i', 1/"EXCLAMATION" — восклицание, 2/"QUESTION" — вопрос, 3/"STOP" — стоп |
| args[4] | Integer | Нет | Номер кнопки по умолчанию (0, 1, 2) |
| args[5] | Boolean | Нет | Системный режим — выводит MessageBox в любом потоке, в т.ч. в транзакции. По умолчанию FALSE |
| args[6] | Integer | Нет | Таймер в мс. По умолчанию 0 (бесконечность) |

**Возвращает**: при 2 кнопках — `Boolean` (OK/Yes = TRUE, Cancel/No = FALSE). При 3 кнопках — `Integer` (Yes = -1, No = 0, Cancel = 1). При текстовом args[2] — `String` ("OK", "CANCEL", "YES", "NO" и т.д.).

Если args = 0 или 1: MessageBox без иконки с одной кнопкой OK.

```EME-L
If (is_message("Заголовок", "Текст сообщения"))
    'Нажали OK'
End If

result = is_message("Вопрос", "Продолжить?", 3, 2)
' result: -1=Yes, 0=No, 1=Cancel

result = is_message("Заголовок", "Текст", "YESNO", "QUESTION")
' result: "YES" или "NO"
```

### is_message_beep — звуковой сигнал

выводит звук в динамики компьютера.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer/String | Да | Тип звука. Строка: "ICONASTERISK", "ICONEXCLAMATION", "ICONERROR", "ICONHAND", "ICONINFORMATION", "ICONQUESTION", "ICONSTOP", "OK". Число: 0-7 (по порядку) |


### is_timer_message — MessageBox с таймером

выводит строку в TimerMessageBox. Параметры аналогичны `is_message`, но args[6] — TimeOut в миллисекундах (по умолчанию 10000).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0..5] | — | — | См. is_message |
| args[6] | Integer | Нет | TimeOut в мс (по умолчанию 10000) |

**Возвращает**: см. is_message.

```EME-L
result = is_timer_message("Заголовок", "Автозакрытие через 5с", 2, 0, 0, FALSE, 5000)
```


## Системные настройки

### is_system — системная настройка

возвращает значение системной настройки по индексу.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Индекс системной настройки |

Индексы:
- 0 — Ссылка на текущего оператора
- 1 — Флаги доступа текущего оператора
- 2 — Идентификатор (имя) текущего оператора
- 3 — Код запуска модуля
- 4 — Код текущего филиала
- 5 — Ссылка на текущий филиал
- 6 — Код проекта
- 7 — is_workstation (TRUE/FALSE)
- 8 — Мозговой компьютер (TRUE/FALSE)

**Возвращает**: значение настройки (тип зависит от индекса).

```EME-L
ref = is_system(0)      'ссылка на оператора
flags = is_system(1)    'флаги доступа
name = is_system(2)     'имя оператора
code = is_system(3)     'код запуска
fcode = is_system(4)    'код филиала
```

### is_sys_param — параметр настройки

возвращает значение настройки по имени. Настройки рабочей станции начинаются с "W://", настройки модулей — с "M://".

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя настройки (W://Имя или M://Имя) |
| args[1] | Any | Нет | Тип (определяется автоматически: строка, целое, вещественное). По умолчанию — строка |
| args[2] | Integer | Нет | Режим: 0 (по умолчанию) — EXCEPT_MODE, 1 — SOFT_MODE |

**Возвращает**: значение настройки в соответствии с типом args[1].

```EME-L
val = is_sys_param("M://MySetting", "")
val = is_sys_param("W://WsSetting", 0)
val = is_sys_param("M://SoftSetting", "", 1)
```

## Права доступа и пользователи

### is_administrator — проверка администратора

проверяет, является ли текущий оператор администратором.

**Параметры**: нет.

**Возвращает**: `Boolean` — TRUE, если оператор — администратор.

```EME-L
If (is_administrator())
    'Администратор'
End If
```

---

### is_access — проверка прав доступа

проверяет наличие прав доступа у текущего оператора.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер бита в 32-битном поле доступа |

**Возвращает**: `Boolean` — TRUE, если право есть.

```EME-L
If (is_access(5))
    'Право доступа 5 есть'
End If
```

---

### is_password — диалог ввода пароля

запускает диалог идентификации оператора по паролю.

**Параметры**: нет.

**Возвращает**: `Boolean` — TRUE при успешной идентификации.

```EME-L
If (is_password())
    'Пароль верный'
End If
```

---

### is_change_password — смена пароля

запускает диалог смены пароля.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Нет | Номер строки пользователя (по умолчанию NULL_REF — текущий) |
| args[1] | Boolean | Нет | Требовать ли старый пароль (по умолчанию TRUE) |

**Возвращает**: ничего.

```EME-L
is_change_password()
is_change_password(user_ref, TRUE)
```

---

### is_set_user_read_only — режим только чтение

устанавливает флаг "только чтение" в памяти у текущего пользователя (бит 1 в поле "32 бита допусков"). В БД не меняет. Запись в БД запрещена.

**Параметры**: нет.

**Возвращает**: ничего.

```EME-L
is_set_user_read_only()
```

---

### is_clr_user_read_only — снять режим только чтение

снимает флаг "только чтение" в памяти у текущего пользователя. В БД не меняет. Запись в БД разрешена.

**Параметры**: нет.

**Возвращает**: ничего.

```EME-L
is_clr_user_read_only()
```

---

### is_disconnect_user — отключение пользователя

отправляет на все рабочие станции команду отключить пользователя.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на пользователя в записи "Список пользователей" |

**Возвращает**: пустое значение.

```EME-L
is_disconnect_user(user_ref)
```

---

### is_impersonate_user — установка пользователя

позволяет в неинтерфейсном потоке задать текущего пользователя. Используется для ЦУП.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на пользователя |

**Возвращает**: пустое значение.

## Цвета

### is_RGB — цвет из RGB

генерирует цвет из трёх значений RGB.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Красный (0-255) |
| args[1] | Integer | Да | Зелёный (0-255) |
| args[2] | Integer | Да | Синий (0-255) |

**Возвращает**: `Integer` — цвет (COLORREF).

```EME-L
color = is_RGB(255, 128, 0)
```

---

### is_RGBA — цвет из RGBA

возвращает RGBA-цвет. Поддерживает два режима вызова.

**Режим 1** — отдельные каналы:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Красный (0-255) |
| args[1] | Integer | Да | Зелёный (0-255) |
| args[2] | Integer | Да | Синий (0-255) |
| args[3] | Integer | Нет | Прозрачность (по умолчанию 0xFF — непрозрачный) |

**Режим 2** — из RGB-цвета:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | RGB-цвет |
| args[1] | Integer | Нет | Прозрачность (по умолчанию 0x00 — прозрачный) |

**Возвращает**: `Integer` — RGBA-цвет.

```EME-L
color = is_RGBA(255, 0, 0, 128)   'полупрозрачный красный
color = is_RGBA(rgb_color, 200)    'из RGB с прозрачностью
```

---

### is_blendRGB — смешивание цветов

смешивает два цвета с заданным коэффициентом.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Первый цвет |
| args[1] | Integer | Да | Второй цвет |
| args[2] | Real | Нет | Коэффициент смешивания (0.0-1.0, по умолчанию 0.5) |

**Возвращает**: `Integer` — результирующий цвет (RGB).

```EME-L
mixed = is_blendRGB(color1, color2, 0.7)
mixed = is_blendRGB(color1, color2)  '50/50
```

---

### is_sys_color — системный цвет Windows

возвращает текущий цвет отображаемого элемента Windows.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Название элемента: "3DDKSHADOW", "3DFACE", "BTNFACE", "ACTIVEBORDER", "ACTIVECAPTION", "HIGHLIGHT", "WINDOW", "WINDOWTEXT" и др. |

**Возвращает**: `Integer` — цвет (COLORREF).

### is_legend_color — цвет из палитры легенды

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Номер элемента палитры (0-21) |

**Возвращает**: `Integer` — цвет (COLORREF).

```EME-L

```

## Транзакции сервера

### is_server_transaction_dump — дамп транзакций

на сервере делает дамп транзакций в объект запроса. На рабочей станции подготавливает запрос.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Object(Query) | Да | Пустой запрос для приёма дампа |
| args[1] | Any | Нет | Имя/номер записи (NULL_RECORD — все) |
| args[2] | Any | Нет | Имя/номер поля (NULL_FIELD — все) |
| args[3] | Any | Нет | Номер строки (NULL_REF — все) |
| args[4] | Integer | Нет | Номер транзакции с (0 — все) |
| args[5] | Integer | Нет | Номер транзакции по |
| args[6] | String | Нет | Данные для поиска |
| args[7] | DateTime | Нет | Дата начала |
| args[8] | DateTime | Нет | Дата окончания |
| args[9] | DateTime | Нет | Время начала |
| args[10] | DateTime | Нет | Время окончания |
| args[11] | Boolean | Нет | Не показывать служебную информацию |
| args[12] | Boolean | Нет | История поиска транзакций |

**Возвращает**: `Boolean` — TRUE при успехе.

```EME-L
q = Object("Query")
result = is_server_transaction_dump(q, NULL_RECORD, NULL_FIELD, NULL_REF)
```

---

**Возвращает**: `String` — данные команды или пустое значение.

```EME-L
data = is_server_transaction_data(trans_no, cmd_no)
```

### is_server_first_transaction — границы транзакций

на сервере возвращает номера первой и последней транзакции в хранилище.

## Внешние программы

### is_shell_execute — запуск внешней программы

аналог ShellExecute в Windows. Запускает внешнюю программу или открывает документ.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Действие: "open", "explore", "print" и др. |
| args[1] | String | Да | Рабочий каталог |
| args[2] | String | Да | Имя приложения, документа или файла |
| args[3] | String | Нет | Параметры запуска |
| args[4] | Integer | Нет | Таймаут ожидания в мс (0 или без — не ждать) |

**Возвращает**: `Integer` — код результата. При ожидании: код выхода процесса. При ошибках: -1 (ShellExecuteEx), -2 (WaitFailed), -3 (Timeout), -4 (другое).

```EME-L

is_shell_execute("open", "", "document.pdf", "", 10000)
```

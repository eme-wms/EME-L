## Сообщения и диалоги

### is_msg — текст сообщения проекта

Возвращает текст сообщения проекта по его имени из словаря сообщений. Выполняет поиск через GetProjectMessage.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя сообщения проекта для поиска в словаре |

**Возвращает**: `String` — текст найденного сообщения проекта. Если сообщение не найдено, возвращает пустую строку.

```EME-L
text = is_msg("MessageName")
```

### is_message — вывод MessageBox

Выводит модальное диалоговое окно MessageBox с заданным заголовком, текстом, набором кнопок, иконкой и опциональным таймаутом. Поддерживает как числовые, так и строковые значения для кнопок и иконок. В системном режиме (systemMode = TRUE) диалог выводится в любом потоке, включая транзакции (в этом случае таймаут должен быть > 0).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Нет | Заголовок окна. При отсутствии — пустой заголовок |
| args[1] | String | Нет | Текст сообщения. При отсутствии — пустое сообщение |
| args[2] | Integer/String | Нет | Набор кнопок (по умолчанию 2 / OKCANCEL). Числа: 1 = OK, 2 = OKCANCEL, 3 = YESNOCANCEL, 4 = YESNO, 5 = YES_NO_YESTOALL_NOTOALL_CANCEL. Строки: "OK", "OKCANCEL", "YESNOCANCEL", "YESNO", "ABORTRETRYIGNORE", "RETRYCANCEL" |
| args[3] | Integer/String | Нет | Тип иконки (по умолчанию INFORMATION). Числа: 0 = INFORMATION, 1 = EXCLAMATION, 2 = QUESTION, 3 = STOP. Строки: "INFORMATION", "EXCLAMATION", "QUESTION", "STOP" |
| args[4] | Integer | Нет | Номер кнопки по умолчанию (по умолчанию 0): 0 = первая кнопка, 1 = вторая кнопка, 2 = третья кнопка |
| args[5] | Boolean | Нет | Системный режим (по умолчанию FALSE). Если TRUE, диалог выводится в любом потоке и в транзакции |
| args[6] | Integer | Нет | Таймаут в миллисекундах (по умолчанию 0 — бесконечность). При истечении таймаута возвращается значение кнопки по умолчанию |

**Возвращает**: при текстовом args[2] — `String` с именем нажатой кнопки ("OK", "CANCEL", "ABORT", "RETRY", "IGNORE", "YES", "NO", "CLOSE", "HELP"). При YESNOCANCEL (3) и числовом args[2] — `Integer` (Yes = -1, No = 0, Cancel = 1). При YES_NO_YESTOALL_NOTOALL_CANCEL (5) — `String` ("YES", "NO", "YESTOALL", "NOTOALL", "CANCEL"). В остальных случаях — `Boolean` (OK/Yes = TRUE, Cancel/No = FALSE).

```EME-L
' Показать MessageBox с заголовком и текстом, TRUE — нажали OK '
result = is_message("Заголовок", "Текст сообщения")

' Диалог «Да/Нет/Отмена» с иконкой вопроса, YES=-1, NO=0, Cancel=1 '
result = is_message("Вопрос", "Продолжить?", 3, 2)

' Диалог «Да/Нет» с иконкой вопроса, строковые параметры, результат «YES» или «NO» '
result = is_message("Заголовок", "Текст", "YESNO", "QUESTION")
```

### is_message_beep — звуковой сигнал

Воспроизводит системный звуковой сигнал через API MessageBeep. Поддерживает как числовые, так и строковые идентификаторы типа звука.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer/String | Да | Тип звукового сигнала. Числа: 0 = ICONASTERISK, 1 = ICONEXCLAMATION, 2 = ICONERROR, 3 = ICONHAND, 4 = ICONINFORMATION, 5 = ICONQUESTION, 6 = ICONSTOP, 7 = OK. Строки: "ICONASTERISK", "ICONEXCLAMATION", "ICONERROR", "ICONHAND", "ICONINFORMATION", "ICONQUESTION", "ICONSTOP", "OK" |

**Возвращает**: `Integer` — результат вызова MessageBeep (ненулевое при успехе).

```EME-L
is_message_beep("ICONEXCLAMATION")
is_message_beep(0)
```

### is_timer_message — MessageBox с автозакрытием

Выводит модальное диалоговое окно MessageBox с заданным заголовком, текстом, набором кнопок, иконкой и автоматическим закрытием по таймауту. Поддерживает как числовые, так и строковые значения для кнопок и иконок. В системном режиме (systemMode = TRUE) диалог выводится в любом потоке, включая транзакции (в этом случае таймаут должен быть > 0). При истечении таймаута окно закрывается автоматически и возвращается значение кнопки по умолчанию.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Нет | Заголовок окна. При отсутствии — пустой заголовок |
| args[1] | String | Нет | Текст сообщения. При отсутствии — пустое сообщение |
| args[2] | Integer/String | Нет | Набор кнопок (по умолчанию 2 / OKCANCEL). Числа: 1 = OK, 2 = OKCANCEL, 3 = YESNOCANCEL, 4 = YESNO, 5 = YES_NO_YESTOALL_NOTOALL_CANCEL. Строки: "OK", "OKCANCEL", "YESNOCANCEL", "YESNO", "ABORTRETRYIGNORE", "RETRYCANCEL" |
| args[3] | Integer/String | Нет | Тип иконки (по умолчанию INFORMATION). Числа: 0 = INFORMATION, 1 = EXCLAMATION, 2 = QUESTION, 3 = STOP. Строки: "INFORMATION", "EXCLAMATION", "QUESTION", "STOP" |
| args[4] | Integer | Нет | Номер кнопки по умолчанию (по умолчанию 0): 0 = первая кнопка, 1 = вторая кнопка, 2 = третья кнопка |
| args[5] | Boolean | Нет | Системный режим (по умолчанию FALSE). Если TRUE, диалог выводится в любом потоке и в транзакции |
| args[6] | Integer | Нет | Таймаут в миллисекундах (по умолчанию 10000). При значении 0 — бесконечность (без автозакрытия) |

**Возвращает**: при текстовом args[2] — `String` с именем нажатой кнопки ("OK", "CANCEL", "ABORT", "RETRY", "IGNORE", "YES", "NO", "CLOSE", "HELP"). При YESNOCANCEL (3) и числовом args[2] — `Integer` (Yes = -1, No = 0, Cancel = 1). При YES_NO_YESTOALL_NOTOALL_CANCEL (5) — `String` ("YES", "NO", "YESTOALL", "NOTOALL", "CANCEL"). В остальных случаях — `Boolean` (OK/Yes = TRUE, Cancel/No = FALSE). При срабатывании таймаута возвращается значение кнопки по умолчанию (args[4]).

```EME-L
result = is_timer_message("Заголовок", "Автозакрытие через 5с", 2, 0, 0, FALSE, 5000)
```

### is_enter_string — ввод строки через диалог

GUI-функция. Выводит диалоговое окно для ввода строки текста и возвращает введённую строку. Если пользователь нажимает клавишу ESC, возвращается пустая строка.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Заголовок диалогового окна |
| args[1] | String | Нет | Начальный текст в поле ввода. По умолчанию пустая строка |

**Возвращает**: `String` — введённый пользователем текст. Пустая строка при нажатии ESC.

```EME-L
' Запросить имя пользователя через диалог '
name = is_enter_string("Введите имя", "Иванов")
' name = "Иванов" если пользователь подтвердил, или "" при отмене '
```

### is_dlg_var — значение переменной диалога

GUI-функция. Возвращает значение переменной диалога. В веб-контексте значение читается из прокси-объекта веб-диалога, в стандартном контексте — из текущего или указанного диалога в иерархии. При отсутствии переменной поведение определяется режимом контроля.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя переменной диалога |
| args[1] | Integer | Нет | Уровень диалога в иерархии. По умолчанию 0 (текущий диалог) |
| args[2] | Integer | Нет | Режим контроля при отсутствии переменной: 0 — генерировать ошибку (по умолчанию), 1 — вернуть пустое значение |

**Возвращает**: тип зависит от типа переменной — `String`, `Integer`, `Date`, `Time`, `Float` или `Boolean`.

```EME-L
' Получить значение переменной MyVar из текущего диалога '
val = is_dlg_var("MyVar")

' Получить переменную из родительского диалога, без ошибки если не найдена '
val = is_dlg_var("ParentVar", 1, 1)
```

| args[1] | Integer | Нет | Базовая строка диалога. По умолчанию NEW_LINE_MARK |

**Возвращает**: `Integer` — код возврата модального диалога.

```EME-L
' Открыть диалог настроек модально '
result = is_dialog("Settings.Dialog")
```

### is_parent_dlg_rec — запись родительского диалога

### is_if — условное выражение

Возвращает значение arg1, если условие arg0 истинно, иначе возвращает значение arg2. Аналог тернарного оператора в других языках программирования.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Boolean | Да | Логическое условие для проверки |
| args[1] | Variant | Да | Значение, возвращаемое при истинном условии |
| args[2] | Variant | Да | Значение, возвращаемое при ложном условии |

**Возвращает**: `Variant` — значение arg1 или arg2 в зависимости от условия.

```EME-L
' Выбрать максимальное из двух чисел '
maxVal = is_if(a > b, a, b)
' maxVal = a если a > b, иначе b '

' Условное присваивание строки '
status = is_if(quantity > 0, "В наличии", "Нет в наличии")
```

### is_exec_if — выполнение условного выражения

Выполняет и возвращает результат выражения EME-L arg1, если условие arg0 истинно, иначе выполняет и возвращает результат выражения arg2. Выражения передаются в виде текстовых строк и компилируются при выполнении.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Boolean | Да | Логическое условие для проверки |
| args[1] | String | Да | Емельное выражение для выполнения при истинном условии |
| args[2] | String | Да | Емельное выражение для выполнения при ложном условии |

**Возвращает**: `Variant` — результат выполнения выбранного выражения EME-L.

```EME-L
' Выполнить разные вычисления в зависимости от условия '
result = is_exec_if(x > 0, "x * 2", "x + 10")
' result = x * 2 если x > 0, иначе x + 10 '

' Условный вызов функции через строку '
val = is_exec_if(mode == 1, "is_calc_total()", "is_calc_average()")
```

Выполняет выражение EME-L с возможностью передачи параметров, доступных через is_arg(). При ошибке компиляции или выполнения выражения генерирует исключение.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Емельное выражение для выполнения |
| args[1] | Variant | Нет | Первый параметр выражения, доступный как is_arg(0) |
| ... | Variant | Нет | Последующие параметры выражения |

**Возвращает**: `Variant` — результат выполнения выражения EME-L.

```EME-L
' Выполнить простое выражение '
result = is_exec("2 + 3")
' result = 5 '

' Выполнить выражение с параметрами '
result = is_exec("is_arg(0) + is_arg(1)", 10, 20)
' result = 30 '

' Выполнить вызов метода через строку '
result = is_exec("is_this().Calculate()")
```

Возвращает системную настройку по указанному индексу.

```EME-L
fref = is_system(5)
```

### is_sys_param — параметр настройки

Возвращает значение системной настройки по имени. Имя настройки должно начинаться с префикса `W://` (настройки рабочей станции) или `M://` (настройки модулей).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя настройки с префиксом W:// или M:// |

### is_RGBA — цвет из RGBA

формирует цветовое значение RGBA с альфа-каналом (прозрачностью). Поддерживает два режима вызова. Вызов qRgba.

**Режим 1** — отдельные каналы (4 аргумента):

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|

| args[2] | Integer | Да | Интенсивность синего канала (0-255) |
| args[3] | Integer | Нет | Альфа-канал (0-255, по умолчанию 0x00 — полностью прозрачный) |

**Режим 2** — из готового RGB-цвета (2 аргумента):

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Готовый RGB-цвет (COLORREF) |
| args[1] | Integer | Нет | Альфа-канал (0-255, по умолчанию 0x00 — полностью прозрачный) |

**Возвращает**: `Integer` — цветовое значение RGBA.

```EME-L
' полупрозрачный красный из каналов '

color = is_RGBA(rgb_color, 200)
```

### is_shell_execute — запуск внешней программы

Запускает внешнее приложение, открывает документ или протокол через ShellExecute/ShellExecuteEx. Если указан таймаут, дожидается завершения процесса и возвращает его exit code. В режиме службы (NT Service) запуск заблокирован.

## Архивация БД

### is_archive_DB — архивация базы данных

выполняет архивацию базы данных: помечает устаревшие строки записей к удалению и создаёт архивный экспорт обрезанной копии базы. в языке EME-L функция должна вызываться в монопольном (MONO) режиме на мозговом компьютере вне транзакции. Если первый параметр пустой, пометка строк к архивации не выполняется — только архивный экспорт ранее помеченных строк. Вызов без аргументов или с одним аргументом недопустим, вызывает ошибку «Слишком мало параметров».

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата, до которой удалять документы и операции |
| args[1] | Date | Да | Срок годности (BBD), до которого удалять документы |
| args[2] | Date | Нет | Значение некорректного BBD, которое нужно игнорировать при фильтрации. По умолчанию максимально возможная дата (0x7FFF год) |
| args[3] | String | Нет | Директория для сохранения файлов архивного экспорта. По умолчанию используется стандартная директория архивного раздела |

**Возвращает**: пустое значение.

```EME-L
' Архивация: удалить документы до 01.01.2024, BBD до 01.01.2023 '
is_archive_DB(Date(2024, 1, 1), Date(2023, 1, 1))

' С указанием некорректного BBD и пути экспорта '
is_archive_DB(Date(2024, 1, 1), Date(2023, 1, 1), Date(2099, 12, 31), "C:\Archive\export")
```

## Удаление документов

### is_delete_documents — удаление закрытых документов

удаляет все закрытые документы WMS, строки документов, пачки заказов, заказы, партии, ГТД и связанные подчинённые записи, у которых дата закрытия меньше или равна указанной дате. в языке EME-L функция работает только в MONO-режиме на мозговом компьютере. Перед удалением отключает отложенное обнуление ссылок, выполняет удаление внутри серверной транзакции, а затем восстанавливает отложенное обнуление ссылок. После удаления выполняет архивную выгрузку базы данных.

**Ограничения:**
- Функция доступна только в MONO-режиме работы станции. В противном случае генерируется ошибка «Архивация БД возможна только в МОНО режиме»
- Функция доступна только на мозговом компьютере (canal brain computer). В противном случае генерируется ошибка «Архивация БД возможна только на мозговом компьютере»

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Date | Да | Дата, до которой (включительно) удаляются закрытые документы и связанные объекты |
| args[1] | String | Нет | Путь для сохранения архивной выгрузки базы данных после удаления. Если не указан или пустая строка, используется стандартный путь архивной выгрузки |

**Возвращает**: пустое значение.

```EME-L
' Удалить все документы, закрытые до 01.01.2024 '
is_delete_documents(Date(2024, 1, 1))

' Удалить с указанием пути архивной выгрузки '
is_delete_documents(Date(2024, 1, 1), "C:\Archive\export")
```

### is_EANCOM_segment — извлечение сегмента EANCOM

извлекает страницы из PDF-файла и сохраняет их в виде изображений в формате JPEG в указанный контейнер данных. в системе EME.WMS поддерживаемые типы контейнеров: CIArchive, CICollection, CIArray и CIDataStorage. При использовании CIDataStorage сохраняется только одна страница.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Имя исходного PDF-файла |
| args[1] | Reference | Да | Контейнер для сохранения изображений: CIArchive (Store), CICollection (элементы по номеру страницы), CIArray (объекты CIDataStorage), CIDataStorage (одна страница) |
| args[2] | Integer | Нет | Номер начальной страницы. 0 — с первой (по умолчанию) |
| args[3] | Integer | Нет | Номер последней страницы. 0 — до последней (по умолчанию) |
| args[4] | Integer | Нет | Разрешение в DPI. По умолчанию 300 |
| args[5] | Integer | Нет | Коэффициент сглаживания: 1 — минимальное, 2 — среднее, 4 — максимальное (по умолчанию) |

**Возвращает**: `Integer` — количество извлечённых страниц при успехе. Отрицательное значение при ошибке Ghostscript.

```EME-L
' Извлечь все страницы в CIArchive '
Dim archive = New CIArchive()
count = is_get_pdfimages("C:\Docs\report.pdf", archive)
' count = количество извлечённых страниц '

' Извлечь страницы 1-3 с разрешением 150 DPI '
count = is_get_pdfimages("C:\Docs\report.pdf", archive, 0, 2, 150)
```

## Мониторинг заказов (табло)

### is_order_monitor — информация о заказе для табло

возвращает JSON-строку с детальной информацией об одном заказе для табло мониторинга. Формирует объект с полями: идентификатор заказа, дата расчёта, клиент, адрес доставки, общее количество, собранное количество, грузчик, статус, упакованное количество и количество заказов. Учитывает только просчитанные заказы (статус >= 200), исключая отклонённые.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Integer | Да | Ссылка на заказ в базе данных. Должен быть валидным идентификатором существующего заказа |
| args[1] | Boolean | Нет | Признак пересчёта количества в литры/вес. `TRUE` — пересчитывать с учётом веса нетто товара. `FALSE` — базовое количество в единицах хранения (по умолчанию) |

**Возвращает**: `String` — JSON-объект с полями `{"id", "calcDate", "customer", "address", "total", "collected", "loaderMan", "status", "qtyPacked", "TotalOrders"}`. Пустая строка при невалидном заказе, отклонённом заказе или статусе < 200.

```EME-L
json = is_order_monitor(orderRef)
' json = {"id":123,"calcDate":"2024-01-15","customer":"ООО Вектор",...} '

' Получить JSON заказа с пересчётом в литры '
json = is_order_monitor(orderRef, TRUE)
```

### is_all_orders_monitor — статус всех заказов для табло

возвращает JSON-массив со статусами всех активных заказов для табло мониторинга. в начале массива добавляется объект с версией метапроекта (major.minor). Включает только просчитанные заказы (статус >= 200), исключая отклонённые, которые ещё не покинули склад. При указании склада фильтрует заказы по конкретному складу.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Boolean | Да | Признак пересчёта количества в литры/вес. `TRUE` — пересчитывать с учётом веса нетто товара. `FALSE` — базовое количество в единицах хранения |
| args[1] | Integer | Нет | Ссылка на склад для фильтрации. 0 или `NULL_REF` — все склады (по умолчанию). Ненулевое значение — только заказы указанного склада |

**Возвращает**: `String` — JSON-массив объектов `[{"version":"X.Y"}, {...заказ1...}, {...заказ2...}]`. `"[]"` при отсутствии заказов.

```EME-L
' Все заказы в единицах хранения '
json = is_all_orders_monitor(FALSE)

' Заказы конкретного склада в литрах '
json = is_all_orders_monitor(TRUE, warehouseRef)
```

## Производительность и питание

### is_calc_speed — измерение производительности компьютера

измеряет производительность компьютера путём выполнения тестового набора арифметических операций на языке EME-L и сравнения фактического времени выполнения с эталонным значением. При необходимости использует все доступные ядра процессора для параллельного измерения.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | Boolean | Да | Признак использования всех ядер процессора. `TRUE` — параллельное измерение на всех ядрах, результат усредняется. `FALSE` — измерение на одном ядре |

**Возвращает**: `Integer` — производительность в процентах от рекомендованной. 100% соответствует эталону (время выполнения теста 11.5 секунд). Больше 100% — выше эталона, меньше — ниже.

```EME-L
' Замер на одном ядре '
speed = is_calc_speed(FALSE)
' speed = 150 — компьютер в 1.5 раза быстрее эталона '

' Замер на всех ядрах (усреднённый результат) '
speed = is_calc_speed(TRUE)
```

## Интернет и коммуникации

### is_httptofile — HTTP-запрос с записью в файл

выполняет HTTP-запрос к указанному серверу и записывает ответ сервера в заданный файл. в системе EME.WMS используется для загрузки данных с внешних HTTP-серверов и сохранения результата на диск.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Адрес HTTP-сервера (IP или доменное имя) |
| args[1] | Integer | Да | Порт прослушивания сервера |
| args[2] | String | Да | Ресурс на HTTP-сервере (URL) |
| args[3] | Integer | Да | Тайм-аут ожидания отклика в секундах. `0` или пустое значение — тайм-аут по умолчанию, задаваемый ядром |
| args[4] | String | Да | Полное имя файла для записи ответа сервера. Если пустое — генерируется ошибка |
| args[5+] | String | Нет | Дополнительные параметры запроса в формате `"имя=значение"`. Можно указать произвольное количество |

**Возвращает**: `String` — результат выполнения запроса:
- `"Ok"` — запрос выполнен успешно, данные записаны в файл
- `"Ошибка при обращении к HTTP серверу"` — общая ошибка HTTP-запроса
- `"Не найден HTTP сервер"` — сервер не найден
- `"На HTTP сервере не найдена запрашиваемая страница"` — запрашиваемый URL не найден
- `"Истек timeout при обращении к HTTP-серверу"` — превышен тайм-аут ожидания ответа
- `<строка_перенаправления>` — адрес перенаправления (при редиректе)
- `"Неизвестная ошибка при обращении к HTTP серверу"` — неизвестная ошибка

```EME-L
' Выполнить HTTP-запрос и сохранить ответ в файл '
result = is_httptofile("192.168.1.10", 8080, "/api/data", 30, "C:\\Temp\\response.xml")
' result = "Ok" при успехе '

' Запрос с дополнительными параметрами '
result = is_httptofile("api.example.com", 80, "/export", 60, "C:\\Data\\export.csv", "format=csv", "date=today")
```

### is_email_message — асинхронная отправка email

отправляет электронное письмо. Отправка всегда выполняется асинхронно (в отдельном потоке), независимо от значения параметра "игнорируется". в системе EME.WMS используется для уведомлений и автоматической рассылки отчётов.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Да | Адрес электронной почты отправителя |
| args[1] | String | Нет | Отображаемое имя отправителя. Если не задано или пустое — `"Без имени"` |
| args[2] | String | Да | Адрес электронной почты получателя. Если пустой — отправка не выполняется |
| args[3] | String | Нет | Отображаемое имя получателя. Если не задано — используется значение `args[2]` |
| args[4] | String | Нет | Тема письма. Если не задана или пустая — `"Без темы"` |
| args[5] | String | Да | Текстовое содержимое письма |
| args[6] | Boolean | Нет | Признак формата текста: `FALSE` — простой текст (по умолчанию), `TRUE` — HTML-формат |
| args[7] | String | Нет | Путь к прикреплённому файлу или несколько путей, разделённых символом `"|"`. По умолчанию пустая строка (вложения отсутствуют) |
| args[8] | Boolean | Нет | Параметр зарезервирован и игнорируется. Принудительно устанавливается в `1` (асинхронный режим) |
| args[9] | Boolean | Нет | Разрешить отправку в любом режиме работы: `FALSE` — только в сетевом режиме (по умолчанию), `TRUE` — отправка выполняется всегда |

**Возвращает**: `Integer` — `1` — отправка успешно поставлена в очередь (асинхронный поток запущен); `0` — отправка не выполнена (адрес получателя пуст или ошибка выделения памяти).

```EME-L
' Отправить простое письмо '
result = is_email_message("sender@example.com", "Система", "admin@example.com", "Администратор", "Отчёт", "Текст отчёта")
' result = 1 при успешной постановке в очередь '

' Отправить HTML-письмо с вложением '
result = is_email_message("noreply@wms.local", "EME.WMS", "user@company.ru", "", "Уведомление", "<h1>Важно</h1>", TRUE, "C:\\Reports\\daily.pdf")
```

**Возвращает**: `Integer` — при асинхронной отправке: `1` — задача поставлена в очередь, `0` — отправка не выполнена; при синхронной отправке: `1` — письмо успешно отправлено, `0` — отправка не удалась.

```EME-L
' Синхронная отправка для проверки соединения '
result = is_test_email_message("test@local", "", "admin@local", "", "Тест", "Проверка связи", FALSE, "", FALSE)
' result = 1 при успешной отправке '

' Асинхронная отправка с HTML '
result = is_test_email_message("system@wms", "EME.WMS", "operator@company.ru", "", "Статус", "<b>Готово</b>", TRUE, "", TRUE)
```

### is_sms_message — отправка SMS

отправляет SMS-сообщение через сервер сообщений EME. в системе EME.WMS используется для экстренных уведомлений операторам и администраторам через SMS-канал.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|

| args[2] | String | Нет | Адрес(а) получателя, разделённые запятой. Можно указать до 100 номеров за один запрос. По умолчанию пустая строка — выбор получателя осуществляется сервером сообщений |

**Возвращает**: `Boolean` — `TRUE` — сообщение успешно добавлено в очередь сервера сообщений; `FALSE` — не удалось добавить сообщение в очередь.

### is_server_job — проверка сервера заданий

**Параметры**: нет.

```EME-L
If (is_server_job())
    ' Логика сервера заданий '
    is_process_background_task()
Else
    ' Логика клиента '
    is_show_status("Ожидание обработки...")
End If
```

### is_syslog_msg — отправка диагностики в формате SYSLOG

отправляет диагностическое сообщение в формате SYSLOG на сервер ArcSight (аналогично записи в историю операций). в системе EME.WMS используется для централизованного сбора диагностики и мониторинга безопасности.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| args[0] | String | Нет | Название продукта. Если пусто — `"EME.WMS"` |
| args[1] | String | Нет | Версия продукта. Если пусто — `"5.0"` |
| args[2] | Integer | Да | Номер тома истории операций |
| args[3] | String | Да | Описание тома истории операций |
| args[4] | Integer | Да | Уровень важности сообщения: `0` — Low (низкий), `1` — Medium (средний), `2` — High (высокий), `3` — VeryHigh (очень высокий) |
| args[5] | String | Да | Текст диагностического сообщения |

**Возвращает**: `Boolean` — `TRUE` — диагностическое сообщение успешно отправлено; `FALSE` — отправка не выполнена.

```EME-L
' Отправить диагностику среднего уровня '
result = is_syslog_msg("EME.WMS", "5.0", 1, "История операций", 1, "Пользователь выполнил вход в систему")
' result = TRUE при успешной отправке '

' Отправка критического сообщения '
result = is_syslog_msg("", "", 2, "Безопасность", 3, "Обнаружена попытка несанкционированного доступа")
```

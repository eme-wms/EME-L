# Класс Reconciler

Класс `Reconciler` в языке EME-L — средство сверки данных между двумя запросами: плановым и фактическим. В системе EME.WMS и связанных решениях он используется для сопоставления ожидаемых и фактических строк, например при сверке маркированных товаров или складских остатков.

В языке EME-L класс `Reconciler` предоставляет механизм сопоставления строк запросов по одному или нескольким полям связи, расчёта сверенного количества и записи результатов сверки обратно в исходные запросы. Для работы в запросах автоматически создаются служебные поля `LinkRef` (перекрёстная ссылка), `ReconcilingQty` (сверяемое количество) и `ReconciledQty` (сверенное количество).

Основной сценарий использования в EME-L: создание объекта, установка планового запроса методом `SetPlanQuery`, установка фактического запроса методом `SetFactQuery`, разметка заголовков обоих запросов методом `MarkupQueryHeader`, настройка связи по общему полю методом `Join`, выполнение сверки методом `Reconcile`. При необходимости обработки оставшихся непривязанных строк факта выполняется сверка излишков методом `ReconcileSurplus`. Метод `UnjoinAll` сбрасывает список полей связи и вызывается автоматически при уничтожении объекта. Метод `GetVersion` возвращает номер версии реализованного алгоритма сверки.

## Создание объекта

В языке EME-L объект класса `Reconciler` создаётся через оператор `Object()`.

```EME-L
'Создает объект средства сверки данных с пустыми настройками связи'
objReconciler = Object("Reconciler");
```

Класс `Reconciler` имеет один конструктор без аргументов. Он создаёт объект с пустым списком полей связи между плановым и фактическим запросами.

## Настройка и управление запросами

В языке EME-L методы этой группы подготавливают запросы и задают правила сопоставления строк.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetPlanQuery(arg0)` | arg0: Query (по ссылке) | Empty | Задаёт запрос с данными по плану. Должен быть вызван до `Join` и `Reconcile`. |
| `SetFactQuery(arg0)` | arg0: Query (по ссылке) | Empty | Задаёт запрос с данными по факту. Должен быть вызван до `Join` и `Reconcile`. |
| `MarkupQueryHeader(arg0)` | arg0: Query (по ссылке) | Empty | Размечает заголовок запроса: добавляет служебные поля `LinkRef` (REF), `ReconcilingQty` (FLOAT) и `ReconciledQty` (FLOAT). |
| `Join(arg0)` | arg0: String (имя поля связи) | Empty | Добавляет одно поле связи между плановым и фактическим запросами в языке EME-L. Поле должно присутствовать в обоих запросах и иметь одинаковые атрибуты. Метод можно вызывать несколько раз. Перед вызовом должны быть заданы оба запроса (`SetPlanQuery` и `SetFactQuery`). Если запросы не заданы или атрибуты полей связи не совпадают, генерируется ошибка выполнения. |
| `UnjoinAll()` | — | Empty | Очищает список полей связи. Вызывается автоматически при уничтожении объекта. |

## Сверка данных

В языке EME-L методы этой группы выполняют собственно сверку плановых и фактических данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Reconcile()` | — | Empty | Выполняет сверку данных без учёта излишков. Каждая строка факта сопоставляется с подходящей строкой плана по полям связи. Строки плана с нулевым значением `ReconcilingQty` пропускаются. Вычисляется минимальное из двух сверяемых количеств, которое считается сверенным. Если сверяемое количество не покрывается полностью, строка расщепляется: в соответствующем запросе добавляется новая строка с остатком и обновляется карта плана. После сверки сверяемое количество обнуляется, сверенное количество записывается в обе строки, а в поля `LinkRef` прописываются перекрёстные ссылки на номера связанных строк. |
| `ReconcileSurplus()` | — | Empty | Выполняет сверку излишков фактических данных. Обрабатывает только строки факта, которые не были связаны ранее (поле `LinkRef` не заполнено). Строки плана с нулевым значением `ReconcilingQty` не пропускаются. Для каждой такой строки ищется подходящая строка плана по полям связи, и всё сверяемое количество факта записывается как сверенное. Сверяемое количество обнуляется, сверенное количество прописывается в обе строки, а в поля `LinkRef` записываются перекрёстные ссылки на номера связанных строк. |

## Сервисный метод

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetVersion()` | — | Real | Возвращает номер версии алгоритма сверки. Текущая реализация возвращает `2.1`. Версия 2.0 добавила карту плана для ускорения сверки. Версия 2.1 доработала расщепление строк плана с обновлением карты плана. |

## Результат сверки и интерпретация LinkRef

В языке EME-L после вызова `Reconcile` и/или `ReconcileSurplus` состояние строк запросов кодируется следующим образом:

- `LinkRef` содержит номер строки связанного запроса. Для строки факта — номер строки плана, с которой она сверена; для строки плана — номер строки факта.
- `LinkRef == NULL_REF` (в EME-L `NULL_REF` равен `-1`) означает, что строка не была связана:
  - в плановом запросе — **недостача** (плановая строка не покрыта фактом);
  - в фактическом запросе — **излишек** (фактическая строка не покрыта планом).
- `ReconcilingQty` обнуляется для всех строк, участвовавших в сверке.
- `ReconciledQty` заполняется сверенным количеством для связанных строк; для не связанных строк остаётся исходным (обычно `0`).

## Примеры

В языке EME-L проверка версии реконсилятора перед использованием:

```EME-L
objReconciler = Object("Reconciler");

'Признак использования C++ реконсилятора'
USE_CPP_RECONCILER = objReconciler.GetVersion() >= 2.1;
```

В языке EME-L базовый сценарий сверки двух запросов:

```EME-L
objReconciler = Object("Reconciler");

'Разметка заголовков планового и фактического запросов'
objReconciler.MarkupQueryHeader(qryPlan);
objReconciler.MarkupQueryHeader(qryFact);

'Настройка полей связи'
objReconciler.Join("GoodsItemRef");
objReconciler.Join("TrueSignRef");

'Установка запросов'
objReconciler.SetPlanQuery(qryPlan);
objReconciler.SetFactQuery(qryFact);

'Основная сверка и сверка излишков'
objReconciler.Reconcile();
objReconciler.ReconcileSurplus();
```

Полный шаблон использования с переключателем между C++ реконсилятором и резервной реализацией в системе EME.WMS:

```EME-L
'Конструктор класса-обёртки ERPReconciler'
m_Reconciler = Object("Reconciler");

'Признак использования C++ реконсилятора'
USE_CPP_RECONCILER = m_Reconciler.GetVersion() >= 2.1;

m_PlanQuery = Object("Query");
m_FactQuery = Object("Query");
m_PlanQuery.Create();
m_FactQuery.Create();

'Разметка заголовков планового и фактического запросов'
MarkupQueryHeader(m_PlanQuery);
MarkupQueryHeader(m_FactQuery);

'Настройка полей связи'
Join("TrueSignRef");
Join("GoodsItemRef");

'Если используется C++ реконсилятор, передаём запросы в объект'
If (USE_CPP_RECONCILER)
    m_Reconciler.SetPlanQuery(m_PlanQuery);
    m_Reconciler.SetFactQuery(m_FactQuery);
End If

'Выполнение сверки'
If (USE_CPP_RECONCILER)
    m_Reconciler.Reconcile();
    m_Reconciler.ReconcileSurplus();
Else
    'Резервная реализация на EME-L'
End If

'Проверка результата: непривязанные строки факта — излишки'
Loop (m_FactQuery)
    If (m_FactQuery.Get("LinkRef") == NULL_REF)
        'Строка m_FactQuery.GetLine() — излишек'
    End If
End Loop
```

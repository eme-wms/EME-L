# Класс DsQueryStorage

> Синонимы для поиска: CIDsQueryStorage DsQueryStorage класс EME-L, хранилище dsQUERY EME-L, dsQUERY dsDB EME-L, пакетная запись EME-L, FlushToDB EME-L, CreateRef CreateChain UpdateChain EME-L, внутренние ссылки EME-L, цепочки записей EME-L, AppendLines EME-L, CompareLines CheckResolving IsUnresolvedLink EnumRecords RemoveAll IsEmpty GetAt GetDsDBAt Add, атомарная запись EME-L, EME.WMS хранилище объектов записей

Класс `DsQueryStorage` в языке EME-L управляет хранилищем объектов записей с источником данных `dsQUERY`. Класс `DsQueryStorage` в языке EME-L предоставляет механизм создания и хранения объектов записей базы данных в памяти с источником данных `dsQUERY`, а также соответствующих им объектов с источником `dsDB`. Поддерживает управление внутренними ссылками и цепочками записей, пакетную запись изменений в базу данных, сравнение данных между `dsQUERY` и `dsDB`, а также отслеживание неразрешенных ссылок.

Класс `DsQueryStorage` в системе EME.WMS используется в прикладных скриптах EME-L для организации пакетной обработки данных в памяти с последующей атомарной записью изменений в базу данных. Объект класса `DsQueryStorage` в языке EME-L создаётся через `Object()` без параметров. При уничтожении объекта все хранимые записи автоматически удаляются.

Объект класса `DsQueryStorage` в языке EME-L не привязан к контексту диалога или браузера — это самостоятельный класс для управления пакетом записей в памяти.

## Создание объекта класса DsQueryStorage

```EME-L
m_Storage = Object("DsQueryStorage");
```

Конструктор без аргументов класса `DsQueryStorage` в языке EME-L создаёт пустое хранилище. Все объекты записей добавляются позже методом `Add`. Повторное использование хранилища (например, при цикличной обработке пачек сообщений) требует обязательной очистки методом `RemoveAll`, чтобы исключить остаточный «мусор» после предыдущей обработки.

## Добавление и доступ к записям

Методы класса `DsQueryStorage` в языке EME-L для добавления записей в хранилище и доступа к объектам записей с источниками `dsQUERY` и `dsDB`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Add(arg0)` | arg0: String (имя класса записи) | CEMERec (ссылка, dsQUERY) | Создаёт объект записи с источником `dsQUERY` и соответствующий объект с источником `dsDB`, добавляя их в хранилище. Если запись с таким номером уже существует — генерирует ошибку. |
| `Add(arg0, arg1)` | arg0: String (имя класса), arg1: String (имя записи) | CEMERec (ссылка, dsQUERY) | То же по имени класса и имени записи. |
| `GetAt(arg0)` | arg0: Integer (номер) или String (имя записи) | CEMERec (ссылка, dsQUERY) или Empty | Возвращает объект записи с источником `dsQUERY` по номеру или имени. `Empty` — запись отсутствует. |
| `GetDsDBAt(arg0)` | arg0: Integer (номер) или String (имя записи) | CEMERec (ссылка, dsDB) или Empty | Возвращает парный объект записи с источником `dsDB` по номеру или имени. `Empty` — запись отсутствует. |
| `EnumRecords(arg0)` | arg0: Array (по ссылке) | Empty | Перечисляет имена всех записей в хранилище `dsQUERY` и заполняет ими массив (перед добавлением массив полностью очищается). |
| `RemoveAll()` | — | Empty | Удаляет все объекты записей, освобождая память. Очищает все внутренние структуры: карты записей, полей и неразрешённых ссылок. |
| `IsEmpty()` | — | Boolean | `TRUE` — хранилище пустое, `FALSE` — в хранилище есть объекты. |

Метод `Add` класса `DsQueryStorage` в языке EME-L создаёт сразу пару объектов — `dsQUERY` (для манипуляций в памяти) и `dsDB` (для связи с базой). Возвращается ссылка на объект `dsQUERY`; соответствующий `dsDB` извлекается отдельно через `GetDsDBAt`. Типичный паттерн в системе EME.WMS — настроить запрос у `dsQUERY`-объекта и инициализировать режимы поиска у парного `dsDB`-объекта.

## Внутренние ссылки и цепочки

Внутренняя ссылка в языке EME-L — это поле объекта `dsQUERY`, значение которого ссылается на строку в другом (или том же) объекте `dsQUERY` хранилища (а не на реальный номер строки БД). Цепочка в системе EME.WMS — это набор связанных полей (первая строка, обратная ссылка, сортировка), описывающих иерархию «заголовок–строки» между записями.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CreateRef(arg0)` | arg0: CEMEFld (ссылка, поле из записи dsQUERY текущего хранилища) | Empty | Регистрирует внутреннюю ссылку для указанного поля. Если ссылка для данного поля уже создана — генерирует ошибку. |
| `CreateChain(arg0)` | arg0: CEMEFld (поле ссылки на первую строку цепочки / заголовок цепочки / поле сортировки-цепочки) | Empty | Создаёт внутреннюю цепочку: находит в хранилище записи заголовка и строк цепочки по конфигурации поля, затем создаёт внутренние ссылки для полей первой строки, обратной ссылки и сортировки цепочки. |
| `UpdateChain(arg0)` | arg0: CEMEFld (как в `CreateChain`) | Empty | Перестраивает связи между строками цепочки на основе актуальных данных обратных ссылок: очищает старые ссылки на первые строки и сортировки, затем прописывает актуальные значения. |

Перед вызовом `CreateRef`, `CreateChain` или `UpdateChain` класса `DsQueryStorage` в языке EME-L объекты записей заголовка и строк цепочки уже должны быть добавлены в хранилище через `Add`, иначе генерируется ошибка «В хранилище нет записи заголовка/строк цепочки».

## Контроль неразрешённых ссылок

Неразрешённая ссылка в языке EME-L — внешняя ссылка (реальный номер строки БД), которая ещё не преобразована во внутреннюю (строка `dsQUERY`). Проверка неразрешённых ссылок перед записью в системе EME.WMS обязательна: метод `FlushToDB` генерирует ошибку, если в хранилище остались неразрешённые ссылки.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsUnresolvedLink(arg0)` | arg0: CEMEFld (ссылка, поле) | Boolean | `TRUE` — поле содержит неразрешённую ссылку, `FALSE` — ссылка разрешена или отсутствует. |
| `CheckResolving()` | — | String | Список неразрешённых ссылок в формате `[запись]:строка <- [запись]:[поле]:строка`. Пустая строка — неразрешённых ссылок нет. |

Контрольная точка перед записью в БД в системе EME.WMS: вызовите `CheckResolving` и прервите выполнение с `is_error`, если возвращаемая строка не пустая.

## Запись в базу данных

Методы класса `DsQueryStorage` в языке EME-L для записи накопленных изменений из хранилища `dsQUERY` в базу данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AppendLines()` | — | Empty | Добавляет новые строки во все записи хранилища в базу данных: для каждой записи находит строки `dsQUERY`, у которых поле самоссылки содержит `NULL_REF`, добавляет соответствующие строки в БД и прописывает полученные номера в поле самоссылки. |
| `FlushToDB()` | — | String | Записывает все изменения из хранилища `dsQUERY` в БД. Перед записью проверяет наличие неразрешённых ссылок и генерирует ошибку, если они есть. Последовательно выполняет: добавление новых строк, преобразование внутренних ссылок во внешние, блочную запись изменённых данных и удаление помеченных строк. Возвращает статистику — количество добавленных, обновлённых и удалённых строк для каждой записи. |
| `CompareLines()` | — | String | Сравнивает данные между объектами `dsQUERY` и соответствующими `dsDB` для всех записей. Возвращает описание найденных расхождений с указанием записей, строк и полей; пустая строка — данные идентичны. Требует предварительного вызова `ConnectFields` у запроса (внутренняя ошибка, если блок записи не инициализирован). |

Метод `FlushToDB` класса `DsQueryStorage` в языке EME-L — атомарная операция записи всего хранилища; вызывается один раз после полной подготовки данных. Мастер-транзакция в системе EME.WMS (если текущая запись является мастер-записью) открывается вызывающим кодом через `is_open_master_transaction()` до вызова `FlushToDB`.

## Примеры

Создание хранилища, добавление записей и настройка ссылок в языке EME-L:

```EME-L
'#DEV_INITIALS 27.06.2025 EME company'
'Инициализация хранилища для импорта товара'
m_Storage = Object("DsQueryStorage");

'Запись заголовка и строк цепочки'
q_ASN As "ASN" = m_Storage.Add("ASN");
r_ASN As "ASN" = m_Storage.GetDsDBAt(q_ASN.GetRecord());
r_ASN.SetFindMode1(r_ASN.GetRegNoFld().GetFieldName());

q_ASNLines As "ASNLines" = m_Storage.Add("ASNLines");
r_ASNLines As "ASNLines" = m_Storage.GetDsDBAt(q_ASNLines.GetRecord());
m_Storage.CreateChain(q_ASNLines.GetASNRefFld());

'Регистрируем ссылки на справочники'
m_Storage.CreateRef(q_ASN.GetSupplierRefFld());
```

Контроль неразрешённых ссылок и запись в БД в системе EME.WMS:

```EME-L
If (m_Storage.IsEmpty())
    Return;
End If

'Контрольная точка перед записью'
ResolvingResult = m_Storage.CheckResolving();
If (ResolvingResult != "")
    is_error(1, "Не разрешены ссылки:\n" + ResolvingResult);
End If

'Мастер-транзакцию открываем по месту'
r_ASN = Object("dsDB", "ASN");
If (r_ASN.IsMaster())
    is_open_master_transaction();
End If

Stats = m_Storage.FlushToDB();
```

Перечисление записей в хранилище в языке EME-L:

```EME-L
arrNames = Object("Array");
m_Storage.EnumRecords(arrNames);
Loop (arrNames)
    Name = arrNames.GetAt(arrNames.GetLine());
    q_Rec = m_Storage.GetAt(Name);
End Loop
```

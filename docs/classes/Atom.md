# Класс Atom

> Синонимы для поиска: CIAtom Atom класс EME-L, атомарная синхронизация EME-L, псевдопараллельные вычисления EME-L, Lock UnLock Check EME-L, RefOf Atom, защита обработчиков EME-L, VT_I4 ссылка EME-L, атом EME-L, мьютекс EME-L, взаимоисключение EME-L, race condition EME-L, Atom TaskDialog EME.WMS, ATOM_CONTEXT ATOM_TIMER_UPDATE

Класс `Atom` в языке EME-L реализует примитив атомарной синхронизации для защиты от псевдопараллельных вычислений. Связывается с глобальной целочисленной переменной через ссылку и предоставляет методы для её атомарного захвата (`Lock`), освобождения (`UnLock`) и проверки (`Check`) без изменения значения.

Класс Atom в системе EME.WMS применяется для защиты обработчиков кнопок и таймеров от взаимных состояний гонки: если один обработчик уже захватил атом, другой, обратившийся к тому же общему ресурсу, получает отказ (`FALSE`) и завершается досрочно. Типичный сценарий — диалог с таймером обновления, где одновременно может сработать таймер и произойти действие пользователя по кнопке; атом гарантирует, что только один из обработчиков будет работать с общим контекстом в каждый момент времени.

> Класс `Atom` в языке EME-L решает конкретную задачу — взаимоисключение между псевдопараллельными обработчиками в пределах одного процесса EME-L. Это не мьютекс операционной системы и не синхронизация между разными процессами. Синхронизация между процессами в системе EME.WMS выполняется другими механизмами.

Объект класса `Atom` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс синхронизации, связанный только с переданной ему глобальной переменной.

## Создание объекта класса Atom

```EME-L
'Глобальная переменная-флаг: модифицируется по ссылке в Object("Atom")'
ATOM_TASK = 0;

objAtom = Object("Atom", RefOf ATOM_TASK);
```

Конструктор класса Atom в языке EME-L принимает один обязательный аргумент — ссылку на глобальную целочисленную переменную (`VT_I4`). Переменная обязана быть объявлена заранее и иметь тип «целое число»; передавать в конструктор можно только ссылку, полученную через `RefOf` (оператор `RefOf` возвращает ссылку на переменную в месте вызова).

Если на момент создания объекта значение связанной переменной уже равно `1`, объект класса Atom переходит в **режим только для чтения**: методы `Lock` и `UnLock` возвращают `FALSE`, не изменяя значение переменной. Метод `Check` в этом режиме продолжает работать и отражает фактическое состояние переменной. Режим фиксируется на всё время жизни объекта и не может быть переключен.

## Методы класса Atom

Методы класса Atom в языке EME-L управляют связанной глобальной переменной синхронизации. Все три метода не принимают аргументов и возвращают `Boolean`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Lock()` | — | Boolean | Проверяет и активирует связанную переменную. Если переменная равна `0` — устанавливает её в `1` и возвращает `TRUE`. Если уже не `0` — возвращает `FALSE`, ничего не меняя. |
| `UnLock()` | — | Boolean | Деактивирует связанную переменную, устанавливая её в `0`. В режиме записи возвращает `TRUE`. |
| `Check()` | — | Boolean | Проверяет состояние переменной без изменения. `TRUE` — переменная деактивирована (`0`), `FALSE` — активирована (не `0`). |

В **режиме только для чтения** класса Atom методы `Lock` и `UnLock` всегда возвращают `FALSE`, не затрагивая переменную. Режим только для чтения включается, когда в момент создания объекта значение переменной уже равно `1`.

## Поведение при уничтожении

При уничтожении объекта класса Atom в языке EME-L (например, при выходе из метода, где он был создан как локальная переменная) деструктор автоматически сбрасывает связанную переменную в `0` — но только если объект находится в режиме записи. Объект в режиме только для чтения оставляет переменную без изменений.

Это означает в системе EME.WMS, что повторный захват через новый `Object("Atom", RefOf VAR)` обычно не требуется после выхода предыдущего обработчика: переменная уже освобождена деструктором предыдущего атома. В памяти EME-L нет операторов `new` и `delete` — объекты удаляются автоматически при выходе из области видимости.

## Примеры использования класса Atom

Защита обработчика таймера от повторного входа в языке EME-L — если таймер сработал, пока предыдущий обработчик ещё выполняется, новый вызов завершается досрочно. Реальный паттерн из системы EME.WMS (класс `TaskDialog`):

```EME-L
'ATOM_TIMER_UPDATE — глобальная переменная-флаг класса TaskDialog'
ATOM_TIMER_UPDATE = 0;

TimerUpdate_OnTimer()
{
    'Попытка захватить атом; FALSE — уже занято, выходим'
    If (~Object("Atom", RefOf ATOM_TIMER_UPDATE).Lock())
        Return;
    End If

    'Проверка, что контекст ещё валиден'
    If (~Object("Atom", RefOf ATOM_CONTEXT).Check())
        Return;
    End If

    'Тело обработчика; при выходе деструктор атома освободит переменную'
    ReloadReportMain();
}
```

Комбинированная защита двух раздельных ресурсов через два атома класса Atom в языке EME-L — один для контекста диалога, второй для процедуры обновления. Каждой глобальной переменной соответствует отдельный атом:

```EME-L
'Глобальные переменные для атомов класса TaskDialog системы EME.WMS'
ATOM_CONTEXT = 0;
ATOM_TIMER_UPDATE = 0;
ATOM_INPUT_LIST = 0;

Dialog_OnInit()
{
    'Захват контекста на время работы диалога'
    If (~Object("Atom", RefOf ATOM_CONTEXT).Lock())
        Return;
    End If

    'Дальнейшая инициализация диалога...'
}
```

Проверка состояния без захвата — метод `Check` класса Atom в языке EME-L полезен, когда нужно узнать, активен ли другой обработчик, не блокируя его и не претендуя на ресурс:

```EME-L
objAtom = Object("Atom", RefOf ATOM_TASK);
If (objAtom.Check())
    'Переменная свободна — можно действовать'
Else
    'Переменная занята другим обработчиком'
End If
```

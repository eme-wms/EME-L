### IrrOctree — класс полигональной сетки (октодерева) движка Irrlicht (язык EME-L)

Класс `IrrOctree` в языке EME-L — это класс полигональной сетки (октодерева) движка Irrlicht. Представляет статический трёхмерный объект, хранимый в виде октодерева (octree) для оптимизации рендеринга и пространственных запросов в сцене Irrlicht. Позволяет загружать полигональную сетку из mesh-файла или формировать её на основе объекта `IrrMeshCreator` с помощью метода `SetMesh`.

Класс `IrrOctree` в системе EME.WMS наследует все методы базового класса `IrrNode` — управление положением, поворотом, масштабом, видимостью, именем, материалом и текстурой узла. Используется в EME-L-скриптах для отображения статических 3D-моделей в сцене Irrlicht, когда важна эффективность отрисовки крупных или детальных сеток: октодерево ускоряет отсечение невидимых полигонов по сравнению с обычным узлом `IrrMesh`.

### Создание объекта IrrOctree

В блоке регистрации `BEGIN_IMPLEMENT(CIIrrOctree)` в C++ исходнике нет зарегистрированных конструкторов (`IMPLEMENT_CTOR` отсутствует), поэтому объект `IrrOctree` в языке EME-L не создаётся через `Object("IrrOctree", ...)` напрямую. Канонический способ получить узел октодерева — вызвать метод `AddOctree(meshSource)` у родительского узла `IrrNode`. Этот метод создаёт дочерний узел октодерева, инициализирует его внутреннее представление Irrlicht и возвращает ссылку на новый объект `IrrOctree`.

```EME-L
'Создать устройство Irrlicht и корневой узел сцены'
objDevice = Object("IrrDevice");
objRoot   = objDevice.GetRootNode();

'Добавить дочерний узел октодерева из mesh-файла'
objOctree = objRoot.AddOctree("D:\\Models\\warehouse.mesh");
```

Метод `AddOctree` в системе EME.WMS принимает на вход тот же тип аргумента, что и `SetMesh` — путь к mesh-файлу (строка) либо объект `IrrMeshCreator`. После создания к узлу сразу применяются масштаб `(1, 1, 1)` и пустое имя; в дальнейшем имя задаётся методом `SetName` базового класса `IrrNode`. Глобальный масштаб узла учитывается так же, как и для остальных узлов Irrlicht.

### Метод SetMesh класса IrrOctree

Класс `IrrOctree` в языке EME-L регистрирует единственный собственный метод — `SetMesh`. Все прочие операции (положение, поворот, масштаб, текстура, видимость, имя) наследуются от базового класса `IrrNode`. Метод `SetMesh` перестраивает внутренний указатель Irrlicht, не меняя остальных свойств узла (имя, положение, текстура).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetMesh(arg0)` | arg0: String (путь к mesh-файлу) **или** IrrMeshCreator (ссылка) | IrrOctree (ссылка на текущий объект) | Загружает или формирует новую полигональную сетку и устанавливает её для текущего узла IrrOctree (октодерева). При передаче строки путь загружается менеджером сцены Irrlicht; при передаче объекта `IrrMeshCreator` сетка строится вызовом `CreateMesh` этого объекта. Возвращает текущий объект `IrrOctree` для цепочки вызовов. |

Тип аргумента `arg0` метода `SetMesh` в языке EME-L определяется по типу переданного значения: строка (path) вызывает загрузку mesh-файла, ссылка на объект `IrrMeshCreator` — программное построение геометрии. Возвращаемая ссылка на `IrrOctree` позволяет объединять настройку в цепочку вызовов (method chaining), например `objOctree.SetMesh(mesh).SetClickable(TRUE)`.

### Примеры использования IrrOctree в языке EME-L

Создание узла октодерева и настройка позиции и текстуры:

```EME-L
'Корневой узел сцены Irrlicht'
objDevice = Object("IrrDevice");
objRoot   = objDevice.GetRootNode();

'Создать октодерево из mesh-файла и настроить свойства'
objOctree = objRoot.AddOctree("D:\\Models\\rack_sector.mesh");
objOctree.SetPosition(10.0, 0.0, 5.0);
objOctree.SetMaterialTexture("D:\\Textures\\metal.png");
objOctree.SetLighting(TRUE);
objOctree.SetName("Rack#A1");
```

Замена сетки через `IrrMeshCreator` с цепочкой вызовов:

```EME-L
'Программно собрать полигональную сетку'
meshCreator = Object("IrrMeshCreator");
meshCreator.AddCube(2.0);

'Установить новую сетку в существующий узел октодерева'
objOctree.SetMesh(meshCreator).SetClickable(TRUE);
```

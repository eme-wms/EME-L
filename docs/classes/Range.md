### Класс Range — работа с ячейками отчёта EME DB в языке EME-L

Класс `Range` (C++ `CIRange`) в языке EME-L — класс для работы с одной или несколькими ячейками шаблона отчёта EME DB. Представоставляет программный доступ к ячейкам отчёта: получение и изменение значений, управление визуальным оформлением, навигация по бандам, строкам и столбцам, а также доступ к источникам данных, связанным с ячейками (биндинг).

В системе EME.WMS объекты `Range` не создаются пользователем напрямую — их порождает класс `Report` (методы `GetCell`, `GetRow`, `GetColumn`, `GetSelectedCell`) либо они получаются клонированием существующего `Range` через `Clone`. Самостоятельный вызов `Object("Range")` возможен, но контекстом служит активное окно отчёта.

Класс доступен только при включённой поддержке отчётов (C++ compile flag `I_REPORT`). Часть методов требует дополнительных флагов компиляции ядра: `SetHeight` — `REPORT_CHANGE_CELL_SIZE_IN_META`; `GetTextWidth`, `CellGoAway` — `REPORT_CALC_TEXT_WIDTH_FOR_CELL`; `FixArea` — `REPORT_VERSION >= 1.8`; `GetBackColor`/`GetTextColor`/`GetFrameColor` (с вариантами) — `REPORT_VERSION >= 1.25`.

Основные возможности класса `Range` в языке EME-L:

- Выделение ячеек: одна ячейка, множество ячеек, столбец (вертикальный банд), строка банда, ячейка по имени или текущее выделение в отчёте.
- Чтение и запись значений ячеек с автоматическим преобразованием типов (строка, целое число, вещественное число, дата/время и др.).
- Управление цветами: фон, текст, рамка — для отдельной ячейки, всех ячеек или диапазона через массив цветов.
- Управление видимостью, шириной, высотой, границами и выравниванием текста ячеек.
- Получение адресной информации: номер строки банда, номера строк родительских бандов, вертикальный индекс, номер ячейки, имя ячейки.
- Навигация и выделение в окне отчёта: `Select`, `FixArea`, `GetSelectedCell`.
- Доступ к источнику привязки (биндингу): объекты запроса, поля источника, номера строк оригинального запроса.
- Клонирование выделения, проверка на пустоту, очистка ячеек, удаление ячеек со сдвигом влево.

### Создание объекта Range в языке EME-L

Конструктор без аргументов создаёт пустой объект `Range`. В прикладных скриптах объекты этого класса обычно получают не прямым конструированием, а как результат методов класса `Report`.

```EME-L
'Создание пустого объекта Range, привязанного к активному окну отчёта'
objRange = Object("Range");
```

```EME-L
'Типичный способ: получить Range ячейки по имени из объекта отчёта'
objRange = objReport.GetCell("price_total");
```

### Получение подчинённых Range — навигация по ячейкам отчёта

Эти методы создают новый объект `Range` на основе текущего выделения и возвращают ссылку на него. В языке EME-L иерархия: `Report` → `Range` (ячейка/строка/столбец) → вложенные `Range`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetCell(arg0)` | arg0: Integer (индекс) | Range (ссылка) | Новый `Range`, содержащий одну ячейку из текущего выделения по её индексу. |
| `GetCell([arg0[, arg1[, arg2…]]])` | arg0: String (имя ячейки), arg1: Integer/Any (вертикальный индекс, необязательный), arg2…: Integer (номера строк родительских бандов, до 10) | Range (ссылка) | Новый `Range` для ячейки, заданной по имени, с уточнением вертикального индекса и строк родительских бандов. Без аргументов диспетчер передаёт пустое имя ячейки — это соответствует критерию «любая ячейка», итоговое выделение зависит от остальных правил обхода. |
| `GetRow(arg0[, arg1…])` | arg0: String (имя банда), arg1…: Integer (номера строк родительских бандов, до 10) | Range (ссылка) | Новый `Range`, содержащий строку заданного банда. |
| `GetRow([arg0])` | arg0: Integer (индекс ячейки, по умолчанию 0) | Range (ссылка) | Новый `Range`, содержащий строку отчёта, в которой находится ячейка с заданным индексом. |
| `GetColumn()` | — | Range (ссылка) | Новый `Range`, содержащий ячейки столбца (вертикального банда), к которому относится первая ячейка выделения. |
| `GetColumn(arg0)` | arg0: Integer/Any | Range (ссылка) | **Фактически это overload для вертикальной строки, а не индекса ячейки.** Диспетчер передаёт `arg0` в перегрузку `GetColumn(CIRange&, COleVariant& vline)`, которая обрабатывает его как вертикальный идентификатор строки (`vline.oleline`). Перегрузка `GetColumn(CIRange&, int index)`, выбирающая столбец по индексу ячейки, существует в C++, но из скрипта недостижима — диспетчер к ней не обращается. Для получения столбца конкретной ячейки сначала получите `Range` этой ячейки через `GetCell(индекс)`, затем вызовите `GetColumn()` без аргументов. |
| `GetSelectedCell()` | — | Range (ссылка) | Новый `Range` с ячейкой, выделенной в отчёте в данный момент. Требует включённого режима выделения (`is_selection_mode()`). Пустой объект, если выделение отсутствует. |
| `GetRect()` | — | Rect (ссылка) | Объект `Rect` с координатами первой ячейки выделения. |
| `GetRect(arg0)` | arg0: Integer (индекс) | Rect (ссылка) | Объект `Rect` с координатами ячейки по индексу. |

| `m_noofreportitems` (default) | `0x7FFFFFFF` | Начальное значение счётчика ячеек в конструкторе — принудительно заставляет `RefreshIndex` перестроить индекс при первом обращении. |
| `ALLCELLSINDEX` | -1 (по соглашению) | Значение индекса «все ячейки» в C++ реализации (`IsEmpty(-1)`, `Clear(-1)`, `IsVisible`). Внешнему пользователю передаётся как опущенный аргумент. |
| `NULL_REF` | -1 | Стандартное «нулевое» значение ссылки строки банда. Возвращается методами `GetLine`, `GetParentLine`, `GetVLine`, `FindBindSourceLine`, когда ячейка вне банда или связи нет. Для индикации ошибки методы используют `NULL_REF - 1` (= -2). |

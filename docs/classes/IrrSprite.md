# Класс IrrSprite

> Синонимы для поиска: CIIrrSprite IrrSprite спрайт Irrlicht EME-L, IrrSprite AddSprite FindSpriteByName LinkToNode SetImage SetAlignment SaveSprite LoadSprite Remove, IrrDevice спрайт Irrlicht, 2D GUI Irrlicht EME-L

Класс `IrrSprite` в языке EME-L — класс спрайта движка Irrlicht: двумерного изображения, отображаемого в графическом интерфейсе сцены. В языке EME-L спрайт представляет собой прямоугольник с заданными координатами, текстурой и режимами выравнивания, который может быть создан как независимый элемент интерфейса либо привязан к трёхмерному узлу сцены `IrrNode` с указанной длиной связи («верёвочки»).

Спрайт не является узлом сцены и не наследует методы `IrrNode` — это самостоятельный элемент 2D-GUI, отображаемый поверх трёхмерной сцены. В системе EME.WMS класс предоставляет установку текстуры из файла или объекта `IrrImage`, настройку выравнивания краёв относительно родительского элемента, сериализацию параметров в дерево свойств `PropertyTree` и удаление из сцены. При привязке к узлу сцены (`LinkToNode`) спрайт отрисовывается рядом с положением узла с заданным смещением; обновление позиции спрайта при движении камеры и узла выполняется устройством `IrrDevice` на каждом кадре отрисовки в языке EME-L.

Используется в прикладных скриптах EME-L для отображения двумерных элементов интерфейса, индикаторов, маркеров, подписей и других изображений, привязанных к сцене или к отдельным узлам — например, курсоров выбора стеллажей, текстовых меток персонала, легенд 3D-вида.

## Создание объекта IrrSprite

В блоке `BEGIN_IMPLEMENT(CIIrrSprite)` нет зарегистрированных конструкторов с аргументами (`IMPLEMENT_CTOR`) — объект `IrrSprite` в языке EME-L не создаётся через `Object("IrrSprite", ...)` напрямую. Канонический способ получить спрайт — вызвать метод `AddSprite()` у устройства `IrrDevice` (см. [IrrDevice](./IrrDevice.md)): этот метод создаёт спрайт, регистрирует его по имени в реестре устройства и возвращает ссылку на новый объект `IrrSprite`.

```EME-L
'Создать устройство Irrlicht и спрайт с прямоугольником положения'
objDevice = Object("IrrDevice");
Rect = Object("Rect", 0, 0, 90, 20);
objSprite = objDevice.AddSprite("Legend", Rect);
```

Вторая перегрузка `AddSprite` принимает дерево свойств `PropertyTree` — спрайт создаётся и сразу загружает параметры (прямоугольник, текстура, привязка) из дерева. В языке EME-L это удобно для сохранения и восстановления сцены между сеансами.

```EME-L
'Создать спрайт из ранее сохранённого дерева свойств'
objSprite = objDevice.AddSprite(propsTree);
```

Повторный вызов `AddSprite` с уже существующим именем возвращает ранее созданный объект, а не создаёт новый — реестр устройств ведётся по имени спрайта.

## Текстура и изображение

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetImage(arg0)` | arg0: String или IrrImage (по ссылке) | Empty | Устанавливает текстуру спрайта. Строка интерпретируется как имя файла текстуры; ссылка на объект `IrrImage` — как готовое изображение, которое преобразуется в текстуру устройства Irrlicht. |

При использовании `IrrImage` типичная цепочка в языке EME-L: создать изображение, отрисовать на нём текст/графику, передать объект в `SetImage`. Текстура обновляется в реальном времени — повторные вызовы `SetImage` заменяют изображение спрайта.

## Привязка к узлу сцены

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `LinkToNode(arg0, arg1)` | arg0: IrrNode (по ссылке), arg1: Integer (длина связи) | Empty | Привязывает спрайт к узлу сцены с заданной длиной связи. При отрисовке устройства спрайт позиционируется относительно мировых координат узла; `arg1` задаёт условное расстояние «верёвочки» от узла до спрайта. Допустимы любые целые значения, включая 0 (спрайт непосредственно на узле). |

Привязка заменяет предыдущую связь спрайта — каждый спрайт привязан не более чем к одному узлу. Для поиска ранее созданного спрайта по имени используйте `IrrDevice.FindSpriteByName` (см. [IrrDevice](./IrrDevice.md)).

## Выравнивание

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetAlignment(arg0, arg1, arg2, arg3)` | arg0–arg3: String или Empty | Empty | Устанавливает режимы выравнивания левого, правого, верхнего и нижнего краёв спрайта относительно родительского элемента. Допустимые значения: `"upperLeft"` (по умолчанию), `"lowerRight"`, `"center"`, `"scale"`. `Empty` или пустая строка интерпретируются как `"upperLeft"`. |

Допустимые значения каждого параметра:

| Значение | Поведение |
|----------|----------|
| `"upperLeft"` | По верхнему или левому краю родителя (используется по умолчанию). |
| `"lowerRight"` | По нижнему или правому краю родителя. |
| `"center"` | По центру родителя. |
| `"scale"` | По размеру родителя — спрайт растягивается. |

В языке EME-L параметры `SetAlignment` можно опускать с конца: `SetAlignment("lowerRight", "lowerRight",, )` — выравнивание только по левому и правому краям, верх/низ остаются по умолчанию.

> **Внимание:** в текущей реализации движка (`i_IrrSprite.cpp`, компилируется под `IRRLICHT_IS_AVAILABLE`) условие проверки аргумента `bottomAlignment` ошибочно ссылается на `args[0]` вместо `args[3]`. При передаче непустого `leftAlignment` вместе с пустым `bottomAlignment` нижнее выравнивание не получит значение по умолчанию `"upperLeft"`, как описано выше, в языке EME-L. Передавайте `bottomAlignment` явно, чтобы обойти дефект в системе EME.WMS.

## Сериализация

Методы сохраняют и восстанавливают полное состояние спрайта (прямоугольник, текстура, привязка) в дерево свойств `PropertyTree`. В языке EME-L используются вместе с перегрузкой `AddSprite(propsTree)` устройства `IrrDevice` для persistence сцены.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SaveSprite(arg0)` | arg0: PropertyTree (по ссылке) | Empty | Сохраняет параметры спрайта в дерево свойств. |
| `LoadSprite(arg0)` | arg0: PropertyTree (по ссылке) | Empty | Загружает параметры спрайта из дерева свойств и восстанавливает его состояние (пересоздаёт изображение, применяет текстуру, привязывает к узлу, если указан `LinkNodeName`). |

Сохраняемые/загружаемые параметры дерева свойств в языке EME-L:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `Name` | String | Имя спрайта. Если имя не задано при создании, генерируется автоматически из адреса объекта. |
| `UpperLeftCorner` | 2di | Координаты верхнего левого угла спрайта. |
| `LowerRightCorner` | 2di | Координаты нижнего правого угла спрайта. |
| `TextureFileName` | String | Имя файла текстуры. |
| `TextureImage` | IrrImage | Объект изображения. Сохраняется только при отсутствии `TextureFileName`; при загрузке — загружается только если `TextureFileName` пуст. |
| `LinkNodeName` | String | Имя узла сцены, к которому привязан спрайт. |
| `RopeLength` | Integer | Длина связи между спрайтом и узлом. |

## Удаление

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Remove()` | — | Empty | Удаляет спрайт из сцены и освобождает объект. После вызова ссылка на объект становится недействительной. |

Метод `Remove` также неявно вызывается устройством при сбросе сцены (`IrrDevice.Reset`) и при удалении узла, к которому привязан спрайт.

## Примеры

Создание спрайта курсора-прицела, привязанного к узлу, с отрисовкой на `IrrImage`:

```EME-L
'Создать спрайт 16x16 и нарисовать на нём прицел'
objSprite = m_Irrlicht.AddSprite("Cursor#point", Object("Rect", 0, 0, 16, 16));
imgPoint = Object("IrrImage", 16, 16);
imgPoint.Fill(is_RGBA(0, 0, 0, 0));
imgPoint.SetAntialiasing(TRUE);
imgPoint.SetPen(is_RGBA(0, 0, 0, 255));
imgPoint.DrawEllipse(0, 0, 15, 15);
imgPoint.SetBrush(is_RGBA(0, 0, 0, 255));
imgPoint.DrawEllipse(4, 4, 7, 7);
objSprite.SetImage(imgPoint);
objSprite.LinkToNode(CursorNode, 0);
```

Создание спрайта FPS-индикатора, выровненного по нижнему правому углу:

```EME-L
'Создать спрайт FPS и выровнять его по нижнему правому углу'
nFPS = m_Irrlicht.GetFPS();
rFPS = Object("Rect", 0, 0, 90, 20);
objSprite = m_Irrlicht.AddSprite("FPS", rFPS);
objSprite.SetAlignment("lowerRight", "lowerRight",, );

imgFPS = Object("IrrImage", 90, 20);
imgFPS.Fill(is_RGBA(255, 255, 255, 127));
If (nFPS < 24)
    imgFPS.SetPen(is_RGBA(255, 0, 0, 255));
Else
    imgFPS.SetPen(is_RGBA(0, 127, 0, 255));
End If
imgFPS.SetFont("Arial", 16);
imgFPS.DrawText(0, 0, 90, 20,, "FPS: " + nFPS);
objSprite.SetImage(imgFPS);
```

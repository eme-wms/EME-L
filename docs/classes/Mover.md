# Класс Mover

> Синонимы для поиска: CIMover Mover класс EME-L, перемещение строк документа EME-L, SrcToBuf BufToDst BufToSrc SrcToDst постановка в буфер отгрузка из буфера возврат из буфера прямая отгрузка, SetConsole SetContext OperationContext, складские операции EME-L, DocLines Move, режим отгрузки EME-L, протокол ошибок Mover

Класс `Mover` в языке EME-L предназначен для выполнения складских операций перемещения строк документов. Управляет жизненным циклом строк документов при выполнении складских операций: постановка строк в буфер, отгрузка из буфера, возврат строк из буфера и прямая отгрузка без использования буфера. Класс Mover в системе EME.WMS работает с записями строк документа `DocLines` и оперирует их номерами, передаваемыми в массиве.

Класс обеспечивает многоступенчатый контроль целостности данных: предварительные проверки вне транзакции, проверки внутри транзакции и собственно выполнение операции. При обнаружении изменений в строках документов во время операции формируется детальный протокол ошибок. В языке EME-L этот протокол возвращается в виде строки.

Поддерживается автоматический контроль доступности ячеек с исключением строк, для которых невозможно выполнить операцию по соображениям доступности. Исключенные строки могут быть собраны в отдельный массив. В системе EME.WMS механизм исключения основан на вызове внутренней функции `ExcludeLinesFromOperation`.

Опционально доступен вывод операционной информации в консоль с настройкой параметров окна (размеры, заголовок). Для передачи дополнительных параметров операции (например, настройки поиска ячеек) в языке EME-L используется объект `OperationContext`, который задаётся методом `SetContext`. Если контекст не задан, создаётся временный объект `OperationContext` со значением по умолчанию `NeedCheckAccessibility = TRUE`.

## Создание объекта класса Mover

Класс `Mover` в языке EME-L имеет два конструктора — один на каждое число аргументов.

```EME-L
'Конструктор без аргументов — объект не привязан к конкретной строке документа'
objMover = Object("Mover");
```

```EME-L
'Конструктор с указанием записи строки документа DocLines'
objMover = Object("Mover", r_DocLines.GetRecord());
```

Конструктор с одним аргументом принимает номер записи строки документа (`DocLines`) и используется для операций над строками этого документа. Конструктор без аргументов создаёт объект без привязки к строке.

## Операции с буфером

Методы `SrcToBuf`, `BufToDst`, `BufToSrc` перемещают строки документа через промежуточный буфер. Все методы возвращают строку с протоколом операции: пустую строку при успехе или текст ошибок для строк, которые не удалось обработать. В языке EME-L проверка результата выполняется сравнением с пустой строкой.

## Режимы выполнения операций

Параметр `режим` в методах `SrcToBuf`, `BufToDst`, `BufToSrc`, `SrcToDst` класса Mover в языке EME-L принимает одно из следующих значений:

| Значение | Описание |
|----------|----------|
| `0` | Предварительные проверки вне транзакции и выполнение операции с повторными проверками внутри транзакции (по умолчанию). |
| `1` | Отгрузка с проверками внутри транзакции; транзакция открывается вызывающим кодом. |
| `2` | Только предварительные проверки без выполнения операции. |

## Примеры

Постановка строк в буфер с контекстом операции — типовой паттерн в языке EME-L:

```EME-L
objContext = Object("OperationContext", TRUE);
objContext.SetDisableCreateMoveOrder(TRUE);
objMover = Object("Mover", r_DocLines.GetRecord());
objMover.SetContext(objContext);
strResult = objMover.SrcToBuf(arrLoadedLines);
If (strResult != "")
    is_message(tr("Ошибка"), strResult, 1);
End If
```

Постановка в буфер с явным режимом и выводом в консоль — расширенный паттерн Mover в языке EME-L:

```EME-L
context = Object("OperationContext", TRUE);
objMover = Object("Mover", r_DocLinesMove.GetRecord());
objMover.SetContext(context);
objMover.SetConsole(TRUE, 500, 500, tr("Постановка в буфер"));
strResult = objMover.SrcToBuf(arrDocLinesMove, 1);
If (strResult != "")
    is_message(tr("Ошибка постановки в буфер"), strResult, 1);
End If
```

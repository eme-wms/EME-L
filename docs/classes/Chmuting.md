# Класс Chmuting

> Синонимы для поиска: CIChmuting Chmuting класс EME-L, чмутинг EME-L, проверка целостности данных EME-L, коррекция данных EME-L, ChmutingVolume EME-L, SetVolume CheckRecord CheckAllRecords ShowBad Run, CChmuting EME-L, целостность базы данных EME.WMS, процедура чмутинга EME-L

Класс `Chmuting` в языке EME-L выполняет высокоуровневую процедуру проверки и коррекции целостности данных (чмутинг). Класс Chmuting в языке EME-L предоставляет механизм выборочной или полной проверки записей базы данных в соответствии с настройками, хранящимися в записях `Chmuting` и `ChmutingVolume`. Поддерживает выбор тома проверки, фильтрацию обрабатываемых записей по отдельности или массово, а также управление отображением некорректных строк в дамп.

Класс Chmuting в системе EME.WMS используется для аудита и восстановления целостности данных в выбранных томах: оператор или планировщик создаёт объект Chmuting, выбирает том через `SetVolume`, отмечает записи для проверки и запускает процедуру `Run`, которая сверяет данные и опционально вносит коррекции.

Объект класса `Chmuting` в языке EME-L создаётся через `Object()` и требует валидного контекста выполнения и объекта-контейнера — без контекста конструктор генерирует ошибку.

## Создание объекта класса Chmuting

```EME-L
'Без явного контейнера — контекст определяется автоматически'
objChm = Object("Chmuting");

'С явным объектом-контейнером'
objChm = Object("Chmuting", rContainer);
```

Конструктор без аргументов класса Chmuting в языке EME-L определяет объект-контейнер из текущего контекста выполнения и генерирует ошибку, если контекст не задан. Конструктор с одним аргументом принимает ссылку на объект-контейнер, в контексте которого будет выполняться чмутинг. При создании инициализируется внутренний битовый массив для отслеживания используемых строк настроек чмутинга, а выражение отображения некорректных строк устанавливается в значение по умолчанию `"1"`.

## Выбор тома и настройка записей

Методы класса Chmuting в языке EME-L для выбора тома проверки и фильтрации обрабатываемых записей.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetVolume(arg0)` | arg0: String (имя тома) | Empty | Устанавливает том чмутинга по имени, загружая настройки из записи `ChmutingVolume`. Сбрасывает внутренний битовый массив выбранных строк. Если том с указанным именем не найден, генерирует ошибку. |
| `CheckRecord(arg0, arg1)` | arg0: String (идентификатор записи), arg1: Boolean | Empty | Включает или исключает отдельную запись из операции чмутинга для текущего тома. Выполняет поиск строки настройки чмутинга, соответствующей заданному идентификатору, и устанавливает бит внутреннего массива. `arg1=TRUE` — включить запись, `FALSE` — исключить. Если том не задан, генерирует ошибку. |
| `CheckAllRecords(arg0)` | arg0: Boolean | Empty | Включает или исключает все записи текущего тома чмутинга сразу. Устанавливает бит внутреннего массива для каждой строки настройки, привязанной к выбранному тому. `arg0=TRUE` — включить все записи, `FALSE` — исключить все. Если том не задан, генерирует ошибку. |

## Управление отображением и запуск процедуры

Методы класса Chmuting в системе EME.WMS для управления выводом некорректных строк и выполнения самой процедуры чмутинга.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ShowBad(arg0)` | arg0: Boolean | Empty | Устанавливает режим отображения некорректных (плохих) строк в дамп при выполнении чмутинга. Управляет внутренним выражением, передаваемым в окружение DUMP-функций. `TRUE` — показывать плохие строки (выражение `"1"`), `FALSE` — скрывать (выражение `"0"`). |
| `Run(arg0)` | arg0: Boolean (режим коррекции) | Integer | Выполняет процедуру чмутинга для выбранного тома с учётом установленных битов используемых строк. Загружает параметры проверки и коррекции из записей `Chmuting`, строит окружение для DUMP-функций и запускает процедуру проверки целостности данных. `TRUE` — чмутинг с коррекцией данных, `FALSE` — только проверка. Если том не задан, генерирует ошибку. |

## Порядок вызова

Основной сценарий использования класса Chmuting в языке EME-L: создание объекта, установка тома через `SetVolume`, настройка набора проверяемых записей через `CheckRecord` (выборочно) или `CheckAllRecords` (массово), при необходимости включение отображения плохих строк через `ShowBad`, и запуск процедуры проверки или коррекции методом `Run`. Требует валидного контекста выполнения и объекта-контейнера при создании.

## Примеры

Проверка целостности всех записей тома в языке EME-L (только режим проверки, без коррекции):

```EME-L
'Создать объект чмутинга в текущем контексте'
objChm = Object("Chmuting");
objChm.SetVolume("Main");
objChm.CheckAllRecords(TRUE);
Result = objChm.Run(FALSE);
'Result — код результата выполнения чмутинга'
```

Выборочная проверка с коррекцией и отображением плохих строк в системе EME.WMS:

```EME-L
objChm = Object("Chmuting", rContainer);
objChm.SetVolume("Archive");
objChm.CheckRecord("DOC_HEADER", TRUE);
objChm.CheckRecord("DOC_LINES", TRUE);
objChm.ShowBad(TRUE);
Result = objChm.Run(TRUE);
'Result — код результата, плохие строки выведены в дамп'
```

> Примеры синтезированы для документации: реальных вызовов `Object("Chmuting")` в выгрузке `Базовая выгрузка проекта/Классы/*.txt` на момент составления документации не обнаружено. Имена томов (`"Main"`, `"Archive"`) и идентификаторы записей (`"DOC_HEADER"`, `"DOC_LINES"`) — иллюстративные.

### Rack — класс для работы с топологией складского стеллажа

Класс `Rack` в языке EME-L — это объектная модель физической структуры складского стеллажа, построенная на основе данных сектора, колонн, ячеек и секций из базы данных EME.WMS. Класс предоставляет доступ к структурным характеристикам стеллажа (количество колонн, рядов, уровней, пролётов), к физическим размерам и координатам элементов (ширина пролётов, высота уровней, координаты стоек и колонн), к ссылкам на ячейки по координатам внутри стеллажа, а также к построению трёхмерной визуализации в движке Irrlicht и сериализации топологии в дерево свойств.

Объект класса `Rack` создаётся через `Object("Rack")` с пустой топологией. Для загрузки структуры необходимо вызвать метод `Attach`, передав ссылку на сектор склада — после этого объект готов к запросам размеров, координат и ссылок на ячейки в языке EME-L.

#### Создание объекта в языке EME-L

```EME-L
'Создать объект стеллажа с пустой топологией'
objRack = Object("Rack");
```

Класс `Rack` имеет единственный конструктор без аргументов — объект создаётся с пустой топологией, привязка к сектору выполняется отдельным методом `Attach`. После создания объект не содержит данных о колоннах, уровнях и ячейках — все методы доступа вернут нулевые значения, пока не вызван `Attach`.

#### Attach — привязка к сектору склада

Привязывает объект стеллажа к указанному сектору склада, загружает из базы данных колонны, ячейки, секции и уровни сектора, строит внутреннюю топологическую модель стеллажа в системе EME.WMS. Рассчитывает физические размеры пролётов, ширины секций, высоты уровней и координаты колонн на основе данных ячеек и их типовых размеров.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer | Да | Ссылка на сектор склада. Должна указывать на существующую запись в классификаторе секторов. |

**Возвращает**: `Empty`.

`Attach` выполняет полную загрузку топологии за один вызов. Если у сектора нет привязки к секциям (`GetSectionRef() == NULL_REF`), пролёты формируются автоматически по три колонны в каждом.

#### Структурные характеристики стеллажа в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetNoOfColumns()` | — | Integer | Общее количество колонн, загруженных в топологию стеллажа после привязки к сектору. |
| `GetNoOfRows(arg0)` | arg0: Integer (индекс колонны, с 0) | Integer | Количество рядов (строк ячеек) в указанной колонне стеллажа. |
| `GetNoOfLevels(arg0, arg1)` | arg0: Integer (индекс колонны, с 0), arg1: Integer (индекс ряда, с 0) | Integer | Количество уровней (ярусов) в указанном ряду заданной колонны стеллажа. |
| `GetBayIndexOfColumn(arg0)` | arg0: Integer (индекс колонны, с 0) | Integer | Индекс пролёта (секции), к которому относится указанная колонна стеллажа. |
| `GetNoOfBays()` | — | Integer | Количество пролётов в стеллаже. Пролёты формируются из секционной привязки или автоматически группируются по три колонны при отсутствии секций. |

#### Уровни и их высоты в пролётах

В системе EME.WMS уровни внутри пролёта могут иметь разную высоту, рассчитанную на основе данных о высотах ячеек из базы данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetNoOfLevelsInBay(arg0, arg1)` | arg0: Integer (индекс пролёта, с 0), arg1: Integer (индекс ряда, с 0) | Integer | Максимальное количество уровней среди всех колонн, входящих в указанный пролёт, для заданного ряда. При недопустимом индексе пролёта генерируется ошибка выполнения. |
| `GetLevelHeightInBay(arg0, arg1)` | arg0: Integer (индекс пролёта, с 0), arg1: Integer (индекс уровня, с 0) | Real | Абсолютная высота указанного уровня относительно пола в метрах. Пустые или отсутствующие высоты заменяются типовыми значениями. При недопустимом индексе уровня генерируется ошибка выполнения. |

#### Доступ к ячейкам по координатам в топологии стеллажа

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetCellRef(arg0, arg1, arg2)` | arg0: Integer (индекс колонны, с 0), arg1: Integer (индекс ряда, с 0), arg2: Integer (индекс уровня, с 0) | Integer | Ссылка на ячейку склада в указанной позиции стеллажа. Возвращает `NULL_REF` (`-1`), если комбинация индексов выходит за пределы загруженной топологии. |

> **Важно:** `NULL_REF` в языке EME-L равно `-1`, а не `0`. Нулевая ссылка является валидной строкой классификатора. Проверяйте результат через `If (cellRef != NULL_REF)`, а не `If (cellRef != 0)`.

#### Физические размеры и координаты элементов стеллажа

Все размеры возвращаются в метрах. Координаты Y пролётов центрируются относительно начала координат в системе EME.WMS. Размеры ячеек и сектора в базе данных хранятся в сантиметрах; класс `Rack` переводит их в метры с коэффициентом `0.01`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetBayWidth(arg0)` | arg0: Integer (индекс пролёта, с 0) | Real | Ширина пролёта стеллажа в метрах. Рассчитывается из суммарной ширины входящих в него ячеек с учётом типовых размеров. |
| `GetUprightWidth()` | — | Real | Ширина стойки стеллажа в метрах. Совпадает с высотой балки (`GetBeamHeight`) — внутренняя реализация делегирует вызов `GetBeamHeight()`. |
| `GetBeamHeight()` | — | Real | Высота балки стеллажа в метрах. Если высота не задана в секторе, по умолчанию `0.1` (10 см). |
| `GetUprightMinY(arg0)` | arg0: Integer (индекс пролёта, с 0) | Real | Y-координата левой (минимальной) границы указанного пролёта в метрах. См. примечание о расхождении ниже — метод работает в продакшен-скриптах, несмотря на отсутствие явно написанного диспетчера C++. |
| `GetUprightMaxY(arg0)` | arg0: Integer (индекс пролёта, с 0) | Real | Y-координата правой (максимальной) границы указанного пролёта в метрах. |
| `GetColumnY(arg0)` | arg0: Integer (индекс колонны, с 0) | Real | Y-координата центра указанной колонны в метрах. Рассчитывается как сумма минимальной Y-координаты пролёта и центрального смещения колонны внутри пролёта. |

#### Поиск индексов колонн и уровней по шифрам

В языке EME-L шифр — это номер элемента (колонны или уровня), как он задан в классификаторе базы данных EME.WMS. Поиск возвращает индекс этого элемента во внутренней топологии стеллажа.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetColumnIndex(arg0)` | arg0: Integer (номер/шифр колонны) | Integer | Индекс колонны в топологии стеллажа. Возвращает `-1`, если колонна с указанным шифром отсутствует. |
| `GetLevelIndex(arg0)` | arg0: Integer (номер/шифр уровня) | Integer | Индекс уровня в топологии стеллажа. Возвращает `-1`, если уровень с указанным шифром отсутствует. |

#### Трёхмерная визуализация и сериализация стеллажа

Метод `AddRack` создаёт полигональную сетку стеллажа для движка Irrlicht (доступна только при сборке с `IRRLICHT_IS_AVAILABLE`). Метод `SaveRack` сериализует топологию в дерево свойств для передачи на клиент или сохранения.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddRack(arg0)` | arg0: IrrNode (родительский узел сцены Irrlicht) | IrrMesh или Empty | Создаёт трёхмерную полигональную сетку стеллажа и добавляет её к родительскому узлу Irrlicht. Формирует геометрию балок для всех уровней всех пролётов с масштабированием по реальным размерам стеллажа, применяет текстуры и нормальные карты. Требует наличия библиотеки Irrlicht и 3D-моделей. При отсутствии поддержки Irrlicht возвращает пустое значение. |
| `SaveRack(arg0)` | arg0: PropertyTree (дерево свойств) | Empty | Сохраняет полную топологию стеллажа в дерево свойств: уровни, колонны, ряды, ячейки, пролёты и их размеры. Для каждой ячейки дополнительно записывает номер, объём, свободный объём, используемый объём, высоту, признак физической пустоты и зазор. Также сохраняет координаты центра сектора, угол поворота, номер сектора и признак расположения табличек. См. [PropertyTree](./PropertyTree.md). |

| Константа | Значение | Назначение |

#### Примеры использования класса Rack в языке EME-L

Загрузка топологии стеллажа и перебор всех ячеек:

```EME-L
'Создать стеллаж и привязать к сектору склада'
objRack = Object("Rack");
objRack.Attach(r_Sector);

'Перебрать все колонны, ряды и уровни'
nColumns = objRack.GetNoOfColumns();
For (c = 0; c < nColumns; c = c + 1)
    nRows = objRack.GetNoOfRows(c);
    For (r = 0; r < nRows; r = r + 1)
        nLevels = objRack.GetNoOfLevels(c, r);
        For (l = 0; l < nLevels; l = l + 1)
            cellRef = objRack.GetCellRef(c, r, l);
            If (cellRef != NULL_REF)
                'cellRef — действительная ссылка на ячейку в БД EME.WMS'
            End If
        End For
    End For
End For
```

Поиск колонны по шифру и получение её физических координат (паттерн из `Wms3DRackComponent.txt`):

```EME-L
'Найти индекс колонны по номеру из классификатора'
colIdx = objRack.GetColumnIndex(r_Column.GetNumber());
If (colIdx != -1)
    yPos = objRack.GetColumnY(colIdx);
    'yPos — координата центра колонны в метрах'
End If
```

Расчёт габаритов и высоты стеллажа по всем пролётам (паттерн из `Wms3DRackComponent.txt`):

```EME-L
'Обойти все пролёты, просуммировать длину и найти максимальную высоту'
dRackLength = 0.0;
MaxHeight = 0;
iNoOfBays = objRack.GetNoOfBays();
For (BayIndex = 0; BayIndex < iNoOfBays; BayIndex = BayIndex + 1)
    NoOfLevels = objRack.GetNoOfLevelsInBay(BayIndex, 0);
    If (NoOfLevels == 1)
        LevelHeight = 0;
    Else
        LevelHeight = objRack.GetLevelHeightInBay(BayIndex, NoOfLevels - 1);
    End If
    If (LevelHeight > MaxHeight)
        MaxHeight = LevelHeight;
    End If
    dRackLength = dRackLength + objRack.GetBayWidth(BayIndex);
End For
```

#### Расхождения с реализацией C++ (i_Rack.cpp)

В файле `i_Rack.cpp` обнаружены следующие расхождения между регистрацией методов в `BEGIN_IMPLEMENT(CIRack)` и их телами в C++:

- **Дубликат диспетчера `GetUprightWidth`.** В строках 683–686 определён корректный диспетчер `COleVariant CIRack::GetUprightWidth(int argn, COleVariant args[])`, возвращающий `m_dBeamHeight`. В строках 699–702 определён **второй** диспетчер с тем же именем и сигнатурой, тело которого вызывает `GetUprightWidth(variant2integer(args[0]))` — но внутренний `double CIRack::GetUprightWidth()` не принимает аргументов. Второе определение либо отбрасывается линкером, либо является мёртвым кодом.
- **Отсутствует явно написанный диспетчер `GetUprightMinY(int, COleVariant[])`.** Внутренний помощник `double CIRack::GetUprightMinY(long lBayIndex)` существует (строки 695–698) и возвращает `m_Bays[lBayIndex].m_dMinY`, но соответствующая функция-диспетчер для EME-L не определена. Тем не менее вызов `Rack.GetUprightMinY(bay)` из EME-L **реально работает** в продакшен-скрипте `Wms3DRackComponent.txt` (функция `GetRackAsJSON`), возвращая корректные Y-координаты. По-видимому, инфраструктура `BEGIN_IMPLEMENT/END_IMPLEMENT` генерирует диспетчер на основе таблицы регистрации `IMPLEMENT_METHOD`, а не требует явно написанной функции-диспетчера с совпадающим именем C++.
- **`GetUprightWidth` — синоним `GetBeamHeight`.** Внутренняя реализация `double CIRack::GetUprightWidth()` просто вызывает `GetBeamHeight()`. Концептуально ширина стойки и высота балки — разные физические величины, но в данной реализации они всегда равны. Это видно и из EME-L-примеров в системе EME.WMS, где `Rack.GetUprightWidth()` используется как толщина стойки, а `Rack.GetBeamHeight()` — как высота балки.

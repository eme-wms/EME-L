# Класс MemoryLeaks

> Синонимы для поиска: CIMemoryLeaks MemoryLeaks класс EME-L, детектор утечек памяти EME-L, утечки памяти EME-L, memory leak detection EME-L, Snapshot Checkpoint DumpAllObjectsSince, снимок памяти EME-L, контрольная точка памяти EME-L, отладочная сборка EME-L, _DEBUG CMemoryState EME-L, катастрофический лог EME-L, developer_error память EME-L, диагностические сообщения утечек EME-L, autodumpname EME-L

Класс `MemoryLeaks` в языке EME-L — диагностический инструмент для обнаружения утечек памяти в отладочной сборке приложения. Класс MemoryLeaks в языке EME-L предоставляет механизм контроля за динамическим выделением памяти путём создания снимков состояния кучи и последующего сравнения с текущим состоянием. При обнаружении утечек класс MemoryLeaks формирует диагностические сообщения с детализацией по блокам памяти и выводит список неосвобождённых объектов.

Класс MemoryLeaks в языке EME-L доступен **только в отладочной сборке** приложения (макрос `_DEBUG` без `_AFX_NO_DEBUG_CRT`). В релизной сборке все методы класса MemoryLeaks являются пустыми операциями (no-op) — тело конструктора, `Snapshot`, `Checkpoint` и `DumpAllObjectsSince` не выполняют никаких действий. Это позволяет оставлять вызовы `MemoryLeaks` в скриптах EME-L без ущерба для производительности production-среды.

Класс MemoryLeaks в системе EME.WMS поддерживает два режима работы:

- **Ручной контроль** — вызов `Snapshot` для фиксации базового состояния, затем `Checkpoint` в контрольных точках для проверки расхождений.
- **Автоматический контроль** — при создании объекта с указанием имени точки автоматически выполняется `Snapshot`, а при уничтожении объекта — `Checkpoint` с этим именем. Если обнаружены утечки, результат выводится в TRACE и катастрофический лог, после чего вызывается `DumpAllObjectsSince`.

## Создание объекта класса MemoryLeaks

Класс `MemoryLeaks` в языке EME-L имеет одну перегрузку конструктора с одним необязательным аргументом.

```EME-L
'Безымянный детектор — ручной режим, автопроверка отключена'
objLeaks = Object("MemoryLeaks");
```

```EME-L
'Именованный детектор — Snapshot при создании, автопроверка при уничтожении'
objLeaks = Object("MemoryLeaks", "ИмпортДанных");
```

При передаче имени точки конструктор класса MemoryLeaks в языке EME-L автоматически фиксирует снимок памяти (`Snapshot`) и настраивает автоматическую проверку: при уничтожении объекта вызывается `Checkpoint` с указанным именем. Если имя не задано (пустая строка по умолчанию), автоматическая проверка отключена — разработчик должен вызывать `Snapshot` и `Checkpoint` вручную.

## Методы класса MemoryLeaks

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Snapshot()` | — | Empty | Фиксирует текущее состояние выделенной памяти (снимок) для последующего сравнения методом `Checkpoint`. Используется как базовая точка отсчёта при проверке утечек памяти в языке EME-L. |
| `Checkpoint(имяТочки)` | имяТочки: String | Empty | Сравнивает текущее состояние памяти с последним снимком, зафиксированным методом `Snapshot`. При обнаружении расхождений формирует диагностическое сообщение (см. поведение ниже). |
| `DumpAllObjectsSince()` | — | Empty | Выводит список объектов, выделенных в памяти с момента последнего вызова `Snapshot`. Результат направляется в отладочный вывод (afxDump/TRACE) в системе EME.WMS. |

### Поведение Checkpoint при обнаружении утечек памяти

Метод `Checkpoint` класса MemoryLeaks в языке EME-L использует два пути обработки в зависимости от источника вызова:

- **При автоматическом вызове из деструктора** (имя точки совпадает с заданным в конструкторе): результат выводится в `TRACE` и катастрофический лог (`catastrophic_log`), после чего вызывается `DumpAllObjectsSince` для детального дампа всех объектов, выделённых с момента снимка.
- **При ручном вызове из скрипта** EME-L: отправляется сообщение разработчику (`developer_error`) с детализацией по блокам памяти — количеством и размером утечек по каждому типу блока (`m_lCounts`, `m_lSizes`).

Если расхождений в состоянии памяти не обнаружено, метод `Checkpoint` класса MemoryLeaks не формирует никаких сообщений.

## Примеры использования класса MemoryLeaks

Ручной режим: снимок перед блоком обработки и проверка после в языке EME-L:

```EME-L
'Создать детектор и зафиксировать состояние памяти до обработки'
objLeaks = Object("MemoryLeaks");
objLeaks.Snapshot();

'...блок кода, потенциально вызывающий утечки...'

'Проверить наличие утечек после обработки'
objLeaks.Checkpoint("ПослеИмпорта");
```

Автоматический режим: именованный детектор проверяет утечки при выходе из области видимости в системе EME.WMS:

```EME-L
'Создать детектор с именем точки — Snapshot выполнится автоматически'
objLeaks = Object("MemoryLeaks", "ЗагрузкаКлиентов");

'...обработка, в ходе которой выделяется память...'

'При уничтожении объекта автоматически вызовется Checkpoint("ЗагрузкаКлиентов")'
'Если обнаружены утечки — вывод в TRACE и катастрофический лог'
```

Пошаговая диагностика с несколькими контрольными точками в классе MemoryLeaks языка EME-L:

```EME-L
objLeaks = Object("MemoryLeaks");
objLeaks.Snapshot();

'...первый этап обработки...'
objLeaks.Checkpoint("Этап1_Импорт");

'...второй этап обработки...'
objLeaks.Checkpoint("Этап2_Конвертация");

'Детальный дамп объектов, оставшихся с момента Snapshot'
objLeaks.DumpAllObjectsSince();
```

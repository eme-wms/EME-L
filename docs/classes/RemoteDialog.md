## Класс RemoteDialog — сериализация диалога в языке EME-L

Класс `RemoteDialog` (CIName `CIRemoteDialog`) в языке EME-L — класс для сериализации состояния диалогового окна и передачи его между клиентом и сервером. Представляет собой снимок диалога, который позволяет заполнять и изменять значения органов управления диалога на сервере без прямого доступа к GUI-объектам. В системе EME.WMS и языке EME-L класс RemoteDialog используется для выполнения серверных процедур, требующих доступа к данным диалога, без необходимости передачи самих GUI-объектов.

Объект класса RemoteDialog привязывается к указанному диалогу, а считывание реальных значений органов выполняется при сериализации для передачи на сервер. На стороне сервера скрипт получает десериализованный объект RemoteDialog, обращается к органам через методы GetControl или C (сокращённое имя), меняет их значения методами Put и PutToRow и управляет доступностью через Disable. При обратной десериализации на клиенте изменения автоматически синхронизируются с исходным диалогом.

### Создание объекта класса RemoteDialog в языке EME-L

Класс `RemoteDialog` в языке EME-L имеет один конструктор — с одним аргументом. Конструктор принимает ссылку на объект диалога, состояние которого требуется сериализовать. Извлекаются внутренние данные диалога и определяется имя объекта для формирования полных имён органов (формат `«Объект.Орган»`). Само считывание значений органов выполняется позже — при последующей сериализации объекта.

```EME-L
'Конструктор с привязкой к существующему диалогу'
rd = Object("RemoteDialog", dlg);
```

После создания на клиенте объект сериализуется для передачи на сервер; на сервере скрипт получает уже десериализованный объект через аргументы метода и работает с ним через методы доступа.

### Методы доступа к органам в языке EME-L

После десериализации на сервере орган диалога представляется объектом `CRemoteControl` (внутренний класс, недоступный для прямого создания через `Object()`). Ссылка на него возвращается методами GetControl или C, после чего через неё вызываются методы Put, PutToRow и Disable класса CRemoteControl. В языке EME-L эти методы работают только с десериализованным снимком — вызов GetControl при наличии привязанного к серверу диалога генерирует внутреннюю ошибку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetControl(arg0)` | arg0: String (полное имя в формате `«Объект.Орган»`) | CRemoteControl | Сериализованный орган диалога по полному имени. Предназначен для вызова на стороне сервера с десериализованным объектом; при наличии привязанного диалога генерируется внутренняя ошибка. |
| `C(arg0)` | arg0: String (сокращённое имя органа без указания объекта) | CRemoteControl | Сериализованный орган по сокращённому имени. Автоматически дополняет имя именем объекта диалога, формируя полное имя `«Объект.Орган»`. |

### Методы класса CRemoteControl в языке EME-L

Методы вызываются на ссылке, возвращённой GetControl или C. Все методы возвращают пустое значение (`null`/Empty). В языке EME-L нумерация строк таблицы начинается с 0.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Put(arg0)` | arg0: любой тип (новое значение органа) | Empty | Устанавливает значение сериализованного органа диалога в строку 0. |
| `PutToRow(arg0, arg1)` | arg0: любой тип (значение), arg1: Integer (номер строки таблицы, с 0) | Empty | Устанавливает значение сериализованного органа диалога в указанную строку таблицы. |
| `Disable(arg0)` | arg0: Boolean / `"1"` / `"0"` / `"Да"` / `"Нет"` | Empty | Устанавливает флаг доступности органа «только для чтения» в строке 0. |

Флаг `только_чтение` метода Disable в языке EME-L принимает любой «булевский» тип. TRUE: `TRUE`, `"1"`, `"Да"` — орган доступен только для чтения. FALSE: `FALSE`, `"0"`, `"Нет"` — орган доступен для редактирования.

### Типы сериализуемых органов в системе EME.WMS

При сериализации на стороне клиента в снимок включаются только органы следующих типов (определяется внутренним флагом `CNTRL_TYPE` органа диалога): `CT_EDITOR` — поле ввода/редактора; `CT_COMBOBOX` — выпадающий список; `CT_BUTTON` — кнопка; `CT_RADIOBUTTON` — переключатель. Органы других типов (статические подписи, рамки и пр.) в снимок не попадают и не могут быть получены через GetControl или C. Для табличных столбцов значения считываются и записываются построчно — методы PutToRow и строка 0 в Put/Disable соответствуют строкам связанной таблицы диалога.

### Примеры работы с RemoteDialog в языке EME-L

Заполнение органа на стороне сервера по полному имени:

```EME-L
'На сервере: rd получен десериализацией аргумента'
ctl = rd.GetControl("Obj.header_field");
ctl.Put("Новое значение");
ctl.Disable(TRUE);
```

То же по сокращённому имени с автоматическим добавлением объекта:

```EME-L
ctl = rd.C("header_field");
ctl.Put("Новое значение");
```

Заполнение табличного органа по строкам:

```EME-L
'Установить значение органа в строки 0, 1, 2'
ctl = rd.C("table_column");
ctl.PutToRow("Значение 1", 0);
ctl.PutToRow("Значение 2", 1);
ctl.PutToRow("Значение 3", 2);
```

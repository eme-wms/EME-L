# Класс DivisionDataStorage

> Синонимы для поиска: CIDivisionDataStorage DivisionDataStorage класс EME-L, межразделный перенос данных EME-L, перенос данных между разделами EME-L, ExportLine AllowNullRef Import GetCount GetDivisionNo GetDivisionNoTo GetContent, NULL_REF, DIVISION DIVISION_TO, smart_find_record smart_find_field, write_import_data, раздел базы данных EME-L, сервер терминалов EME.WMS

Класс `DivisionDataStorage` в языке EME-L предназначен для переноса данных раздела из одного раздела базы данных в другой. Класс `DivisionDataStorage` в системе EME.WMS используется в межразделных процедурах: один раздел (например, сервер терминалов с номером 2) наполняет объект экспортируемыми строками, а другой раздел (основной, с номером 0) импортирует их в свою базу данных. Класс реализует контролируемый перенос: фиксируются номера исходного и целевого разделов, а разименование ссылок при импорте выполняется по строгим правилам.

Объект класса `DivisionDataStorage` в языке EME-L создаётся через `Object()` без аргументов — раздел, выполняющий импорт, создаёт пустой объект и передаёт его в процедуру раздела-экспортёра. Объект не существует вне межразделного сценария: его единственное назначение — аккумулировать строки записей на стороне одного раздела для последующей записи в базу другого раздела.

Номер целевого раздела (`DIVISION_TO`) фиксируется при создании объекта и равен текущему разделу. Номер исходного раздела (`DIVISION`) обновляется при **каждом** вызове метода `ExportLine` и соответствует разделу, из которого выполняется экспорт. Все вызовы `ExportLine` в один объект должны идти из одного раздела — иначе номер раздела-экспортёра будет переписан последним вызовом. Эти номера сохраняются при сериализации объекта и доступны через геттеры `GetDivisionNo` и `GetDivisionNoTo`.

## Создание объекта класса DivisionDataStorage

Конструктор класса `DivisionDataStorage` в языке EME-L параметров не имеет. Объект должен быть создан разделом, который будет выполнять `Import` в свою базу — именно поэтому номер целевого раздела автоматически устанавливается равным `current_division_no()` на момент создания.

```EME-L
'Создать пустое хранилище для приёма данных из другого раздела'
objStorage = Object("DivisionDataStorage");
```

В деструкторе объект освобождает внутренние структуры автоматически — операторов `new`/`delete` в EME-L нет. Передача объекта между разделами выполняется через процедурный механизм: объект передаётся аргументом в процедуру, исполняемую в контексте другого раздела.

## Экспорт данных класса DivisionDataStorage

Методы наполнения объекта данными на стороне раздела-экспортёра в языке EME-L. Раздел-экспортёр — это раздел, из которого строки записей переносятся в объект `DivisionDataStorage`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ExportLine(arg0, arg1)` | arg0: Integer (номер записи), arg1: Integer (номер строки) | Empty | Экспортирует строку записи из текущего раздела в объект. При **каждом** вызове обновляет номер текущего раздела как раздел-экспортёр (`DIVISION`). |
| `AllowNullRef(arg0, arg1)` | arg0: Integer/String (запись), arg1: Integer/String (поле) | Empty | Разрешает для указанного ссылочного поля неразименованные ссылки при импорте: вместо исключения записывается `NULL_REF` (-1). Каждый вызов добавляет одну пару (запись, поле). |

Метод `ExportLine` в языке EME-L при каждом вызове фиксирует номер текущего раздела в качестве раздела-экспортёра (`current division`). Номер раздела переписывается при каждом вызове, поэтому вызовы из разных разделов в один объект приводят к некорректному состоянию — все вызовы `ExportLine` должны идти из одного раздела.

При экспорте данных, содержащих ссылки, они должны быть логически замкнуты: ссылки должны указывать либо на мастер-данные, либо на строки, также добавленные в объект. Например, при экспорте строки документа необходимо также экспортировать заголовок документа — иначе разименование ссылки на заголовок при импорте завершится неудачей и (если поле не в списке `AllowNullRef`) сгенерирует исключение.

Метод `AllowNullRef` в системе EME.WMS принимает как номер записи/поля, так и их имена. При передаче строки выполняется поиск записи по имени через `smart_find_record`, а поля — через `smart_find_field(rec, ...)`. Поиск по имени делает вызов устойчивым к перенумерации записей и полей между разделами.

## Импорт данных класса DivisionDataStorage

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Import()` | — | Empty | Выполняет импорт данных из объекта в базу данных текущего раздела. Для каждой экспортированной строки разименовывает ссылочные поля. |

Метод `Import` в языке EME-L должен вызываться на стороне целевого раздела (`DIVISION_TO`). Для каждой экспортированной строки выполняется разименование ссылочных полей. Если разименование ссылки не удалось и поле не указано в списке `AllowNullRef`, генерируется исключение. Для полей из списка `AllowNullRef` вместо исключения в значение ссылочного поля записывается `NULL_REF` (-1).

Импорт данных в системе EME.WMS выполняется внутри транзакции: открывается `is_transaction(1, "описание")`, затем вызывается `Import()`, затем `is_transaction(-1)`. Все проверки и подготовка данных должны быть выполнены до открытия транзакции.

## Инспекция объекта класса DivisionDataStorage

Методы чтения состояния объекта `DivisionDataStorage` в языке EME-L. Эти методы можно вызывать на любой стороне (экспортёра или импортёра) для контроля наполнения объекта.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetCount()` | — | Integer | Количество экспортированных строк записей, хранящихся в объекте. |
| `GetDivisionNo()` | — | Integer | Номер раздела базы данных, из которого выполняется экспорт. Обновляется при каждом вызове `ExportLine`. |
| `GetDivisionNoTo()` | — | Integer | Номер раздела базы данных, в который будет выполнен импорт. При создании объекта устанавливается равным текущему разделу. |
| `GetContent()` | — | String | Текстовое представление содержимого для отладки. Формат: `DIVISION=<n>; DIVISION_TO=<n>; <запись>:<строка>; ...`. Пустая строка при отсутствии данных. |

Метод `GetContent` в языке EME-L предназначен для отладки: строка формируется только при наличии данных, иначе возвращается пустая строка. Формат включает номер раздела-экспортёра, целевого раздела и список пар «запись:строка».

## Примеры

> Примеры синтезированы на основе сигнатур и описаний методов в C++ — реальных использований класса `DivisionDataStorage` в выгрузке проекта не найдено. Класс узкоспециализированный и применяется во внутренних межразделных процедурах EME.WMS.

Межразделный перенос данных в системе EME.WMS. Раздел 0 (основной) создаёт объект и передаёт его в процедуру раздела 2 (сервер терминалов), который наполняет его строками подтверждённых приказов, затем раздел 0 импортирует данные в свою базу:

```EME-L
'Сторона раздела 0: создать пустой объект и вызвать процедуру раздела 2'
objStorage = Object("DivisionDataStorage");
is_run_procedure_in_division(2, "ExportOrdersToStorage", objStorage);

'Сторона раздела 0: импорт данных в базу раздела 0'
is_transaction(1, "Импорт данных из раздела 2");
objStorage.Import();
is_transaction(-1);
```

Процедура `ExportOrdersToStorage` на стороне раздела 2 наполняет объект:

```EME-L
'Сторона раздела 2: наполнить объект строками подтверждённых приказов'
Loop (objQuery)
    nRec = objQuery.Get_r();
    nLine = objQuery.Get_l();
    objStorage.ExportLine(nRec, nLine);
End Loop

'Разрешить неразименованные ссылки в поле «Документ-основание» записи заказа'
objStorage.AllowNullRef("Order", "SourceDoc");
```

Проверка содержимого объекта перед импортом в языке EME-L:

```EME-L
If (objStorage.GetCount() > 0)
    'Логирование: номер раздела-экспортёра и количество строк'
    sLog = objStorage.GetContent();
    is_message("Импорт", sLog);
End If
```

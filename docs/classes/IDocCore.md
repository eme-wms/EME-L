# Класс IDocCore

> Синонимы для поиска: CIIDocCore IDocCore класс EME-L, IDoc ядро EME-L, XML документ по схеме EME-L, PropertyTree по схеме EME-L, схема IDoc EME-L, типизированный доступ к XML EME-L, Import XSD WSDL JSON схема EME-L, Compress схема IDoc EME-L, AsQuery AsXml схема EME-L, GetSchemaItem SchemaItem EME-L, DefineNamespace CastToXml EME-L, SetStrictedOrder EnableEmptyValues EME-L, GetAttrib SetAttrib EME-L, addRoot getRoot getContent EME-L, Reset IDocCore EME-L, сегменты IDoc EME-L, пространства имён IDoc EME-L

Класс `IDocCore` в языке EME-L — базовый программный интерфейс к структурированным данным IDoc. Объект класса `IDocCore` в системе EME.WMS связывается с записью «Схемы IDoc» по коду, загружает структуру схемы и обеспечивает типизированный доступ к узлам XML-документа или дерева свойств (PropertyTree). Класс `IDocCore` в языке EME-L предоставляет операции для чтения и записи значений узлов с автоматическим приведением к типу, заданному схемой; добавления сегментов и узлов; навигации по списку дочерних узлов; управления атрибутами, пространствами имён, режимом строгого порядка сегментов и разрешением пустых тегов.

Класс `IDocCore` в системе EME.WMS поддерживает импорт схемы из внешних источников (XSD, WSDL, JSON, XML), сжатие схемы и выгрузку её структуры в объект Query или Xml. Класс `IDocCore` используется в прикладных скриптах EME-L как низкоуровневый слой для работы с XML/PropertyTree по схеме. Класс `IDoc` (отдельный класс, не рассматривается на этой странице) наследует функциональность `IDocCore` и расширяет её динамическими методами `Get<Имя>`, `Put<Имя>`, `Add<Имя>`, `Tune<Имя>`, формируемыми по именам полей схемы.

Объект `IDocCore` в языке EME-L создаётся через `Object()`. Источник данных указывается аргументом конструктора: объект `Xml` для работы с XML-документом либо объект `PropertyTree` для работы с деревом свойств. Часть методов (атрибуты, пространства имён, `CastToXml`) реализована только для источника Xml и не применима к PropertyTree.

## Создание объекта класса IDocCore

Класс `IDocCore` в языке EME-L имеет два конструктора — по числу аргументов.

```EME-L
'Конструктор без аргументов — пустой объект без схемы и источника'
objIDoc = Object("IDocCore");
```

```EME-L
'Конструктор с кодом схемы и источником данных'
objIDoc = Object("IDocCore", "ORDERS05", objXml);
```

Конструктор без аргументов класса `IDocCore` в языке EME-L создаёт объект без привязки к схеме. Такой объект может быть инициализирован позже методом `Reset`. Конструктор с двумя аргументами немедленно загружает схему по коду из записи «Схемы IDoc» и привязывает источник данных: объект `Xml` или объект `PropertyTree`.

## Навигация и работа с узлами IDocCore

### Получение содержимого и корневого узла

Методы класса `IDocCore` в языке EME-L для получения XML-представления документа и корневого узла схемы.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `getContent()` | — | String / Empty | XML-представление документа (текст XML). Для источника PropertyTree возвращает Empty. |
| `getRoot()` | — | IDocCore (ссылка) | Объект, настроенный на корневой узел схемы. Корневой элемент должен иметь тип Root; для PropertyTree генерируется ошибка. |
| `addRoot()` | — | IDocCore (ссылка) | Добавляет корневой узел, описанный схемой. Применим только к самому XML-документу и только при типе корня Root. |
| `addRoot(namespace)` | namespace: String | IDocCore (ссылка) | То же с указанием пространства имён корневого узла. |

### Чтение и запись значений узлов

Методы `Get` и `Put` класса `IDocCore` в языке EME-L обеспечивают типизированный доступ к узлам и атрибутам. Значение автоматически приводится к типу, заданному схемой (String, Integer, Real, Date, Time, Boolean). Если узел является сегментом, вызов `Put` генерирует ошибку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Get(nodeName)` | nodeName: String | String / Integer / Real / Date / Time / Boolean / IDocCore | Содержимое узла или атрибута. Для сегмента возвращает ссылку на объект, настроенный на список дочерних узлов. Для скалярного поля — значение, приведённое к типу схемы. |
| `Put(nodeName, value)` | nodeName: String, value: любой тип | Empty | Задаёт значение узла или атрибута с автоматическим приведением типа. Если узел — сегмент, генерируется ошибка. |
| `Put(nodeName, value, namespace)` | nodeName: String, value: любой тип, namespace: String | Empty | То же с указанием пространства имён создаваемого узла. |

### Добавление узлов и проверка существования

Методы класса `IDocCore` в языке EME-L для добавления новых узлов в текущий сегмент и проверки их наличия. Метод `Add` применим только к сегментам.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Add(nodeName)` | nodeName: String | IDocCore (ссылка) | Добавляет новый узел в текущий сегмент и возвращает объект, настроенный на добавленный узел. |
| `Add(nodeName, namespace)` | nodeName: String, namespace: String | IDocCore (ссылка) | То же с указанием пространства имён создаваемого узла. |
| `Tune(nodeName)` | nodeName: String | Boolean | Проверяет существование узла или атрибута. TRUE — существует, FALSE — отсутствует. |

### Перебор дочерних узлов

Методы `SetFirstLine`, `IsValidLine`, `SetNextLine` класса `IDocCore` в языке EME-L образуют стандартный цикл перебора дочерних узлов. Для XML указатель перемещается по дочерним узлам, для PropertyTree — по строкам дерева.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает указатель на первый элемент списка. |
| `IsValidLine()` | — | Boolean | TRUE — текущий узел/строка валиден. FALSE — достигнут конец списка. |
| `SetNextLine()` | — | Empty | Перемещает указатель на следующий элемент списка. |

Для итерации по записям и массивам в языке EME-L предпочтительнее оператор `Loop`, который автоматически вызывает методы `SetFirstLine` / `IsValidLine` / `SetNextLine`.

## Управление схемой IDocCore

### Импорт и преобразование схемы

Методы класса `IDocCore` в языке EME-L для импорта схемы из внешних источников, сжатия и выгрузки структуры. Метод `Reset` переинициализирует объект без создания нового.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Reset(schemaCode, sourceObject)` | schemaCode: String, sourceObject: Xml / PropertyTree | Empty | Переинициализирует объект: уничтожает текущий XML-документ и заново связывает объект с указанной схемой и источником. |
| `Import(source, prevSchema, format, emelClassRef)` | source: Xml / String / PropertyTree, prevSchema: String, format: String, emelClassRef: Integer | String | Импортирует схему IDoc из внешнего источника, объединяет с предыдущей версией и возвращает текст обновлённой схемы. |
| `Compress(schemaString)` | schemaString: String | String | Удаляет из схемы пустые элементы без дочерних узлов и возвращает сжатую схему. |
| `AsQuery(schemaString, query)` | schemaString: String, query: Query | Empty | Выгружает структуру схемы в запрос. Если запрос пустой, добавляет поля ID, ParentID, Level, Tag, Name, Type, Help. |
| `AsXml(xmlObject)` | xmlObject: Xml | Empty | Выгружает структуру схемы в объект Xml в виде узлов SchemaItem. |
| `QueryToString(query)` | query: Query | String | Преобразует запрос (с полями структуры схемы) в текст схемы IDoc. |
| `XmlToString(xmlObject)` | xmlObject: Xml | String | Преобразует объект Xml с узлами SchemaItem в текст схемы IDoc. |

Значения аргумента `format` метода `Import` класса `IDocCore` в языке EME-L:

| Код | Источник | Описание |
|-----|----------|----------|
| `"IDOC"` | Xml / String (путь) | XSD-схема в формате IDoc |
| `"SOAP"` | Xml / String (путь) | XSD-схема в формате SOAP |
| `"XSD"` | Xml / String (путь) | Файл XSD-схемы |
| `"WSDL"` | Xml / String (путь) | Файл WSDL |
| `"JSON"` | PropertyTree | Дерево свойств JSON |
| `"XML"` | Xml / String (путь) | XML-документ-образец |

### Элемент схемы SchemaItem

Метод `GetSchemaItem` класса `IDocCore` в языке EME-L возвращает объект `SchemaItem`, описывающий один элемент схемы (поле, атрибут или сегмент). Объект `SchemaItem` предоставляет доступ к тегу, программному имени, типу данных и справочному описанию элемента.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetSchemaItem(nodeName)` | nodeName: String (по умолчанию пустая строка) | SchemaItem (ссылка) | Элемент схемы для указанного узла. При пустой строке — текущий элемент схемы. |

Методы объекта `SchemaItem` класса `IDocCore` в языке EME-L:

| Метод | Аргументы | Возвращает | Описание |

| `GetTag()` | — | String | Тег элемента — имя поля, атрибута или сегмента в исходном XML/JSON. |
| `GetName()` | — | String | Программное имя элемента, используемое для формирования имён методов (`Get<Имя>`, `Put<Имя>`, `Add<Имя>`, `Tune<Имя>`) и навигации по схеме. |
| `GetType()` | — | String | Тип данных элемента. |
| `GetHelp()` | — | String | Справочное описание элемента. Пустая строка, если описание не задано. |

Типы элементов схемы, возвращаемые методом `GetType()` объекта `SchemaItem` в языке EME-L:

| Значение | Описание |

## Атрибуты и пространства имён IDocCore

Методы работы с атрибутами и пространствами имён класса `IDocCore` в языке EME-L реализованы только для источника Xml. Для источника PropertyTree эти методы не доступны и их вызов может привести к ошибке.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetAttrib(name)` | name: String | String | Значение атрибута текущего XML-узла. |
| `GetAttrib(name, defaultValue)` | name: String, defaultValue: String | String | Значение атрибута или `defaultValue`, если атрибут отсутствует. |
| `SetAttrib(name, value)` | name: String, value: String | Xml (ссылка) | Задаёт значение атрибута текущего XML-узла. |
| `DefineNamespace(alias, namespace)` | alias: String, namespace: String | Empty | Регистрирует псевдоним для пространства имён в схеме и XML-документе. |
| `CastToXml()` | — | Xml (ссылка) | Преобразует текущий узел IDoc в объект Xml. Результат кэшируется внутри объекта. |

## Режимы работы IDocCore

Методы класса `IDocCore` в языке EME-L для управления режимами записи: строгим порядком добавления сегментов и разрешением пустых значений для нестроковых полей.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetStrictedOrder(flag)` | flag: Boolean | Empty | Включает (TRUE) или отключает (FALSE) режим строгого порядка: дочерние узлы можно добавлять только в порядке, определённом схемой. По умолчанию FALSE. |
| `EnableEmptyValues(flag)` | flag: Boolean | Empty | Разрешает (TRUE) запись пустых тегов для полей, тип которых отличен от строки. FALSE (по умолчанию) — пустые значения для нестроковых полей игнорируются. |

## Примеры использования IDocCore

Создание объекта с XML-источником и запись значения узла в языке EME-L:

```EME-L
'Создать Xml-документ и привязать схему IDoc'
objXml = Object("Xml");
objIDoc = Object("IDocCore", "ORDERS05", objXml);

'Добавить корневой узел и записать поле'
objRoot = objIDoc.addRoot();
objRoot.Put("DOCNUM", "1234567890");
```

Чтение значения узла из существующего XML в языке EME-L:

```EME-L
objXml = Object("Xml");
objXml.LoadFile("D:\\Data\\order.xml");

objIDoc = Object("IDocCore", "ORDERS05", objXml);
objRoot = objIDoc.getRoot();
DocNum = objRoot.Get("DOCNUM");
```

Перебор дочерних сегментов класса `IDocCore` в системе EME.WMS:

```EME-L
objSegment = objRoot.Get("E1EDP01");
objSegment.SetFirstLine();
While (objSegment.IsValidLine())
    Matnr = objSegment.Get("MATNR");
    objSegment.SetNextLine();
End While
```

Проверка существования узла и работа с атрибутами в языке EME-L:

```EME-L
If (objRoot.Tune("E1EDK01"))
    objHeader = objRoot.Get("E1EDK01");
    Msgfn = objHeader.GetAttrib("MSGFN", "");
End If
```

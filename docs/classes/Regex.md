### Regex — класс регулярных выражений в языке EME-L

Класс `Regex` (C++ `CIRegex`) в языке EME-L — класс для работы с регулярными выражениями на базе библиотеки boost::regex. Предназначен для поиска, проверки сопоставления и замены текста по шаблонам регулярных выражений. Поддерживает группы захвата, нумерованные подстановки в строке замены (спецкоды `%N`), а также поиск отдельных и множественных вхождений в строке.

В системе EME.WMS класс `Regex` используется в прикладных скриптах EME-L для валидации строковых данных (например, e-mail, телефонов, артикулов), разбора строк на компоненты через группы захвата и переформатирования текста. Объект класса `Regex` создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс.

Основной сценарий использования класса `Regex` в языке EME-L: создание объекта с установкой регулярного выражения через `SetExpression`, поиск соответствий методами `Match` и `Search`, получение позиции, длины и содержимого групп через `Position`, `Length` и `What`, а также замена найденных фрагментов методами `Replace` и `ReplaceAfterSearch`. Объект создаётся без параметров или с указанием регулярного выражения и опционального флага soft mode.

Внимание: при некорректном синтаксисе регулярного выражения поведение может быть непредсказуемым, так как обработка ошибок в текущей реализации ограничена. Регулярные выражения должны быть корректными до передачи в `SetExpression`.

Класс `Regex` является wrapper'ом к библиотеке `regex.dll` (скомпилирована в VS2010, использует `boost::regex`). Везде используется прямой вызов методов boost через wrapper'ы; собственные доработки сосредоточены в группе методов `Replace*`. Синтаксис регулярных выражений соответствует `boost::regex`.

### Создание объекта класса Regex в языке EME-L

Класс `Regex` регистрирует один CTOR с переменным числом аргументов — в скриптах EME-L доступны три формы вызова.

```EME-L
'Конструктор без аргументов — выражение не задано'
re = Object("Regex");
```

```EME-L
'Конструктор с регулярным выражением'
re = Object("Regex", "^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+$");
```

```EME-L
'Конструктор с регулярным выражением и флагом soft mode'
re = Object("Regex", "^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+$", TRUE);
```

Флаг soft mode (arg1) в классе `Regex` языка EME-L подавляет вывод системной ошибки при невозможности загрузить `regex.dll` — в этом случае методы класса будут возвращать «нет соответствия» вместо прерывания. Без soft mode ошибка загрузки библиотеки выводится через `smart_error`. Если выражение задано в конструкторе, оно компилируется немедленно; для смены выражения используйте `SetExpression`.

### Управление выражением класса Regex в EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetExpression(arg0)` | arg0: String | Boolean | Компилирует и устанавливает регулярное выражение в синтаксисе `boost::regex`. TRUE — выражение скомпилировано и готово к использованию, FALSE — ошибка компиляции. |

### Поиск и сопоставление в классе Regex языка EME-L

Метод `Match` класса `Regex` проверяет полное соответствие всей строки выражению; `Search` ищет первое или N-е вхождение подстроки. Результаты последнего вызова `Match` или `Search` хранятся в объекте и используются методами `Position`, `Length`, `What`, `Matched` для извлечения деталей соответствия.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Match(arg0)` | arg0: String | Boolean | TRUE — строка полностью соответствует выражению, FALSE — не соответствует. |
| `Search(arg0)` | arg0: String | Boolean | Ищет первое вхождение выражения. TRUE — найдено (позицию и длину дают `Position` и `Length`), FALSE — соответствий нет. |
| `Search(arg0, arg1)` | arg0: String, arg1: Integer | Boolean | Ищет вхождение по порядковому номеру: 0 — первое вхождение, 1 — второе и т.д. TRUE — соответствие с указанным номером найдено, FALSE — не найдено. |

### Группы захвата класса Regex в EME-L

Группа 0 — основное соответствие целиком. Группы 1 и выше — нумерованные группы захвата, заданные круглыми скобками `(...)` в регулярном выражении. Методы без аргументов эквивалентны вызову с аргументом 0.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Position()` | — | Integer | Позиция начала основного соответствия (группа 0). 0, если соответствие не найдено. |
| `Position(arg0)` | arg0: Integer | Integer | Позиция начала указанной группы захвата. 0, если группа не найдена или соответствие отсутствует. |
| `Length()` | — | Integer | Длина основного соответствия (группа 0) в символах. 0, если соответствие не найдено. |
| `Length(arg0)` | arg0: Integer | Integer | Длина указанной группы захвата в символах. 0, если группа не найдена или соответствие отсутствует. |
| `What(arg0)` | arg0: Integer | String | Текст указанной группы захвата. Пустая строка, если группа не найдена, соответствие отсутствует или произошла внутренняя ошибка. |
| `Matched()` | — | Boolean | TRUE — основное соответствие (группа 0) найдено в результате последнего `Search`/`Match`. |
| `Matched(arg0)` | arg0: Integer | Boolean | TRUE — указанная группа захвата существует и содержит данные. |

Метод `What` класса `Regex` читает текст группы во внутренний буфер размером 1024 байта (`REGEX_WHAT_BUFFER_SIZE`). При превышении буфера возвращается усечённое значение.

### Замена в классе Regex языка EME-L

Методы замены класса `Regex` в языке EME-L поддерживают спецкоды в строке замены:

| Код | Что вставляется |
|-----|------------------|
| `%0` | Текст всего найденного соответствия (группа 0) |
| `%1`, `%2`, ... | Текст соответствующей группы захвата |
| `%%` | Одиночный символ `%` |

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Replace(arg0, arg1)` | arg0: String (исходная), arg1: String (замена) | String | Заменяет **все** вхождения выражения в исходной строке. Если соответствий нет, возвращает исходную строку без изменений. |
| `Replace(arg0, arg1, arg2)` | arg0: String, arg1: String (замена), arg2: Integer | String | Заменяет выбранное вхождение: -1 — все вхождения, 0 — первое, 1 — второе и т.д. Если указанное соответствие не найдено, возвращает исходную строку без изменений. |
| `ReplaceAfterSearch(arg0)` | arg0: String (замена) | String | Заменяет первое вхождение в строке, использованной при последнем вызове `Search` или `Match`. Если соответствие не найдено, возвращает исходную строку без изменений. |
| `ReplaceAfterSearch(arg0, arg1)` | arg0: String (замена), arg1: Integer | String | То же с выбором номера соответствия: -1 — все вхождения в строке последнего поиска, 0 — первое, 1 — второе и т.д. |

Метод `ReplaceAfterSearch` класса `Regex` оперирует строкой, сохранённой в объекте при последнем вызове `Search`/`Match`, и не требует повторной передачи исходной строки. Это удобно для цепочки «поиск — анализ — замена» в скриптах EME-L.

### Примеры использования класса Regex в EME-L

Проверка формата e-mail и извлечение компонентов через группы захвата:

```EME-L
'Проверить формат e-mail и разобрать на имя и домен'
re = Object("Regex", "^([A-Za-z0-9._%+-]+)@([A-Za-z0-9.-]+)$");
If (re.Match("user@example.com"))
    userName = re.What(1);
    domain = re.What(2);
    'userName = "user", domain = "example.com"'
End If
```

Поиск N-го вхождения и получение позиции:

```EME-L
'Найти второе число в строке'
re = Object("Regex", "\\d+");
If (re.Search("abc 10 def 20 ghi 30", 1))
    pos = re.Position();
    len = re.Length();
    value = re.What(0);
    'value = "20", pos = позиция "20" в строке'
End If
```

Замена с подстановкой групп:

```EME-L
'Переформатировать дату из DD.MM.YYYY в YYYY-MM-DD'
re = Object("Regex", "(\\d{2})\\.(\\d{2})\\.(\\d{4})");
result = re.Replace("Дата: 27.06.2026", "%3-%2-%1");
'result = "Дата: 2026-06-27" — все вхождения заменены'
```

Замена конкретного вхождения через `ReplaceAfterSearch`:

```EME-L
'Заменить только второе число в строке'
re = Object("Regex", "\\d+");
If (re.Search("10 20 30", 1))
    result = re.ReplaceAfterSearch("[X]");
    'result = "10 [X] 30"'
End If
```

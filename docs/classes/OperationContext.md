# Класс OperationContext

> Синонимы для поиска: CIOperationContext OperationContext класс EME-L, контекст операции EME-L, складские операции EME-L, Mover SetContext, ячейки запрещенные ячейки, принудительная ячейка, трассировка поиска ячейки, удаление адреса ячейки, затруднения ошибки операции, инвентаризация EME-L, приказы на ТСД, округление по слоям, ячейки перехвата

Класс `OperationContext` в языке EME-L — это контекст группы отгружаемых строк документа, предназначенный для управления параметрами и состоянием складских операций: приход, расход, перемещение и инвентаризация. В системе EME.WMS объект `OperationContext` передаётся в объекты складских операций, например в [`Mover`](./Mover.md), методом `SetContext`. Если контекст не задан, внутри `Mover` создаётся временный объект `OperationContext` со значением по умолчанию `NeedCheckAccessibility = TRUE`.

В языке EME-L класс `OperationContext` обеспечивает контроль доступности ячеек, управление списком запрещённых ячеек, установку принудительных ячеек для размещения товара без поиска. Он поддерживает трассировку поиска ячеек и режим удаления адреса ячейки. Контекст позволяет регистрировать и обрабатывать затруднения (ошибки) операций, включая получение кода ошибки, номера строки документа, максимального и выбранного кода ответа оператора. Ошибки могут быть получены в виде форматированного текста или заполнены в объект запроса `Query` с полями `DocLineRef` и `ErrorText`.

Класс `OperationContext` в системе EME.WMS поддерживает специализированные режимы: новая инвентаризация, первая инвентаризация ячейки, округление количества по количеству в слое, автоматический перевод заказа на статус «Подобран», а также управление созданием приказов на терминалы сбора данных (ТСД) и их привязкой к терминалам.

## Создание объекта класса OperationContext

В языке EME-L объект `OperationContext` создаётся оператором `Object`. Хотя в реестре `BEGIN_IMPLEMENT` для `OperationContext` нет макросов `IMPLEMENT_CTOR`, C++-конструктор объекта поддерживает три формы вызова.

```EME-L
'Конструктор без аргументов — проверка доступности выключена, запись строк не задана'
objContext = Object("OperationContext");
```

```EME-L
'Конструктор с флагом проверки доступности ячеек'
objContext = Object("OperationContext", TRUE);
```

```EME-L
'Конструктор с флагом проверки и записью отгружаемых строк'
objContext = Object("OperationContext", TRUE, r_DocLines.GetRecord());
```

Конструктор с одним аргументом в языке EME-L принимает признак проверки доступности ячеек (`Boolean`) и эквивалентен созданию объекта без аргументов с последующим вызовом `SetCheckAccessibility`. Конструктор с двумя аргументами дополнительно задаёт запись строк документа `DocLines` (имя или номер записи), вызывая `SetRecord`. Значения по умолчанию, устанавливаемые при создании объекта `OperationContext` в системе EME.WMS: проверка доступности ячеек — `FALSE`, автоматический перевод заказа на статус «Подобран» — `TRUE`, тип ячеек для прихода навалом — `CELL_TYPE_NONADDR`, трассировка поиска ячейки и удаление адреса ячейки — `FALSE`.

## Запись строк документа в OperationContext

Методы класса `OperationContext` в языке EME-L для привязки к записи строк документа:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetRecord(arg0)` | arg0: String или Integer | Empty | Задаёт запись отгружаемых строк документа `DocLines`, с которой будет работать контекст. Принимает имя записи (строка) или номер записи (целое). При недопустимом номере генерируется ошибка. |

В языке EME-L метод `SetRecord` является обязательной преднастройкой для `GetErrorText` и `GetErrorsAsQuery` — без него генерируется ошибка «Контекст не настроен на запись строк».

## Запрещённые ячейки в OperationContext

В системе EME.WMS список запрещённых ячеек управляет тем, какие ячейки исключаются из поиска при размещении или подборке товара. Методы класса `OperationContext` в языке EME-L для работы со списком запрещённых ячеек:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ForbidCell(arg0)` | arg0: Integer (ссылка на Cells) | Empty | Добавляет ячейку в список запрещённых для поиска. |
| `RemoveAllForbidCell()` | — | Empty | Удаляет все ячейки из списка запрещённых, если не установлен флаг сохранения (`SetNotRemoveForbidden`). |
| `CancelForbidCell(arg0)` | arg0: Integer (ссылка на Cells) | Empty | Удаляет указанную ячейку из списка запрещённых. |
| `IsCellForbidden(arg0)` | arg0: Integer (ссылка на Cells) | Boolean | Проверяет, запрещена ли ячейка. `TRUE` — запрещена, `FALSE` — разрешена. |
| `SetNotRemoveForbidden(arg0)` | arg0: Integer (0/1) | Empty | Устанавливает флаг сохранения запрещённых ячеек при вызове `RemoveAllForbidCell`. `1` — не удалять, `0` — удалять. В реестре метод описан как принимающий `Boolean`, однако C++-тело метода преобразует аргумент через `variant2integer` и сравнивает с нулём, поэтому в языке EME-L следует передавать целое `0` или `1`. |

| `SetUserCell(arg0, arg1)` | arg0: Integer (ссылка на Cells), arg1: Boolean (проверять) | Empty | То же с явным указанием флага проверки возможности размещения товара в указанную ячейку. По умолчанию `TRUE`. |
| `RemoveUserCell()` | — | Empty | Отменяет принудительную замену ячейки. |

## Затруднения (ошибки операций) в OperationContext

В системе EME.WMS затруднения (`troubles`) — это ошибки, возникающие в ходе операции и требующие ответа оператора. Каждое затруднение хранит номер строки документа, код ошибки, максимальный и выбранный код ответа. Методы класса `OperationContext` в языке EME-L для работы со затруднениями:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetNoOfTroubles()` | — | Integer | Количество зарегистрированных затруднений. |
| `GetTroubleLine(arg0)` | arg0: Integer (индекс) | Integer | Номер строки документа, в которой возникло затруднение с указанным индексом. |
| `GetTroubleErrorCode(arg0)` | arg0: Integer (индекс) | Integer | Код ошибки затруднения с указанным индексом. |
| `GetTroubleMaxChoice(arg0)` | arg0: Integer (индекс) | Integer | Максимальный допустимый код ответа оператора для затруднения с указанным индексом. |
| `GetTroubleChoice(arg0)` | arg0: Integer (индекс) | Integer | Текущий (выбранный) код ответа оператора для затруднения с указанным индексом. |
| `PutTroubleChoice(arg0, arg1)` | arg0: Integer (индекс), arg1: Integer (ответ) | Empty | Устанавливает код ответа оператора. Значение должно быть больше `itcUNKNOWN` и не превышать максимальный допустимый код для данного затруднения. При недопустимом значении генерируется ошибка. |
| `DeleteAllTroubles()` | — | Empty | Удаляет все зарегистрированные затруднения. |

## Ошибки и протокол в OperationContext

В языке EME-L ошибки (`errors`) — это текстовые сообщения об ошибках, привязанные к строкам документа. В отличие от затруднений, ошибки не требуют ответа оператора и используются для формирования протокола операции. Методы класса `OperationContext` для работы с ошибками:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetErrorText()` | — | String | Формирует текстовый отчёт по всем ошибкам, сгруппированный по документам и строкам. Включает номера строк, описания товаров и тексты ошибок. Требует предварительной настройки на запись строк методом `SetRecord`. |
| `GetErrorsAsQuery(arg0)` | arg0: Query (по ссылке) | Empty | Заполняет переданный запрос структурированной информацией об ошибках. Заполняет поля `DocLineRef` (ссылка на строку документа) и `ErrorText` (текст ошибки, строка длиной до 256 символов). В запрос попадают только ошибки с непустым текстом. Требует предварительной настройки на запись строк методом `SetRecord`. |
| `DeleteAllErrors()` | — | Empty | Удаляет все зарегистрированные ошибки. |

## Примеры использования OperationContext в языке EME-L

В языке EME-L создание контекста с запретом создания приказов и передача его в `Mover` выглядит так:

```EME-L
'Создать контекст с проверкой доступности ячеек'
objContext = Object("OperationContext", TRUE);
'Запретить создание приказов на ТСД для этой операции'
objContext.SetDisableCreateMoveOrder(TRUE);
'Передать контекст в объект складской операции'
objMover = Object("Mover", r_DocLines.GetRecord());
objMover.SetContext(objContext);
strResult = objMover.SrcToBuf(arrLoadedLines);
If (strResult != "")
    is_message(tr("Ошибка"), strResult, 1);
End If
```

В языке EME-L сбор ошибок в объект `Query` для дальнейшей обработки выполняется следующим образом:

```EME-L
objContext = Object("OperationContext", TRUE, r_DocLines.GetRecord());
'...выполнение складской операции...'
'Получить структурированный список ошибок'
objQuery = Object("Query");
objContext.GetErrorsAsQuery(objQuery);
Loop (objQuery)
    lDocLineRef = objQuery.DocLineRef;
    csErrorText = objQuery.ErrorText;
    'Обработать ошибку для строки lDocLineRef'
End Loop
```

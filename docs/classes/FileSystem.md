### Класс FileSystem — клиент файловых систем в языке EME-L

Класс `FileSystem` (C++ класс `CIFileSystem`) в языке EME-L — это унифицированный клиент для работы с файловыми системами различных типов: локальной дисковой, FTP (включая FTPS с шифрованием TLS) и WinSCP. В системе EME.WMS класс применяется для организации файлового обмена, импорта и экспорта данных, интеграции с внешними файловыми хранилищами.

Класс `FileSystem` предоставляет единый интерфейс для подключения к хранилищам файлов, выбора файлов по маске, последовательного навигационного перебора выбранных файлов, чтения и записи бинарного содержимого через объекты `CDataStorage`, а также копирования, перемещения и удаления файлов. Объект создаётся без параметров через `Object("FileSystem")`; перед любой файловой операцией необходимо установить подключение методом `Connect`.

Большинство файловых операций (`Connect`, `PutData`, `GetData`, `DeleteFile`, `CopyFile`, `MoveFile`, `SelectFiles`) в языке EME-L возвращают пустую строку при успешном выполнении либо строку с описанием ошибки при неудаче. Методы `IsConnected` и `IsValidLine` возвращают `Boolean`, `SetFirstLine` и `SetNextLine` возвращают пустое значение, а `GetFileName` — имя файла строкой.

### Создание объекта FileSystem

В языке EME-L объект `FileSystem` создаётся конструктором без аргументов. После создания клиент не привязан ни к какой файловой системе — требуется вызов `Connect`. Перегрузок конструктора с аргументами нет (C++ `IMPLEMENT_CTOR(CIFileSystem, 0, ...)`).

```EME-L
i_FS = Object("FileSystem");
'Клиент создан, но не подключён — далее вызовите Connect'
```

### Подключение и состояние — методы Connect / Disconnect / IsConnected / GetLastError

Группа методов управления жизненным циклом подключения клиента файловой системы в EME-L.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Connect(arg0, arg1)` | arg0: String (строка подключения), arg1: Boolean (создавать каталоги) | String | Подключает клиента к файловой системе. Пустая строка — успех, иначе текст ошибки. Если клиент уже подключён, генерируется ошибка. |
| `Disconnect()` | — | String | Отключает от файловой системы и освобождает ресурсы. Пустая строка — успех. Если клиент не подключён, генерируется ошибка. |
| `IsConnected()` | — | Boolean | TRUE — клиент подключён, FALSE — подключение отсутствует. |
| `GetLastError()` | — | String | Текст последней ошибки. Пустая строка, если ошибок не было. |

#### Строка подключения Connect и типы драйверов

Тип драйвера файловой системы определяется префиксом строки подключения — подстрокой до первого двоеточия, сравнение выполняется без учёта регистра. Это ключевая константа поведения класса `FileSystem` в EME-L.

| Префикс | Драйвер | Формат строки после префикса |
|---------|---------|------------------------------|
| `FTP:` | FTP | `ftp://user:password@host:port/path` |
| `FTPS:` | FTP + TLS | `ftps://user:password@host:port/path` — принудительно включаются SSL-режим (`SetSslMode`), TLS-режим (`SetTlsMode`), пассивный режим (`SetPassiveMode`), отключается проверка сертификата (`SetDisableCheckSert`). |
| `WINSCP:` | WinSCP | `<параметры команды open>;<папка на сервере>;<локальная папка>` — три части, разделённые точкой с запятой. Требуется установленная утилита WinSCP в папке `<DBDir>\WinSCP\Exe\WinSCP.com`. |
| `EME:` / `EMEL:` | — | Зарезервированные префиксы. В текущей версии вызывают `internal_error()` — драйверы не реализованы. |
| без префикса / нераспознанное двоеточие | Дисковый драйвер | Полный или относительный путь к каталогу: `C:\Data`, `.\Files`. Дисковый драйвер выбирается также для строк с двоеточием, чей префикс не совпал ни с одним из зарегистрированных (двоеточие в таких путях не отсекается). |

Параметр `arg1` (флаг создания каталогов, `Boolean`) действует на дисковый и WinSCP-драйверы: `TRUE` — создать недостающие каталоги (`MakeSureDirectoryPathExists`), `FALSE` — вернуть ошибку об отсутствии пути (поведение по умолчанию).

### Выборка и навигационный перебор файлов — SelectFiles / SetFirstLine / IsValidLine / SetNextLine / GetFileName

Выборка файлов выполняется по маске в текущем каталоге подключения. После вызова `SelectFiles` список доступен для последовательного навигационного перебора методами `SetFirstLine`, `IsValidLine`, `SetNextLine` с получением имени текущего файла через `GetFileName`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SelectFiles(arg0)` | arg0: String (маска имени) | String | Выбирает файлы по маске, например `*.txt` или `data_*.xml`. Пустая строка — успех. Для WinSCP файлы предварительно скачиваются с удалённого сервера в локальную папку и удаляются там (команда `get <mask> -delete`). |
| `SetFirstLine()` | — | Empty | Устанавливает указатель на первый файл выборки. |
| `IsValidLine()` | — | Boolean | TRUE — текущий файл существует и доступен, FALSE — список пуст или достигнут конец выборки. |
| `SetNextLine()` | — | Empty | Сдвигает указатель на следующий файл выборки. |
| `GetFileName()` | — | String | Имя текущего файла выборки. Пустая строка, если указатель не установлен. |

Особенность дискового драйвера: при переборе каталоги и файлы нулевой длины пропускаются, а также проверяется право на чтение и запись (`_access(..., 6)`) — это исключает выбор файлов, которые ещё записываются другим процессом.

### Чтение и запись данных — GetData / PutData

Чтение и запись файлов выполняются через бинарные объекты `CDataStorage`. Для FTP обмен выполняется в бинарном режиме (`FTP_TRANSFER_TYPE_BINARY`).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetData(arg0)` | arg0: CDataStorage (по ссылке) | String | Читает содержимое текущего файла выборки в переданный объект. Пустая строка — успех. |
| `GetData(arg0, arg1)` | arg0: String (имя файла), arg1: CDataStorage (по ссылке) | String | Читает содержимое указанного файла. Путь задаётся относительно каталога подключения. |
| `PutData(arg0, arg1)` | arg0: String (имя файла), arg1: CDataStorage (по ссылке) | String | Записывает данные из объекта в файл. Существующий файл перезаписывается (открывается с `modeCreate | modeWrite | shareExclusive`). Пустая строка — успех. |

### Операции с файлами — CopyFile / MoveFile / DeleteFile

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CopyFile(arg0, arg1)` | arg0: String (имя файла), arg1: FileSystem (целевой объект, по ссылке) | String | Копирует файл в целевую файловую систему с сохранением имени: читает данные в промежуточный `CDataStorage` и вызывает `PutData` целевого объекта. Целевой объект должен быть предварительно подключён. |
| `MoveFile(arg0, arg1)` | arg0: String (имя файла), arg1: FileSystem (целевой объект, по ссылке) | String | Перемещает файл: вызывает `CopyFile`, при успехе удаляет файл из исходной системы. |
| `DeleteFile(arg0)` | arg0: String (имя файла) | String | Удаляет файл из текущего каталога подключения. Пустая строка — успех. |

### Примеры использования класса FileSystem в EME-L

Подключение к FTP, запись файла и отключение:

```EME-L
i_FS = Object("FileSystem");
Result = i_FS.Connect("FTP:ftp://user:pass@host:21/inbox", TRUE);
If (Result != "")
    is_message("FileSystem", "Ошибка подключения: " + Result);
Else
    Result = i_FS.PutData("export.xml", d_Data);
    If (Result != "")
        is_message("FileSystem", "Ошибка записи: " + Result);
    End If
    i_FS.Disconnect();
End If
```

Выборка и перебор файлов по маске с чтением содержимого каждого файла:

```EME-L
i_FS = Object("FileSystem");
Result = i_FS.Connect("WINSCP:ftps://eme.wms:pass@host:990 -certificate=*;inbox;C:\\WMS\\inbox", FALSE);
If (Result == "")
    Result = i_FS.SelectFiles("*.xml");
End If
If (Result == "")
    i_FS.SetFirstLine();
    While (i_FS.IsValidLine())
        FileName = i_FS.GetFileName();
        d_File = Object("DataStorage");
        Result = i_FS.GetData(d_File.Get());
        If (Result != "")
            Break
        End If
        'd_File содержит бинарные данные файла FileName'
        i_FS.SetNextLine();
    End While
End If
```

# Класс 1CItem

Класс `1CItem` в языке EME-L — это интерфейс для чтения, записи и манипулирования объектами данных системы 1С:Предприятие версии 7.7. Предоставляет методы для навигации по выборкам справочников и документов, получения и установки атрибутов объектов, поиска элементов по коду, наименованию и произвольным атрибутам, создания новых элементов справочников и документов, а также работы с табличными частями документов.

Объекты класса `1CItem` в языке EME-L создаются через конструктор с необязательным параметром — ссылкой на объект 1С, полученной методом `CreateObject` класса [`1C`](./1C.md). Этот класс используется для интеграции прикладной логики EME с учетными данными, хранящимися в базе 1С:Предприятие 7.7. C++ имя класса — `CI1CItem` (исходный файл `i_1CItem7_7.cpp`); script name, используемый в `Object("…")`, — `1CItem`.

## Создание объекта

В системе EME.WMS объект `1CItem` создаётся оператором `Object`. Конструктор без аргументов создаёт объект без привязки к данным 1С — ссылка устанавливается позже методом `Attach`. Конструктор с одним аргументом сразу привязывает объект к данным 1С, полученным от `ConnectionTo1C.CreateObject(...)`.

| Конструктор | Аргументы | Возвращает | Описание |
|-------------|-----------|------------|----------|
| `1CItem()` | — | 1CItem | Создаёт объект интерфейса без привязки к объекту 1С. |
| `1CItem([объект_1С])` | объект_1С: ссылка на объект данных 1С:Предприятие (необязательный) | 1CItem | Создаёт объект интерфейса и привязывает его к объекту данных 1С, полученному методом `CreateObject` класса `1C`. |

```EME-L
'Создать 1CItem без привязки — ссылка будет установлена через Attach'
objItem = Object("1CItem");
'Создать 1CItem с немедленной привязкой к справочнику Склады'
objItem = Object("1CItem",
    ConnectionTo1C.CreateObject("Справочники.Склады"));
```

## Привязка объекта 1С

В языке EME-L метод `Attach` используется, когда объект `1CItem` создан конструктором без аргументов и ссылку на объект 1С нужно установить отдельно.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Attach(объект_1С)` | объект_1С: ссылка на объект данных 1С:Предприятие | Boolean | Устанавливает ссылку на объект данных 1С для текущего экземпляра. TRUE — привязка выполнена успешно, FALSE — не выполнена. |

## Навигация по выборке справочника

Методы `SelectItems` и `GetItem` в системе EME.WMS обеспечивают обход элементов справочника. `SelectItems` открывает выборку с учётом или без учёта иерархии, `GetItem` последовательно позиционируется на следующий элемент; при достижении конца выборки возвращается FALSE.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SelectItems([режим])` | режим: Integer (1 — с учётом иерархии по умолчанию, 0 — без учёта) | Boolean | Выполняет выборку элементов справочника. TRUE — выборка выполнена, FALSE — нет. |
| `GetItem([режим])` | режим: Integer (1 — включать подчинённые элементы по умолчанию, 0 — не включать) | Boolean | Получает очередной элемент из выборки. TRUE — элемент получен, FALSE — достигнут конец выборки. |
| `GetCurrentItem()` | — | значение (ссылка/строка/целое) | Возвращает значение текущего позиционированного элемента справочника. Тип возвращаемого значения зависит от типа справочника в конфигурации 1С. |

## Атрибуты (реквизиты) объекта 1С

В языке EME-L имена атрибутов берутся из конфигуратора 1С — это строковые имена реквизитов, заданные в метаданных конфигурации. Тип возвращаемого `GetAttrib` значения соответствует типу атрибута в 1С (строка, целое, вещественное, дата, Boolean или ссылка).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetAttrib(имя_атрибута)` | имя_атрибута: String | значение (строка/целое/вещественное/дата/Boolean/ссылка) | Возвращает значение атрибута (реквизита) текущего объекта по имени из конфигуратора 1С. |
| `SetAttrib(имя_атрибута, значение)` | имя_атрибута: String, значение: соответствующего типа | Boolean | Устанавливает значение атрибута. TRUE — значение установлено успешно, FALSE — не установлено. |

## Поиск элементов справочника

Класс `1CItem` в языке EME-L предоставляет четыре метода поиска: по коду, по значению, по наименованию и по произвольному реквизиту. Все методы возвращают Boolean (TRUE — найден, FALSE — нет) и позиционируют объект на найденном элементе.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FindByCode(код)` | код: String | Boolean | Поиск элемента справочника по коду. |
| `FindItem(значение)` | значение: строка/целое/ссылка/другое | Boolean | Поиск элемента справочника по значению. Тип зависит от типа справочника. |
| `FindByDescr(наименование, [поиск_в_подчинении], [точное_соответствие])` | наименование: String; поиск_в_подчинении: Integer (1 — внутри родителя по умолчанию, 0 — по всему справочнику); точное_соответствие: Integer (1 — точное совпадение, 0 — по первым символам по умолчанию) | Boolean | Поиск элемента справочника по наименованию. |
| `FindByAttribute(имя_реквизита, значение, [область_поиска])` | имя_реквизита: String; значение: соответствующего типа; область_поиска: Integer (0 — в пределах подчинения по умолчанию, 1 — по всему справочнику) | Boolean | Поиск элемента справочника по значению произвольного реквизита. |

## Создание и запись объектов 1С

В системе EME.WMS типичный цикл работы с документом 1С через `1CItem`: `New()` → заполнение атрибутов через `SetAttrib` → `Write()`. Для проведения документа вместо `Write()` используется `MakeActions`. При создании нового элемента справочника код можно назначить через `SetNewCode` с префиксом.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `New()` | — | Boolean | Создаёт новый элемент справочника или новый документ. TRUE — создан успешно, FALSE — нет. |
| `Write()` | — | Boolean | Сохраняет текущий элемент справочника или документ в базе 1С. TRUE — записан успешно, FALSE — нет. |
| `MakeActions([режим_проведения], [дополнительный_параметр])` | режим_проведения: Integer (0 — без сдвига ТА по умолчанию, 1 — проводить непроведённый реальным временем, 2 — перепроводить проведённый реальным временем, 3 — проводить любой реальным временем); дополнительный_параметр: произвольный (например, строка «Документ проведён из EME») | Boolean | Проводит текущий документ 1С с управлением точкой актуальности (ТА). TRUE — проведён, FALSE — нет. |
| `SetNewCode(префикс)` | префикс: String | Boolean | Устанавливает новый код элемента справочника с указанным префиксом. TRUE — код установлен, FALSE — нет. |
| `NewLine()` | — | Boolean | Создаёт новую строку в табличной части текущего документа 1С. TRUE — строка создана, FALSE — нет. |

## Периодические значения и перечисления 1С

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `UseDate([дата])` | дата: DateTime (необязательный) | значение (дата/ссылка) | Устанавливает дату для получения периодических значений реквизитов документа 1С. |
| `GetValueByIdentifier(идентификатор)` | идентификатор: String | значение перечисления (ссылка/целое) | Возвращает значение перечисления 1С по его строковому идентификатору из конфигуратора. Тип возвращаемого значения зависит от типа перечисления. |

## Иерархия и подчинение справочника

Методы `UseOwner` и `UseParent` возвращают предыдущее значение на момент до выполнения — это позволяет сохранить и восстановить контекст выборки при вложенной навигации по справочнику.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `UseOwner([владелец])` | владелец: ссылка или другой тип (необязательный) | значение (ссылка) | Устанавливает выборку элементов справочника по указанному владельцу — элементу связанного справочника, которому подчинён данный. Возвращает предыдущее значение владельца. |
| `UseParent([группа])` | группа: ссылка или другой тип (необязательный) | значение (ссылка) | Устанавливает выборку элементов справочника по указанной группе (родителю). Возвращает предыдущее значение группы. |

## Примеры

Подключение к 1С:Предприятие 7.7 и поиск элемента справочника по наименованию в системе EME.WMS:

```EME-L
'Подключиться к 1С:Предприятие 7.7'
ConnectionTo1C = Object("1C");
OneCInitString = "/d\"C:\\DATABASE_1C\" /nПользователь /p";
ConnectionTo1C.Connect(0, OneCInitString);
'Создать 1CItem и привязать к справочнику Склады'
objWarehouse1C = Object("1CItem",
    ConnectionTo1C.CreateObject("Справочники.Склады"));
'Найти склад по наименованию СВХ внутри текущего подчинения'
objWarehouse1C.FindByDescr("СВХ", 1, 1);
```

Создание нового документа с заполнением шапки и записью в базе 1С:

```EME-L
'Создать новый документ приходного ордера'
objDoc1C = Object("1CItem",
    ConnectionTo1C.CreateObject("Документы.ПриходныйОрдерНаТовары"));
objDoc1C.New();
objDoc1C.SetAttrib("Склад", objWarehouse1C.GetCurrentItem());
objDoc1C.SetAttrib("Номер", "12353");
'Записать документ в базе 1С:Предприятие'
objDoc1C.Write();
```

Поиск номенклатуры по произвольному реквизиту (артикулу) по всему справочнику и чтение атрибута:

```EME-L
'Создать 1CItem для справочника Номенклатура'
objGood1C = Object("1CItem",
    ConnectionTo1C.CreateObject("Справочники.Номенклатура"));
'Найти по артикулу 11623341 по всему справочнику (область_поиска=1)'
objGood1C.FindByAttribute("Артикул", "11623341", 1);
Name = objGood1C.GetAttrib("НаименноеПолное");
```

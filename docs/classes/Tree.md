### Tree — класс абстрактного дерева в языке EME-L

Класс `Tree` в языке EME-L — это класс абстрактного дерева для организации иерархических структур данных в системе EME.WMS. Класс `Tree` в языке EME-L представляет собой контейнер, состоящий из узлов, организованных в древовидную структуру: каждый узел может хранить произвольное значение, набор именованных свойств и ссылки на дочерние узлы, образующие иерархию отношений родитель-потомок. В языке EME-L объект `Tree` создаётся через `Object("Tree")` без аргументов; при создании инициализируется корневой узел без данных, который автоматически становится текущим узлом дерева.

Класс `Tree` в системе EME.WMS используется для организации иерархических коллекций данных: деревьев отделов, классификаторов, конфигурационных структур. Узлы дерева в языке EME-L представлены объектами вспомогательного класса `Node` (внутренний C++ класс `CINode`). Объекты `Node` не создаются напрямую через `Object("Node")` — они получаются из методов класса `Tree` и из методов других объектов `Node`.

#### Создание объекта Tree в языке EME-L

```EME-L
'Создать пустое дерево с корневым узлом'
objTree = Object("Tree");
```

Класс `Tree` в языке EME-L имеет единственный конструктор без аргументов. Объект создаётся с пустым деревом, состоящим из одного корневого узла без данных. При уничтожении объекта `Tree` в системе EME.WMS открытые узлы автоматически освобождаются деструктором.

#### Навигация по дереву в языке EME-L

Методы класса `Tree` в языке EME-L для перемещения между узлами дерева и получения текущего узла.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetRoot()` | — | Node | Устанавливает текущий узел дерева в корневой узел и возвращает его в языке EME-L. |
| `GetRoot()` | — | Node | Возвращает объект корневого узла дерева в языке EME-L. |
| `Get()` | — | Node | Возвращает объект текущего узла дерева в языке EME-L. |
| `Set(arg0)` | arg0: Node | Node | Устанавливает указанный узел в качестве текущего. Должен быть валидным узлом, полученным из этого дерева. Возвращает установленный узел. |

**Важно:** в реализации C++ `i_Tree.cpp` метод `Set` не проверяет внутренними средствами, что переданный узел действительно принадлежит данному дереву. Передача узла из другого дерева может привести к неопределённому поведению. В языке EME-L всегда используйте узлы, полученные из методов того же объекта `Tree`.

#### Поиск в дереве в языке EME-L

Методы класса `Tree` в языке EME-L для рекурсивного поиска значений по всем узлам дерева.

|-------|-----------|------------|----------|

| `IsExist(arg0)` | arg0: любой тип | Boolean | Выполняет рекурсивный поиск указанного значения в дереве, начиная с корневого узла. Текущий узел дерева не изменяется. Возвращает `TRUE` (-1), если значение присутствует в дереве, и `FALSE` (0) в противном случае. |

#### Удаление и очистка дерева в языке EME-L

Методы класса `Tree` в системе EME.WMS для удаления текущего поддерева и полной очистки дерева.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Remove()` | — | Node | Удаляет текущий узел дерева со всеми его дочерними узлами (поддеревом). После удаления текущим узлом становится родительский узел удалённого. Если удаляется корневой узел (родитель отсутствует), дерево переинициализируется с созданием нового пустого корневого узла. Возвращает объект нового текущего узла. |
| `RemoveAll()` | — | Node | Полностью очищает дерево, удаляя все узлы включая корневой, и создаёт новое пустое дерево с единственным корневым узлом. Возвращает объект нового корневого узла. |
| `IsEmpty()` | — | Boolean | Проверяет наличие дочерних узлов у корневого узла. Возвращает `TRUE` (-1), если корневой узел имеет дочерние узлы, и `FALSE` (0), если дерево содержит только корневой узел без потомков. |

**Примечание:** семантика метода `IsEmpty` класса `Tree` в языке EME-L обратная интуитивной — `TRUE` означает «в дереве есть узлы», а не «дерево пустое». Это соответствует контракту реестра и реализации C++ в `i_Tree.cpp`.

### Класс Node — узел дерева в языке EME-L

Объекты `Node` в языке EME-L возвращаются методами класса `Tree` (`GetRoot`, `Get`, `Set`, `SetRoot`, `Remove`, `RemoveAll`) и другими методами `Node` (`AddTail`, `AddHead`, `InsertBefore`, `InsertAfter`, `GetParent`, `GetFirstChild`, `GetLastChild`, `GetPrevChild`, `GetNextChild`). Напрямую создать объект `Node` через `Object("Node")` нельзя — это вспомогательный класс `CINode`, зарегистрированный в системе EME.WMS через `REGISTRY_EMEL_CLASS_FOR_HELP`.

#### Добавление дочерних узлов в языке EME-L

Методы класса `Node` в языке EME-L для создания и добавления дочерних узлов к текущему узлу.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddTail()` | — | Node | Создаёт новый узел без данных и добавляет его в конец списка дочерних узлов текущего узла. Возвращает созданный дочерний узел. |
| `AddTail(arg0)` | arg0: любой тип | Node | Создаёт новый узел с указанными данными и добавляет его в конец списка дочерних узлов. Возвращает созданный дочерний узел. |
| `AddHead()` | — | Node | Создаёт новый узел без данных и добавляет его в начало списка дочерних узлов. Возвращает созданный дочерний узел. |
| `AddHead(arg0)` | arg0: любой тип | Node | Создаёт новый узел с указанными данными и добавляет его в начало списка дочерних узлов. Возвращает созданный дочерний узел. |
| `InsertBefore(arg0)` | arg0: Node | Node | Создаёт новый узел без данных и вставляет его перед указанным дочерним узлом текущего узла. Указанный узел должен присутствовать в списке дочерних узлов. Если узел не найден, генерируется ошибка выполнения. Возвращает созданный узел. |
| `InsertBefore(arg0, arg1)` | arg0: Node, arg1: любой тип | Node | Создаёт новый узел с данными `arg1` и вставляет его перед указанным дочерним узлом `arg0`. |
| `InsertAfter(arg0)` | arg0: Node | Node | Создаёт новый узел без данных и вставляет его после указанного дочернего узла текущего узла. Указанный узел должен присутствовать в списке дочерних узлов. Если узел не найден, генерируется ошибка выполнения. Возвращает созданный узел. |
| `InsertAfter(arg0, arg1)` | arg0: Node, arg1: любой тип | Node | Создаёт новый узел с данными `arg1` и вставляет его после указанного дочернего узла `arg0`. |

**Важно:** в текущей реализации C++ `i_Tree.cpp` одноаргументные формы `InsertBefore(node)` и `InsertAfter(node)` содержат ошибку: тело метода читает `args[1]`, которое выходит за пределы переданного массива аргументов (должно быть `argn > 1 ? args[1] : пустое значение`). Фактически при вызове с одним аргументом вместо пустых данных может быть использовано мусорное значение из стека. В языке EME-L для вставки узла без данных рекомендуется использовать `AddTail()` / `AddHead()` или двухаргументную форму `InsertBefore(node, Empty)` / `InsertAfter(node, Empty)`.

#### Примеры использования класса Tree и Node в языке EME-L

Построение дерева отделов и обход дочерних узлов:

```EME-L
'Создать дерево отделов в системе EME.WMS'
objTree = Object("Tree");
rootNode = objTree.GetRoot();
rootNode.PutData("Компания");

'Добавить дочерние узлы'
salesNode = rootNode.AddTail("Отдел продаж");
logisticsNode = rootNode.AddTail("Отдел логистики");

'Добавить подчинённые узлы'
salesNode.AddTail("Менеджер 1");
salesNode.AddTail("Менеджер 2");
logisticsNode.AddTail("Курьер");

'Сохранить коды отделов как именованные свойства'
salesNode.Put("code", "SALES");
logisticsNode.Put("code", "LOG");

'Найти узел и обойти его дочерние узлы'
found = objTree.Find("Отдел продаж");
If (found)
    foundNode = objTree.Get();
    child = foundNode.GetFirstChild();
    While (child)
        employeeName = child.GetData();
        'employeeName = имя сотрудника'
        child = foundNode.GetNextChild();
    End While
End If
```

Проверка существования и удаление поддерева в языке EME-L:

```EME-L
'Проверить, есть ли в дереве узел с данными "Курьер"'

'Установить текущий узел на логистику и удалить его поддерево'
objTree.SetRoot();
found = objTree.Find("Отдел логистики");
If (found)
    objTree.Remove();
End If
```

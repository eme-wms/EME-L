# Класс Set

## Назначение класса Set в языке EME-L

Класс `Set` в языке EME-L — класс для работы с набором строк записи базы данных. Полное имя C++ — `CISetCore`, script name в REGISTRY — `Set`. Предоставляет механизм формирования, фильтрации и выполнения теоретико-множественных операций над подмножествами строк записи.

В языке EME-L объект класса `Set` создаётся для конкретной записи базы данных и позволяет накладывать условия отбора строк, перебирать отобранные строки, получать ссылки из ссылочных полей, а также выполнять объединение, пересечение, симметрическую разность и дополнение наборов строк.

Класс `Set` в системе EME.WMS и языке EME-L поддерживает динамическую генерацию методов сравнения для полей и ссылочных полей записи (вида `GetИмяПоляEQ`, `GetИмяПоляNE`, `GetИмяПоляGT`, `GetИмяПоляLT`, `GetИмяПоляGE`, `GetИмяПоляLE`, `GetИмяRefNE`, `GetИмяRefEQ`, `GetИмяRefSet`, `IsИмяEQ`, `IsИмяNE`), что позволяет накладывать ограничения в краткой форме без явного вызова `Constrain`.

Класс `Set` в языке EME-L используется в прикладных скриптах для организации выборок строк записей, множественной фильтрации и обхода отобранных строк в циклах. Объект создаётся через `Object("Set", ...)` с обязательным указанием имени или номера записи базы данных.

## Создание объекта класса Set

Конструктор класса `Set` в языке EME-L имеет одну основную форму с переменным числом аргументов. Допустимы только формы с 1 или 3 аргументами; вызов без аргументов или с 2 аргументами вызывает ошибку `too_rare`.

```EME-L
'Конструктор с указанием имени записи — без условия отбора'
setRows = Object("Set", "Goods");
```

```EME-L
'Конструктор с указанием условия отбора и параметра сравнения'
setRows = Object("Set", "Goods", "GetGroup() ==", 15);
```

```EME-L
'Пустая строка в качестве имени записи — набор без привязки к конкретной записи'
setEmpty = Object("Set", "");
```

Конструктор с одним аргументом в языке EME-L создаёт набор строк для указанной записи без наложения условий — в набор входят все строки записи. Конструктор с тремя аргументами дополнительно накладывает условие на строки через вызов `Constrain`. Пустая строка в качестве имени записи (`""`) создаёт набор без привязки к конкретной записи — внутренний объект `CEMERec(dsDB)` создаётся пустым. Передача двух аргументов вызывает ошибку: при числе аргументов больше одного требуется полный набор из трёх (запись, условие, параметр).

В языке EME-L внутренний итератор набора строк инициализируется в неизвестное состояние. До вызова `SetFirstLine` методы `GetLine()` и `IsValidLine()` возвращают специальное значение отсутствия строки и `FALSE` соответственно.

## Ограничение и фильтрация строк в классе Set

Методы класса `Set` в языке EME-L для наложения ограничений и выборки строк.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Constrain(condition, param)` | condition: String, param: Variant | Set | Накладывает ограничение на текущий набор строк, оставляя только строки, удовлетворяющие условию. Модифицирует текущий объект и возвращает его. Формат условия: `"ИмяМетодаПоля() <оператор> "`, где оператор — `<`, `>`, `<=`, `>=`, `==`, `!=`. Поддерживаются поля (FLD), битовые поля (BIT) и ссылочные поля (REF). Для ссылочного поля при `==` параметр может быть целым числом (ссылка) или другим набором `Set`, строки которого должны входить в ссылочное поле. |
| `Pick(condition, param)` | condition: String, param: Variant | Set | Возвращает **новый** набор строк, отобранных из текущего по заданному условию. Исходный набор не изменяется. Формат условия и поддерживаемые операторы аналогичны `Constrain`. Тип значения параметра должен соответствовать типу сравниваемого поля (целое число, строка, boolean, вещественное число, дата, время). |

Метод `Constrain` в языке EME-L работает через систему фильтров записи (`SetSkipMode`), поэтому последовательные вызовы суммируют ограничения. Метод `Pick` выполняет независимую выборку итеративным обходом с применением оператора сравнения к каждой строке, не затрагивая исходный набор.

## Навигация по строкам набора Set

Методы класса `Set` в языке EME-L для обхода строк набора.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает итератор набора строк на первую строку в соответствии с текущим состоянием отбора. |
| `IsValidLine()` | — | Boolean | `TRUE` — итератор указывает на валидную строку. `FALSE` — строка отсутствует или итератор не инициализирован. |
| `SetNextLine()` | — | Empty | Перемещает итератор набора строк на следующую строку в соответствии с текущим состоянием отбора. |
| `GetLine()` | — | Integer | Номер текущей строки объекта записи, на которую установлен итератор. Возвращает `NULL_REF` (специальное значение отсутствия строки), если текущая строка не валидна. |

Для обхода строк набора в языке EME-L используйте цикл `Loop`, который автоматически вызывает `SetFirstLine` / `IsValidLine` / `SetNextLine`:

```EME-L
'Обход отобранных строк набора в языке EME-L'
setRows.SetFirstLine();
Loop (setRows)
    nLine = setRows.GetLine();
    'Обработка строки nLine записи'
End Loop
```

## Информация о записи в классе Set

Методы класса `Set` в языке EME-L для получения информации о записи.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetRecord()` | — | Integer | Номер записи базы данных, строки которой входят в набор. Возвращает `NULL_RECORD` (специальное значение отсутствия записи), если объект записи не задан. |
| `GetRecordObject()` | — | CEMERec | Объект записи базы данных, для которой сформирован набор строк. Пустое значение, если запись не задана. |
| `GetRefSet(fieldName)` | fieldName: String | Set | Возвращает **новый** набор строк, содержащий ссылки из ссылочного поля текущей записи. `fieldName` — имя метода-получателя ссылочного поля в форме `GetИмяRefFld()`. Должно соответствовать ссылочному полю записи. |

> **Внимание:** в языке EME-L параметр метода `GetRefSet` — это не произвольное «емельное выражение», а именно имя метода-получателя ссылочного поля, например `"GetGoodsRefFld()"`. Внутри метода по этому имени находится ссылочное поле, и для каждой строки текущего набора из него извлекается ссылка.

## Теоретико-множественные операции класса Set

Класс `Set` в языке EME-L предоставляет четыре операции над наборами строк. Все операции возвращают **новый** объект набора строк. Оба набора-операнда должны относиться к одной и той же записи базы данных — попытка объединить наборы от разных записей вызывает ошибку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Or(otherSet)` | otherSet: Set | Set | Объединение: строки, присутствующие хотя бы в одном из наборов. |
| `And(otherSet)` | otherSet: Set | Set | Пересечение: строки, присутствующие в обоих наборах одновременно. |
| `Xor(otherSet)` | otherSet: Set | Set | Симметрическая разность: строки, присутствующие только в одном из наборов (не в обоих). |
| `Not()` | — | Set | Дополнение: все строки записи, отсутствующие в текущем наборе. |

> **Примечание о реализации `Not()` в системе EME.WMS:** в C++-теле метода используется выражение `~pNotSet->m_bb_Lines;` — унарное побитовое отрицание без присваивания результата. По контракту реестра метод должен возвращать дополнение набора строк, однако фактическое поведение зависит от того, как компилятор обрабатывает это выражение. Используйте `Not()` с осторожностью и проверяйте результат на целевой версии ядра.

## Динамические методы сравнения класса Set

Класс `Set` в языке EME-L генерирует методы сравнения для каждого поля записи на лету через механизм `FindMethod` / `CallMethod`. Суффикс метода определяет оператор сравнения. Это позволяет накладывать ограничения в краткой форме без явного вызова `Constrain`.

### Поля данных (FLD) в классе Set

Для каждого поля данных записи в языке EME-L генерируются шесть методов:

| Метод | Оператор | Эквивалент через Constrain |
|-------|----------|----------------------------|
| `GetИмяПоляEQ(param)` | `==` | `Constrain("GetИмяПоля() ==", param)` |
| `GetИмяПоляNE(param)` | `!=` | `Constrain("GetИмяПоля() !=", param)` |
| `GetИмяПоляGT(param)` | `>` | `Constrain("GetИмяПоля() >", param)` |
| `GetИмяПоляLT(param)` | `<` | `Constrain("GetИмяПоля() <", param)` |
| `GetИмяПоляGE(param)` | `>=` | `Constrain("GetИмяПоля() >=", param)` |
| `GetИмяПоляLE(param)` | `<=` | `Constrain("GetИмяПоля() <=", param)` |

Все методы принимают один параметр типа Variant и возвращают текущий объект `Set` с наложенным ограничением.

### Ссылочные поля (REF) в классе Set

Для каждого ссылочного поля записи в языке EME-L генерируются три метода:

| Метод | Описание |
|-------|----------|
| `GetИмяRefNE(param)` | Оставляет строки, у которых значение ссылочного поля не равно `param` (целое число — ссылка на запись). |
| `GetИмяRefEQ(param)` | Оставляет строки, у которых значение ссылочного поля равно `param` (целое число). |
| `GetИмяRefSet()` | Возвращает новый объект `Set`, содержащий ссылки из этого ссылочного поля (эквивалент `GetRefSet("GetИмяRefFld()")`). Не принимает параметров. |

Для ссылочных полей в языке EME-L поддерживаются только операторы `==` и `!=`, а также метод получения набора ссылок `GetИмяRefSet`. Операторы `>`, `<`, `>=`, `<=` для ссылок недоступны.

### Битовые поля (BIT) в классе Set

Для каждого битового поля записи в языке EME-L генерируются два метода:

| Метод | Описание |
|-------|----------|
| `IsИмяEQ(param)` | Оставляет строки, где бит установлен в 1 (`param = TRUE`) или сброшен в 0 (`param = FALSE`). |
| `IsИмяNE(param)` | Оставляет строки, где бит сброшен в 0 (`param = TRUE`) или установлен в 1 (`param = FALSE`). Инверсия `IsИмяEQ`. |

## Внутренние константы класса Set

Класс `Set` в языке EME-L использует следующие специальные значения:

- `NULL_RECORD` — специальное значение, возвращаемое методом `GetRecord()`, если объект записи не задан (создан набор без привязки к записи или с пустой строкой в качестве имени записи).
- `NULL_REF` — специальное значение, возвращаемое методом `GetLine()`, если итератор не установлен на валидную строку.

Внутренняя реализация класса `Set` в системе EME.WMS использует три типа итератора (недоступны в скриптах EME-L напрямую): `UnknownIterator` (начальное состояние), `RecordIterator` (обход через систему фильтров записи), `BitBufferIterator` (обход через битовый буфер `BitBuffer`). Режим `ArrayIterator` зарезервирован, но не используется. Теоретико-множественные операции и метод `Pick` переключают итератор в режим `BitBufferIterator`.

Методы класса `Set`, возвращающие новый объект `Set` (`Constrain`, `Pick`, `GetRefSet`, `Or`, `And`, `Xor`, `Not`) или объект `CEMERec` (`GetRecordObject`), реализованы как `IMPLEMENT_MTYPED` и возвращают типизированный объект.

## Примеры использования класса Set

Фильтрация строк записи по условию и обход результата в языке EME-L:

```EME-L
'Создать набор строк товара с группой 15'
setGoods = Object("Set", "Goods", "GetGroup() ==", 15);
setGoods.SetFirstLine();
Loop (setGoods)
    nLine = setGoods.GetLine();
    'Чтение данных строки nLine записи Goods'
End Loop
```

Использование динамических методов и теоретико-множественных операций в языке EME-L:

```EME-L
'Два набора: активные и отмеченные товары'
setActive = Object("Set", "Goods");
setActive.IsActiveEQ(TRUE);
setMarked = Object("Set", "Goods");
setMarked.IsFlagEQ(TRUE);
'Пересечение: активные и одновременно отмеченные'
setBoth = setActive.And(setMarked);
```

> В примере выше предполагается, что в записи `Goods` есть битовые поля `IsActive` и `IsFlag`. Для поля данных `Group` краткая форма в языке EME-L выглядела бы как `setGoods.GetGroupEQ(15)`.

Получение набора ссылок из ссылочного поля в классе Set:

```EME-L
'Все строки заказа с заданным номером'
setOrder = Object("Set", "OrderHeader", "GetDocNo() ==", nDocNo);
setOrder.SetFirstLine();
Loop (setOrder)
    'Получить набор строк товаров из ссылочного поля'
    setGoods = setOrder.GetRefSet("GetGoodsRefFld()");
    setGoods.SetFirstLine();
    Loop (setGoods)
        nGoodsLine = setGoods.GetLine();
    End Loop
End Loop
```

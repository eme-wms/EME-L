# Класс DBWatch — дозор базы данных в языке EME-L

Класс `DBWatch` (C++ — `CIDBWatch`) в языке EME-L реализует механизм «дозора базы данных»: обнаружение изменений записей, полей и строк между двумя точками выполнения сценария. Объект накапливает список наблюдаемых объектов, а затем за одно обращение возвращает текстовый отчёт обо всех изменившихся элементах.

В системе EME.WMS класс применяется прежде всего для защиты от задвоения ключей при конкурентной записи: до транзакции формируется снимок заинтересованных полей, после открытия транзакции вызывается `Check()` — пустой отчёт разрешает запись, непустой требует перечитать данные.

Объект `DBWatch` создаётся через `Object()` без аргументов и не привязан к контексту диалога или браузера. Это самостоятельный класс EME-L; в памяти управляется автоматически (нет операторов `new`/`delete`).

## Создание объекта

```EME-L
'Создать дозор базы данных'
DBWatch = Object("DBWatch");
```

Конструктор создаёт объект с пустым списком наблюдения и выключенными счётчиками добавленных строк.

## Наблюдение за изменениями (группа методов Watch*)

Методы группы `Watch*` в языке EME-L регистрируют объекты базы данных в списке дозора. Наблюдение можно установить на запись целиком, на отдельное поле, на строку в записи или на поле в конкретной строке. Идентификатором записи/поля может служить целое число (номер), строка (наименование) или объект `CEMERec` / `CEMEFld`.

При повторной установке наблюдения за тем же объектом более детальные наблюдения сбрасываются: например, повторный `WatchRecord` очищает ранее установленные наблюдения за полями и строками этой записи. Если наблюдается вся запись, добавление полей и строк игнорируется; если наблюдается всё поле — игнорируются конкретные строки этого поля.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `WatchRecord(arg0)` | arg0: Integer/String/CEMERec | Empty | Наблюдение за записью в целом. Сбрасывает ранее установленные наблюдения за полями и строками этой записи. |
| `WatchField(arg0, arg1)` | arg0: Integer/String (запись), arg1: Integer/String (поле) | Empty | Наблюдение за полем. Если наблюдается вся запись — игнорируется. |
| `WatchField(arg0)` | arg0: CEMEFld | Empty | Наблюдение за полем через объект поля; запись и номер поля извлекаются из `CEMEFld`. |
| `WatchLine(arg0, arg1)` | arg0: Integer/String (запись), arg1: Integer (строка) | Empty | Наблюдение за строкой записи. Если наблюдается вся запись — игнорируется. |
| `WatchLine(arg0)` | arg0: CEMERec | Empty | Наблюдение за текущей строкой через объект записи. |
| `WatchFieldLine(arg0, arg1, arg2)` | arg0: Integer/String (запись), arg1: Integer/String (поле), arg2: Integer (строка) | Empty | Наблюдение за полем в конкретной строке. Если наблюдается вся запись или всё поле — игнорируется. |
| `WatchFieldLine(arg0)` | arg0: CEMEFld | Empty | Наблюдение за полем в конкретной строке через объект поля. |
| `UnwatchAll()` | — | Empty | Снимает все объекты с наблюдения и очищает внутренние структуры. Вызывается автоматически при уничтожении объекта. |

> Поля переменной длины (VLF, VARIABLE_LENGTH_FIELD_BIT) нельзя проверить на модификацию. Попытка установить наблюдение за VLF-полем через `WatchField` или `WatchFieldLine` вызывает ошибку ядра EME-L.

## Проверка и отладка содержимого дозора

`Check()` — основной рабочий метод класса `DBWatch` в EME-L. Возвращает многострочный текст: пустая строка означает отсутствие изменений, непустая — перечень изменившихся объектов с наименованиями записи и поля, например `Изменено поле [Рег.No] в записи [Документы]`. Проверка идёт последовательно по уровням: запись → поле → строка → поле в строке.

`GetContent()` — отладочный снимок текущего состояния дозора, не зависит от факта изменений. Форматы элементов: запись — `{[Наименование]}`; поле — `{[Запись],[Поле]}`; строка записи — `{[Запись],Номер}`; поле в строке — `{[Запись],[Поле],Номер}`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Check()` | — | String | Перечень изменившихся объектов; пустая строка — изменений нет. |
| `GetContent()` | — | String | Структурированное содержимое дозора для отладки. |

## Счётчики добавленных строк

Отдельный механизм `DBWatch` — накопительные счётчики добавленных строк для каждой записи базы данных в EME-L. Счётчики управляются парой `TurnOnAppendedCounters` / `TurnOffAppendedCounters`: при включении фиксируется снимок текущего количества строк каждой записи, при выключении разница между снимком и текущим количеством прибавляется к накопленному значению. Назначение — учёт строк, добавленных за период между двумя операциями сценария.

`ResetAppendedCounters` обнуляет счётчики и пересоздаёт внутренний массив под текущее количество записей в базе. Повторное включение уже включённых счётчиков вызывает ошибку, как и выключение неотключённых.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ResetAppendedCounters()` | — | Empty | Обнуляет счётчики всех записей. |
| `TurnOnAppendedCounters()` | — | Empty | Включает механизм; сохраняет снимок количества строк. Повторный вызов — ошибка. |
| `TurnOffAppendedCounters()` | — | Empty | Выключает механизм и прибавляет к счётчикам разницу со снимком. Если не включены — ошибка. |
| `GetAppendedCounters(arg0)` | arg0: Array/Map (по ссылке) | Empty | Записывает значения в контейнер. Для `Array` — все индексы, для `Map` — только ненулевые (ключ — номер записи). Иной тип — ошибка. |

## Примеры

Контроль изменений записи по наименованию и отладка содержимого дозора (реальный тест из `Базовая выгрузка проекта/Классы/ERPTests.txt`):

```EME-L
DBWatch = Object("DBWatch");
DBWatch.WatchRecord("Товарная единица");
DBWatch.WatchField("Документы", "Рег.No");
DBWatch.WatchLine("Строки документа", 12);
DBWatch.WatchFieldLine("Строки документа", "@Заголовок", 12);
Content = DBWatch.GetContent();
'Content — структурированный список наблюдаемых объектов'
```

Защита от задвоения SSCC при конкурентной записи (реальный пример из `Базовая выгрузка проекта/Классы/Examples.txt`): поле наблюдения задано через объект `CEMEFld`, проверка выполняется после открытия транзакции.

```EME-L
'Создать дозор и подключить ядро'
DBWatch = Object("DBWatch");
is_db_watch(1);
'Поле, которое должно остаться неизменным'
DBWatch.WatchField(r_SSCC.GetPalletIDFld());

'Сначала вне транзакции ищем все SSCC'
bNeedCreate = FindSSCC(arrSSCC, r_SSCC);

If (bNeedCreate)
    'Открыть транзакцию и проверить, что поле не изменилось'
    is_transaction(1, "Создание новых SSCC");
    is_db_watch(-1);
    If (DBWatch.Check() != "")
        'Поле изменилось — перечитать данные'
        bNeedCreate = FindSSCC(arrSSCC, r_SSCC);
    End If
    If (bNeedCreate)
        CreateSSCC(arrSSCC, r_SSCC);
    End If
End If
```

# Класс DBImpExp

> Синонимы для поиска: CIDBImpExp DBImpExp класс EME-L, экспорт данных EME-L, импорт данных EME-L, срез данных EME-L, Export Import AddRecord AddLine Backup GetWorkDir GetTempDir GetArchiveDir, DBImpExp Export slice, экспорт БД EME.WMS, импорт среза EME.WMS, перенос данных между филиалами, резервная копия БД EME-L

Класс `DBImpExp` в языке EME-L выполняет экспорт и импорт среза данных базы данных в архивный файл. Класс DBImpExp в системе EME.WMS позволяет выгрузить указанные записи и их строки, схему структуры БД, текущие настройки и сопутствующие скрипты в архив, а затем восстановить их в другой базе — вместе с предварительной очисткой, проверкой схемы и корректировкой дат.

Класс `DBImpExp` в языке EME-L используется для переноса данных между филиалами в системе EME.WMS, создания резервных копий среза и загрузки преднастроенных срезов (включая настройки пользователей, модулей и параметров).

Объект класса `DBImpExp` в языке EME-L создаётся через `Object()` без аргументов. При создании объект инициализирует рабочие каталоги (`Exchange\`, `Exchange\Temp\`, `Exchange\Archive\` внутри каталога данных проекта) и очищает временный каталог от мусора.

## Создание объекта класса DBImpExp

```EME-L
m_DBImpExp = Object("DBImpExp");
```

Конструктор без аргументов создаёт объект и подготавливает каталоги для операций экспорта/импорта в языке EME-L.

## Экспорт и импорт

Методы экспорта и импорта класса DBImpExp в языке EME-L — основные операции для переноса среза данных между базами.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Export(arg0)` | arg0: String (путь к архиву) | Integer | Выполняет экспорт данных среза в архивный файл. Выгружает указанные через `AddRecord`/`AddLine` записи и их строки, схему структуры БД, текущие настройки, список записей и сопутствующие скрипты. При успехе возвращает `0` (`EXP_SUCCESS`), при ошибке — код ошибки экспорта. |
| `Import(arg0)` | arg0: String (путь к архиву) | Integer | Выполняет импорт данных среза из архивного файла. Распаковывает архив, выполняет предварительный скрипт очистки, загружает данные в рамках серверной транзакции, выполняет завершающий скрипт восстановления настроек и корректировки дат. При успехе возвращает `0`, при ошибке — код ошибки импорта. Проверка схемы перед импортом выполняется по умолчанию. |
| `Import(arg0, arg1)` | arg0: String (архив), arg1: Boolean (проверка схемы) | Integer | То же; arg1=TRUE (по умолчанию) проверяет структуру БД перед импортом, FALSE — пропускает проверку. |
| `Import(arg0, arg1, arg2)` | + arg2: String (наименование среза) | Integer | То же с наименованием среза для отображения в консоли вывода. Пустая строка — наименование не выводится. |
| `Backup()` | — | String или Empty | Создаёт резервную копию базы данных. При успехе возвращает путь к созданному файлу резервной копии, при ошибке — пустое значение. |

> Экспорт/импорт класса DBImpExp в языке EME-L требуют монопольного режима (`is_workstation_mode() == "MONO"`) и прав администратора. Перед импортом в системе EME.WMS рекомендуется вызвать `is_enable_multiple_transactions()`, чтобы защитить БД от повреждений при случайном прерывании.

## Формирование списка записей для экспорта

Перед вызовом `Export` класса DBImpExp в языке EME-L необходимо указать, какие записи и строки выгружать. Если для записи вызван только `AddRecord` (без `AddLine`), выгружаются все строки этой записи.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddRecord(arg0)` | arg0: Integer (номер записи) | String | Добавляет запись в список выгрузки по её номеру и сохраняет внутреннюю структуру с пустым списком строк. Возвращает имя добавленной записи. |
| `AddLine(arg0, arg1)` | arg0: Integer (номер записи), arg1: Integer (номер строки) | Empty | Добавляет конкретную строку к записи в списке выгрузки. Строка будет включена в экспорт при выгрузке указанной записи. |

Номер записи в языке EME-L получают через `is_get_record_by_name("Имя записи")`.

## Каталоги

Методы класса DBImpExp в языке EME-L для получения путей к рабочим каталогам экспорта/импорта:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetWorkDir()` | — | String | Полный путь к рабочему каталогу экспорта/импорта. Подкаталог `Exchange` в каталоге данных проекта. Каталог создаётся автоматически при инициализации объекта. |
| `GetTempDir()` | — | String | Полный путь к временному каталогу. Подкаталог `Temp` в рабочем каталоге. Каталог очищается автоматически при инициализации объекта. |
| `GetArchiveDir()` | — | String | Полный путь к архивному каталогу. Подкаталог `Archive` в рабочем каталоге. Каталог создаётся автоматически при инициализации объекта. |

## Примеры

Базовый экспорт среза со всеми строками указанных записей методом Export класса DBImpExp в языке EME-L:

```EME-L
m_DBImpExp = Object("DBImpExp");

'Собрать список записей для выгрузки'
m_DBImpExp.AddRecord(is_get_record_by_name("Документы"));
m_DBImpExp.AddRecord(is_get_record_by_name("Строки документа"));
m_DBImpExp.AddRecord(is_get_record_by_name("Ячейки"));

FileName = m_DBImpExp.GetWorkDir() + "EME_slice.zip";
Result = m_DBImpExp.Export(FileName);
'Result = 0 (EXP_SUCCESS) при успехе, иначе код ошибки'
```

Импорт среза методом Import класса DBImpExp в языке EME-L с пропуском проверки схемы:

```EME-L
m_DBImpExp = Object("DBImpExp");

If (is_workstation_mode() != "MONO" | ~is_administrator())
    is_message(tr("Внимание"), tr("Импорт выполняется только в монопольном режиме"));
    Return;
End If

'Защита БД при случайном прерывании импорта'
is_enable_multiple_transactions();

Result = m_DBImpExp.Import(FileName, False, tr("Срез демо-данных"));
'Result = 0 при успехе'
```

Резервное копирование БД методом Backup класса DBImpExp в языке EME-L:

```EME-L
m_DBImpExp = Object("DBImpExp");
BackupPath = m_DBImpExp.Backup();
'BackupPath — путь к файлу резервной копии, пусто при ошибке'
```

Реальный пример — формирование среза в каталог архива с последующей записью в базу через класс DBImpExp в языке EME-L:

```EME-L
m_DBImpExp = Object("DBImpExp");

'Сформировать файл среза в каталоге архива'
FileName = m_DBImpExp.GetArchiveDir() + is_format("EME_%d_%s.zip", DataSliceRec.GetLine(), is_fdatetime(is_dos_date() + is_dos_time(), "%d%m%y_%H%M%S"));

'Заполнить список записей перед экспортом'
RecordsMap = Object("Map", 1001);
Loop (RecordsMap)
    m_DBImpExp.AddRecord(RecordsMap.GetValue());
End Loop

'Выгрузить срез в архив'
m_DBImpExp.Export(FileName);

'Перейти в монопольный режим и импортировать срез'
If (is_workstation_mode() == "MONO")
    is_switch_workstation_mode();
    is_ws_align_data();
    m_DBImpExp.Import(FileName);
End If
```

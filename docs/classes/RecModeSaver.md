### RecModeSaver — класс-хранитель состояния обхода записи в языке EME-L

Класс `RecModeSaver` (C++ `CIRecModeSaver`) в языке EME-L — это хранитель состояния обхода записи (или таблицы диалога) семейства `CEMERec`. Предназначен для временного сохранения и автоматического восстановления параметров обхода записи при выходе объекта из области видимости. Запоминает способ обхода, текущую строку, цепочку, обратную ссылку, признак использования обратной ссылки, список пропущенных строк (skip-записей) и признак проверки валидности.

В системе EME.WMS класс `RecModeSaver` используется в прикладных скриптах EME-L для защиты состояния обхода записи в локальных фрагментах кода (методах, обработчиках событий), где требуется временно переключить режим traversal и гарантированно вернуть запись в исходное состояние после завершения блока. Класс не поддерживает создание из записи, находящейся в режиме обхода по массиву ссылок (`rmArrayMode`) — в этом случае в конструкторе генерируется ошибка «Обход по массиву ссылок не поддерживается».

### Создание объекта класса RecModeSaver в языке EME-L

Класс `RecModeSaver` имеет один конструктор — ровно с одним обязательным аргументом, передаваемым **по ссылке** на запись.

```EME-L
'Конструктор с передачей записи по ссылке — захват состояния обхода'
saver = Object("RecModeSaver", r_DocLines);
```

Конструктор принимает ссылку на объект записи семейства `CEMERec`, состояние обхода которого требуется сохранить. Передача по ссылке обязательна — значение arg0 должно быть ссылкой на запись, а не её копией. При уничтожении объекта сохранённое состояние автоматически восстанавливается в исходной записи, включая позицию текущей строки.

### Состав сохраняемого состояния в EME-L

При создании объекта `RecModeSaver` в языке EME-L захватываются следующие элементы состояния записи `CEMERec`:

| Элемент | Поле C++ | Описание |
|---------|----------|----------|
| Способ обхода | `m_rmRecMode` | Один из режимов `rmRealMode`, `rmSortMode`, `rmChainMode`, `rmSkipMode`. `rmArrayMode` не поддерживается — генерируется ошибка в конструкторе. |
| Текущая строка | `cur_line` | Позиция текущей строки на момент создания хранителя. |
| Цепочка | `CEMEChain` | Полное состояние динамической цепочки обхода. |
| Обратная ссылка | `back_ref` | Ссылка на родительскую запись. |
| Признак использования обратной ссылки | `m_fUseBackRef` | Флаг активности обратной ссылки. |
| Список пропущенных строк | `m_arrSkipEntries` | Массив skip-записей — указатели переносятся между записями с сохранением счётчика ссылок (`AddRef` при копировании в хранитель, `Release` при сбросе исходного массива в деструкторе). |
| Признак проверки валидности | `m_fMustBeValidChecked` | Требование проверки валидности строк. |

Восстановление выполняется строго в дестукторе объекта `CIRecModeSaver`. Порядок восстановления в системе EME.WMS: способ обхода (`m_rmRecMode`), затем цепочка (`CEMEChain`), затем обратная ссылка (`back_ref`), признак её использования (`m_fUseBackRef`), затем освобождение и перенос skip-записей (`m_arrSkipEntries`), затем признак проверки валидности (`m_fMustBeValidChecked`), и **в последнюю очередь** — установка текущей строки через метод `CEMERec::SetLine` (не прямой записью в `cur_line`). Текущая строка устанавливается **после** переноса skip-списка — это важно для корректной работы внутренних инвариантов `CEMERec`.

### Методы класса RecModeSaver в языке EME-L

#### Управление сохранённой позицией

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetLine(arg0)` | arg0: Integer (по значению) | Empty | Устанавливает номер текущей строки, который будет восстановлен в записи при уничтожении хранителя. |
| `GetLine()` | — | Integer | Возвращает номер текущей строки, сохранённый в хранителе. |

Метод `SetLine` в EME-L не меняет позицию записи немедленно — он лишь подменяет сохранённое значение `m_lCurrentLine`, которое будет установлено в запись через `CEMERec::SetLine` в момент уничтожения хранителя (после настройки skip-списка). Используйте `GetLine`, чтобы прочитать ранее сохранённую позицию без запуска восстановления.

Методы `SetLine`/`GetLine` хранителя в языке EME-L управляют **сохранённой** позицией, а не текущей позицией записи. Для прямой установки текущей строки записи используйте `SetLine` у самого объекта записи (`r_DocLines.SetLine(...)`), а не у хранителя. В кодовой базе EME.WMS методы `SetLine`/`GetLine` хранителя не имеют реальных примеров вызова — типовой сценарий использует только конструктор для автоматического восстановления.

### Способы обхода, поддерживаемые классом RecModeSaver

Класс различает пять способов обхода записи `CEMERec` в языке EME-L. Эти режимы соответствуют внутренним константам `CEMERec::rmRealMode`, `rmSortMode`, `rmChainMode`, `rmSkipMode`, `rmArrayMode`. Строковые имена режимов (`"Real"`, `"Sort"`, `"Chain"`, `"Skip"`, `"Array"`) используются только во внутренней бинарной сериализации состояния хранителя (через `CEMEArchive`) — в прикладном коде EME-L их передавать не нужно.

| Способ обхода | Поддержка хранителем |
|---------------|----------------------|
| `rmRealMode` — реальный физический порядок строк | Да |
| `rmSortMode` — отсортированный порядок | Да |
| `rmChainMode` — обход по цепочке | Да |
| `rmSkipMode` — обход с пропусками строк | Да |
| `rmArrayMode` — обход по массиву ссылок | Нет (генерируется ошибка в конструкторе) |

При десериализации в системе EME.WMS неизвестное строковое значение режима вызывает ошибку «Неизвестный способ обхода в архиве - %s» — это защищает целостность состояния при чтении повреждённых архивов.

### Примеры использования RecModeSaver в языке EME-L

Защита состояния обхода перед временным переключением в системе EME.WMS:

```EME-L
'Сохраняем текущий способ обхода строк записи, иначе UploadHTML переключит его'
rms = Object("RecModeSaver", dsDB);

dsDB.SetLine(is_line());

If (~dsDB.IsValidLine())
    Return;
End If

dsDB.UploadHTML();
```

Обход ведущих строк (MasterLine) с сохранением исходной позиции вызывающей стороны:

```EME-L
'Заполняет массив ссылок на MasterLine для строки r_DocLines, от старшей к младшей'
rms = Object("RecModeSaver", r_DocLines);

While (r_DocLines.IsValidLine())
    lMasterLineRef = r_DocLines.GetMasterLineRef();
    If (lMasterLineRef == NULL_REF)
        Break;
    End If
    arrMasterLines.InsertAt(0, lMasterLineRef);
    r_DocLines.SetLine(lMasterLineRef);
End While
```

Подмена восстанавливаемой позиции через `SetLine` хранителя (метод не имеет реальных примеров в кодовой базе EME.WMS, показан канонический шаблон использования):

```EME-L
'Сохранить состояние обхода, но при восстановлении вернуть запись на строку 0'
rms = Object("RecModeSaver", r_DocLines);

r_DocLines.SetFirstLine();
While (r_DocLines.IsValidLine())
    'Обработка строк в собственном режиме обхода'
    r_DocLines.SetNextLine();
End While

'Переопределить восстанавливаемую строку: при уничтожении rms запись вернётся на строку 0'
rms.SetLine(0);
```

# Класс VendorEnhancement

> Синонимы для поиска: CIVendorEnhancement VendorEnhancement класс EME-L, SAP соответствия вендора EME-L, таблица дополнительных соответствий, Object("VendorEnhancement"), TuneVendorIfNeed, GetStatusFromLocation, GetLocationFromStatus, GetSAPStatusFromStatus, GetMvtCode, GetTrsCode, GetSimpleCode, SLOC MVT SIMP, PRIMESTOCK BADSTOCK

Класс `VendorEnhancement` в языке EME-L — это класс-интерфейс для работы с таблицей дополнительных соответствий организаций (вендоров). В системе EME.WMS он предоставляет доступ к специфичным для конкретной организации настройкам соответствия между кодами SAP и внутренними статусами/локейшенами системы EME.

Основные возможности класса `VendorEnhancement` в языке EME-L включают преобразование пар SAP-локейшен + SAP-статус в складской статус EME и обратно; получение кодов движений (Movement Type) и транзакций (Transaction Code) SAP по начальным и конечным статусам/локейшенам; получение произвольных кодов из таблицы дополнительных соответствий по ключу; автоматическую настройку на организацию с контролем актуальности кэшированных данных. Данные в системе EME.WMS загружаются из таблицы `VendorEnhancement` (код записи «Дополнительные соответствия») и кэшируются в памяти с отслеживанием изменений по штампу записи организации. В языке EME-L объект `VendorEnhancement` не предназначен для прямого редактирования таблицы соответствий — это объект только для чтения; изменение настроек вендора выполняется через справочник «Дополнительные соответствия» (запись таблицы `VendorEnhancement`).

## Создание объекта класса VendorEnhancement

В языке EME-L класс `VendorEnhancement` создаётся через `Object()` без аргументов. Перед использованием методов необходимо вызвать `TuneVendorIfNeed`, чтобы настроить объект на организацию и загрузить актуальные соответствия из таблицы `VendorEnhancement`.

```EME-L
'Создание объекта настройки вендора в системе EME.WMS'
Vendor_Obj = Object("VendorEnhancement");

'Настройка на организацию по ссылке на запись Clients'
Vendor_Obj.TuneVendorIfNeed(rVendor.GetLine());
```

Конструктор без аргументов в языке EME-L создаёт объект, связанный с общим кэшем таблицы дополнительных соответствий. Все вызовы методов в системе EME.WMS выполняются относительно последней настроенной организации; объект не привязан к конкретному диалогу или браузеру и живёт до конца метода, в котором был создан (память EME-L освобождается автоматически — операторов `new` и `delete` нет).

## Методы класса VendorEnhancement

Методы класса `VendorEnhancement` в языке EME-L разделены по группам: настройка на вендора, соответствия статусов и локейшенов (ключ таблицы `SLOC`), коды движений и транзакций (ключ таблицы `MVT`), произвольные коды (ключ таблицы `SIMP`).

### Настройка на вендора

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `TuneVendorIfNeed(nVendorRef)` | arg0: Integer — ссылка на запись организации (`Clients`) | Empty | Выполняет настройку объекта на указанную организацию (вендора), если текущая настройка устарела или отсутствует. В системе EME.WMS проверяет актуальность кэшированных данных по штампу записи организации и при необходимости перезагружает внутренние карты соответствий из таблицы `VendorEnhancement`. |

### Соответствия статусов и локейшенов (ключ таблицы `SLOC`)

В языке EME-L группа методов `GetStatusFromLocation`, `GetLocationFromStatus` и `GetSAPStatusFromStatus` работает с записями таблицы `VendorEnhancement`, у которых первое поле равно строке `SLOC`. Эти записи связывают SAP-локейшен + SAP-статус с односимвольным складским статусом EME.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetStatusFromLocation(szLocation, szSAPStatus)` | arg0: String (SAP-локейшен), arg1: String (SAP-статус) | String | Возвращает односимвольный складской статус системы EME (например, `'U'`, `'B'`, `'Q'`) по паре SAP-локейшен и SAP-статус. Возвращает пустую строку, если соответствие в таблице `VendorEnhancement` не найдено. |
| `GetLocationFromStatus(szStatus, szWarehouseOrReason)` | arg0: String (складской статус EME), arg1: String или Integer (тип склада или ризон-код) | String | Возвращает SAP-локейшен по складскому статусу EME и типу склада (или ризон-коду). В языке EME-L, если `arg1` — целое число, `0` трактуется как склад брака (`BADSTOCK`), любое ненулевое значение — как основной склад (`PRIMESTOCK`). Возвращает пустую строку, если соответствие не найдено. |
| `GetSAPStatusFromStatus(szStatus, szWarehouseOrReason)` | arg0: String (складской статус EME), arg1: String или Integer (тип склада или ризон-код) | String | Возвращает односимвольный SAP-статус по складскому статусу EME и типу склада. В системе EME.WMS, если `arg1` — целое число, `0` трактуется как склад брака (`BADSTOCK`), любое ненулевое значение — как основной склад (`PRIMESTOCK`). Если соответствие не найдено, возвращает значение по умолчанию `'B'` (блокирован). |

### Коды движений и транзакций (ключ таблицы `MVT`)

В языке EME-L группа методов `GetMvtCode` и `GetTrsCode` работает с записями таблицы `VendorEnhancement`, у которых первое поле равно строке `MVT`. Эти записи связывают начальный и конечный статус/локейшен с кодом движения (Movement Type) или транзакции (Transaction Code) SAP.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetMvtCode(szFrom, szTo)` | arg0: String (начальный статус SAP или локейшен), arg1: String (конечный статус SAP или локейшен) | String | Возвращает код движения SAP (Movement Type) по начальному и конечному параметрам. В системе EME.WMS, если передано числовое значение, отличное от `0`, автоматически подставляется ключ `SLOC`. Возвращает пустую строку, если соответствие не найдено. |
| `GetTrsCode(szFrom, szTo)` | arg0: String (начальный статус SAP или локейшен), arg1: String (конечный статус SAP или локейшен) | String | Возвращает код транзакции SAP (Transaction Code) по начальному и конечному параметрам. В системе EME.WMS, если передано числовое значение, отличное от `0`, автоматически подставляется ключ `SLOC`. Возвращает пустую строку, если соответствие не найдено. |

### Произвольные коды (ключ таблицы `SIMP`)

В языке EME-L метод `GetSimpleCode` работает с записями таблицы `VendorEnhancement`, у которых первое поле равно строке `SIMP`. Эти записи хранят произвольные пары «ключ — значение» для организации.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetSimpleCode(szKey)` | arg0: String (ключ поиска, соответствующий второму полю таблицы) | String | Возвращает значение произвольного кода (третье поле таблицы) по ключу (второму полю таблицы `VendorEnhancement`). Возвращает пустую строку, если соответствие не найдено. |

## Константы и внутренние ключи таблицы VendorEnhancement

В языке EME-L таблица дополнительных соответствий (`VendorEnhancement`, код записи «Дополнительные соответствия») различает записи по значению первого поля. Внутренние ключи, которые использует класс `VendorEnhancement` в системе EME.WMS:

| Ключ | Назначение |
|------|------------|
| `SLOC` | Соответствия между SAP-локейшеном + SAP-статусом и складским статусом/локейшеном EME. |
| `MVT` | Соответствия между начальным и конечным статусом/локейшеном и кодом движения/транзакции SAP. |
| `SIMP` | Произвольные коды: поиск по ключу (второе поле) возвращает значение (третье поле). |
| `FUNC` | Зарезервированный ключ для апплетов/обработчиков (не используется через публичные методы класса `VendorEnhancement` в EME-L). |
| `PRIMESTOCK` | Основной склад; в системе EME.WMS используется как значение по умолчанию, когда целочисленный аргумент `szWarehouseOrReason` не равен `0`. |
| `BADSTOCK` | Склад брака; в системе EME.WMS используется как значение по умолчанию, когда целочисленный аргумент `szWarehouseOrReason` равен `0`. |
| `U` / `B` / `Q` | Типичные односимвольные складские статусы EME, возвращаемые методом `GetStatusFromLocation`. |
| `B` | Значение SAP-статуса по умолчанию для метода `GetSAPStatusFromStatus`, если соответствие в таблице не найдено. |

## Примеры использования класса VendorEnhancement

В системе EME.WMS класс `VendorEnhancement` в языке EME-L часто используется для проверки произвольных настроек вендора. Пример ниже повторяет реальный паттерн из `Document.txt`: создаётся объект, настраивается на организацию и проверяется произвольный код `USE_PERIOD_SCHEME`.

```EME-L
'Получить объект настройки вендора в языке EME-L'
Vendor_Obj = Object("VendorEnhancement");

'Настроиться на организацию из текущей строки документа'
Vendor_Obj.TuneVendorIfNeed(rVendor.GetLine());

'Проверить, включена ли схема периодов'
If (Vendor_Obj.GetSimpleCode("USE_PERIOD_SCHEME") == "")
    Return 0;
End If
```

Корректировка даты с учётом настройки вендора в системе EME.WMS — реальный паттерн из `EANCOM.txt`:

```EME-L
'Создать и настроить объект вендора в языке EME-L'
Vendor_Obj = Object("VendorEnhancement");
Vendor_Obj.TuneVendorIfNeed(VendorRef);

UsePeriodScheme = False;

'Проверить флаг использования периодической схемы'
If (Vendor_Obj.GetSimpleCode("USE_PERIOD_SCHEME") == "ON")
    UsePeriodScheme = True;
End If
```

Получение SAP-статуса по складскому статусу и типу склада в системе EME.WMS:

```EME-L
Vendor_Obj = Object("VendorEnhancement");
Vendor_Obj.TuneVendorIfNeed(rVendor.GetLine());

'Основной склад: szWarehouseOrReason не равен 0'
SAPStatus = Vendor_Obj.GetSAPStatusFromStatus("U", 1);
'SAPStatus — односимвольный SAP-статус или "B" по умолчанию'

'Склад брака: szWarehouseOrReason равен 0'
SAPStatusBad = Vendor_Obj.GetSAPStatusFromStatus("U", 0);
```

Получение кода движения между статусами/локейшенами в языке EME-L:

```EME-L
Vendor_Obj = Object("VendorEnhancement");
Vendor_Obj.TuneVendorIfNeed(rVendor.GetLine());

'Код движения из статуса в статус'
MvtCode = Vendor_Obj.GetMvtCode("U", "B");

'Код движения с числовой подстановкой SLOC (число не 0)'
MvtCodeSLoc = Vendor_Obj.GetMvtCode("1", "B");
```

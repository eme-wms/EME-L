## Класс Map — ассоциативный массив (хэш-таблица) в языке EME-L

Класс `Map` в языке EME-L — это класс ассоциативного массива (хэш-таблицы) для хранения пар «ключ-значение» и быстрого доступа к данным по ключу. В системе EME.WMS и языке EME-L класс Map предоставляет контейнер, в котором каждое значение ассоциировано с уникальным ключом. Поддерживает ключи четырёх типов: строка (String), целое число (Integer/Long), вещественное число (Real/Double) и 64-битное целое (Long Integer/LongLong). Тип ключа задаётся при создании объекта или изменяется позже; при смене типа содержимое объекта полностью очищается.

Объект класса Map создаётся через `Object("Map", …)` и не привязан к контексту диалога или браузера. Это самостоятельный класс для построения словарей, кэшей, индексов и промежуточных коллекций, требующих произвольного доступа к элементам по идентификатору. Используется в прикладных скриптах EME-L для организации быстрого поиска данных по ключу.

## Создание объекта класса Map

Класс Map в языке EME-L имеет три конструктора — один на каждое число аргументов (0, 1 или 2).

```EME-L
'Без аргументов — размер хэш-таблицы по умолчанию 17, тип ключа String'
mapNames = Object("Map");
```

```EME-L
'С размером хэш-таблицы — тип ключа String'
mapNames = Object("Map", 100);
```

```EME-L
'С размером хэш-таблицы и типом ключа — Integer'
mapIds = Object("Map", 100, 1);
```

Размер хэш-таблицы должен быть больше 0 и приблизительно равен ожидаемому количеству элементов, умноженному на 1.2. Нулевой размер вызывает ошибку «Значение размера хэш-таблицы должно быть больше 0…». Избыточно большой размер приводит к неиспользуемым ячейкам памяти, а слишком маленький замедляет поиск из-за последовательного перебора ключей с совпадающим хэшем.

Тип ключа в языке EME-L определяется типом второго аргумента конструктора:

| Аргумент `keyType` | Тип ключа |
|--------------------|-----------|
| `1` (целое число) | Integer |
| `1.0` (вещественное число) | Real |
| `"str"` (строка) | String |
| `is_int64(1)` (Long Integer) | Long Integer |

## Доступ по ключу — методы Lookup, IsKeyExists, SetAt, RemoveKey, RemoveAll

Методы доступа по ключу в языке EME-L обеспечивают поиск, добавление и удаление пар «ключ-значение» в объекте Map. Тип ключа должен соответствовать типу ключа объекта Map, заданному при создании. Для типа ключа String допускается передача целого или вещественного числа — оно будет преобразовано в строку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Lookup(key)` | key: Integer / Real / String / Long Integer | любой тип или Empty | Значение по ключу либо `Empty`, если ключ отсутствует. Тип ключа должен соответствовать типу ключа объекта Map в языке EME-L. |
| `Lookup(key, defaultValue)` | key: ключ, defaultValue: любой тип, кроме Object | любой тип | Значение по ключу либо `defaultValue`. Выбрасывает исключение, если в качестве значения по умолчанию передан Object — используйте `Lookup` с одним параметром и проверяйте результат на `is_empty()`. |
| `IsKeyExists(key)` | key: ключ | Boolean | TRUE — ключ найден, FALSE — ключ отсутствует. |
| `SetAt(key, value)` | key: ключ, value: любой тип | Empty | Добавляет пару «ключ-значение» в объект Map. Если ключ уже существует, значение перезаписывается. Для типа ключа String допускается передача целого или вещественного числа, которое будет преобразовано в строку. Выбрасывает исключение при несовпадении типа ключа (кроме случая String + Integer/Real). В режиме отладки (`is_debug_mode()`) или для системного филиала при превышении количества элементов над начальным размером хэш-таблицы выдаётся предупреждение о необходимости задать размер через `InitHashTable`. |
| `RemoveKey(key)` | key: ключ | Boolean | Удаляет ключ и связанное значение из объекта Map. TRUE — ключ удалён, FALSE — ключ отсутствовал. При удалении ключа, на который указывает итератор `NextKey`, итератор автоматически смещается на следующий элемент. |
| `RemoveAll()` | — | Empty | Полностью очищает объект Map, удаляя все пары «ключ-значение». |

## Перебор элементов — FirstKey, NextKey, KeyFound, FirstValue, NextValue

В языке EME-L объект Map поддерживает два способа последовательного перебора пар. Первый — через методы `FirstKey`/`NextKey`/`KeyFound` (перебор с возвратом ключей) и `FirstValue`/`NextValue` (перебор с возвратом значений).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FirstKey()` | — | ключ или Empty | Устанавливает итератор на первую пару объекта Map, возвращает её ключ. Empty — объект пуст. |
| `NextKey()` | — | ключ или Empty | Перемещает итератор на следующую пару объекта Map, возвращает её ключ. Empty — достигнут конец коллекции. |
| `FirstValue()` | — | любой тип или Empty | Устанавливает итератор на первую пару объекта Map, возвращает её значение. Empty — объект пуст. |
| `NextValue()` | — | любой тип или Empty | Перемещает итератор на следующую пару объекта Map, возвращает её значение. Empty — достигнут конец коллекции. |
| `KeyFound()` | — | Boolean | Результат предыдущей итерации по объекту Map. TRUE — предыдущий переход на пару выполнен успешно, FALSE — достигнут конец коллекции. |

## Перебор через Loop — SetFirstLine, SetNextLine, IsValidLine, GetKey, GetValue

Второй способ перебора пар в объекте Map в языке EME-L — через механизм текущей «строки» методами `SetFirstLine`/`SetNextLine`/`IsValidLine` с чтением ключа и значения текущей пары методами `GetKey` и `GetValue`. Этот способ совместим с конструкцией `Loop (map)`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает внутренний итератор на первую пару объекта Map. |
| `SetNextLine()` | — | Empty | Перемещает внутренний итератор на следующую пару объекта Map. |
| `IsValidLine()` | — | Boolean | Проверяет, указывает ли итератор на допустимую пару объекта Map. TRUE — итератор на существующей паре, FALSE — достигнут конец коллекции. |
| `GetKey()` | — | ключ | Возвращает ключ текущей пары объекта Map. Выбрасывает исключение, если объект Map пустой или текущая строка недопустима. |
| `GetValue()` | — | любой тип | Возвращает значение текущей пары объекта Map. Выбрасывает исключение, если объект Map пустой или текущая строка недопустима. |

## Метаданные и тип ключа — GetCount, IsEmpty, GetHashTableSize, InitHashTable, SetKeyType, GetKeyType, GetContent

Методы метаданных в языке EME-L возвращают информацию о состоянии объекта Map: количество элементов, признак пустоты, размер хэш-таблицы и тип ключа.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetCount()` | — | Integer | Возвращает количество пар «ключ-значение» в объекте Map. |
| `IsEmpty()` | — | Boolean | Проверяет, содержит ли объект Map пары. TRUE — объект пуст, FALSE — содержит хотя бы одну пару. |
| `GetHashTableSize()` | — | Integer | Возвращает текущий размер хэш-таблицы объекта Map. |
| `InitHashTable(hashSize)` | hashSize: Integer | Empty | Задаёт размер хэш-таблицы объекта Map. **Очищает содержимое**, поэтому вызывается сразу после конструктора, до добавления элементов. Размер должен быть больше 0. |
| `SetKeyType(keyType)` | keyType: Integer / Real / String / Long Integer | Boolean | Изменяет тип данных ключа объекта Map. **При изменении типа весь объект очищается.** TRUE — тип изменён и объект очищен, FALSE — переданный тип совпадает с текущим. |
| `GetKeyType()` | — | ключ | Возвращает пустое значение, тип которого соответствует типу ключа объекта Map (Integer/Real/String/Long Integer). |
| `GetContent()` | — | String | Формирует строковое представление содержимого объекта Map для вывода в отладчике EME-L. Использовать только для отладки. |

## Копирование и сериализация — Copy, ToBase64, FromBase64

Методы копирования и сериализации в языке EME-L обеспечивают обмен содержимым между объектами Map и преобразование в строковый формат Base64 для строковых ключей.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Copy(sourceMap)` | sourceMap: Map (по ссылке) | String | Копирует все пары из другого объекта Map в текущий. Текущий объект полностью перезаписывается, включая тип ключа и размер хэш-таблицы. При пустом источнике размер хэш-таблицы устанавливается в 19. Выбрасывает исключение, если аргумент не является объектом Map. Возвращаемое значение (пустая строка) не используется — метод работает ради побочного эффекта копирования. |
| `ToBase64()` | — | String | Сериализует объект Map с типом ключа String в строку формата Base64. Доступен с версии ядра 47.4. Выбрасывает исключение, если тип ключа отличен от String. |
| `FromBase64(base64String)` | base64String: String | Empty | Загружает содержимое строки формата Base64 в объект Map с типом ключа String. Текущий объект полностью перезаписывается. Доступен с версии ядра 47.4. Выбрасывает исключение, если тип ключа отличен от String. |

## Примеры использования класса Map в языке EME-L

Построение словаря «код товара → количество» с целочисленным ключом и значением по умолчанию при отсутствии ключа:

```EME-L
'Создать Map с целочисленным ключом и заполнить парами код-количество'
mapQty = Object("Map", 100, 1);
mapQty.SetAt(1001, 5);
mapQty.SetAt(1002, 12);
'Lookup возвращает 0, если ключа нет в коллекции'
nQty = mapQty.Lookup(1001, 0);
```

Перебор элементов объекта Map через конструкцию `Loop` с получением ключей и значений текущей строки:

```EME-L
'Словарь «фамилия → склад» с строковым ключом'
mapNames = Object("Map", 17);
mapNames.SetAt("Иванов", "Склад 1");
mapNames.SetAt("Петров", "Склад 2");
'Loop автоматически вызывает SetFirstLine/SetNextLine/IsValidLine'
Loop (mapNames)
    csKey = mapNames.GetKey();
    csVal = mapNames.GetValue();
End Loop
```

# Класс Container1C

> Синонимы для поиска: CIContainer1C Container1C класс EME-L, контейнер 1С EME-L, CF EPF ERF CFE EME-L, чтение конфигурации 1С, метаданные 1С EME.WMS, File1C CIContainer1C::File1C, Tree1C CIContainer1C::Tree1C, GetFileByIndex GetFilesCount Open AsContainer AsString AsTree, Get GetCount IsTree FindValue, DataSerializer 1С, MetC_CLR.dll, сжатый контейнер CompressedContainer, десериализация дерева 1С, внешние обработки 1С EME-L, интеграция 1С без COM

Класс `Container1C` в языке EME-L предназначен для чтения и навигации по контейнерам 1С:Предприятие (файлы форматов CF, EPF, ERF, CFE). Класс Container1C в языке EME-L предоставляет доступ к внутренней структуре контейнера, позволяя получать количество содержащихся в нём файлов и извлекать отдельные файлы по индексу для дальнейшего анализа их содержимого, имени и типа.

Класс Container1C в системе EME.WMS используется для интеграции с внешними отчётами и обработками 1С, хранящимися в контейнерных файлах, а также для программного доступа к метаданным и данным конфигураций 1С без использования COM-соединения. Класс является обёрткой над внутренним объектом C++ `CIContainer1C` и требует наличия библиотеки `MetC_CLR.dll` для открытия контейнеров.

## Иерархия объектов класса Container1C

Класс `Container1C` в языке EME-L образует трёхуровневую объектную иерархию:

1. **`Container1C`** — корневой контейнер файла CF/EPF/ERF/CFE. Открывается методом `Open`, содержит список файлов.
2. **`File1C`** — отдельный файл внутри контейнера. Получается методом `GetFileByIndex` класса Container1C в языке EME-L, предоставляет имя, тип и содержимое (как строка, контейнер или дерево).
3. **`Tree1C`** — дерево данных, десериализованное из строкового содержимого файла. Позволяет перемещаться по уровням, проверять тип узла и искать значения.

## Создание объекта класса Container1C

```EME-L
objContainer = Object("Container1C");
Result = objContainer.Open(FileName);
```

Конструктор без аргументов класса Container1C в языке EME-L создаёт пустой объект. Открытие файла выполняется методом `Open`, который возвращает пустую строку при успехе или описание ошибки.

## Методы Container1C — открытие контейнера

Метод `Open` класса Container1C в языке EME-L открывает файл контейнера для чтения.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Open(fileName)` | fileName: String | String | Открывает контейнер 1С (CF, EPF, ERF, CFE) для чтения и доступа к внутренним файлам. Перед открытием освобождает ранее загруженные данные. Для работы требуется `MetC_CLR.dll`. Возвращает `""` при успехе или описание ошибки (например, не удалось загрузить `MetC_CLR.dll` или найти точку входа). |

## Навигация по файлам класса Container1C

Методы класса Container1C в системе EME.WMS для перебора файлов внутри открытого контейнера.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetFilesCount()` | — | Integer | Количество файлов в открытом контейнере. |
| `GetFileByIndex(index)` | index: Integer | File1C | Возвращает файл контейнера по индексу (с 0). При первом обращении создаёт внутренний кэш файлов; объекты файлов кэшируются для повторного использования. |

## Класс File1C — файл внутри контейнера 1С

Объект `File1C` в языке EME-L возвращается методом `GetFileByIndex` контейнера и не создаётся напрямую через `Object()`. Описывает отдельный файл внутри контейнера 1С и предоставляет доступ к его имени, типу и содержимому.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetFileName()` | — | String | Имя файла в контейнере. |
| `GetBodyType()` | — | String | Тип содержимого файла (например, `CompressedContainer`). |
| `AsContainer()` | — | Container1C | Содержимое файла в виде вложенного контейнера 1С. При первом вызове создаёт объект контейнера и кэширует его для последующих обращений. |
| `AsString()` | — | String | Содержимое файла в виде строки. |
| `AsTree()` | — | Tree1C | Содержимое файла в виде дерева данных. При первом вызове выполняет десериализацию строкового содержимого и кэширует объект дерева. |

Тип содержимого `CompressedContainer` класса File1C в языке EME-L означает, что файл является сжатым вложенным контейнером — для доступа к его файлам используйте `AsContainer()` и работайте с возвращённым объектом `Container1C` так же, как с корневым.

## Класс Tree1C — дерево данных контейнера 1С

Объект `Tree1C` в языке EME-L возвращается методом `AsTree()` класса `File1C` и представляет десериализованное содержимое файла в виде иерархического дерева. Каждый узел дерева класса Tree1C в системе EME.WMS — это либо лист со строковым значением, либо поддерево с дочерними узлами.

### Доступ к узлам класса Tree1C

Методы класса Tree1C в языке EME-L для навигации по уровням дерева.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Get(index1[, index2[, ...]])` | indexN: Integer (один или более) | String или Tree1C | Возвращает содержимое узла по цепочке индексов. Если конечный узел — лист, возвращает его строковое значение; если поддерево — возвращает объект `Tree1C`. Вызов без аргументов недопустим. |
| `GetCount()` | — | Integer | Количество дочерних узлов на текущем уровне дерева. |
| `IsTree(index)` | index: Integer | Boolean | TRUE — узел содержит поддерево; FALSE — узел является листом (содержит строковое значение). |

### Поиск значения в классе Tree1C

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FindValue(value)` | value: String | Tree1C или Empty | Выполняет рекурсивный поиск значения по всем уровням вложенности. Возвращает поддерево, содержащее искомое значение; если значение не найдено — пустое значение. |

## Примеры использования класса Container1C

Открытие контейнера CFE и чтение списка файлов класса Container1C в языке EME-L:

```EME-L
'Открыть контейнер 1С и получить количество файлов'
Container = Object("Container1C");
Result = Container.Open("D:\\Distrib\\EmeWms.cfe");
If (Result == "")
    FilesCount = Container.GetFilesCount();
    For (Index = 0; Index < FilesCount; Index = Index + 1)
        File = Container.GetFileByIndex(Index);
        is_catastrophic_log(File.GetFileName() + " " + File.GetBodyType());
    End For
End If
```

Чтение имени конфигурации из файла `configinfo` через дерево класса Container1C в системе EME.WMS:

```EME-L
'Найти корневой файл конфигурации и извлечь имя из дерева'
FilesMap = Object("Map", is_max(FilesCount * 1.2, 19));
For (Index = 0; Index < Container.GetFilesCount(); Index = Index + 1)
    File = Container.GetFileByIndex(Index);
    FilesMap.SetAt(File.GetFileName(), Index);
    If (File.GetFileName() == "configinfo")
        ConfigTree = File.AsTree();
        ConfigName = ConfigTree.Get(1, 1);
    End If
End For
```

Спуск во вложенный сжатый контейнер класса File1C в языке EME-L:

```EME-L
'Тип CompressedContainer — файл является вложенным контейнером'
File = Container.GetFileByIndex(Index);
If (File.GetBodyType() == "CompressedContainer")
    Nested = File.AsContainer();
    For (Index2 = 0; Index2 < Nested.GetFilesCount(); Index2 = Index2 + 1)
        NestedFile = Nested.GetFileByIndex(Index2);
        is_catastrophic_log(NestedFile.GetFileName());
    End For
End If
```

Проверка типа узла и поиск значения в дереве класса Tree1C в языке EME-L:

```EME-L
'Обойти дочерние узлы дерева и проверить, какие из них являются поддеревьями'
Tree = File.AsTree();
For (Index = 0; Index < Tree.GetCount(); Index = Index + 1)
    If (Tree.IsTree(Index))
        Subtree = Tree.Get(Index);
        Found = Subtree.FindValue("ИскомоеЗначение");
    End If
End For
```

# Класс PrintJob в языке EME-L — формирование и выполнение групповой печати

Класс `PrintJob` (C++ `CIPrintJob`) в языке EME-L — класс для формирования и выполнения групповой печати документов на принтер. Позволяет создать пакет печати, задать имя принтера, добавить файлы (PDF и другие), настроить параметры печати (количество копий, ориентация, двусторонняя печать, скрепление) и отправить задание на печать напрямую, через драйвер Windows или с использованием библиотеки Mupdf (в текущей реализации режим Mupdf не реализован). Также предоставляет текст последней ошибки, позволяет задавать имя задания и объединять несколько PDF-файлов в один.

В системе EME.WMS класс `PrintJob` используется скриптами пакетной печати для отправки на принтер комплектов документов — сертификатов, деклараций, сопроводительных листов, экспортированных в PDF. Объект накапливает пакет файлов и общие параметры, а запуск печати выполняется одним вызовом `Execute`.

## Создание объекта PrintJob в EME-L

### PrintJob — конструктор без аргументов

Создаёт объект группового задания печати в языке EME-L. После создания у объекта нет ни принтера, ни файлов в пакете — всё задаётся методами. Объект не привязан к контексту диалога или браузера.

```EME-L
'Создать объект группового задания печати'
r_PrintJob = Object("PrintJob");
```

## Настройка задания печати в EME-L

Методы для задания общих параметров пакета печати. Вызываются до наполнения пакета и запуска `Execute`.

### SetPrinterName — задать имя принтера

Устанавливает имя принтера, на который будет отправлено задание печати в языке EME-L. Если метод не вызывать, используется принтер по умолчанию системы.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | String | Да | Имя принтера |

**Возвращает**: `Boolean` — TRUE, если имя принтера задано; FALSE при ошибке.

### SetJobName — задать имя пакета печати

Задаёт имя пакета печати в языке EME-L. **Внимание:** в текущей реализации фактическая установка имени отключена (подозрение, что ломает печать) — вызов игнорируется, метод всегда возвращает TRUE.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | String | Да | Имя задания печати |

**Возвращает**: `Boolean` — всегда TRUE; в настоящее время фактическая установка имени не выполняется.

### SetStaple — режим скрепления листов

Устанавливает режим скрепления листов в задании печати в языке EME-L.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer | Да | Режим скрепления: 0 — без скрепления, 1 — одна скрепка, 2 — скрепка в углу, 3 — две скрепки, 4 — три скрепки, 5 — шесть скрепок |

**Возвращает**: `Boolean` — TRUE, если режим установлен; FALSE при ошибке.

### SetAddBlankPageForDuplexPrint — пустая страница при двусторонней печати (не реализован)

Предназначен для управления добавлением пустой страницы при двусторонней печати в языке EME-L. **Не реализован**: при вызове генерируется сообщение об ошибке «функция не реализована», метод возвращает пустое значение. Оставлен в интерфейсе класса для совместимости со старыми скриптами, где вызов встречался в виде комментария.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Boolean | Да | Признак добавления пустой страницы |

**Возвращает**: пустое значение (`Empty`).

## Наполнение пакета печати в EME-L

### AddFileToJob — добавить файл в пакет печати

Добавляет файл в пакет печати в языке EME-L. Метод вариативный — принимает от одного до пяти аргументов; пропущенные параметры принимают значения по умолчанию.

| Перегрузка | Параметры | Возвращает |
|------------|-----------|------------|
| `AddFileToJob(arg0)` | arg0: String (полный путь к файлу) | Boolean |
| `AddFileToJob(arg0, arg1)` | + arg1: Integer (копии, по умолчанию 1) | Boolean |
| `AddFileToJob(arg0, arg1, arg2)` | + arg2: Integer (ориентация: 0 — книжная, 1 — альбомная; по умолчанию 0) | Boolean |
| `AddFileToJob(arg0, arg1, arg2, arg3)` | + arg3: Integer (режим печати: 0 — односторонняя, 1 — двусторонняя; по умолчанию 0) | Boolean |
| `AddFileToJob(arg0, arg1, arg2, arg3, arg4)` | + arg4: Boolean (конвертировать в PDF; по умолчанию TRUE) | Boolean |

**Возвращает**: `Boolean` — TRUE, если файл добавлен; FALSE при ошибке.

При вызове только с именем файла используются значения по умолчанию для всех остальных параметров: одна копия, книжная ориентация, односторонняя печать, конвертация в PDF включена. Для пакетной печати в EME.WMS вызов чаще всего выглядит как `r_PrintJob.AddFileToJob(strTempFile, 1, 0, 1)` — одна копия, книжная ориентация, двусторонняя печать.

## Запуск печати и диагностика в EME-L

### Execute — запустить печать пакета

Запускает печать сформированного пакета в языке EME-L. Режим печати задаётся целочисленным аргументом.

| Перегрузка | Параметры | Возвращает |
|------------|-----------|------------|
| `Execute()` | — (по умолчанию режим 0) | Boolean |
| `Execute(arg0)` | arg0: Integer (режим) | Boolean |

**Режимы печати** (значения `arg0`):

- `0` — прямая печать PDF (обход драйвера Windows). Принтер должен поддерживать прямой приём PDF.
- `1` — печать через драйвер Windows.
- `2` — режим Mupdf. **Не реализован**: при вызове генерируется ошибка «функция не реализована», возвращается FALSE.
- Значения меньше 0 воспринимаются как режим 1 для совместимости со старыми скриптами, где параметр передавался как TRUE/FALSE (TRUE в EME-L равно -1).

**Возвращает**: `Boolean` — TRUE при успехе, FALSE при ошибке или выборе нереализованного режима.

### GetLastError — текст последней ошибки

Возвращает текстовое описание последней ошибки, возникшей при работе с заданием печати в языке EME-L. Пустая строка, если ошибок не было. После `Execute` результат этого метода позволяет диагностировать причину сбоя — например, отказ принтера или нереализованный режим.

**Возвращает**: `String` — описание ошибки или пустая строка.

## Объединение PDF-файлов в EME-L

### ConvertAndUnitePDFFiles — объединить несколько PDF в один

Преобразует переданные PDF-файлы и объединяет их в один результирующий PDF-файл в языке EME-L. Добавлен в класс `PrintJob` для подготовки пакетов документов перед печатью.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Array (по ссылке) | Да | Массив путей к исходным PDF-файлам |

**Возвращает**: `String` — полный путь к объединённому PDF-файлу.

## Примеры использования PrintJob в EME-L

Пакетная печать нескольких экспортированных PDF на принтер по умолчанию:

```EME-L
r_PrintJob = Object("PrintJob");
r_PrintJob.SetJobName("Пакетная печать EME.WMS.");
'Третий аргумент — ориентация (0 — книжная);'
'четвёртый — режим печати (1 — двусторонняя)'
r_PrintJob.AddFileToJob(strCertFile, 1, 0, 1);
r_PrintJob.AddFileToJob(strDeclFile, 1, 0, 1);
'Односторонняя печать комплекта'
r_PrintJob.AddFileToJob(strOrgFile, 1, 0, 0);
r_PrintJob.Execute(0);
err = r_PrintJob.GetLastError();
If (err != "")
    ArrayErrors.Add(err);
End If
```

Объединение нескольких PDF в один файл:

```EME-L
arrPdfs = Object("Array");
arrPdfs.Add("D:\\Data\\doc1.pdf");
arrPdfs.Add("D:\\Data\\doc2.pdf");
r_PrintJob = Object("PrintJob");
strUnited = r_PrintJob.ConvertAndUnitePDFFiles(arrPdfs);
'strUnited — полный путь к объединённому PDF'
```

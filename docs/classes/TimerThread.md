# Класс TimerThread

> Синонимы для поиска: CITimerThread TimerThread класс EME-L, периодический таймер EME-L, фоновый поток EME-L, StartTimer StopTimer Trace, таймер в отдельном потоке EME-L, EME-L TimerThread, запуск метода по таймеру EME-L

Класс `TimerThread` в языке EME-L — класс для запуска периодического таймера в отдельном потоке выполнения. В языке EME-L класс TimerThread предназначен для автоматического вызова указанного метода EME-L-класса с заданным интервалом времени в фоновом режиме.

В системе EME.WMS класс TimerThread предоставляет конструктор с необязательным именем потока, метод `StartTimer` для запуска периодического вызова, метод `StopTimer` для остановки и освобождения ресурсов, а также метод `Trace` для включения трассировки в журнале катастрофических событий. Деструктор `TimerThread` в языке EME-L не останавливает таймер автоматически — для корректного завершения работы необходимо явно вызвать `StopTimer()`.

## Создание объекта класса TimerThread в языке EME-L

Класс `TimerThread` в языке EME-L имеет два конструктора — без аргументов и с одним необязательным аргументом.

```EME-L
'Таймер без имени потока — внутреннее имя объекта будет сформировано как "ClassName."
objTimer = Object("TimerThread");

'Таймер с именем потока — внутреннее имя объекта будет "ClassName.MyThread"
objTimer = Object("TimerThread", "MyThread");
```

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `threadName` | String | Нет | Имя потока таймера. В языке EME-L используется для формирования уникального имени объекта EME-L-класса при вызове `StartTimer`. По умолчанию — пустая строка. |

## Управление таймером класса TimerThread в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `StartTimer(interval, className, methodName[, description])` | `interval`: Integer, `className`: String, `methodName`: String, `description`: String | Empty или Integer | Запускает таймер в отдельном потоке выполнения. В системе EME.WMS при успешном запуске возвращает пустое значение. Если таймер уже был запущен, возвращает `-1`. |
| `StopTimer()` | — | Integer | Останавливает таймер и удаляет объект потока. В языке EME-L при успешной остановке возвращает `0`. Если таймер не был запущен, возвращает `-1`. |
| `Trace(enable)` | `enable`: Boolean | Integer | Включает (`True`) или отключает (`False`) трассировку выполнения метода таймера в журнале катастрофических событий. В языке EME-L возвращает `0`. Если таймер не запущен, вызов игнорируется. |

### Параметры метода StartTimer в языке EME-L

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `interval` | Integer | Да | Интервал срабатывания таймера в миллисекундах. В языке EME-L должен быть больше `0`. |
| `className` | String | Да | Имя EME-L-класса, объект которого будет создан и чей метод будет вызываться. |
| `methodName` | String | Да | Имя метода указанного класса для периодического вызова. |
| `description` | String | Нет | Текстовое описание таймера для журналирования. В системе EME.WMS по умолчанию — `"<UNKNOWN>"`. |

### Имя создаваемого внутреннего объекта в языке EME-L

При запуске таймера внутри создаётся объект EME-L-класса `IClass` с именем, составленным из имени класса и имени потока. В языке EME-L пример ниже показывает, как имя потока влияет на внутреннее имя объекта.

```EME-L
'Если threadName = "MyThread", а className = "MyTimerClass",
'внутри будет создан объект с именем "MyTimerClass.MyThread"
objTimer = Object("TimerThread", "MyThread");
objTimer.StartTimer(60000, "MyTimerClass", "OnTimer", "Фоновая задача");
```

В языке EME-L, если имя потока не задано, внутреннее имя объекта заканчивается точкой (`"ClassName."`). Это имя используется только внутри движка; из скрипта EME-L объект таймера управляется через переменную, в которой сохранена ссылка.

## Примеры использования класса TimerThread в языке EME-L

Минимальный таймер, вызывающий метод `OnTimer` класса `MyBackgroundClass` раз в минуту:

```EME-L
'Создаём таймер с именем потока для уникальности объекта
objTimer = Object("TimerThread", "BackgroundSync");

'Запускаем вызов OnTimer каждые 60 секунд
objTimer.StartTimer(60000, "MyBackgroundClass", "OnTimer", "Синхронизация остатков");

'Включаем трассировку — в журнал будут писаться начало/конец каждого вызова
objTimer.Trace(True);

'Позже, при завершении работы, останавливаем таймер явно
objTimer.StopTimer();
```

Проверка состояния перед запуском и корректная остановка в языке EME-L:

```EME-L
objTimer = Object("TimerThread");

'Если таймер уже запущен, StartTimer вернёт -1
result = objTimer.StartTimer(30000, "HeartbeatClass", "Ping", "Heartbeat");
If (result = -1)
    is_message(tr("Внимание"), tr("Таймер уже запущен"), 1, 1);
    Return;
End If

'...по завершении работы
If (objTimer.StopTimer() = 0)
    'Таймер остановлен
End If
```

## Особенности поведения класса TimerThread в языке EME-L

- Первый вызов метода происходит **сразу** после запуска таймера, до первого срабатывания таймера. После этого метод вызывается с заданным интервалом.
- Метод таймера в системе EME.WMS выполняется в отдельном потоке, поэтому внутри него должны соблюдаться правила потокобезопасности: использование семафоров (`Semaphore`), атомарных операций (`Atom`) и транзакций при записи в базу данных.
- `StopTimer()` в языке EME-L возвращает `0` только если таймер был запущен. После остановки повторный запуск на том же объекте `TimerThread` может не сработать, поскольку внутренний флаг состояния не сбрасывается в методе `StopTimer`. В языке EME-L рекомендуется создавать новый объект `TimerThread` для каждого цикла запуска/остановки.

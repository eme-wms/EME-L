# Класс FTP

> Синонимы для поиска: CIFTP FTP класс EME-L, FTP-клиент EME-L, Connect Close SetCurrentDirectory GetCurrentDirectory SetParentDirectory CreateDirectory RemoveDirectory GetList GetFile PutFile Rename RemoveFile IsExistFile IsExistPath FindFile EnableLog SetLogName GetLogName GetServerSystem SetOperationTimeout GetOperationTimeout SetPassiveMode GetPassiveMode SetTLSMode GetLastError FTPS PASV TLS EME-L, файловый обмен FTP EME-L, бинарный режим ASCII FTP EME-L, пассивный режим FTP EME-L

Класс `FTP` в языке EME-L — клиент для доступа к файлам и каталогам на удалённом FTP-сервере. В системе EME.WMS класс FTP предоставляет полный набор операций для работы с файловой системой FTP-сервера: установка и разрыв соединения, навигация по каталогам, создание и удаление каталогов, копирование файлов между локальной системой и сервером, переименование и удаление файлов, проверка существования файлов и путей.

Класс FTP в языке EME-L поддерживает активный и пассивный режимы передачи данных, FTPS (TLS), а также ASCII и бинарный режимы передачи файлов. Включает механизм ведения лога операций для диагностики и отладки взаимодействия с сервером. Позволяет получать информацию о типе операционной системы FTP-сервера (UNIX, WINDOWS). Используется для автоматизации обмена файлами с внешними системами через протокол FTP/FTPS — например, для импорта и экспорта XML-документов, выгрузки отчётов на FTP заказчика, синхронизации файлов конфигурации.

Объект класса `FTP` в языке EME-L создаётся через `Object()` без аргументов. Для начала работы необходимо вызвать метод `Connect` с параметрами подключения. Все операции с файлами выполняются в контексте текущего рабочего каталога на сервере. Объект класса FTP не привязан к контексту диалога или браузера — это самостоятельный сетевой клиент в языке EME-L.

## Создание объекта класса FTP

Класс `FTP` в языке EME-L имеет единственный конструктор — без аргументов. После создания объекта вызовите `Connect` для установки соединения, при необходимости предварительно настроив пассивный режим, TLS или таймаут.

```EME-L
'Создание FTP-клиента в языке EME-L — соединение ещё не установлено'
objFTP = Object("FTP");
```

В системе EME.WMS объект класса FTP часто создаётся как член класса (через `Const` в конструкторе), чтобы переиспользовать одно соединение на протяжении жизненного цикла объекта-обёртки. Соединение устанавливается один раз методом `Connect`, после чего выполняются файловые операции.

## Соединение с FTP-сервером

Методы установки и разрыва соединения с FTP-сервером в языке EME-L.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Connect(arg0, arg1, arg2)` | arg0: String (URL), arg1: String (логин), arg2: String (пароль) | Integer | Устанавливает соединение и выполняет авторизацию. Если соединение уже установлено, возвращает 1 без повторного подключения. URL автоматически дополняется префиксом `ftp://` при отсутствии. Логин и пароль могут быть заданы как в URL, так и через отдельные параметры; при совпадении приоритет у отдельных параметров. Если в URL указан каталог, выполняется переход в него после авторизации. 1 — успех, 0 — ошибка. |
| `Close()` | — | Empty | Закрывает активное соединение и сбрасывает флаг подключения. После вызова операции с сервером невозможны до повторного `Connect`. |

Метод `Connect` в языке EME-L принимает URL в формате `ftp://IP_адрес/каталог`. Если порт не указан, используется стандартный для FTP — 21. Если в URL заданы логин и пароль (например, `ftp://login:password@host/path`), они используются при отсутствии отдельных параметров; при совпадении приоритет отдается отдельным параметрам.

## Навигация по каталогам FTP-сервера

Методы класса FTP в языке EME-L для навигации по каталогам на удалённом FTP-сервере.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetCurrentDirectory(arg0)` | arg0: String (директория) | Integer | Изменяет текущий рабочий каталог. Рекомендуется указывать путь без завершающего слэша. 1 — успех, 0 — каталог не существует или недоступен. |
| `GetCurrentDirectory()` | — | String | Полный путь текущего рабочего каталога (например, `/home/user/docs`). Пустая строка при ошибке. |
| `SetParentDirectory()` | — | Integer | Переход в родительский каталог (команда CDUP). 1 — успех, 0 — сбой или текущий каталог корневой. |
| `CreateDirectory(arg0)` | arg0: String (директория) | Integer | Создаёт новый каталог в текущем рабочем каталоге. 1 — успех, 0 — каталог уже существует, недостаточно прав или другая ошибка. |
| `RemoveDirectory(arg0)` | arg0: String (директория) | Empty | Зарезервированный метод-заглушка. Текущая реализация не выполняет удаление каталога и всегда возвращает пустое значение без фактических действий. |

Метод `RemoveDirectory` класса FTP в языке EME-L объявлен в регистре методов, но его тело не содержит фактической операции удаления — это заглушка, всегда возвращающая пустой результат.

## Операции с файлами на FTP-сервере

Методы скачивания, загрузки, переименования и удаления файлов класса FTP в языке EME-L. Все операции выполняются относительно текущего рабочего каталога FTP-сервера.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetFile(arg0, arg1)` | arg0: String (имя на FTP), arg1: String (локальный путь) | Integer | Копирует файл с FTP в локальную систему в бинарном режиме. 1 — успех, 0 — файл не найден, нет прав на запись, сетевой сбой. |
| `GetFile(arg0, arg1, arg2)` | + arg2: Integer (режим) | Integer | То же с указанием режима передачи. 1 — ASCII (текстовые файлы), 2 — бинарный (по умолчанию). |
| `PutFile(arg0, arg1)` | arg0: String (локальный путь), arg1: String (имя на FTP) | Integer | Копирует файл из локальной системы на FTP в бинарном режиме. 1 — успех, 0 — локальный файл не найден, нет прав, сетевой сбой. |
| `PutFile(arg0, arg1, arg2)` | + arg2: Integer (режим) | Integer | То же с указанием режима передачи. 1 — ASCII, 2 — бинарный (по умолчанию). |
| `Rename(arg0, arg1)` | arg0: String (старое имя), arg1: String (новое имя) | Integer | Переименовывает файл или каталог в текущем каталоге. 1 — успех, 0 — файл не найден, новое имя занято или недостаточно прав. |
| `RemoveFile(arg0)` | arg0: String (имя файла) | Integer | Удаляет файл в текущем рабочем каталоге. 1 — успех, 0 — файл не найден или недостаточно прав. |

Режим передачи файлов в языке EME-L: значение 1 — ASCII-режим для текстовых файлов, значение 2 — бинарный режим для всех типов файлов (используется по умолчанию, если аргумент `mode` опущен).

## Поиск и проверка существования файлов на FTP

Методы класса FTP в языке EME-L для поиска файлов и проверки существования файлов и каталогов.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FindFile(arg0)` | arg0: Array (по ссылке) | Integer | Ищет файлы в текущем каталоге по маске `*.*` и записывает их имена в массив. Подкаталоги не включаются. Массив очищается перед заполнением. Возвращает количество найденных файлов. |
| `FindFile(arg0, arg1)` | arg0: Array (по ссылке), arg1: String (маска) | Integer | То же с указанной маской. Поддерживает подстановочные символы `*` и `?`. |
| `IsExistFile(arg0)` | arg0: String (путь к файлу) | Integer | Проверяет существование файла. 1 — существует, 0 — не существует или ошибка. |
| `IsExistPath(arg0)` | arg0: String (путь к каталогу) | Integer | Проверяет существование каталога или пути. 1 — существует, 0 — не существует или ошибка. |
| `GetList()` | — | String | Подробный список содержимого текущего каталога (права, размер, дата, имя). Формат зависит от типа FTP-сервера. Пустая строка при ошибке. |
| `GetServerSystem()` | — | String | Тип ОС FTP-сервера: `"UNIX"`, `"WINDOWS"` или `"UNKNOWN"`. |

Метод `FindFile` класса FTP в языке EME-L очищает массив перед заполнением, поэтому предыдущее содержимое теряется. Поиск выполняется только в текущем рабочем каталоге FTP-сервера; подкаталоги в результат не включаются. Метод `GetServerSystem` возвращает строковый идентификатор типа операционной системы: `"UNIX"` для UNIX-подобных ОС, `"WINDOWS"` для Windows, `"UNKNOWN"` если тип не определён.

## Настройки передачи данных и таймаут

Методы настройки режима передачи данных класса FTP в языке EME-L.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetPassiveMode(arg0)` | arg0: Integer (режим) | Empty | Включает или отключает пассивный режим (PASV). В пассивном режиме клиент инициирует соединение для передачи данных, что обеспечивает работу через межсетевые экраны и NAT. 0 — активный режим (PORT), не 0 — пассивный (PASV). |
| `GetPassiveMode()` | — | Integer | Текущий режим передачи. 1 — пассивный (PASV), 0 — активный (PORT). |
| `SetTLSMode(arg0)` | arg0: Integer (режим) | Empty | Включает или отключает защищённое соединение FTPS (FTP over TLS/SSL). 0 — обычное FTP, не 0 — FTPS. |
| `SetOperationTimeout(arg0)` | arg0: Integer (мс) | Empty | Максимальное время ожидания выполнения операций. 0 — системное значение по умолчанию. |
| `GetOperationTimeout()` | — | Integer | Текущий таймаут операций в миллисекундах. 0 — системное значение по умолчанию. |

Настройки `SetPassiveMode` и `SetTLSMode` в языке EME-L рекомендуется вызывать до метода `Connect`, чтобы они действовали с момента установки соединения.

## Логирование и диагностика FTP-клиента

Методы класса FTP в языке EME-L для ведения журнала операций и получения информации об ошибках.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `EnableLog(arg0)` | arg0: Integer (флаг) | Empty | Включает или отключает ведение журнала операций FTP-клиента. 0 — отключить, не 0 — включить. При включённом логе все команды и ответы сервера записываются в лог-файл. |
| `SetLogName(arg0)` | arg0: String (имя лога) | Empty | Устанавливает полный путь и имя файла журнала в локальной файловой системе. |
| `GetLogName()` | — | String | Полный путь и имя текущего лог-файла. Пустая строка, если не задан. |
| `GetLastError()` | — | String | Текстовое описание последней ошибки на русском языке (сетевой сбой, отказ в доступе, неверный логин и т. п.). Пустая строка, если ошибок не было. |

Метод `GetLastError` класса FTP в языке EME-L возвращает текст ошибки на русском языке — это полезно для диагностики проблем при работе с FTP-сервером в системе EME.WMS. Вызывайте его после любой операции, вернувшей 0, чтобы получить подробное описание причины сбоя.

## Примеры

Подключение к FTP-серверу и скачивание файла методом GetFile класса FTP в языке EME-L:

```EME-L
'Создать FTP-клиент и подключиться к серверу'
objFTP = Object("FTP");
objFTP.SetPassiveMode(TRUE);
Result = objFTP.Connect("ftp://10.138.4.25:21/inbox", "user", "password");
If (Result == 0)
    is_catastrophic_log("FTP: " + objFTP.GetLastError());
    Return False;
End If

'Скачать файл из текущего каталога FTP в локальную папку'
objFTP.GetFile("report.xml", "D:\\Data\\report.xml", 1);
objFTP.Close();
```

Загрузка файла на FTP методом PutFile и поиск файлов по маске методом FindFile в языке EME-L:

```EME-L
objFTP = Object("FTP");
objFTP.SetPassiveMode(TRUE);
objFTP.SetTLSMode(TRUE);
Result = objFTP.Connect("ftp://ftptest.example.com/outbound", "login", "pass");
If (Result == 0)
    ErrorText = objFTP.GetLastError();
    Return;
End If

'Загрузить локальный файл на сервер в бинарном режиме'
objFTP.PutFile("D:\\Export\\orders.xml", "orders.xml");

'Найти все XML-файлы в текущем каталоге FTP'
arrFiles = Object("Array");
Count = objFTP.FindFile(arrFiles, "*.xml");

objFTP.Close();
```

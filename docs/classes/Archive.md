# Класс Archive

> Синонимы для поиска: CIArchive Archive класс EME-L, сериализация EME-L, бинарная сериализация EME-L, Base64 EME-L, Store Load Empty GetText PutText AsString PutRPC SeekToBegin, CArchive EME-L, RPC сериализация EME-L, DataStorage EME-L, CString long int сериализация EME-L, архивирование данных EME-L

Класс `Archive` в языке EME-L выполняет бинарную сериализацию данных для хранения, передачи и восстановления значений различных типов. Класс Archive в языке EME-L предоставляет методы для записи и чтения данных во внутренний буфер памяти с использованием механизма архивирования CArchive. Поддерживает сериализацию произвольных значений, явную запись и извлечение примитивных типов C++ (`CString`, `long`, `int`), а также преобразование содержимого архива в текстовое представление Base64 и обратно.

Класс Archive в системе EME.WMS используется для упаковки данных перед передачей через RPC, сохранения состояния объектов и организации потокового чтения/записи бинарных данных в прикладных скриптах EME-L.

Объект класса `Archive` в языке EME-L создаётся через `Object()` и не имеет параметров конструктора — это самостоятельный класс, не привязанный к контексту диалога или браузера.

## Создание объекта класса Archive

```EME-L
objArchive = Object("Archive");
```

Конструктор без аргументов класса Archive в языке EME-L создаёт пустой объект архива. Внутренние буферы и файлы не инициализируются до первого вызова методов записи (`Store`) или чтения (`Load`, `PutText`, `PutRPC`).

## Управление состоянием

Методы класса Archive в языке EME-L для управления внутренним состоянием объекта архива.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Empty()` | — | Empty | Очищает архив, удаляя все внутренние данные, буферы памяти и объекты архива. После вызова архив возвращается в пустое состояние, аналогичное состоянию после создания. |
| `Store()` | — | Empty | Подготавливает архив к записи данных. Если внутренний архив ещё не создан, инициализирует буфер памяти и объект архива в режиме записи (store). Если архив уже существует и находится в режиме чтения — вызывает ошибку. |
| `SeekToBegin()` | — | Empty | Устанавливает позицию чтения в начало внутреннего буфера памяти и пересоздаёт объект архива в режиме чтения. Используется для повторного чтения данных из архива, который уже был заполнен. Архив должен содержать данные. |

## Запись данных

Метод `Store` класса Archive в языке EME-L последовательно сериализует значения в архив. Первый вызов `Store` (с аргументом или без) инициализирует архив в режиме записи; последующие вызовы `Load` на этом же объекте вызовут ошибку — направление сериализации (чтение/запись) нельзя сменить. Для перечитывания записанного архива используйте `SeekToBegin`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Store(данные)` | данные: любой тип | Empty | Сериализует указанное значение и записывает его в архив. Если архив ещё не инициализирован, автоматически создаёт внутренний буфер памяти и переводит архив в режим записи. |
| `Store(данные, тип_данных)` | данные: String/Integer, тип_данных: String | Empty | Записывает значение с явным указанием примитивного типа C++ для корректной сериализации строк и целых чисел. |

Значения аргумента `тип_данных` класса Archive в языке EME-L:

| Код типа | Тип C++ | Что сериализуется |
|----------|---------|-------------------|
| `"CString"` | `CString` | Строковое значение |
| `"long"` | `long` | Длинное целое число |
| `"int"` | `int` | Целое число |

## Чтение данных

Метод `Load` класса Archive в языке EME-L последовательно извлекает значения из архива в порядке их записи. При извлечении объекта (без указания типа) он автоматически помещается в пул объектов текущего контекста. В системе EME.WMS это позволяет восстановленным объектам сразу использоваться в вычислениях без дополнительной регистрации.

> Если при вызове `Load()` без аргументов извлекается объект и текущий контекст невалиден, метод сообщает об устаревшем контексте. В этом случае задайте контекст явно методом `SetContext`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Load()` | — | любой тип | Извлекает очередное значение путём десериализации. Архив должен быть предварительно заполнен и находиться в режиме чтения. Если извлечённое значение является объектом, оно помещается в пул объектов текущего контекста. |
| `Load(тип_данных)` | тип_данных: String | String или Integer | Извлекает значение указанного примитивного типа C++. Принимает те же коды `"CString"`, `"long"`, `"int"`, что и `Store`. |

## Преобразование в Base64 и обратно

Эти методы класса Archive в языке EME-L связывают бинарный буфер архива и внешнее текстовое представление (Base64), используемое для передачи и хранения. Base64 — стандартное текстовое кодирование бинарных данных, применяемое в системе EME.WMS для передачи сериализованных значений через RPC и файловые каналы.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetText()` | — | String | Завершает запись, закрывает архив и преобразует содержимое внутреннего буфера в строку Base64, полностью освобождая ресурсы. После вызова архив становится пустым. |
| `AsString()` | — | String | Возвращает текущее содержимое буфера в виде строки Base64 **без** изменения состояния архива и без освобождения ресурсов. В отличие от `GetText`, не закрывает и не удаляет архив. |
| `PutText(текст)` | текст: String (Base64) | Empty | Заполняет архив данными из строки Base64. Декодирует текст во внутренний буфер и создаёт объект архива в режиме чтения. Архив должен быть пустым до вызова. |
| `PutRPC(объект_данных)` | объект_данных: DataStorage (по ссылке) | Empty | Заполняет архив данными из объекта низкоуровневого хранилища `DataStorage`, полученного через RPC. Декодирует содержимое во внутренний буфер и создаёт объект архива в режиме чтения. Архив должен быть пустым. |

## Порядок вызова методов

Типичные сценарии использования класса Archive в языке EME-L:

**Запись (сериализация):** `Store(...)` → `GetText()` или `AsString()` для получения Base64.

**Чтение (десериализация):** `PutText(...)` или `PutRPC(...)` → последовательно `Load(...)` до исчерпания данных.

**Перечитывание:** после `Load` вызвать `SeekToBegin()` и снова `Load(...)`.

## Примеры

Упаковка строки и целого числа в архив класса Archive в языке EME-L и получение Base64:

```EME-L
objArchive = Object("Archive");
objArchive.Store("значение для передачи", "CString");
objArchive.Store(42, "long");
Base64 = objArchive.GetText();
'Base64 содержит сериализованные данные, архив пуст'
```

Обратное чтение значений из архива класса Archive в языке EME-L:

```EME-L
objArchive = Object("Archive");
objArchive.PutText(Base64);
Str = objArchive.Load("CString");
Num = objArchive.Load("long");
'Порядок Load должен совпадать с порядком Store'
```

Реальный пример из системы EME.WMS — чтение файла-архива EMEArc, где `Archive` десериализует бинарное тело, отрезанное от сигнатуры:

```EME-L
objFile = Object("File", FileName);
objArchive = Object("Archive");
objArchive.PutText(objFile.GetDataFromBegin(SignLen, objFile.GetSize() - SignLen).AsString());

While (TRUE)
    RecordName = objArchive.Load("CString");
    If (RecordName == "NULL_RECORD")
        Break;
    End If
    LineKey = objArchive.Load("CString");
End While
```

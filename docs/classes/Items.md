# Класс Items

> Синонимы для поиска: CIItems Items класс EME-L, набор элементов интегратора EME-L, DBIntegrator элементы EME-L, дерево интегратора EME-L, AttachToIntegrator AttachToIntegratorEx AddItem SetCurrentIcon SetCurrentCategory SetCategoryProperties SetCategoryRef SetCategorySkip FindCategoryProperties FindCategoryRef FindCategorySkip GetCategory GetCategoryRef GetText GetCategoryProperties SetTextColor SetBkColor, фильтрация элементов интегратора EME-L, категория элемента EME-L, ссылка REF категории EME-L, CSkip Skip метод EME-L, NULL_REF MAXDWORD __REF__ __CLASS__ __SKIP__, иерархия элементов DBIntegrator EME.WMS, навигационная панель EME-L, древовидный справочник EME-L

Класс `Items` в языке EME-L — класс-оболочка для формирования и управления набором абстрактных элементов в DBIntegrator. Предназначен для программного создания дерева элементов интегратора, привязки к ним EME-L класса-обработчика, настройки категорий, иконок, цветов и фильтрующих методов, а также для последующей вставки элементов в DBIntegrator в виде корневых узлов или иерархического дерева.

Класс Items в системе EME.WMS обеспечивает динамическое построение содержимого интегратора: навигационных панелей, древовидных справочников и иерархических структур данных. Элементы набора связываются с EME-L классом-обработчиком, методы которого вызываются для инициализации (`OnInit`) и фильтрации элементов.

## Создание объекта класса Items

Конструктор класса Items в языке EME-L имеет две перегрузки, доступные из прикладного скрипта через `Object()`.

```EME-L
'Набор без родителя — корневой уровень интегратора'
objItems = Object("Items", "MyHandlerClass");

'Набор с родительским объектом Items — дочерний уровень'
objItems = Object("Items", "MyHandlerClass", objParentItems);
```

Конструктор класса Items в языке EME-L создаёт пустой набор элементов, привязывает его к указанному EME-L классу-обработчику и немедленно вызывает метод `OnInit` этого класса. Если метод `OnInit` не найден — генерируется ошибка. Если имя класса — пустая строка, набор создаётся без обработчика (метод `OnInit` не вызывается), и элементы можно добавлять в интегратор напрямую. Внутренний индекс текущего элемента инициализируется значением -1.

## Управление элементами класса Items

Методы класса Items в языке EME-L для добавления элементов в набор и настройки их визуального представления.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddItem(text, className)` | text: String, className: String | Empty | Добавляет элемент в набор. Новому элементу присваиваются текущие иконка и категория, если они были заданы через `SetCurrentIcon`/`SetCurrentCategory` до вызова. |
| `SetCurrentIcon(icon)` | icon: String | Empty | Устанавливает иконку для всех последующих элементов. Значение сохраняется до следующего вызова. |
| `SetCurrentCategory(category)` | category: String | Empty | Устанавливает категорию для всех последующих элементов. Категория используется при иерархической вставке через `AttachToIntegratorEx`. |
| `SetTextColor(color)` | color: Integer (RGB) | Empty | Устанавливает цвет текста последнего добавленного элемента. |
| `SetBkColor(color)` | color: Integer (RGB) | Empty | Устанавливает цвет фона последнего добавленного элемента. |

При создании элемента через `AddItem` класса Items в языке EME-L цвета текста и фона по умолчанию устанавливаются в `MAXDWORD` — значение, означающее «цвет не задан», при котором используется стандартный цвет элемента управления.

## Свойства категории класса Items

Методы работы со свойствами категории последнего добавленного элемента набора Items. Свойства хранятся в виде строки параметров (формат `CParameters`) и используются при фильтрации и навигации в DBIntegrator.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetCategoryProperties(properties)` | properties: String | Empty | Задаёт строку свойств категории для последнего добавленного элемента. |
| `SetCategoryRef(ref)` | ref: Integer | Empty | Записывает числовую ссылку REF в свойства категории. Сохраняется под именем `__REF__`. |
| `SetCategorySkip(methodName)` | methodName: String | Empty | Задаёт фильтрующий метод для категории. В свойства записываются имя текущего EME-L класса (`__CLASS__`) и имя метода-фильтра (`__SKIP__`). |

## Поиск по цепочке родительских наборов в классе Items

Методы поиска класса Items в языке EME-L поднимаются по цепочке родительских наборов `Items`, пока не найдут элемент с совпадающей категорией. Это позволяет дочерним наборам получать свойства, установленные на более высоком уровне иерархии.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FindCategoryProperties(category)` | category: String | String или Empty | Строка свойств категории при успешном поиске. Пустое значение — категория не найдена. |
| `FindCategoryRef(category)` | category: String | Integer | Значение REF из свойств категории. `NULL_REF` — категория не найдена или свойство `__REF__` отсутствует. |
| `FindCategorySkip(category)` | category: String | CSkip | Объект `CSkip`, настроенный на фильтрующий метод найденной категории. Объект кэшируется внутри набора — повторные вызовы с той же категорией возвращают ранее созданный объект. |

## Доступ к текущему элементу класса Items

Методы доступа класса Items в языке EME-L к свойствам текущего (активного) элемента набора.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetText()` | — | String | Отображаемый текст текущего элемента. |
| `GetCategory()` | — | String | Категория текущего элемента. |
| `GetCategoryProperties()` | — | String или Empty | Строка свойств категории текущего элемента. Пустое значение — текущий элемент отсутствует. |
| `GetCategoryRef()` | — | Integer | Значение REF из свойств категории текущего элемента. `NULL_REF` — свойство `__REF__` отсутствует. |

## Вставка в DBIntegrator из класса Items

Методы класса Items в языке EME-L для вставки набора элементов в интегратор. Вставка выполняется либо плоским списком (корневые узлы), либо с построением иерархического дерева по категориям.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AttachToIntegrator(integrator)` | integrator: DBIntegrator (по ссылке) | Empty | Вставляет все элементы набора в интегратор как корневые (верхнеуровневые) объекты. |
| `AttachToIntegratorEx(integrator)` | integrator: DBIntegrator (по ссылке) | Empty | Вставляет элементы в виде иерархического дерева. Сначала добавляются элементы без категории как корневые узлы, затем элементы, чья категория совпадает с текстом уже добавленного элемента, вставляются как его дочерние узлы. Процесс повторяется до построения всех уровней дерева. |

## Вложенный класс CSkip в классе Items

Объект `CSkip` в языке EME-L возвращается методом `FindCategorySkip` и предназначен для динамической фильтрации элементов дерева интегратора через прикладной метод EME-L. Объект создается с привязкой к EME-L классу и методу, имена которых были сохранены через `SetCategorySkip`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Skip(arg0)` | arg0: любой тип | Boolean или произвольный | Вызывает фильтрующий метод EME-L класса, заданного при создании объекта, и передаёт в него аргумент. При отсутствии класса или метода возвращает `FALSE`. Тип и семантика возвращаемого значения определяются реализацией фильтрующего метода в EME-L классе. |

## Константы класса Items

Константы, используемые методами класса Items в языке EME-L и системой EME.WMS.

| Константа | Описание |
|-----------|----------|
| `NULL_REF` | Значение ссылки по умолчанию: возвращается методами `FindCategoryRef` и `GetCategoryRef`, если категория или свойство `__REF__` не найдены. |
| `MAXDWORD` | Цвет текста и фона по умолчанию для новых элементов: означает «цвет не задан», используется стандартный цвет элемента управления. |
| `__REF__` | Имя свойства категории, в котором `SetCategoryRef` сохраняет числовую ссылку REF. |
| `__CLASS__` | Имя свойства категории, в котором `SetCategorySkip` сохраняет имя EME-L класса-обработчика. |
| `__SKIP__` | Имя свойства категории, в котором `SetCategorySkip` сохраняет имя метода-фильтра. |

## Примеры работы с классом Items

Создание набора элементов с категориями и иерархическая вставка в DBIntegrator в языке EME-L:

```EME-L
'Создать корневой набор с обработчиком'
objItems = Object("Items", "TreeHandler");
objItems.SetCurrentIcon("folder");
objItems.SetCurrentCategory("");
objItems.AddItem("Склад 1", "WarehouseItem");
objItems.SetCurrentCategory("Склад 1");
objItems.AddItem("Зона A", "ZoneItem");
objItems.AddItem("Зона B", "ZoneItem");
'Вставить как иерархическое дерево в DBIntegrator'
objItems.AttachToIntegratorEx(integrator);
```

Поиск REF родительской категории по цепочке наборов класса Items в системе EME.WMS:

```EME-L
objItems = Object("Items", "ChildHandler", objParentItems);
'Найти REF категории «Склад» в родительских наборах'
nRef = objItems.FindCategoryRef("Склад");
'Если nRef не NULL_REF — категория найдена'
If (nRef != NULL_REF)
    RefValue = nRef;
End If
```

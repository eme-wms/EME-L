# Класс CryptoPro

> Синонимы для поиска: CICryptoPro CryptoPro класс EME-L, электронная подпись EME-L, ЭП EME-L, CADES-BES EME-L, КриптоПро EME-L, ГОСТ Р 34.11-2012 EME-L, cades.dll, SetCertificate GetCertificate Sign SetGOST GetLastError, PCCERT_CONTEXT, Base64 подпись EME-L, открепленная подпись detached EME-L, хранилище сертификатов MY CA ROOT SPC, OID алгоритма хеширования, серийный номер сертификата

Класс `CryptoPro` в языке EME-L обеспечивает формирование электронной подписи CADES-BES с использованием сертификатов КриптоПро и библиотеки `cades.dll`. Класс CryptoPro в языке EME-L предоставляет методы для выбора сертификата из системных хранилищ Windows, создания усовершенствованной подписи в соответствии с российскими криптографическими стандартами ГОСТ Р 34.11-2012, настройки алгоритма хеширования и получения информации об ошибках.

Объект класса `CryptoPro` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный криптографический класс. Для работы требуются установленная библиотека `cades.dll` и действующий сертификат в системном хранилище Windows.

## Создание объекта класса CryptoPro

Класс `CryptoPro` в языке EME-L имеет три перегрузки конструктора.

```EME-L
'Конструктор без аргументов — сертификат выбирается позже методом SetCertificate'
objCrypto = Object("CryptoPro");

'Конструктор с серийным номером — сертификат ищется в хранилище "MY"'
objCrypto = Object("CryptoPro", "02B537A6001BACA59848C12354E25B4985");

'Конструктор с серийным номером и именем хранилища'
objCrypto = Object("CryptoPro", "02B537A6001BACA59848C12354E25B4985", "MY");
```

Серийный номер указывается в шестнадцатеричном представлении в языке EME-L; поиск выполняется без учёта регистра. Имя хранилища по умолчанию — `"MY"` (личные сертификаты пользователя).

## Выбор сертификата

Методы класса CryptoPro в языке EME-L для работы с системными хранилищами сертификатов Windows.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetCertificate()` | — | Boolean | Выполняет поиск сертификата в хранилище `"MY"` с пустым серийным номером. |
| `SetCertificate(arg0)` | arg0: String (серийный номер) | Boolean | Ищет сертификат по серийному номеру в хранилище `"MY"`. |
| `SetCertificate(arg0, arg1)` | arg0: String (серийный номер), arg1: String (хранилище) | Boolean | Ищет сертификат по серийному номеру в указанном хранилище. |
| `GetCertificate()` | — | Reference | Системный контекст сертификата Windows (`PCCERT_CONTEXT`), установленный ранее методом `SetCertificate` или конструктором. Возвращает `0` (нулевую ссылку), если сертификат не установлен. |

Допустимые имена хранилищ сертификатов в классе CryptoPro языка EME-L: `"MY"` (личные сертификаты пользователя, по умолчанию), `"CA"` (сертификаты центров сертификации), `"ROOT"` (доверенные корневые сертификаты), `"SPC"` (хранилище отозванных сертификатов). Метод `SetCertificate` возвращает `TRUE` (-1 в языке EME-L), если сертификат найден и установлен, и `FALSE` (0), если хранилище недоступно или сертификат не найден — в этом случае текст ошибки доступен через `GetLastError`.

## Формирование подписи

Метод `Sign` класса CryptoPro в языке EME-L формирует усовершенствованную электронную подпись CADES-BES для переданных данных с использованием установленного сертификата и алгоритма хеширования ГОСТ.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Sign(arg0)` | arg0: String или CDataStorage | String | Формирует присоединённую подпись CADES-BES. Base64 без символов перевода строки. |
| `Sign(arg0, arg1)` | arg0: String или CDataStorage, arg1: Boolean | String | То же; `arg1=TRUE` — откреплённая (detached) подпись, `FALSE` (по умолчанию) — присоединённая. |

Параметр данных `arg0` метода `Sign` класса CryptoPro в языке EME-L принимает строку (подписывается содержимое строки в кодировке текущей системной локали) или ссылку на объект `CDataStorage` с бинарными данными (передаётся ссылка на объект, содержащий массив байт). При ошибке метод `Sign` возвращает пустую строку — текст ошибки доступен через `GetLastError`. При успешном формировании возвращается подписанное сообщение в кодировке Base64 без символов перевода строки.

## Настройка алгоритма хеширования

Метод `SetGOST` класса CryptoPro в языке EME-L устанавливает идентификатор алгоритма хеширования ГОСТ, используемый при формировании электронной подписи.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetGOST(arg0)` | arg0: String (OID алгоритма) | Integer | Устанавливает алгоритм хеширования ГОСТ. Возвращает `1` — установлен, `0` — аргумент не передан. |
| `SetGOST()` | — | Integer | Без аргументов возвращает `0`, алгоритм не изменяется. |

По умолчанию класс CryptoPro в языке EME-L использует ГОСТ Р 34.11-2012 с длиной хеша 256 бит (OID `1.2.643.7.1.1.2.2`). Передаётся строковый OID, например `"1.2.643.7.1.1.2.2"` для ГОСТ Р 34.11-2012 256 бит или `"1.2.643.7.1.1.2.3"` для 512 бит.

## Информация об ошибках

Метод `GetLastError` класса CryptoPro в языке EME-L возвращает текст последней ошибки, возникшей при выполнении операций с сертификатами или формировании подписи.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetLastError()` | — | String | Описание ошибки на языке системной локали. Пустая строка — ошибок не было с момента создания объекта или последнего вызова. |

## Примеры

Установка сертификата и формирование откреплённой подписи в классе CryptoPro языка EME-L (шаблон из реального использования, серийный номер и данные заменены на плейсхолдеры):

```EME-L
objCrypto = Object("CryptoPro");
objCrypto.SetCertificate("02B537A6001BACA59848C12354E25B4985", "MY");
Sign = objCrypto.Sign(DocumentText, True);
If (Sign == "")
    is_catastrophic_log("Ошибка подписи " + objCrypto.GetLastError());
End If
```

Использование конструктора с серийным номером и формирование присоединённой подписи для авторизационного запроса в системе EME.WMS:

```EME-L
m_objCrypto = Object("CryptoPro", CERT_SN, "MY");
SignData = m_objCrypto.Sign(Data, False);
If (SignData == "")
    is_catastrophic_log("RequestAuthToken Ошибка подписи " + m_objCrypto.GetLastError());
    Return "";
End If
```

Явная смена алгоритма хеширования перед подписанием в языке EME-L:

```EME-L
objCrypto = Object("CryptoPro", "02B537A6001BACA59848C12354E25B4985");
objCrypto.SetGOST("1.2.643.7.1.1.2.3");
Sign = objCrypto.Sign(Payload, False);
```

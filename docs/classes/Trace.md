# Класс Trace

Класс `Trace` (C++ `CITrace`) в языке EME-L — класс для трассировки (логирования) строковых сообщений в процессе выполнения скриптов EME-L. В системе EME.WMS класс `Trace` предоставляет механизм накопления текстовых строк в именованном буфере трассировки с возможностью последующего просмотра, извлечения и очистки истории.

В языке EME-L объект класса `Trace` создаётся через `Object()` с одним обязательным аргументом — именем трассировщика. Имя используется для идентификации буфера в глобальном менеджере трассировки. В системе EME.WMS буфер существует в течение жизни объекта и автоматически освобождается при его уничтожении.

> Класс `Trace` в языке EME-L доступен в версиях ядра EME.WMS 16.49 и выше (условие компиляции `LAST_VERSION >= MAKE_VERSION(16, 49)`). Большинство методов опираются на внутренний менеджер трассировки, который создаётся в конструкторе только при наличии `KERNEL_16_52`. На ядрах 16.49–16.51, где `KERNEL_16_52` не определён, вызовы методов `Empty`, `GetFirstString`, `GetLastString`, `GetNextString`, `GetPreviousString` и `IsStringFound` обращаются к неинициализированному менеджеру — фактически класс неработоспособен. Метод `OpenInBrowser` дополнительно требует `KERNEL_16_55`.

## Создание объекта Trace в EME-L

Класс `Trace` в языке EME-L имеет один конструктор — с одним аргументом.

```EME-L
'Создаёт объект трассировщика с именем буфера'
objTrace = Object("Trace", "MyTrace");
```

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `name` | String | Да | Имя трассировщика для идентификации буфера в глобальном менеджере трассировки. |

Конструктор `Object("Trace", name)` в EME-L принимает ровно один аргумент. При попытке вызвать `Object("Trace")` без аргументов ядро генерирует ошибку `too rare`. В C++ классе `CITrace` присутствуют также конструкторы без аргументов и с `LPCTSTR`, но они не зарегистрированы через `IMPLEMENT_CTOR` и из EME-L недоступны.

## Методы работы с буфером трассировки в языке EME-L

В языке EME-L класс `Trace` предоставляет методы для добавления строк, получения всего содержимого, очистки буфера и открытия трассировки в браузере.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddLine(text)` | text: String | Empty | Добавляет строку текста в буфер трассировки в EME-L. |
| `GetAsSingleString()` | — | String или Empty | Возвращает все строки трассировки, объединённые в одну строку. В системе EME.WMS, если трассировщик недоступен (например, ядро ниже 16.52), возвращает пустое значение. |
| `Empty()` | — | Empty | Удаляет всю историю трассировки в EME-L, очищая буфер. |
| `OpenInBrowser()` | — | Empty | Открывает содержимое трассировки в окне браузера. Доступен в EME-L только в ядре 16.55 и выше. |

## Построчный перебор трассировки в языке EME-L

Методы перебора класса `Trace` в языке EME-L используют внутренний курсор буфера. Перед перебором в прямом направлении вызовите `GetFirstString`, в обратном — `GetLastString`. После каждого вызова проверяйте `IsStringFound`: если предыдущий вызов вернул строку из буфера, метод возвращает `TRUE`, иначе `FALSE` (буфер пуст или достигнута граница).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetFirstString()` | — | String или Empty | Возвращает первую строку истории трассировки в EME-L и устанавливает курсор на начало буфера. |
| `GetLastString()` | — | String или Empty | Возвращает последнюю строку истории трассировки в EME-L и устанавливает курсор на конец буфера. |
| `GetNextString()` | — | String или Empty | Возвращает следующую строку истории трассировки в EME-L относительно текущей позиции курсора и сдвигает курсор вперёд. |
| `GetPreviousString()` | — | String или Empty | Возвращает предыдущую строку истории трассировки в EME-L относительно текущей позиции курсора и сдвигает курсор назад. |
| `IsStringFound()` | — | Boolean | Проверяет в EME-L, была ли найдена строка при предыдущем вызове метода получения строки. |

В системе EME.WMS метод `IsStringFound()` возвращает `TRUE` после успешного вызова `GetFirstString`, `GetLastString`, `GetNextString` или `GetPreviousString`, если он вернул строку из буфера. Если буфер пуст или достигнута граница, возвращается `FALSE`.

## Примеры использования класса Trace в EME-L

Пример использования трассировщика при поиске ячейки в EME-L — реальный паттерн из классов `ARXIncoming`, `Defect`, `Incoming` и других:

```EME-L
'Создать трассировщик с именем дампа и очистить старые записи'
objTrace = Object("Trace", dbLines.GetDumpName());
objTrace.Empty();

'Включить трассировку поиска в контексте'
context.SetTracingFind(1);

'После поиска проверить, остались ли строки в трассировке'
objTrace.GetFirstString();
If (objTrace.IsStringFound())
    'Есть сообщения трассировки — можно показать диалог или отчёт'
End If
```

Пример ручного накопления и чтения трассировки в EME-L:

```EME-L
'Создать трассировщик для отладки метода'
objTrace = Object("Trace", "Debug");

'Добавить несколько диагностических строк'
objTrace.AddLine("Начало обработки");
objTrace.AddLine("Шаг 1 выполнен");
objTrace.AddLine("Обработка завершена");

'Получить всё содержимое одной строкой'
strFull = objTrace.GetAsSingleString();

'Перебрать строки от первой к последней'
For (str = objTrace.GetFirstString(); objTrace.IsStringFound(); str = objTrace.GetNextString())
    'Обработать очередную строку трассировки'
End For
```

Пример открытия трассировки в браузере в EME-L:

```EME-L
'Добавить диагностику и показать её в окне браузера'
objTrace = Object("Trace", "MyTrace");
objTrace.AddLine("Проверка параметров");
objTrace.AddLine("Поиск завершён");
objTrace.OpenInBrowser();
```

### Класс DSigXML — цифровая подпись XML-документов в языке EME-L

Класс `DSigXML` в языке EME-L предназначен для создания цифровой подписи XML-документов с использованием внешнего исполняемого файла на C#. Класс применяется для формирования электронной подписи XML-запросов в системе ФГИС Зерно.

Для работы класса `DSigXML` в языке EME-L требуются: установленный КриптоПро CSP 5.0, .NET 6.0 и действующий сертификат с закрытым ключом в личном хранилище сертификатов Windows (хранилище «MY»). Внешний C#-процесс обменивается данными с EME-L через временные XML-файлы — он читает неподписанный XML и записывает подписанный результат или файл ошибки.

Класс зарегистрирован в `p_System.cpp` (`REGISTRY_EMEL_CLASS(DSigXML, ...)`) под меткой `#ZIA 11.07.2023`. C++-имя класса — `CIDSigXML`.

#### Создание объекта

В языке EME-L класс `DSigXML` имеет два overload конструктора. Объект создаётся через `Object()` с опциональным указанием серийного номера сертификата:

```EME-L
'Без аргументов — сертификат задаётся позже методом SetCertificate'
objDSig = Object("DSigXML");
'С серийным номером сертификата — поиск в хранилище «MY» выполняется при создании'
objDSig = Object("DSigXML", "123ABCDEF");
```

Overload 1 (`Object("DSigXML")`, argCount = 0): создаёт объект без установленного сертификата. Каталог для работы с внешним исполняемым файлом определяется из параметра модуля `Предпочтения.УТМ.СДИЗ.Путь к каталогу генератора подписи для ФГИС Зерно`; при пустом значении параметра используется каталог по умолчанию (`m_sDirName`).

Overload 2 (`Object("DSigXML", "123ABCDEF")`, argCount = 1): сразу выполняет поиск сертификата в системном хранилище Windows «MY» по серийному номеру без учёта регистра. При успешном поиске сертификат устанавливается для последующего подписания. Если сертификат не найден, объект создаётся, но `GetLastError` вернёт сообщение об ошибке («Сертификат не найден» или текст системной ошибки `GetLastError`).

Серийный номер сертификата сравнивается с возвратом `CertFindCertificateInStore` в верхнем регистре (приведение `MakeUpper()` для обеих сторон сравнения), поэтому регистр символов в аргументе не имеет значения.

#### Методы

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetCertificate(arg0)` | arg0: String (серийный номер сертификата) | Boolean | Выполняет поиск и установку сертификата в хранилище «MY» по серийному номеру без учёта регистра. TRUE — сертификат найден и установлен, FALSE — сертификат не найден или хранилище недоступно. |
| `Sign(arg0)` | arg0: String (XML-содержимое) | String | Формирует цифровую подпись XML через внешний C#-процесс. Строго 1 аргумент (`too_rare()`/`too_many()` при нарушении). Возвращает подписанный XML-документ. При ошибке — пустая строка. |
| `GetLastError()` | — | String | Текст последней ошибки, возникшей при работе с сертификатом или подписании XML. Пустая строка при отсутствии ошибок. |

#### SetCertificate — установка сертификата в системе EME.WMS

Метод `SetCertificate` выполняет поиск сертификата в системном хранилище Windows «MY» (имя хранилища захардкожено как `"MY"`) по серийному номеру без учёта регистра символов. При невозможности открыть хранилище метод возвращает FALSE и устанавливает сообщение `«Недоступно хранилище сертификатов»`. При возврате FALSE сообщение об ошибке доступно через `GetLastError`. Если параметр не указан или пустой, поиск в хранилище выполняется, но совпадения с пустой строкой не находится — метод возвращает FALSE. При отсутствии системной ошибки будет установлено сообщение `«Сертификат не найден»` (в исходном C++ написано с устаревшим твёрдым знаком «Сертификатъ не найден» — это опечатка).

Перед вызовом `Sign` сертификат должен быть установлен — либо через конструктор с аргументом, либо методом `SetCertificate`. Если подписать XML при отсутствии установленного сертификата, `Sign` возвращает пустую строку, а `GetLastError` возвращает `«Сертификат не был найден»`.

#### Sign — формирование цифровой подписи в языке EME-L

Метод `Sign` создаёт цифровую подпись XML-документа с использованием установленного сертификата. Обмен данными с внешним C#-процессом выполняется через временные XML-файлы: XML-содержимое сохраняется в файл `tobesignedxml.xml`, а результат читается из `signedxml.xml`. Если после выполнения процесса обнаружен файл-журнал ошибок (`m_sFileErrorLog`), его содержимое попадает в `GetLastError`, а подпись не возвращается. Если исполняемый файл C# (`m_sSignerExeName`) отсутствует в каталоге параметра модуля, `Sign` возвращает пустую строку, а `GetLastError` возвращает `«<m_sSignerExeName> не найден»`. Если после выполнения процесса не найден ни файл-журнал ошибки, ни `signedxml.xml`, устанавливается сообщение `«Не найден файл с подписью или файл для подписания некорректный»`.

#### Примеры

Установка сертификата и подписание XML. Реальный пример из производственного класса `SDIZ` (метод `SigningOfXML`):

```EME-L
'Создать объект DSigXML и установить сертификат по серийному номеру'
XMLSigner = Object("DSigXML");
If (~XMLSigner.SetCertificate(CERT_SN))
    ErrorArr.Add(XMLSigner.GetLastError());
    Return "";
End If
'Подписать XML-документ — возвращает подписанный XML или пустую строку'
ResultXML = XMLSigner.Sign(XMLText);
If (ResultXML == "")
    ErrorArr.Add(XMLSigner.GetLastError());
    Return "";
End If
Return ResultXML;
```

Создание объекта с сертификатом в конструкторе в системе EME.WMS:

```EME-L
objDSig = Object("DSigXML", "123ABCDEF");
LastError = objDSig.GetLastError();
If (LastError != "")
    ErrorArr.Add(LastError);
End If
```

#### Константы и файлы из реализации в языке EME-L

Следующие конкретные значения и имена файлов зафиксированы в теле методов C++ (`c_emel_class_method/i_DSigXML.cpp`) и определяют поведение класса `DSigXML` в системе EME.WMS:

- **Имя системного хранилища сертификатов**: `"MY"` — захардкожено в конструкторе с аргументом и в методе `SetCertificate`.
- **Имя параметра модуля** для каталога генератора подписи: `Предпочтения.УТМ.СДИЗ.Путь к каталогу генератора подписи для ФГИС Зерно`. При пустом значении параметра используется каталог по умолчанию (`m_sDirName`).
- **Входной XML-файл** для внешнего C#-процесса: `tobesignedxml.xml` (в каталоге параметра модуля).
- **Выходной подписанный XML-файл**: `signedxml.xml`.
- **Файл-журнал ошибок** внешнего процесса: `m_sFileErrorLog` — если этот файл существует после выполнения процесса, его содержимое попадает в `GetLastError`, а подпись не возвращается.
- **Исполняемый файл C#**: `m_sSignerExeName` — должен существовать в каталоге параметра модуля (проверка через `EMEFindFileCatalogue`).
- **Точные сообщения об ошибках**, устанавливаемые в `GetLastError` в языке EME-L:
  - `«Сертификат не был найден»` — при вызове `Sign` без установленного сертификата.
  - `«Сертификат не найден»` — после неудачного поиска сертификата в `SetCertificate` (если CertFindCertificateInStore ничего не нашёл; в C++ написано с устаревшим твёрдым знаком «Сертификатъ не найден» — это опечатка в исходнике).
  - `«Недоступно хранилище сертификатов»` — если `CertOpenSystemStore` вернул NULL.
  - `«<m_sSignerExeName> не найден»` — если исполняемый файл C# отсутствует в каталоге параметра модуля.
  - `«Не найден файл с подписью или файл для подписания некорректный»` — если после выполнения процесса не найден ни файл-журнал ошибки, ни `signedxml.xml`.
- **Сравнение серийного номера** в системе EME.WMS выполняется в верхнем регистре (`MakeUpper()` применяется к обоим операндам).
- **Возврат Boolean** из `SetCertificate` реализован как `COleVariant((long)Result, VT_BOOL)` — соответствует правилу EME-L `TRUE = -1`.

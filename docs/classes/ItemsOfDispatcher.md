### Класс ItemsOfDispatcher — класс-оболочка для построения дерева диспетчера терминалов в языке EME-L

Класс `ItemsOfDispatcher` (C++ `CIItemsOfDispatcher`) — это класс-оболочка для формирования иерархических наборов элементов диспетчера терминалов сбора данных в интеграторе DBIntegrator. Производный от базового класса `Items`, он предназначен для использования в EME-L-скриптах, управляющих диспетчером терминалов сбора данных в системе EME.WMS. В языке EME-L создаётся через `Object("ItemsOfDispatcher", …)` и подключается к интегратору DBIntegrator унаследованным методом `AttachToIntegrator`.

Класс предоставляет методы для создания элементов дерева, настройки их свойств (привязка к записям `CEMERec`, ссылки на терминалы, приоритеты очереди приказов, произвольные пользовательские поля), а также для управления фильтрацией и сортировкой элементов. Он поддерживает операции привязки, отвязки и удаления приказов из очереди терминалов, заполнения массивов ссылок на строки очереди приказов для отображения в браузерах. Используется для построения пользовательского интерфейса диспетчера терминалов, включая формирование древовидных структур заказов, документов, пачек и приказов на перемещение.

Важное правило: все методы настройки свойств группы `Set*` воздействуют на **последний добавленный** элемент дерева DBIntegrator (добавленный через `AddAndFillItem`), если в описании метода явно не указано иное. Каждый сеттер возвращает **предыдущее** значение свойства, что позволяет восстановить его при откате изменений.

### Создание объекта — конструкторы класса ItemsOfDispatcher в языке EME-L

Класс `ItemsOfDispatcher` имеет два конструктора — на одно и на два аргумента. Второй аргумент передаётся в функцию `OnInit()` EME-L-класса-обработчика при инициализации.

```EME-L
'Конструктор с указанием имени EME-L-класса-обработчика'
objRootItems = Object("ItemsOfDispatcher", "LeftRootDocsItems");
objRootItems.AttachToIntegrator(IntegratorSRC);
```

```EME-L
'Конструктор с произвольными данными, передаваемыми в OnInit()'
objRootItems = Object("ItemsOfDispatcher", "RightRootTerminalsItems", m_FilterOption);
objRootItems.AttachToIntegrator(IntegratorDST);
```

Второй аргумент конструктора (`данные`) может иметь любой тип — целое число, строка, объект. В реальных скриптах диспетчера терминалов EME.WMS через него обычно передают код режима фильтрации левой или правой панели.

### Добавление элементов — метод AddAndFillItem в языке EME-L

Метод `AddAndFillItem` добавляет новый элемент в дерево DBIntegrator и автоматически заполняет его свойства на основе контекста. Автоматика проставляет ссылку на терминал (свойство `__VTR__`), если переданный объект является валидной записью `DataPreparationTerminals` или если такая ссылка уже есть в родительском элементе дерева.

| Перегрузка | Аргументы | Возвращает | Описание |
|------------|-----------|------------|----------|
| `AddAndFillItem(arg0, arg1)` | arg0: String (текст), arg1: String (классЭлемента) | Empty | Добавляет элемент с отображаемым текстом и именем класса дочернего элемента для следующего уровня дерева. Ссылка на терминал наследуется от родителя. |
| `AddAndFillItem(arg0, arg1, arg2)` | + arg2: CEMERec (запись, по ссылке) | Empty | Дополнительно извлекает из записи имя класса CEMERec (`__REC__`) и номер строки (`__REF__`) для открытия связанных документов. |
| `AddAndFillItem(arg0, arg1, arg2, arg3)` | + arg3: Integer (строкаОчереди) | Empty | Дополнительно устанавливает прямую ссылку `__QOL__` на строку очереди приказов. Значение меньше 0 игнорируется (свойство не устанавливается). Если элемент конечный, интегратор выполнит пропуск браузера и автоматическую привязку/отвязку приказов. |

### Внутренние механизмы привязки, отвязки и удаления приказов в диспетчере терминалов EME.WMS

Класс `ItemsOfDispatcher` (C++ `CIItemsOfDispatcher`) реализует четыре операции, которые вызывает сам интегратор DBIntegrator при работе с деревом диспетчера терминалов, а не EME-L-скрипт напрямую: `AttachTerminal`, `DetachTerminal`, `DeleteFromRFDispatcher`, `IsExistQueueOrderWithTerminalForDelete`. Каждая операция сначала пытается делегировать выполнение EME-L-классу-обработчику через соответствующий callback (см. ниже «EME-L-обработчики (callbacks)»); если callback не задан или возвращает `-1`, выполняется стандартная реализация ядра EME.WMS. Эти методы не имеют записей `IMPLEMENT_METHOD` в источнике `i_ItemsOfDispatcher.cpp` и потому не вызываются напрямую из EME-L — для скрипта доступны только методы настройки свойств (см. выше) и `FillQueueLinesArray`.

### EME-L-обработчики (callbacks) — расширение поведения ItemsOfDispatcher в скриптах

При наступлении событий в дереве диспетчера терминалов EME.WMS ядро ищет в EME-L-классе-обработчике, указанном в конструкторе `ItemsOfDispatcher`, методы с фиксированными именами. Все callbacks необязательны: при их отсутствии ядро использует стандартную реализацию. Соглашение о возврате: строка, трактуемая как целое; значение `-1` означает «использовать стандартную реализацию ядра», любое другое значение заменяет результат.

| Callback | Аргументы | Когда вызывается |
|----------|-----------|------------------|
| `OnFillQOLinesArray(this, массивСсылок)` | `this` — объект `ItemsOfDispatcher`; `массивСсылок` — объект `Array` по ссылке | Из `FillQueueLinesArray`, когда свойство `__QOL__` не задано напрямую. Должен заполнить массив ссылками на строки очереди приказов, относящихся к элементу. |
| `OnAttachTerminal(this, терминал, предСтрока, ошибки)` (4 арг.) или `OnAttachTerminal(this, терминал, предСтрока)` (3 арг.) | `this` — объект; `терминал` — строка записи терминала (`Integer`); `предСтрока` — позиция вставки (`Integer`); `ошибки` — объект `Map` по ссылке для накопления ошибок по строкам | При привязке элемента к терминалу. Ядро сначала пробует 4-аргументную сигнатуру, при отсутствии — 3-аргументную. |
| `OnDetachTerminal(this)` | `this` — объект | При отвязке элемента от терминала. |
| `OnDeleteFromRFDispatcher(this, массив)` (2 арг.) или `OnDeleteFromRFDispatcher(this)` (1 арг.) | `this` — объект; `массив` — объект `Array` по ссылке | При удалении элемента из диспетчера. Ядро сначала пробует 2-аргументную сигнатуру, при отсутствии — 1-аргументную. |

Параметр `this` передаётся в callback как сам объект `ItemsOfDispatcher`, что позволяет в EME-L-обработчике читать его свойства (`GetQueueOrdersLine()`, `GetValidTerminalRef()`, `GetImplRecordName()` и т. п.) и принимать решение о составе возвращаемых ссылок.

### Категории свойств элемента дерева DBIntegrator и служебная категория __NGI__

Методы настройки записывают значения во внутреннюю строку категорий элемента в формате `__KEY__=value`, разделённых `\r\n`. Помимо категорий, управляемых явными методами (`__REC__`, `__REF__`, `__QOL__`, `__VTR__`, `__FQP__`, `__DOT__`, `__FLD__`, `__FFS__`, `__MOD__`), метод `AddAndFillItem` автоматически проставляет служебную категорию `__NGI__=1` (NoGroupItem — «схлопнуть группировочный элемент»), если унаследована от родителя или если переданный объект является записью `DataPreparationTerminals` и включена настройка `Предпочтения.Диспетчер терминалов.Схлопывать группировочные элементы в правой части`. Эта категория считывается интегратором DBIntegrator при отрисовке дерева и напрямую из EME-L не переопределяется.

### Заполнение браузеров диспетчера — метод FillQueueLinesArray в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FillQueueLinesArray(arg0)` | arg0: Array (по ссылке) | Empty или String | Заполняет массив ссылками на строки очереди приказов текущего элемента для отображения в браузерах диспетчера терминалов. Если напрямую задано свойство `__QOL__` и терминал совпадает, добавляет только эту ссылку; иначе вызывает EME-L-метод `OnFillQOLinesArray` класса-обработчика. Возвращает `Empty` при успехе или строку с ошибкой: `"Не задан метод EME-L"`, `"Не задан метод EME-L: OnFillQOLinesArray"`. |

### Константы типа записи — возвращаемые значения GetCurrentRecord в языке EME-L

Метод `GetCurrentRecord()` возвращает целочисленную константу C++ в зависимости от значения свойства `__REC__`. Эти константы соответствуют внутренним идентификаторам записей WMS; в EME-L они используются как целые значения для сравнения.

| Значение свойства `__REC__` | Константа C++ | Назначение |
|------------------------------|----------------|------------|
| `"Orders"` | `ORDER_HEADER_RECORD` | Заказ |
| `"OrdersBatch"` | `ORDERS_BATCH_RECORD` | Пачка заказов |
| `"Document"` | `DOCUMENTS_RECORD` | Документ |
| `"DataPreparationTerminals"` | `DATA_PREPARATION_TERMINALS_RECORD` | Терминал сбора данных |
| `"QueueOrders"` | `QUEUE_ORDERS_RECORD` | Строка очереди приказов |
| `"GoodsItem"` | `GOODS_ITEM_RECORD` | Товарная позиция |
| не задано / пусто | `NULL_RECORD` | Привязка отсутствует |

Сеттеры целочисленных свойств (`SetValidTerminalRef`, `SetQueueOrdersLine`, `SetDispOrderType`, `SetFilterMode`) возвращают константу `NULL_REF`, когда свойство не было задано ранее. `NULL_REF` в языке EME-L представляется целочисленным значением, используемым как «нет ссылки» (внутреннее значение определено в ядре EME.WMS).

### Примеры использования ItemsOfDispatcher в скриптах EME-L

Создание корневого набора элементов и подключение к интегратору диспетчера терминалов:

```EME-L
'objRootItems — корневой набор для левой панели диспетчера'
objRootItems = Object("ItemsOfDispatcher", "LeftRootDocsItems", nFilterOption);
objRootItems.AttachToIntegrator(IntegratorSRC);
```

Добавление дочернего элемента с привязкой к записи `OrdersBatch` и приоритетом очереди:

```EME-L
'Заполнить элемент пачки с автоматической привязкой к терминалу родителя'
objItems.AddAndFillItem(GetOrdersBatchItemText(r_OrdersBatch, -1, Count), "OrdersBatchItemsCons", r_OrdersBatch);
objItems.SetFirstQueuePriority(r_OrdersBatch.GetMoveOrder().GetQueueOrders().GetPriority());
```

Привязка конечного элемента к строке очереди приказов через 4-аргументный `AddAndFillItem`:

```EME-L
'arg3 — номер строки очереди приказов; при значении < 0 свойство __QOL__ не устанавливается'
objItems.AddAndFillItem(r_QueueOrders.GetItemText(), "", r_QueueOrders, r_QueueOrders.GetLine());
objItems.SetFirstQueuePriority(dPriority);
```

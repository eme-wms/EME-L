# Класс SmartPointer

> Синонимы для поиска: CISmartPointer SmartPointer класс EME-L, умный указатель EME-L, shared_ptr EME-L, умный указатель на объект EME-L, владение объектом EME-L, разделяемое владение объектом EME-L, shared pointer EME-L, перенаправление вызовов EME-L, обёртка над объектом EME-L, SmartPointer className конструктор, SmartPointer(sourceSmartPointer), передача объекта между методами EME-L, CISmartPointer методы, автоматическое удаление объекта EME-L, время жизни объекта EME-L

Класс `SmartPointer` в языке EME-L — умный указатель на объект, обеспечивающий совместное владение объектом EME-L через механизм `shared_ptr`. Класс SmartPointer в системе EME.WMS позволяет создавать экземпляры других классов EME-L по имени класса и передавать их между переменными и объектами, не заботясь об освобождении памяти: объект автоматически удаляется, когда уничтожается последний `SmartPointer`, который на него ссылается.

Класс `SmartPointer` в языке EME-L поддерживает два способа создания: создание нового объекта заданного класса с передачей параметров его конструктору либо разделение владения с уже существующим объектом `SmartPointer`. Все вызовы методов, выполненные через SmartPointer в языке EME-L, перенаправляются целевому объекту — сам класс SmartPointer не объявляет собственных методов. Это делает SmartPointer в системе EME.WMS прозрачной обёрткой: вызывающий код работает с интерфейсом целевого объекта, а временем жизни управляет умный указатель.

Используется в прикладных скриптах EME-L для передачи сложных объектов между функциями и методами, а также для организации коллекций объектов с автоматическим управлением временем жизни. Элементы [класса ArrayOfObjects](./ArrayOfObjects.md) в языке EME-L хранятся именно как SmartPointer, что обеспечивает освобождение объектов при удалении из массива или уничтожении самого массива.

Объект класса `SmartPointer` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс-обёртка над произвольным объектом EME-L.

## Создание объекта класса SmartPointer

Класс `SmartPointer` в языке EME-L имеет два конструктора — по числу перегрузок `IMPLEMENT_CTOR`.

### Конструктор 1 — создание нового объекта

`SmartPointer(className[, arg1[, arg2[, ...]]])` — создаёт объект указанного строкой `className` класса и сохраняет на него умный указатель (`shared_ptr`). Созданный объект принадлежит новому экземпляру SmartPointer; при уничтожении последнего владельца объект автоматически удаляется в языке EME-L.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| className | String | Да | Имя класса создаваемого объекта. Должно соответствовать зарегистрированному в проекте имени класса EME-L. |
| arg1, arg2, ... | произвольный | Нет | Параметры, передаваемые конструктору создаваемого объекта. Тип и количество параметров определяются целевым классом. |

**Возвращает**: новый объект SmartPointer, владеющий созданным объектом (ссылка).

**Вызов без аргументов недопустим** — при отсутствии аргументов возникает ошибка выполнения.

```EME-L
'Создаёт объект указанного класса и сохраняет на него умный указатель'
sp = Object("SmartPointer", "MD5");
```

```EME-L
'Та же перегрузка с параметрами конструктора целевого класса'
sp = Object("SmartPointer", "AnvizFace", "192.168.1.50", 5010, 1);
```

### Конструктор 2 — разделение владения

`SmartPointer(sourceSmartPointer)` — создаёт новый объект SmartPointer, разделяющий владение объектом с существующим указателем SmartPointer. Оба указателя ссылаются на один и тот же целевой объект; при уничтожении последнего владельца объект удаляется.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| sourceSmartPointer | SmartPointer (ссылка) | Да | Существующий объект SmartPointer, владение которым будет разделено. Если передан не SmartPointer, строковое значение используется как имя класса для создания нового объекта без дополнительных параметров. |

**Возвращает**: новый объект SmartPointer, ссылающийся на тот же целевой объект (ссылка).

**Вызов без аргументов недопустим** — конструктор требует ровно один аргумент.

```EME-L
'Создаёт второй SmartPointer, ссылающийся на тот же целевой объект'
sp2 = Object("SmartPointer", sp);
```

## Методы класса SmartPointer

Класс `SmartPointer` в языке EME-L не объявляет собственных методов: все вызовы, адресованные объекту SmartPointer из EME-L, перенаправляются целевому объекту. Метод, фактически вызываемый при обращении `sp.Method(...)`, определяется интерфейсом объекта, на который ссылается умный указатель.

Например, если `sp` владеет объектом `MD5`, допустимо `sp.Calculate(str)` — вызов будет перенаправлен методу `MD5.Calculate` в языке EME-L. Аналогично, если SmartPointer ссылается на объект `File`, доступны методы `GetFilePath`, `GetData`, `PutData` и остальные методы класса File в системе EME.WMS.

## Примеры SmartPointer в языке EME-L

Создание объекта и разделение владения:

```EME-L
'Первый SmartPointer создаёт объект MD5'
sp = Object("SmartPointer", "MD5");
Hash = sp.Calculate("password");
'sp перенаправляет вызов методу MD5.Calculate'

'Второй SmartPointer разделяет владение тем же объектом'
sp2 = Object("SmartPointer", sp);
Hash2 = sp2.Calculate("password");
'sp и sp2 указывают на один и тот же объект MD5'
```

Передача параметров конструктору целевого класса:

```EME-L
'Создаёт объект AnvizFace и сразу передаёт ему IP, порт, deviceId'
sp = Object("SmartPointer", "AnvizFace", "192.168.1.50", 5010, 1);
```

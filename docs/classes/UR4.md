### UR4 — класс работы с RFID-сканером UR4 в языке EME-L

Класс `UR4` (C++ `CIUR4`) в языке EME-L предназначен для работы с RFID-сканером UR4 через TCP-соединение. В системе EME.WMS класс `UR4` предоставляет механизм подключения к устройству по IP-адресу и порту, выполнения инвентаризации RFID-меток и получения данных о найденных метках. Для работы требуется наличие библиотеки `UHFAPI.dll`.

В языке EME-L объект класса `UR4` создаётся через `Object()` с указанием IP-адреса и порта устройства, опционально с таймаутом ожидания ответа, пороговым значением уровня сигнала и списком антенн. Класс `UR4` поддерживает фильтрацию меток по уровню сигнала и по заданному набору антенн. Результаты инвентаризации записываются в объект `Query`, поля которого создаются автоматически.

### Создание объекта UR4 в EME-L

Класс `UR4` в языке EME-L имеет четыре конструкторные перегрузки. Реестр `IMPLEMENT_CTOR` в `i_UR4.cpp` описывает формы с 2, 3 и 4 аргументами; тело конструктора фактически принимает и пятый аргумент — список антенн через запятую.

```EME-L
'Подключение к сканеру по IP-адресу и порту в EME-L'
objUR4 = Object("UR4", "192.168.1.100", 9090);

'Та же конфигурация с явным таймаутом 5000 мс'
objUR4 = Object("UR4", "192.168.1.100", 9090, 5000);

'С таймаутом и порогом сигнала'
objUR4 = Object("UR4", "192.168.1.100", 9090, 5000, -75.0);

'С ограничением сканирования антеннами 1 и 2 (hex-список через запятую)'
objUR4 = Object("UR4", "192.168.1.100", 9090, 5000, -75.0, "1,2");
```

Если IP-адрес задан пустой строкой, в EME-L используется адрес по умолчанию. Если порт не передан, используется порт по умолчанию. Таймаут и порог сигнала при отсутствии также заменяются системными значениями по умолчанию. В C++ исходнике эти значения именуются `DEFAULT_IP_ADRESS`, `DEFAULT_PORT`, `DEFAULT_TIMEOUT` и `DEFAULT_SIGNAL_THRESHOLD`; их точные числовые значения не удалось верифицировать из доступных исходников. Список антенн — строка с номерами антенн в шестнадцатеричном виде, разделёнными запятой; при его задании в `GetData` записываются только метки от указанных антенн.

### Методы подключения и отключения UR4 в EME-L

В языке EME-L класс `UR4` предоставляет методы для управления TCP-соединением со сканером.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Connect()` | — | Boolean | Устанавливает TCP-подключение к сканеру по заданным IP-адресу и порту. Если соединение уже установлено, возвращает TRUE без повторного подключения. TRUE — подключение установлено или уже существует; FALSE — не удалось установить подключение. |
| `Disconnect()` | — | Boolean | Выполняет отключение от сканера. TRUE — отключение выполнено; FALSE — в библиотеке отсутствует функция отключения. |

### Методы инвентаризации и отладки UR4 в EME-L

В системе EME.WMS класс `UR4` используется для сбора RFID-меток и диагностики работы сканера.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetData(query)` | query: Query (по ссылке) | Boolean | Выполняет инвентаризацию RFID-меток и заполняет переданный объект `Query` данными о найденных метках. Перед записью очищает существующие данные в запросе, автоматически подключается к сканеру, выполняет инвентаризацию и отключается. Поля запроса создаются автоматически: `Id` (String), `Signal` (String), `Antenna` (String), `User` (String), `TID` (String). TRUE — операция завершена (инвентаризация выполнена или не запущена, запрос может быть пустым); FALSE — не удалось подключиться к сканеру или передан недействительный объект Query. |
| `SetDebug(enable)` | enable: Boolean | Empty | Включает или отключает режим отладки объекта UR4. При включенном режиме в лог записывается дополнительная информация о работе сканера: данные антенн, параметры найденных меток, причины фильтрации. |

### Примеры использования класса UR4 в EME-L

Пример сканирования меток и обработки результата в языке EME-L:

```EME-L
'Создать объект UR4 в EME-L с явным адресом, портом и таймаутом'
objUR4 = Object("UR4", "192.168.1.100", 9090, 5000);

'Подключиться к сканеру UR4'
If (objUR4.Connect())
    objQuery = Object("Query");
    'Выполнить инвентаризацию RFID-меток и заполнить запрос'
    If (objUR4.GetData(objQuery))
        'Обработать найденные метки в EME-L'
        Loop (objQuery)
            strId = objQuery.GetData("Id");
            strSignal = objQuery.GetData("Signal");
            strAntenna = objQuery.GetData("Antenna");
        End Loop
    End If
    objUR4.Disconnect();
End If
```

Пример сканирования с ограничением антенн и порогом сигнала:

```EME-L
'Сканер UR4 с таймаутом 5000 мс, порогом -75.0 и антеннами 1,2'
objUR4 = Object("UR4", "192.168.1.100", 9090, 5000, -75.0, "1,2");
objUR4.SetDebug(TRUE);

objQuery = Object("Query");
If (objUR4.GetData(objQuery))
    nTags = objQuery.GetNoOfLines();
End If
```

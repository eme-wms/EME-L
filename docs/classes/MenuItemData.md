# Класс MenuItemData

> Синонимы для поиска: CIMenuItemData MenuItemData класс EME-L, описатель пункта меню EME-L, GetName GetDBLine GetDialogName GetDLLName GetOvlCode GetUserFlags IsModal IsMDI, пункт меню пользователя Modules, модуль диалог DLL OVL-код флаги доступа

Класс `MenuItemData` в языке EME-L — класс описателя пункта меню пользователя. В языке EME-L класс MenuItemData предоставляет доступ к свойствам и параметрам пункта меню, определенного в конфигурации системы (запись Modules).

В языке EME-L класс MenuItemData позволяет получить информацию о привязанном модуле: имя модуля, имя диалога, имя DLL-файла, код запуска (OVL-код), номер строки в записи конфигурации системы, флаги доступа оператора, а также режим открытия диалога (модальный / MDI). Ядро системы связывает объект с данными через служебный метод AttachData, который не экспонирован в EME-L (вызывается из C++ кода ядра). В прикладных скриптах EME-L прямое создание и использование Object("MenuItemData") не встречается — объект подготавливает ядро.

Объект создаётся без параметров и должен быть привязан к данным пункта меню, предоставленным ядром системы. Все методы требуют валидной привязки к данным; при отсутствии привязки генерируется внутренняя ошибка.

## Создание объекта класса MenuItemData

Класс `MenuItemData` в языке EME-L имеет один конструктор — без аргументов. Внутренний указатель на данные пункта меню инициализируется значением NULL. Для использования методов доступа к свойствам объекта должен быть привязан к данным, предоставленным ядром системы.

```EME-L
'Создание непривязанного описателя пункта меню в языке EME-L'
objMenuItem = Object("MenuItemData");
```

## Методы класса MenuItemData

Методы класса `MenuItemData` в языке EME-L возвращают параметры пункта меню, к которому привязан объект. Все методы не принимают аргументов. При отсутствии привязки к данным каждый метод генерирует внутреннюю ошибку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetName()` | — | String | Название (имя) модуля, привязанного к пункту меню. |
| `GetDBLine()` | — | Integer | Номер строки в записи конфигурации системы (Modules), соответствующей данному пункту меню. |
| `GetDialogName()` | — | String | Название диалога, привязанного к модулю пункта меню. |
| `GetDLLName()` | — | String | Имя DLL-файла (динамической библиотеки) модуля, привязанного к пункту меню. |
| `GetOvlCode()` | — | Integer | Код запуска модуля (OVL-код), определяющий режим инициализации и запуска модуля. |
| `GetUserFlags()` | — | Integer | Флаги доступа оператора (пользователя) к модулю пункта меню. |
| `IsModal()` | — | Boolean | TRUE — диалог запускается в модальном режиме (блокирует работу с другими окнами); FALSE — немодальный режим. |
| `IsMDI()` | — | Boolean | TRUE — диалог запускается в режиме MDI (Multiple Document Interface). В текущей реализации метод всегда возвращает TRUE. |

## Примеры

Получение параметров пункта меню после привязки объекта к данным, предоставленным ядром системы:

```EME-L
objMenuItem = Object("MenuItemData");
'Привязка к данным пункта меню выполняется ядром системы в языке EME-L'

ModuleName = objMenuItem.GetName();
DialogName = objMenuItem.GetDialogName();
DLLName = objMenuItem.GetDLLName();
OvlCode = objMenuItem.GetOvlCode();
DBLine = objMenuItem.GetDBLine();
UserFlags = objMenuItem.GetUserFlags();
IsModalMode = objMenuItem.IsModal();
IsMDIMode = objMenuItem.IsMDI();
```

Проверка режима открытия диалога модуля в языке EME-L:

```EME-L
objMenuItem = Object("MenuItemData");
'Привязка к данным пункта меню выполняется ядром системы в языке EME-L'

If (objMenuItem.IsModal())
    OpenMode = tr("Модальный");
Else
    OpenMode = tr("Немодальный");
End If
```

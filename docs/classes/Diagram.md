## Класс Diagram — программное построение графиков в EME-L

Класс `Diagram` в языке EME-L предназначен для программного построения и настройки графиков и диаграмм в ячейках отчёта. В системе EME.WMS класс Diagram предоставляет инструментарий для создания линейных графиков, столбчатых диаграмм, сплайнов и сплайнов Безье, а также детальной настройки осей координат, легенды, цветов, толщины линий, подписей пиков и внешнего вида диаграммы.

Объект класса `Diagram` в языке EME-L создаётся без параметров в контексте выполнения отчёта и привязывается к графической ячейке. Класс Diagram используется в прикладных скриптах EME-L для визуализации данных в виде двумерных графиков непосредственно в отчётных формах. Класс зарегистрирован под именем `Diagram` (C++ класс `CIDiagram`); регистрация `REGISTRY_EMEL_CLASS(Diagram, …)` находится в `p_System.cpp`.

### Создание объекта Diagram в EME-L

```EME-L
'Создать инструментарий Diagram для текущей ячейки отчёта'
Diagram = Object("Diagram");
```

Конструктор класса `Diagram` в языке EME-L аргументов не принимает. При попытке передать аргументы генерируется ошибка «Слишком много аргументов». Объект Diagram привязывается к текущему контексту выполнения отчёта — если контекст не задан, выводится ошибка «Не задан контекст». Конструктор не возвращает самостоятельный файловый или диалоговый объект; он создаёт инструментарий для работы с графиком в текущей ячейке отчёта.

### Добавление графиков в класс Diagram

Методы добавляют новый график в ячейку диаграммы в языке EME-L и возвращают его порядковый номер (индекс). Все три перегрузки класса Diagram принимают одинаковый набор параметров; различие — в поведении при совпадении имени.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddDiagram([имя[, цвет[, толщина[, вид[, флаги]]]]])` | имя: String; цвет: Integer (COLORREF, по умолч. 0); толщина: Integer (по умолч. 1); вид: String (по умолч. `"SIMPLE_LINE"`); флаги: String | Integer | Добавляет график всегда. Индекс добавленного графика или `-1`, если не добавлен. |
| `AddDiagramIfNotExists([имя[, цвет[, толщина[, вид[, флаги]]]]])` | те же | Integer | Добавляет график только при отсутствии графика с таким именем. Индекс или `-1` (график с именем уже существует или не добавлен). |
| `AddDiagramAndOverwrite([имя[, цвет[, толщина[, вид[, флаги]]]]])` | те же | Integer | Добавляет график; при совпадении имени перезаписывает его. Индекс добавленного или перезаписанного графика, либо `-1`. |

**`вид`** — тип визуализации графика в языке EME-L:

| Значение | Тип графика |
|----------|-------------|
| `"SIMPLE_LINE"` | Линейный график (по умолчанию) |
| `"BAR_DIAGRAM"` | Столбчатая диаграмма |
| `"SIMPLE_SPLINE"` | Сплайн |
| `"SIMPLE_SPLINE_BEZIER"` | Сплайн Безье |

**`флаги`** — дополнительные параметры отображения класса Diagram, перечисленные через `;`:

| Значение | Действие |
|----------|----------|
| `"SUPERSCRIBE"` | Подписи в пиках (максимальных значениях) |
| `"INSCRIBE"` | Подписи во всех точках |
| `"MONOCHROME"` | Подписи всегда чёрные |
| `"TRANSPARENT"` | Прозрачные столбики столбчатых диаграмм |

При пустом или неуказанном имени график добавляется без имени. В языке EME-L несколько флагов `флаги` объединяются в одну строку через точку с запятой, например `"SUPERSCRIBE;MONOCHROME"`.

### Точки графика класса Diagram

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `PutPoint(номер_графика, X, Y)` | номер_графика: Integer; X: Real или DateTime; Y: Real или DateTime | Empty | Отмечает точку на графике. Последовательные вызовы формируют линию. DateTime автоматически приводится к числу дней от Рождества Христова. |
| `SetBarBaseAndWidth(начало, ширина)` | начало: Real; ширина: Real | Empty | Начальное смещение и ширина столбика столбчатой диаграммы в значениях оси X. |

Метод `PutPoint` класса Diagram в языке EME-L — основной способ наполнить график данными. Значение типа DateTime по любой координате автоматически преобразуется во внутреннее числовое представление (количество дней от Рождества Христова), поэтому ось можно настраивать как временную без ручной конвертации.

### Настройка осей координат класса Diagram

Во всех методах осей класса Diagram в языке EME-L `номер_оси` равный `0` соответствует оси OX (горизонтальной).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetAxisDataType(номер_оси, тип[, формат_даты])` | номер_оси: Integer; тип: String (`"DT_DATE"`, `"DT_TIME"` или др. — float); формат_даты: String | Empty | Тип данных оси. `формат_даты` используется при `тип = "DT_DATE"`. |
| `SetAxisStartValue(номер_оси, значение[, тип])` | номер_оси: Integer; значение: Real или DateTime; тип: String (`"DT_DATE"` — дата, иначе Real) | Empty | Начальное значение (минимум) оси. |
| `SetAxisEndValue(номер_оси, значение[, тип])` | номер_оси: Integer; значение: Real или DateTime; тип: String | Empty | Конечное значение (максимум) оси. |
| `SetAxisRange(номер_оси, начало, конец[, тип])` | номер_оси: Integer; начало: Real/DateTime; конец: Real/DateTime; тип: String | Empty | Одновременно задаёт начало и конец оси. |
| `SetAxisOrigin(номер_оси, значение[, тип])` | номер_оси: Integer; значение: Real/DateTime; тип: String | Empty | Точка пересечения с другими осями. Если точка не лежит строго между началом и концом оси, пересекающие оси не рисуются. |
| `SetAxisStep(номер_оси, шаг)` | номер_оси: Integer; шаг: Real | Empty | Шаг (расстояние между делениями) оси. |
| `GetAxisStep(номер_оси)` | номер_оси: Integer | Real | Текущий шаг оси. |
| `SetAxisAutoScaleFlag(номер_оси, флаг)` | номер_оси: Integer; флаг: Integer (`>0` — включить, `0` — отключить) | Empty | Автомасштабирование оси. |
| `SetAxisGridFlag(номер_оси, флаг)` | номер_оси: Integer; флаг: Integer (`>0` — показать, `0` — скрыть) | Empty | Отображение сетки по оси. |
| `SetAxisTimeStyle(номер_оси[, флаг1[, флаг2[, флаг3]]])` | номер_оси: Integer; флагN: String | Empty | Формат вывода времени по оси. Флаги можно комбинировать в разных аргументах. Вызов с одним аргументом сбрасывает все флаги. |
| `ForceAxisTextPlacement(номер_оси, положение)` | номер_оси: Integer; положение: String (`"DOWN"` — снизу) | Empty | Положение подписи оси. Непризнанные значения — положение по умолчанию. |
| `SetAxisText(номер_оси, текст)` | номер_оси: Integer; текст: String | Empty | Текстовая подпись оси. |
| `SetAxisRangeColor(номер_оси, начало, конец, цвет)` | номер_оси: Integer; начало: Real; конец: Real; цвет: Integer (COLORREF) | Empty | Цвет фона диапазона значений. Для цвета можно использовать `is_RGB(r, g, b)`. |
| `PutScaleText(номер_оси, значение[, подпись[, текст]])` | номер_оси: Integer (0 — OX, 1 — OY); значение: Real; подпись: String; текст: String | Empty | Пользовательский текст на шкале в заданной позиции. При пустых `подпись`/`текст` — значения по умолчанию. |

Параметр `тип` со значением `"DT_DATE"` в методах осей класса Diagram в языке EME-L указывает, что передаваемое значение является датой (и должно интерпретироваться через DATE_FORMAT); любое другое значение или пустая строка — вещественное число. В системе EME.WMS оси графиков чаще всего настраивают как временные (`"DT_DATE"` с форматом вроде `"%d.%m"`) для отображения динамики показателей во времени.

**Флаги `SetAxisTimeStyle` класса Diagram в языке EME-L:**

| Значение | Действие |
|----------|----------|
| `"HIDESECONDS"` | Скрыть секунды |
| `"HIDEMINUTES"` | Скрыть минуты |
| `"AUTO_HIDE_MS"` | Автоматически скрывать миллисекунды |
| `"FLEXIBLE"` | Гибкий формат |
| `"SEPARATOR_HMS"` | Разделитель «ч:м:с» |
| `"SEPARATOR_HMSPACE"` | Разделитель с пробелом |
| `"SEPARATOR_NONE"` | Без разделителя |
| `"FORCE_2_DIGITS_PER_HOUR"` | Две цифры для часа |

### Примеры использования класса Diagram в EME-L

Минимальный график с одной кривой в языке EME-L:

```EME-L
'Создать диаграмму в контексте текущей ячейки отчёта'
Diagram = Object("Diagram");

'Добавить линейный график зелёным цветом, толщина 2'
Curve = Diagram.AddDiagram("Сумма", is_RGB(0, 128, 0), 2, "SIMPLE_LINE", 0);

'Настроить ось OX как временную'
Diagram.SetAxisDataType(0, "DT_DATE", "%d.%m");
Diagram.SetAxisText(0, "Дата");
Diagram.SetAxisAutoScaleFlag(0, 1);

'Поставить точки'
Diagram.PutPoint(Curve, 45030.0, 100.5);
Diagram.PutPoint(Curve, 45031.0, 120.0);
Diagram.PutPoint(Curve, 45032.0, 90.25);
```

Несколько кривых из запроса в системе EME.WMS (на основе `StaffJobKpi.txt`, реальный пример из кодовой базы):

```EME-L
/* #DEV_INITIALS 25.06.2026 EME company
   Модуль: Отчёт KPI сотрудников
   Заполнить диаграмму данными запроса по оси дат */
FillDiagram(FieldName, AxisName)
{
    Diagram = Object("Diagram");
    SumQuery = Object("Query", "StaffJob.SumQuery");
    SumQuery.Execute();

    StartOX = GetMinValue(SumQuery, "Дата");
    FinalOX = GetMaxValue(SumQuery, "Дата");
    'Ось OX — дата, диапазон из запроса'
    Diagram.SetAxisStartValue(0, StartOX, "DT_DATE");
    Diagram.SetAxisEndValue(0, FinalOX, "DT_DATE");
    Diagram.SetAxisDataType(0, "DT_DATE", "%d.%m");

    'Ось OY — фиксированный минимум 0, максимум из запроса'
    Diagram.SetAxisRange(1, 0, GetMaxValue(SumQuery, FieldName), "");
    Diagram.SetAxisAutoScaleFlag(0, 1);
    Diagram.SetAxisAutoScaleFlag(1, 1);
    Diagram.SetAxisText(0, "Дата");
    Diagram.SetAxisText(1, AxisName);

    'Один график «Сотрудник» по сумме'
    Curve = Diagram.AddDiagram("Сотрудник", is_RGB(0, 255, 0), 5, "SIMPLE_LINE", 0);
    For (Line = 0; Line < SumQuery.GetNoOfLines(); Line = Line + 1)
        'PutPoint сам приводит дату к числу дней от Р.Х.'
        Diagram.PutPoint(Curve, SumQuery.GetData("Дата", Line), SumQuery.GetData(FieldName, Line));
    End For
}
```

Столбчатая диаграмма с подписями пиков в языке EME-L:

```EME-L
Diagram = Object("Diagram");
Bar = Diagram.AddDiagram("Объём", is_RGB(60, 120, 200), 1, "BAR_DIAGRAM", "SUPERSCRIBE");

'Формат подписи над столбцом — один знак после запятой'
Diagram.SetDiagramSuperscriptionFormat(Bar, "%.1f");

'Ширина столбца и отступ в значениях оси X'
Diagram.SetBarBaseAndWidth(0.2, 0.6);

Diagram.PutPoint(Bar, 1.0, 25.0);
Diagram.PutPoint(Bar, 2.0, 40.5);
```

### Примечания по классу Diagram

Метод `PutPoint` класса Diagram в языке EME-L принимает значение типа DateTime по любой координате — оно автоматически преобразуется во внутреннее числовое представление (количество дней от Рождества Христова), поэтому ось можно настраивать как временную без ручной конвертации. При совпадении имён графиков выбор метода добавления (`AddDiagram`, `AddDiagramIfNotExists`, `AddDiagramAndOverwrite`) определяет, будет ли создан дубликат, пропущен новый график или перезаписан существующий. Цвета везде задаются в формате COLORREF; для формирования значения удобно использовать функцию `is_RGB(r, g, b)`.

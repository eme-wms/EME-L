# Класс TimerParallel

> Синонимы для поиска: TimerParallel CITimerParallel Start Stop GetInterval SetUseLog параллельный таймер EME-L измерение времени perf timer log

Класс `TimerParallel` в языке EME-L — параллельный таймер для измерения временных интервалов выполнения действий в миллисекундах. В системе EME.WMS класс TimerParallel предназначен для профилирования производительности скриптов и операций базы данных. Поддерживает параллельный запуск нескольких таймеров с различными именами действий, что позволяет измерять время вложенных или одновременных операций.

Основные возможности в языке EME-L:

- Запуск и остановка таймеров с указанием имени действия.
- Поддержка стекового порядка остановки при наличии нескольких действий с одинаковым именем.
- Ведение журнала работы таймера в текстовый файл.
- Режим подробного логирования и режим точного измерения (без логирования).

Класс TimerParallel в EME-L доступен только в версиях ядра EME.WMS 45.1 и выше (`LAST_VERSION >= MAKE_VERSION(45, 1)`). Объект класса TimerParallel создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный вспомогательный класс для измерения производительности в языке EME-L.

## Создание объекта класса TimerParallel

Класс `TimerParallel` в языке EME-L имеет один конструктор с необязательным параметром `fileLog`.

```EME-L
'Таймер с журналом по умолчанию'
timer = Object("TimerParallel");
```

```EME-L
'Таймер с явным путём к файлу журнала'
timer = Object("TimerParallel", "D:/Logs/timer.log");
```

По умолчанию файл журнала в классе TimerParallel располагается в директории данных проекта (`GetProjectDataDir()`) и называется `timer.log`. Подробное логирование включено по умолчанию (внутренний конструктор `TimerParallel` в C++ задаёт `isUseLog = true`). Логирование снижает точность измерения примерно на 1% из-за времени записи в файл; для максимальной точности в языке EME-L вызовите `SetUseLog(false)` сразу после создания объекта.

## Методы класса TimerParallel в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Start()` | — | Empty | Запускает измерение для действия «БЕЗ НАЗВАНИЯ». |
| `Start(arg0)` | arg0: String | Empty | Запускает измерение для указанного действия. |
| `Stop()` | — | Integer | Останавливает последнее запущенное действие «БЕЗ НАЗВАНИЯ» и возвращает интервал в миллисекундах. |
| `Stop(arg0)` | arg0: String | Integer | Останавливает последнее запущенное действие с указанным именем. |
| `GetInterval()` | — | Integer | Возвращает `0` для действия «БЕЗ НАЗВАНИЯ». |
| `GetInterval(arg0)` | arg0: String | Integer | Возвращает `0` для указанного действия. |
| `SetUseLog(arg0)` | arg0: Boolean | Empty | Включает (`true`) или отключает (`false`) подробное логирование. |

`Start` и `Stop` в классе TimerParallel работают со стеком таймеров. Если несколько действий с одним именем запущены, `Stop` останавливает последнее начатое. Время фиксируется в конце вызова `Start` и перед началом записи результатов в `Stop`, чтобы подготовительные операции в системе EME.WMS не искажали измерение.

`Stop` возвращает неинициализированное значение, если действие с указанным именем не было запущено; в журнал при этом записывается ошибка. В языке EME-L это означает, что результат при отсутствии запущенного таймера не определён и не гарантируется равным `0`.

`GetInterval` всегда возвращает `0` в текущей реализации класса TimerParallel (заглушка).

## Примеры использования TimerParallel в языке EME-L

### Простое измерение

```EME-L
timer = Object("TimerParallel");
timer.Start();
'... выполняемая операция ...'
elapsed = timer.Stop();
```

### Измерение с именем действия и явным журналом

```EME-L
timer = Object("TimerParallel", "D:/Logs/perf.log");
timer.Start("импорт данных");
'... импорт ...'
importTime = timer.Stop("импорт данных");
```

### Режим точного измерения

```EME-L
timer = Object("TimerParallel");
'timer.SetUseLog(false);'
'timer.Start("быстрая операция");'
'... операция ...'
'elapsed = timer.Stop("быстрая операция");'
```

### Параллельные таймеры

```EME-L
timer = Object("TimerParallel");
timer.Start("внешняя операция");
timer.Start("внутренняя операция");
innerTime = timer.Stop("внутренняя операция");
outerTime = timer.Stop("внешняя операция");
```

## Примечания

В реестре C++ (`IMPLEMENT_CTOR`) конструктор декларирован с `argCount = 0` и подписью `CITimerParallel([fileLog])`, но тело конструктора и реальные скрипты в системе EME.WMS принимают 0 или 1 аргумент. Документация в языке EME-L отражает фактическое поведение — оба варианта вызова `Object("TimerParallel")` и `Object("TimerParallel", fileLog)` работают.

## Класс ExpImp64 — 64-разрядный интерфейс к файлам ядерного экспорта/импорта EME

Класс `ExpImp64` в языке EME-L — это 64-разрядный интерфейс для чтения файлов ядерного экспорта/импорта системы EME. Класс ExpImp64 в языке EME-L предоставляет методы для открытия экспортных файлов, последовательного извлечения записей с построчным чтением данных в объекты `Query`, а также получения метаданных файла: версия формата, филиал-источник, дата и время формирования, история предыдущих выгрузок.

В отличие от базового класса `ExpImp`, класс ExpImp64 в языке EME-L поддерживает **поэтапное чтение записи**: сначала извлечение структуры методом `ExtractNextRecord`, затем чтение порциями строк методом `ExtractNextLines`. Также предоставляет доступ к информации о количестве строк в записи (`GetNoOfLines`) и текущей позиции чтения (`GetLine`). Класс используется в скриптах EME-L для программного импорта данных из файлов экспорта, созданных ядром системы или другими филиалами, **с поддержкой 64-битных ссылочных полей** (ссылочные атрибуты приводятся к 64-битному целому ключу).

Класс ExpImp64 в системе EME.WMS используется при организации репликации и импорта данных между филиалами: ядро выгружает данные в бинарный формат `EXIM` (сигнатура заголовка файла), а прикладной скрипт через `ExpImp64` читает записи и заполняет объект `Query` для последующей записи в целевую базу. Каждая запись в файле начинается с заголовка с сигнатурой `HDR3`, а каждое поле записи — с заголовка с сигнатурой `HDR4`.

Объект класса `ExpImp64` в языке EME-L создаётся через `Object("ExpImp64")` без аргументов и не привязан к контексту диалога или браузера. Данные из файла читаются во внешний объект `Query`, который передаётся по ссылке в метод `ExtractNextLines`.

### Создание объекта класса ExpImp64

```EME-L
i_ExpImp = Object("ExpImp64");
```

Конструктор без аргументов класса ExpImp64 в языке EME-L создаёт объект с закрытым файлом. Перегрузок у конструктора нет. Открытие файла выполняется методом `LoadFile`, после чего доступны методы чтения записей и метаданных. Повторный вызов `LoadFile` автоматически закрывает предыдущий открытый файл.

### Открытие файла и управление чтением класса ExpImp64

Методы класса ExpImp64 в языке EME-L для открытия файла экспорта EME и последовательного извлечения записей и строк. Файл экспорта EME — бинарный файл с сигнатурой `EXIM` в заголовке, содержащий записи, каждая из которых состоит из полей и строк данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `LoadFile(filePath)` | filePath: String | String | Открывает файл экспорта EME и проверяет сигнатуру `EXIM`. При успехе возвращает пустую строку; при ошибке — текст сообщения (открытие файла, неверная сигнатура). Если файл уже был открыт, предыдущий закрывается автоматически. |
| `ExtractNextRecord()` | — | Boolean | Извлекает следующую запись из файла и подготавливает структуру полей. `TRUE` (в языке EME-L равно -1) — запись извлечена; `FALSE` (0) — записи закончились. При неверном формате записи (отсутствие сигнатур `HDR3` заголовка записи или `HDR4` заголовка поля) генерирует исключение. |
| `ExtractNextLines(query, lineCount)` | query: Query (по ссылке), lineCount: Integer | Empty | Извлекает `lineCount` строк текущей записи и заполняет переданный запрос данными. Перед вызовом необходимо извлечь запись методом `ExtractNextRecord`. Поддерживает специальную обработку полей «Ключи» (64-битное целое, `INT64_ATTRIBUTE`) и «Флаги» (32-битное целое, `INT_ATTRIBUTE`), а также коррекцию атрибутов для ссылочных полей; длины текстовых полей ограничиваются размером поля в базе данных. |
| `GetNoOfLines()` | — | Integer | Общее количество строк в текущей записи, извлечённой методом `ExtractNextRecord`. |
| `GetLine()` | — | Integer | Текущая позиция чтения в записи (0-based) после вызова `ExtractNextLines`. Нумерация начинается с 0. |

### Поведение метода ExtractNextLines по типам полей

Реализация метода `ExtractNextLines` класса ExpImp64 в языке EME-L (вспомогательные процедуры `PrepareQuery`, `ReadData`, `WriteData`) определяет следующие правила обработки полей в зависимости от их атрибута в файле экспорта:

- **Поля «Ключи»** всегда читаются как 64-битное целое (`INT64_ATTRIBUTE`) независимо от атрибута, записанного в заголовке поля файла.
- **Поля «Флаги»** всегда читаются как 32-битное целое (`INT_ATTRIBUTE`).
- **Ссылочные поля** (атрибут, для которого `this_is_ref_attr` возвращает true) приводятся к `INT64_ATTRIBUTE` — 64-битному ключу записи.
- **Атрибуты `FLAGS_ATTRIBUTE` и `ARRAY_ATTRIBUTE`** приравниваются к `INT_ATTRIBUTE` и читаются как целое число.
- **Массивы ссылок** (`ARRAY_OF_REFERENCE`) пропускаются при подготовке запроса — данные таких полей не импортируются.
- **Текстовые поля** (`TEXT_STRING_ATTRIBUTE`): длина строки ограничивается размером соответствующего поля в целевой базе данных (через `get_field_length` по имени записи и поля); если размер в базе неизвестен или равен нулю, используется значение по умолчанию **256** байт, к которому добавляется 1 байт на завершающий символ (итого 257).
- **Поля нераспознанных типов**: если атрибут не поддерживается ядром (`CEMEQuery::GetNameOfAttribute` возвращает то же, что и для `-1`), используется фиксированный размер **64** байта.
- **Поля переменной длины** (флаг `VARIABLE_LENGTH_FIELD_BIT` в заголовке поля) читаются порционно: для каждой строки из файла сначала считывается 4-байтная длина (`__int32`), затем сами данные указанного размера. То же правило применяется для полей с атрибутом `DATA_STORAGE_ATTRIBUTE`, если версия файла (возвращаемая `GetVersion`) больше либо равна константе `IMPORT_FILE_VERSION4`.

### Метаданные файла класса ExpImp64

Все методы метаданных класса ExpImp64 в языке EME-L требуют предварительно открытого файла методом `LoadFile`, иначе генерируют исключение «Файл не открыт» (проверка `CheckFileLoading`).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetVersion()` | — | Integer | Версия формата файла экспорта EME. |
| `GetExportType()` | — | String | Тип экспорта: два последних символа имени файла, если статус филиала-источника равен `SYS_EXPORT_STATUS` (системный экспортный файл — например, `"s1"`, `"s2"`). Пустая строка, если файл не является системным экспортным файлом. |
| `GetFilialNo()` | — | Integer | Номер филиала, сформировавшего файл экспорта. |
| `GetFilialStatus()` | — | Integer | Код статуса филиала-источника файла экспорта. |
| `GetFileName()` | — | String | Имя файла экспорта из заголовка открытого файла. |
| `GetDate()` | — | DateTime | Дата формирования файла экспорта. |
| `GetTime()` | — | DateTime | Время формирования файла экспорта. |
| `GetRecordName()` | — | String | Имя текущей записи, извлечённой методом `ExtractNextRecord`. Пустая строка, если запись ещё не была извлечена. |

### История предыдущего экспорта класса ExpImp64

Методы возвращают значения из заголовка текущего файла экспорта о предыдущей выгрузке по истории. В системе EME.WMS позволяют проверить целостность цепочки импорта — имя и время предыдущего файла должны совпадать с последним импортированным. Требуют предварительно открытого файла методом `LoadFile`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetPrevFileName()` | — | String | Имя предыдущего файла экспорта по истории. Пустая строка, если информация о предыдущем файле отсутствует. |
| `GetPrevDate()` | — | DateTime | Дата предыдущего файла экспорта по истории. |
| `GetPrevTime()` | — | DateTime | Время предыдущего файла экспорта по истории. |

### Порядок вызова методов класса ExpImp64

Типичный цикл импорта файла ядерного экспорта EME в языке EME-L:

1. `LoadFile(path)` — открыть файл, проверить сигнатуру `EXIM`.
2. В цикле: `ExtractNextRecord()` — извлечь запись; возврат `FALSE` означает конец файла.
3. `GetRecordName()` / `GetNoOfLines()` — опционально, для маршрутизации по типу записи.
4. `ExtractNextLines(query, n)` — заполнить порцию строк в объекте `Query`.
5. Повторять шаг 4, пока `GetLine()` не достигнет `GetNoOfLines()`, затем вернуться к шагу 2.

### Примеры использования класса ExpImp64 в языке EME-L

Открытие файла ядерного экспорта EME и проверка результата в языке EME-L:

```EME-L
i_ExpImp = Object("ExpImp64");
Result = i_ExpImp.LoadFile(FileName);
If (Result != "")
    is_message(tr("Импорт файла"), Result, "OK", "STOP");
    Return;
End If
```

Последовательное извлечение записей и строк в объект `Query` в языке EME-L:

```EME-L
i_ExpImp = Object("ExpImp64");
Result = i_ExpImp.LoadFile(FileName);
If (Result == "")
    Query = Object("Query");
    Query.Create();

    While (i_ExpImp.ExtractNextRecord())
        'ExtractNextRecord возвращает TRUE, пока есть записи'
        RecordName = i_ExpImp.GetRecordName();

        While (i_ExpImp.GetLine() < i_ExpImp.GetNoOfLines())
            'Заполнить пакет строк в Query'
            i_ExpImp.ExtractNextLines(Query, 100);
            'Query содержит строки — обработать и очистить'
            Query.Delete();
            Query.Create();
        End While
    End While
End If
```

Реальный пример из системы EME.WMS — свёрка имени предыдущего файла с последним импортированным. Используется при репликации для контроля непрерывности цепочки выгрузок:

```EME-L
i_ExpImp = Object("ExpImp64");
Result = i_ExpImp.LoadFile(Folder + FileName);
If (Result == "")
    PrevName = i_ExpImp.GetPrevFileName();
    'PrevName содержит имя предыдущего файла экспорта по истории'
End If
```

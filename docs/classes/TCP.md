# Класс TCP

Класс `TCP` в языке EME-L — класс для работы с TCP/IP-соединениями через сокеты. В системе EME.WMS класс TCP предоставляет возможность установления и разрыва TCP-соединений с удаленными хостами, отправки и получения текстовых данных, настройки таймаутов операций, а также асинхронной отправки сообщений в отдельном потоке. Поддерживает разрешение DNS-имен в IP-адреса.

В языке EME-L объект `TCP` создаётся с указанием IP-адреса и порта хоста или с параметрами по умолчанию. Соединение устанавливается явным вызовом метода `Connect` и разрывается методом `Disconnect`. При уничтожении объекта активное соединение автоматически закрывается.

## Создание объекта TCP в языке EME-L

Класс `TCP` в языке EME-L имеет два конструктора — один на каждое число аргументов.

```EME-L
'Конструктор без аргументов — используются константы по умолчанию'
objTcp = Object("TCP")
```

```EME-L
'Конструктор с явным IP-адресом (или DNS-именем) и портом'
objTcp = Object("TCP", "192.168.1.10", 8080)
```

Начальные значения в конструкторе без аргументов берутся из констант `TCP_DEFAULT_HOST_IP_ADRESS`, `TCP_DEFAULT_HOST_PORT`, `TCP_DEFAULT_CONNECTION_TIMEOUT`, `TCP_DEFAULT_DISCONNECTION_TIMEOUT`, `TCP_DEFAULT_SEND_TIMEOUT` и `TCP_DEFAULT_RECIEVE_TIMEOUT`. В доступных исходных файлах числовые значения этих констант не указаны.

## Методы класса TCP в языке EME-L

### Подключение и управление соединением

В языке EME-L методы класса TCP работают с текущими настройками хоста и порта.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Connect()` | — | Boolean | Устанавливает TCP-соединение с текущими `HostIp` и `HostPort`. Возвращает `TRUE` (-1) при успехе, `FALSE` (0) — иначе. |
| `Disconnect()` | — | Boolean | Разрывает установленное TCP-соединение. Возвращает `TRUE` при успехе или если соединение уже разорвано, `FALSE` — при неудаче. |
| `SetHostIpAndPort(HostIp, HostPort)` | HostIp: String, HostPort: Integer | Empty | Устанавливает IP-адрес или DNS-имя хоста и порт для последующих операций. |
| `AsyncConnectSendDisconnect(TextToSend)` | TextToSend: String | String | В отдельном потоке выполняет подключение, отправку текстового сообщения и закрытие соединения. Возвращает пустую строку сразу, не дожидаясь завершения. |

### Настройка таймаутов

Таймауты класса TCP в языке EME-L задаются в миллисекундах.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetTimeForConnection(Timeout)` | Timeout: Integer | Empty | Максимальное время ожидания установления TCP-соединения. |
| `SetTimeForDisconnection(Timeout)` | Timeout: Integer | Empty | Максимальное время ожидания завершения разрыва TCP-соединения. |
| `SetTimeForRead(Timeout)` | Timeout: Integer | Empty | Максимальное время ожидания получения данных от хоста. |
| `SetTimeForWrite(Timeout)` | Timeout: Integer | Empty | Максимальное время ожидания завершения отправки данных. |

### Обмен данными

В языке EME-L класс TCP отправляет текст в UTF-8 и получает ответ в виде строки.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Send(Text)` | Text: String | Boolean | Отправляет текстовое сообщение удаленному хосту; текст предварительно преобразуется в UTF-8. Возвращает `TRUE` (-1) при успехе, `FALSE` (0) — иначе. |
| `Recieve()` | — | String | Получает ответ от удаленного хоста. В случае неудачи или отсутствия данных возвращает пустую строку. |

### Служебные методы

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetHostIpFromName(HostName)` | HostName: String | String | Выполняет разрешение DNS-имени хоста в IP-адрес. При неудаче возвращает пустую строку. |

## Константы по умолчанию класса TCP

В системе EME.WMS класс TCP использует следующие символические константы:

- `TCP_DEFAULT_HOST_IP_ADRESS` — IP-адрес хоста по умолчанию (опечатка в исходнике: `ADRESS` вместо `ADDRESS`).
- `TCP_DEFAULT_HOST_PORT` — порт подключения по умолчанию.
- `TCP_DEFAULT_CONNECTION_TIMEOUT` — таймаут установления соединения.
- `TCP_DEFAULT_DISCONNECTION_TIMEOUT` — таймаут разрыва соединения.
- `TCP_DEFAULT_SEND_TIMEOUT` — таймаут отправки данных.
- `TCP_DEFAULT_RECIEVE_TIMEOUT` — таймаут получения данных (опечатка в исходнике: `RECIEVE` вместо `RECEIVE`).

Числовые значения этих констант не указаны в доступных исходных файлах.

## Примеры использования класса TCP в языке EME-L

### Разрешение DNS-имени текущего компьютера

```EME-L
'Реальный пример: получить IP-адрес текущей рабочей станции'
ip = Object("TCP").GetHostIpFromName(is_computer_name())
```

### Полный цикл TCP-запроса

```EME-L
'Создание объекта с явным хостом и портом'
tcp = Object("TCP", "example.com", 80)

'Подключение к удаленному хосту'
If (tcp.Connect())
    'Отправка текстового запроса (преобразуется в UTF-8)'
    tcp.Send("GET / HTTP/1.1\r\nHost: example.com\r\n\r\n")

    'Получение ответа'
    answer = tcp.Recieve()

    'Закрытие соединения'
    tcp.Disconnect()
End If
```

# Класс StringBuilder

> Синонимы для поиска: CIStringBuilder StringBuilder класс EME-L, построение строк EME-L, накопление строк EME-L, буфер строк EME-L, конкатенация строк EME-L, Append Add AppendLine AppendFormat AppendFormatLine ToString AsString GetLength GetSize IsEmpty Mid SetString SetLength GetGrowBy SetGrowBy Replace, строковый буфер EME-L, текстовый отчёт EME-L, printf EME-L

Класс `StringBuilder` в языке EME-L — класс для эффективного построения и накопления строковых данных. Предоставляет механизм последовательного добавления строк, чисел, дат и других типов данных в расширяемый буфер с минимизацией операций выделения памяти. Поддерживает форматированный вывод по шаблону `printf`, добавление с переводом строки, получение длины, извлечение подстрок и установку содержимого.

Объект создаётся с произвольным количеством начальных значений, которые автоматически добавляются в буфер при конструировании. В языке EME-L класс `StringBuilder` используется для эффективной конкатенации строк и построения текстовых отчётов, особенно при накоплении больших объёмов текста в циклах, где многократная конкатенация через `+` приводит к избыточным выделениям памяти.

Объект класса `StringBuilder` создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс без параметров контекста. Внутренний буфер инициализируется размером приращения 1024 байта (значение по умолчанию `SB_DEFAULT_GROWBY` в системе EME), которое можно изменить методом `SetGrowBy`.

## Создание объекта класса StringBuilder

Конструктор в языке EME-L принимает произвольное количество начальных значений — каждое добавляется в буфер в порядке передачи. Типы значений те же, что и у метода `Append`: строки, целые числа, вещественные числа, даты, время, ссылки.

```EME-L
'Конструктор без аргументов — пустой буфер'
sb = Object("StringBuilder");
```

```EME-L
'Конструктор с начальным значением — строка добавляется в буфер'
sb = Object("StringBuilder", "Заголовок отчёта");
```

```EME-L
'Конструктор с несколькими начальными значениями'
sb = Object("StringBuilder", "Строка 1", 42, 3.14);
'Буфер содержит конкатенацию всех переданных значений'
```

Конструктор без аргументов создаёт пустой буфер. При передаче одного или более значений каждое последовательно добавляется в буфер, как при вызове `Append` для каждого аргумента.

## Добавление данных

Методы для последовательного накопления данных в буфере. Метод `Append` и его синоним `Add` в языке EME-L принимают значения различных типов: строки, целые и вещественные числа, даты (`DATE_FORMAT`), время (`TIME_FORMAT`) и ссылки. При добавлении значения без явного формата используется формат по умолчанию для типа: `%d` для целых чисел (буфер 64 байта), `%f` для вещественных (буфер 64 байта), `ДД.ММ.ГГГГ` для дат, `ЧЧ:ММ:СС` для времени, `ДД.ММ.ГГГГ ЧЧ:ММ:СС` для комбинированного значения даты-времени.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Append(value)` | value: String / Integer / Real / DateTime / DATE_FORMAT / TIME_FORMAT / ссылка | Integer | Добавляет значение в конец буфера. Возвращает количество добавленных символов. |
| `Append(value, format)` | value: любой тип, format: String (printf-спецификатор) | Integer | То же с явной строкой формата. Если формат не задан, используется формат по умолчанию для типа. |
| `Add(value)` | value: любой тип | Integer | Синоним `Append`. |
| `Add(value, format)` | value: любой тип, format: String | Integer | Синоним `Append` с форматом. |
| `AppendLine(value)` | value: любой тип | Integer | Добавляет значение и символ перевода строки `\n`. Возвращает количество символов включая перевод строки. |
| `AppendLine(value, format)` | value: любой тип, format: String | Integer | То же с явным форматом. |
| `AppendFormat(format[, arg1[, arg2[, ...]]])` | format: String (printf-шаблон), arg1..argN: любые типы | Integer | Добавляет результат форматирования по шаблону `printf`. При одном аргументе эквивалентен `Append(format)`. Возвращает количество добавленных символов. |
| `AppendFormatLine(format[, arg1[, arg2[, ...]]])` | format: String, arg1..argN: любые типы | Integer | То же, что `AppendFormat`, с добавлением перевода строки. Возвращает количество символов включая `\n`. |

Поддерживаемые типы значений при добавлении в системе EME.WMS:

| Тип EME-L | Формат по умолчанию | Пример формата |
|-----------|---------------------|----------------|
| String | как есть | — |
| Integer / Long | `%d` | `"%05d"` — с ведущими нулями |
| Real (double) | `%f` | `"%.2f"` — два знака после запятой |
| DATE_FORMAT | `ДД.ММ.ГГГГ` (`%02d.%02d.%04d`) | произвольный `printf` |
| TIME_FORMAT | `ЧЧ:ММ:СС` (`%02d:%02d:%02d`) | произвольный `printf` |
| DateTime (COleDateTime) | авто: дата, время или дата+время | строка формата `COleDateTime::Format` |
| ссылка (void*) | `0x%08p` | — |

## Получение содержимого и длины

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ToString()` | — | String | Возвращает собранное содержимое буфера в виде строки. |
| `AsString()` | — | String | Синоним `ToString`. |
| `GetLength()` | — | Integer | Текущая длина собранной строки в символах. |
| `GetSize()` | — | Integer | Синоним `GetLength`. |
| `IsEmpty()` | — | Boolean | TRUE — буфер пуст, FALSE — содержит данные. |
| `Mid(start)` | start: Integer (позиция с 0) | String | Возвращает подстроку от позиции `start` до конца строки. Если `start` больше или равен длине, возвращает пустую строку. |
| `Mid(start, count)` | start: Integer, count: Integer | String | Возвращает `count` символов начиная с позиции `start`. |

## Управление буфером

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetLength(newLength)` | newLength: Integer | Integer | Устанавливает длину строки. Если больше текущей — дополняется пробелами. Если меньше — усекается. Возвращает предыдущую длину. |
| `GetGrowBy()` | — | Integer | Текущий размер приращения буфера. По умолчанию 1024. |
| `SetGrowBy(growBy)` | growBy: Integer | Integer | Устанавливает размер приращения буфера при расширении. Возвращает предыдущее значение. |
| `SetString(value)` | value: String | Empty | Полностью заменяет содержимое буфера. Предыдущее содержимое теряется. |

## Примеры

Построение текстового сообщения с форматированием в языке EME-L:

```EME-L
'Сборка сообщения об ошибках валидации'
sb = Object("StringBuilder", "Ошибки проверки:\n");
sb.AppendFormat("Запись: %s, поле: %s, значение: %d\n",
    sRecName, sFieldName, nValue);
sb.AppendLine("-- конец списка --");
Message = sb.ToString();
```

Построчное накопление данных с блочной записью в файл (по 10 000 строк) — типичный сценарий в системе EME.WMS при экспорте больших объёмов данных:

```EME-L
'Накопление строк в буфере, запись блоками в файл'
sbData = Object("StringBuilder");
nQtyLines = 0;

Loop (arrLines)
    sbData.AppendLine(arrLines.GetCurLine());
    If (nQtyLines == 10000)
        'Сбросить накопленный блок в файл'
        fileFld.PutEndFileData(sbData.ToString());
        'Пересоздать буфер для следующего блока'
        sbData = Object("StringBuilder");
        nQtyLines = 0;
    End If
    nQtyLines = nQtyLines + 1;
End Loop
'Дописать остаток'
fileFld.PutEndFileData(sbData.ToString());
```

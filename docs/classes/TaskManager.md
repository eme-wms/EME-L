# Класс TaskManager

> Синонимы для поиска: CITask TaskManager класс EME-L, диспетчер задач EME-L, параллельное выполнение EME-L, AddLocalTask RunTasks EnterSection LeaveSection, пул потоков EME-L, CITask GetTaskManager SetInfo, асинхронные задачи EME-L, TASKMODE_LOCAL_ONLY TASKMODE_NET_ONLY, критическая секция EME-L, многопоточность EME-L, CEMELTask CITaskManager, фоновые задачи EME.WMS, version gate LAST_VERSION 52.0 KERNEL_12_00

Класс `TaskManager` в языке EME-L — это диспетчер задач для управления асинхронным выполнением методов EME-L классов в отдельных потоках. Класс TaskManager в языке EME-L предоставляет механизм создания пула рабочих потоков, добавления задач на выполнение и запуска их параллельного исполнения. Каждая задача представляет собой вызов метода указанного EME-L класса с заданными аргументами.

Класс TaskManager в системе EME.WMS используется для организации параллельной обработки данных, выполнения фоновых операций и распределения вычислительной нагрузки между потоками. Поддерживает синхронизацию через критическую секцию для защиты разделяемых данных при параллельном выполнении.

Объект класса `TaskManager` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера. Класс зарегистрирован в C++ под именем `CITaskManager` (CIName=`CITask` для объекта задачи), в скриптах EME-L используется имя `TaskManager`. Внутри вызываемого метода доступен объект текущей задачи (CITask), через который можно получить ссылку на родительский диспетчер и задать текстовое описание хода выполнения.

## Условная компиляция по версии ядра (критично)

В языке EME-L реализация класса `TaskManager` в C++ (`i_TaskManager.cpp`) разделена директивой препроцессора `#if LAST_VERSION < MAKE_VERSION(52, 0)`. Файлы `i_TaskManager.cpp` и `i_TaskManager_utf8.cpp` в `c_emel_class_method/` байт-идентичны — обе копии содержат одну и ту же условно-компилируемую реализацию, отдельной utf8-специфичной функциональности нет.

- **Версии ядра ниже 52.0** (сборки с `KERNEL_12_00`): полный диспетчер задач — три конструктора, методы `AddLocalTask`, `RunTasks`, `EnterSection`, `LeaveSection`, а также вспомогательные C++ классы `CEMELTask` (исполнитель задачи) и `CITask` (объект текущей задачи). Разделы «Добавление и запуск задач» и «Объект текущей задачи CITask» относятся к этому варианту.
- **Версии ядра 52.0 и выше**: класс редуцирован до заглушки — доступен только один конструктор без аргументов и методы `EnterSection`/`LeaveSection`, использующие внутреннюю `CRITICAL_SECTION`. Методы `AddLocalTask`, `RunTasks` и вспомогательный класс `CITask` отсутствуют. Конструктор с любыми аргументами вызывает `too_many()`.

Регистрация `REGISTRY_EMEL_CLASS(TaskManager, …)` в `p_System.cpp` также обёрнута в `#ifdef KERNEL_12_00` — вне этой сборки класс не экспортируется в EME-L вообще. Описание в `REGISTRY` соответствует «полной» реализации (перечисляет «Добавление локальных задач», «Запуск всех добавленных задач») и не упоминает редукцию в v52.0+, что является расхождением registry↔body, зафиксированным в `CPP_DISCREPANCIES_REPORT.md`.

## Создание объекта класса TaskManager

### Версии ядра ниже 52.0

Класс `TaskManager` в языке EME-L имеет три конструктора — по числу аргументов от 0 до 2.

```EME-L
'Конструктор без аргументов — параметры по умолчанию'
objTM = Object("TaskManager");
```

```EME-L
'Конструктор с размером пула локальных потоков'
objTM = Object("TaskManager", 4);
```

```EME-L
'Конструктор с размером пула и режимом выполнения'
objTM = Object("TaskManager", 4, "LOCAL");
```

Конструктор без аргументов класса TaskManager в языке EME-L создаёт диспетчер с параметрами по умолчанию. При указании размера пула (первый аргумент) устанавливается количество рабочих потоков в локальном пуле. Второй аргумент — строковый режим выполнения: `"LOCAL"` ограничивает выполнение локальными потоками, `"NET"` — сетевым выполнением. При указании более двух аргументов вызывается ошибка `too_many()`.

### Версии ядра 52.0 и выше

В языке EME-L для версий ядра 52.0+ доступен только один конструктор:

```EME-L
'Единственный доступный конструктор в v52.0+ — без аргументов'
objTM = Object("TaskManager");
```

Любые аргументы приводят к ошибке `too_many()`. Объект создаёт собственную критическую секцию, но не имеет пула потоков и не выполняет задачи.

## Добавление и запуск задач

> Раздел относится только к версиям ядра ниже 52.0. В v52.0+ методы `AddLocalTask` и `RunTasks` отсутствуют.

Методы класса TaskManager в языке EME-L для добавления задач и их запуска на параллельное исполнение.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddLocalTask(className, methodName)` | className: String, methodName: String | Empty | Добавляет локальную задачу на выполнение указанного метода EME-L класса без дополнительных аргументов. Задача будет выполнена в одном из потоков локального пула при вызове RunTasks. |
| `AddLocalTask(className, methodName, arg1, ..., argN)` | className: String, methodName: String, arg1..argN: произвольный тип | Empty | Добавляет задачу с произвольным числом аргументов, передаваемых вызываемому методу. Минимум 2 обязательных аргумента; далее — необязательные параметры вызываемого метода. |

Метод `AddLocalTask` класса TaskManager в языке EME-L добавляет задачу в очередь диспетчера. Фактическое выполнение происходит только при вызове `RunTasks`. Задача всегда выполняется в режиме `TASKMODE_LOCAL_ONLY`, когерентность отключена. При менее чем двух аргументах вызывается ошибка `too_rare()`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `RunTasks()` | — | String | Запускает все добавленные задачи на параллельное исполнение и ожидает их завершения. |

**Возвращаемое значение `RunTasks`** класса TaskManager в языке EME-L зависит от результата выполнения:

| Сценарий | Возвращаемое значение |
|----------|----------------------|
| Успешное выполнение всех задач | Пустая строка |
| Прерывание пользователем | `"Прервано пользователем"` |
| Ошибки в задачах | Многострочная строка; каждая строка формата `ЗадачаN: инфо (ошибка)`, где `N` — индекс задачи, `инфо` — последний текст SetInfo, `ошибка` — сообщение ошибки |

## Синхронизация

Методы `EnterSection`/`LeaveSection` класса TaskManager в языке EME-L доступны в обеих ветвях условной компиляции, но используют разные внутренние объекты критической секции.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `EnterSection()` | — | Empty | Входит в критическую секцию диспетчера задач для синхронизации доступа к разделяемым ресурсам. Блокирует выполнение других потоков, ожидающих входа в эту же секцию, до вызова LeaveSection. |
| `LeaveSection()` | — | Empty | Выходит из критической секции диспетчера задач. Разблокирует доступ для других потоков, ожидающих входа в эту секцию. |

Критическая секция класса TaskManager в языке EME-L — это единая секция на объект TaskManager. В версиях ниже 52.0 она разделяется потоками пула, выполняющими задачи. В v52.0+ критическая секция остаётся рабочей (на основе `CRITICAL_SECTION`), но без пула потоков практическое применение ограничено внешней синхронизацией из скрипта. Каждый вызов `EnterSection` должен быть парен с соответствующим `LeaveSection`.

## Объект текущей задачи CITask

> Раздел относится только к версиям ядра ниже 52.0. В v52.0+ класс `CITask` не компилируется.

Внутри метода EME-L класса, вызванного через `AddLocalTask`, в языке EME-L доступен объект текущей задачи с CIName `CITask`. Этот объект не создаётся через `Object()` — он передаётся исполняющей системой в нулевой аргумент вызываемого метода (`args[0]`, тип `VT_NULL` с `byref` на `CITask*`). Класс зарегистрирован в C++ как `CITask`.

Методы объекта CITask в языке EME-L:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetTaskManager()` | — | Ссылка на TaskManager | Возвращает объект диспетчера задач, управляющего выполнением текущей задачи. |
| `SetInfo(info)` | info: String | Empty | Устанавливает текстовую информацию о текущей задаче, отображаемую в окне диспетчера задач и используемую в журнале ошибок RunTasks. |

Объект `CITask` класса TaskManager в языке EME-L в скриптах принимает имя, заданное в сигнатуре метода (обычно `Task`). Реальный пример использования обоих методов — класс `QAModule.QATasks` (см. раздел «Примеры»).

## Константы

Значения режима выполнения класса TaskManager в языке EME-L, задаваемые вторым аргументом конструктора `TaskManager(размерПула, режим)`:

| Строковое значение | Внутренняя константа | Описание |
|--------------------|----------------------|----------|
| `"LOCAL"` | `TASKMODE_LOCAL_ONLY` | Задачи выполняются только локально. |
| `"NET"` | `TASKMODE_NET_ONLY` | Задачи выполняются только по сети. |

Любое иное значение аргумента режима приводит к ошибке `Неизвестное значение для параметра task mode`. Эти значения применимы только в версиях ядра ниже 52.0, где доступен конструктор с двумя аргументами.

Возвращаемые строковые константы метода `RunTasks` в языке EME-L:

| Константа | Значение | Условие |
|-----------|----------|---------|
| Строка прерывания | `"Прервано пользователем"` | Пользователь прервал выполнение. |
| Строка ошибки задачи | Формат `"Задача%d: %s (%s)"` | Индекс задачи, текст SetInfo, сообщение ошибки. |

## Примеры

Шаблон диспетчера (создание объекта `TaskManager` класса TaskManager в языке EME-L и вызов `AddLocalTask`/`RunTasks`) ниже синтетический — в поставляемой базе конструкт `Object("TaskManager"` в скриптах EME-L не встречается (проверка выгрузки классов `Базовая выгрузка проекта/Классы/*.txt`). Однако API объекта задачи (`CITask`) используется в реальном классе `QAModule.QATasks` (см. пример ниже), методы которого вызываются через `AddLocalTask` из вызывающего кода вне выгрузки скриптов.

Параллельное выполнение методов двух классов в языке EME-L:

```EME-L
'Создать диспетчер с пулом из 4 потоков в локальном режиме'
objTM = Object("TaskManager", 4, "LOCAL");

'Добавить задачи — метод ProcessData класса MyWorker без аргументов'
objTM.AddLocalTask("MyWorker", "ProcessData");

'Добавить задачу с аргументами — метод Export класса ReportHandler'
objTM.AddLocalTask("ReportHandler", "Export", "D:\\Data\\out.txt", 100);

'Запустить все задачи и дождаться завершения'
strResult = objTM.RunTasks();

If (strResult <> "")
    is_message("Ошибки выполнения", strResult);
End If
```

Использование критической секции и SetInfo внутри метода задачи в языке EME-L:

```EME-L
'Внутри метода класса MyWorker — объект CITask передан в args[0]'
objTM = args[0].GetTaskManager();

'Защита разделяемого ресурса через критическую секцию TaskManager'
objTM.EnterSection();
arrShared.SetAt("key", Value);
objTM.LeaveSection();

'Обновить описание хода выполнения для окна диспетчера задач'
args[0].SetInfo("Обработано записей: " + nDone);
```

### Реальный пример: `QAModule.QATasks` (поставляемая база)

Класс `QAModule.QATasks` в системе EME.WMS — реальный потребитель API `CITask`. Его методы (`RunGlobal`, `RunLocal` и обёртки `RunGlobalGlossary`/`RunLocalHistory`) вызываются через `AddLocalTask` и принимают объект задачи первым аргументом. Внутри методов используется `Task.SetInfo()` для индикации хода выполнения и `Task.GetTaskManager().EnterSection()`/`LeaveSection()` для защиты слияния результатов запроса в разделяемый объект `Query`:

```EME-L
RunGlobal(QueryName, Task AS "Invariant", Query AS "Query", HandShakingQuery AS "Query", QueryLotKeyIn, BranchRef)
{
    Branch = Object("dsDB", "Branch");
    Branch.SetLine(BranchRef);

    QueryLotKey = QueryLotKeyIn;
    IPAddress = Branch.GetIP();
    Port = Branch.GetPort();

    'Индикация хода выполнения — текст виден в окне диспетчера'
    Task.SetInfo(Branch.GetTextCode() + ": " + "запрос к " + IPAddress);

    Object("System").InitQueryCache();
    GlobalQuery = Object("Query", QueryName);
    GlobalQuery.Execute();

    Task.SetInfo(Branch.GetTextCode() + ": " + "обработка запроса");

    'Синхронизация доступа к разделяемому объекту Query'
    TaskManager AS "Invariant" = Task.GetTaskManager();
    TaskManager.EnterSection();
    Query.Merge(GlobalQuery);
    Line = HandShakingQuery.AddLine();
    HandShakingQuery.PutData("Filial", Line, Branch.GetCode());
    TaskManager.LeaveSection();

    Task.SetInfo(Branch.GetTextCode() + ": " + "запрос завершен");
}
```

Шаблон класса QATasks в языке EME-L показывает типичное применение CITask: задача выполняет удалённый/локальный запрос к филиалу, затем под критической секцией диспетчера сливает результат в общий объект `Query` и фиксирует квитирование в `HandShakingQuery`. Поисковый ключ `QueryLotKey`, `IPAddress`, `Port` и `PalletID` сохраняются в полях класса для последующего опроса вызывающим кодом.

# Класс Semaphore

> Синонимы для поиска: CISemaphore Semaphore класс EME-L, серверный семафор EME-L, мастер-семафор EME-L, межстанционная синхронизация EME-L, Set Delete IsSet Check GetName GetUserName GetHostName GetDateTime GetDivision, блокировка на сервере EME-L, многопользовательский режим EME-L, диагностика семафора EME-L

Класс `Semaphore` в языке EME-L — класс для работы с серверными семафорами в многопользовательском режиме. В языке EME-L класс Semaphore предназначен для защиты прикладного кода от одновременного выполнения несколькими пользователями или рабочими станциями.

В системе EME.WMS класс Semaphore предоставляет механизм установки именованной блокировки на сервере, проверки состояния блокировки и снятия блокировки. Поддерживает мастер-семафоры для централизованной блокировки на мастер-сервере. При уничтожении объекта семафор автоматически снимается, если он был установлен.

Включает методы для получения информации об установившем семафор пользователе: имя пользователя, имя компьютера, дата и время установки, а также наименование раздела (филиала). В монопольном режиме работы (MONO) все операции с семафором возвращают пустое значение и не выполняют блокировок.

> Класс `Semaphore` в языке EME-L решает задачу **межстанционной** синхронизации — блокировка видна всем рабочим станциям, подключённым к серверу. Для защиты от псевдопараллельного выполнения обработчиков в пределах одного процесса используйте класс [Atom](./Atom.md) — он легче и не требует round-trip на сервер.

## Создание объекта класса Semaphore

Класс `Semaphore` в языке EME-L имеет один конструктор — с двумя обязательными аргументами.

```EME-L
'Семафор на текущем сервере (не мастер)'
objSem = Object("Semaphore", "MO_PROCESSING", False);

'Мастер-семафор — блокировка устанавливается на мастер-сервере'
objSem = Object("Semaphore", "MO_PROCESSING", True);
```

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `name` | String | Да | Имя семафора для идентификации блокировки на сервере. Не может быть пустым — конструктор вызывает `smart_error` при пустом имени. |
| `master` | Boolean | Да | Флаг мастер-семафора. `True` — семафор устанавливается на мастер-сервере; `False` — на текущем сервере. |

При уничтожении объекта деструктор автоматически снимает семафор с сервера, если имя было задано и станция не работает в монопольном режиме (MONO). В языке EME-L семафоры с пустым именем игнорируются деструктором — это защита от случайного вызова разблокировки при загрузке метапроекта.

## Управление блокировкой класса Semaphore

Все методы этой группы в режиме MONO возвращают пустое значение и не выполняют серверных запросов.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Set()` | — | Boolean или Empty | Устанавливает серверный семафор с именем из конструктора. `True` — успешно; `False` — занят другим пользователем; Empty — станция в режиме MONO. |
| `Delete()` | — | Empty | Снимает ранее установленный семафор. В режиме MONO ничего не делает. Возвращает пустое значение. |
| `IsSet()` | — | Boolean или Empty | Проверяет, удерживается ли семафор **текущим пользователем**. `True` — установлен нами; `False` — не установлен или установлен другим; Empty — режим MONO. |
| `Check()` | — | Boolean или Empty | Проверяет, занят ли семафор **любым пользователем**. `True` — занят; `False` — свободен; Empty — режим MONO. |

Разница между `IsSet` и `Check` в языке EME-L: `IsSet` подтверждает, что блокировка наша (можно безопасно снять через `Delete`); `Check` лишь сообщает, что имя занято — без указания владельца. Для диагностики «кто держит» используйте методы группы «Информация о владельце».

## Информация о владельце семафора в языке EME-L

Методы возвращают данные о пользователе, установившем семафор. Если семафор не установлен или информация недоступна, возвращается пустая строка. Значения заполняются при успешном `Set` или `Check`/`IsSet`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetName()` | — | String | Имя семафора, переданное в конструкторе. |
| `GetUserName()` | — | String | Имя пользователя, установившего семафор. |
| `GetHostName()` | — | String | Имя компьютера (рабочей станции), на котором установлен семафор. |
| `GetDateTime()` | — | String | Дата и время установки семафора в формате `DD.MM.YYYY - HH:MM:SS`. |
| `GetDivision()` | — | String | Наименование раздела (филиала), установившего семафор. Поиск раздела выполняется по внутреннему номеру `divisionNo`; если раздел не найден — возвращается пустая строка. |

Метод `GetDivision` в языке EME-L доступен только в версии ядра 50.0 и выше — в более ранних версиях возвращается пустая строка.

## Примеры использования класса Semaphore в языке EME-L

Защита длительной операции от параллельного выполнения — типичный сценарий для фоновой обработки заданий. Если другой пользователь уже запустил операцию с тем же именем семафора, `Set` вернёт `FALSE` и текущий вызов завершится досрочно:

```EME-L
'Уникальное имя семафора на основе номера документа'
SemaphoreName = is_format("MO_PAUSE#%d", MoveOrderRef);
Semaphore = Object("Semaphore", SemaphoreName, False);

'Попытка установить блокировку; FALSE — занято, выходим'
If (~Semaphore.Set())
    Return;
End If

'Тело критической секции; при выходе из области видимости
'деструктор Semaphore сам снимет блокировку с сервера'
```

Мастер-семафор для глобальной операции заливки изменений в проекте — блокировка ставится на мастер-сервере, чтобы её видели все станции сети независимо от того, к какому серверу они подключены. Перед запуском операции проверяем, не удерживает ли её кто-то ещё:

```EME-L
ChangesCommitSemaphore = Object("Semaphore", "Заливка изменений", True);

'Проверяем, не занят ли семафор другим пользователем'
If (ChangesCommitSemaphore.Check())
    Message = tr("Заливка изменений начата ") +
        ChangesCommitSemaphore.GetDivision() + " " +
        ChangesCommitSemaphore.GetUserName() + " " +
        ChangesCommitSemaphore.GetHostName() + " " +
        ChangesCommitSemaphore.GetDateTime();
    is_message(tr("Внимание!"), Message, 1, 1);
    Return;
End If

'Семафор свободен — захватываем'
If (~ChangesCommitSemaphore.Set())
    is_message(tr("Внимание!"),
        tr("Не удалось установить семафор для заливки изменений."), 1, 3);
    Return;
End If

'Тело заливки; по завершении вызываем Delete явно, чтобы
'не ждать выхода из области видимости'
```

# Класс FreeBrowser — отвязанный браузер в языке EME-L

в языке EME-L класс `FreeBrowser` (внутреннее имя C++ — `CIFreeBrowser`) — это браузер, не привязанный к записи базы данных (отвязанный браузер). Регистрируется макросом `REGISTRY_EMEL_CLASS(FreeBrowser, …)` в `p_System.cpp` и предоставляет механизм отображения данных из запросов EME DB в табличном виде без привязки к конкретной записи или диалогу. Источником данных для `FreeBrowser` всегда выступает объект запроса (`Query`), в отличие от обычного класса `Browser`, который привязывается к шаблону браузера по имени.

Класс поддерживает создание на основе объекта запроса с необязательным указанием имени браузера, имени EME-L-класса и сетевой настройки. Позволяет настраивать источник данных (метод `Tune`), управлять заголовком окна и софтовым именем (`SoftName`), сохранять и загружать сетевые (`SaveNetSettings`/`LoadNetSettings`) и базовые (`SaveBaseSettings`) настройки, программно управлять сохранением фильтров (`SetFilterSaving`) и сортировок (`SetSortSaving`), а также устанавливать форматы отображения колонок (`SetColumnFormat`).

в системе EME.WMS класс `FreeBrowser` применяется в прикладных скриптах для организации произвольного просмотра табличных данных из запросов — например, модальный выбор строки из результата запроса (`Object("FreeBrowser", Query, "Приходы без ASN").Run()`), отображение истории импорта, выбор группы расстановки товаров и т. п.

Внимание! Для того чтобы настройки браузера можно было сохранить, ему нужно дать имя и записать в запись настроек. Обычный браузер делает это при его создании в редакторе диалогов. Здесь нужно зайти в меню браузера «Настройки» -> «Сохранить настройки браузера в шаблон». Если после этого браузер переименовать, его настройки потеряются. Доступен начиная с версии ядра 25.13 (`#if LAST_VERSION >= MAKE_VERSION(25, 13)`).

## Создание объекта — конструкторы (5 перегрузок)

в языке EME-L объект класса `FreeBrowser` создаётся оператором `Object("FreeBrowser", …)`. Класс имеет пять перегрузек конструктора — на каждое число аргументов от 0 до 4. Поведение описано по контракту из тел конструктора (`CIFreeBrowser::CIFreeBrowser` для версии ядра ≥ 51.9).

| Конструктор | Аргументы | Возвращает | Описание |
|-------------|-----------|------------|----------|
| `FreeBrowser()` | — | FreeBrowser | Отвязанный браузер, подключённый к текущему контексту выполнения. Используется, когда браузер создаётся внутри диалога, уже содержащего отвязанный браузер. Если контекст не содержит браузер — объект остаётся пустым. |
| `FreeBrowser(query)` | query: Query (зарезервирован) | FreeBrowser | Пустой объект `FreeBrowser` без привязки к источнику данных. В текущей версии параметр `query` зарезервирован, но не используется при создании браузера. |
| `FreeBrowser(query, name)` | query: Query, name: String | FreeBrowser | Отвязанный браузер на основе объекта запроса с указанием имени. Имя используется для сохранения настроек и отображения заголовка окна (по умолчанию заголовок совпадает с именем). При создании автоматически включается режим сохранения настроек (`setSaveSettingsFlag(true)`). |
| `FreeBrowser(query, name, className)` | + className: String | FreeBrowser | То же с указанием имени класса EME-L. Устанавливает прокси-обработчик (`SetProxy`) для маршрутизации сообщений браузера через `bs_main_proxy`. При создании автоматически включается режим сохранения настроек. |
| `FreeBrowser(query, name, className, netSettings)` | + netSettings: String | FreeBrowser | То же с загрузкой сетевой настройки из базы данных (`load_net_browse_parameters_from_DB(netSettings)`). Устанавливает прокси-обработчик. **В отличие от трёхаргументной формы**, режим сохранения настроек автоматически **не** включается. |

Безаргументная форма (`FreeBrowser()`) привязывается к `context->browse` — существующему отвязанному браузеру текущего диалога. При её использовании вне контекста, уже содержащего отвязанный браузер, объект остаётся пустым (`object` не устанавливается, `ObjectIsCreated()` вернёт `false`).

```EME-L
'Без аргументов — привязка к текущему контексту выполнения'
objFB = Object("FreeBrowser");

'Только объект запроса (query зарезервирован)'
objFB = Object("FreeBrowser", objQuery);

'Объект запроса + имя (включает режим сохранения настроек)'
objFB = Object("FreeBrowser", objQuery, "Приходы без ASN");

'Объект запроса + имя + класс EME-L (прокси-обработчик)'
objFB = Object("FreeBrowser", objQuery, "Приходы", "GoodsItem");

'Объект запроса + имя + класс + сетевая настройка'
objFB = Object("FreeBrowser", objQuery, "Приходы", "GoodsItem", "ASN_Layout");
```

## Методы — управление настройками

в языке EME-L булевы методы `FreeBrowser` возвращают `Boolean` в формате EME-L: TRUE = -1, FALSE = 0. Возвращаемое `false` для этих методов означает «браузер не найден» (объект пустой или внутренний указатель `browse` невалиден). Метод `SetSaveSettingsFlag` управляет глобальным флагом сохранения, а `SaveNetSettings`/`LoadNetSettings`/`SaveBaseSettings` работают с записями в базе данных EME.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetSaveSettingsFlag(flag)` | flag: Boolean | Boolean | Управляет флагом сохранения настроек отвязанного браузера в базе данных. `true` — сохранять настройки, `false` — не сохранять. |
| `SaveBaseSettings()` | — | Boolean | Сохраняет базовую настройку отвязанного браузера в базе данных (`write_base_browse_settings_to_DB`). |
| `SaveNetSettings(name, forAllUsers)` | name: String, forAllUsers: Boolean | Boolean | Сохраняет сетевую настройку с именем `name` (`write_net_browse_settings_to_DB`). `forAllUsers=true` — доступна всем пользователям, `false` — только текущему. |
| `LoadNetSettings(name)` | name: String | Boolean | Загружает сетевую настройку `name` (`load_net_browse_parameters_from_DB`). После успешной загрузки флаг сохранения настроек автоматически сбрасывается в `false`. |

## Методы — источник данных и идентификация

Метод `Tune` полиморфен: строковый аргумент трактуется как имя шаблона браузера, объект `Query` — как источник данных. Методы `SetSoftName`/`SetTitle` управляют софтовым именем (ключом сохранения настроек) и заголовком окна соответственно.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Tune(source)` | source: String или Query | Boolean | Настраивает источник данных. Строковый аргумент — имя шаблона браузера; объект `Query` — подключается в качестве источника данных через `CEMEDataQuery`. |
| `SetSoftName(name)` | name: String | Boolean | Устанавливает софтовое имя браузера (`SoftName`), используемое для сохранения настроек. |
| `SetTitle(title)` | title: String | Empty | Устанавливает заголовок окна браузера. |

## Методы — сохранение фильтров и сортировок

в языке EME-L методы `SetFilterSaving`/`SetSortSaving` управляют режимом сохранения соответствующих аспектов настроек браузера. Возвращают `Empty` (не путать с методами настройки, возвращающими `Boolean`). Доступны при включённой компиляции `NEW_BROWSE_SETTINGS`. Константы режима: `SET_SAVE_FILTERS`/`SET_NO_SAVE_FILTERS` и `SET_SAVE_SORTINGS`/`SET_NO_SAVE_SORTINGS`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFilterSaving(saveFlag)` | saveFlag: Boolean | Empty | Управляет режимом сохранения фильтров. `true` — сохранять фильтры всегда, `false` — не сохранять. |
| `SetSortSaving(saveFlag)` | saveFlag: Boolean | Empty | Управляет режимом сохранения сортировок. `true` — сохранять сортировки всегда, `false` — не сохранять. |

## Методы — формат колонок

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetColumnFormat(columnName, format)` | columnName: String, format: String | Empty | Устанавливает формат отображения ячеек колонки `columnName`. Примеры форматов: `"%s"` — строковое представление, `"%.2f"` — вещественное число с двумя знаками после запятой. При отсутствии браузера или ненайденной колонке (`get_column_number` вернул < 0) генерируется ошибка через `smart_error` (в режиме `except`). |

## Методы базового класса Browser

в языке EME-L объект `FreeBrowser` наследует полноценный интерфейс запуска и навигации от базового класса [`Browser`](./Browser.md) (`CIBrowser`/`CUniBrowse`) — в частности `Run`, `Close`, `RemoveColumn`, `GetName`, `GetColumn` и др. Это позволяет использовать `FreeBrowser` как обычный браузер: открыть модально, выбрать строку, программно скрыть колонки и т. п. См. полный список унаследованных методов на странице [`Browser.md`](./Browser.md).

## Примеры

в системе EME.WMS типичный сценарий — создать отвязанный браузер по объекту запроса и открыть его модально для выбора строки. Номер выбранной строки возвращается методом `Run` (в терминах EME-L это метод базового класса `Browser`). Реальный пример из класса `AltWGoodsItem`:

```EME-L
FreeBrowser = Object("FreeBrowser", PlacingGroupsQuery, "Группы расстановки");
FreeBrowser.RemoveColumn("@Группы расстановки");
QueryLine = FreeBrowser.Run(0);
'QueryLine — номер выбранной строки БД или -1 при отмене'
PlacingGroupsRef = If(QueryLine > -1, PlacingGroupsQuery.GetData("@Группы расстановки", QueryLine), NULL_REF);
```

Краткая форма создания и немедленного запуска (результат `Run` отбрасывается) — реальный пример из `ERPInterface`:

```EME-L
Object("FreeBrowser", Query, "Свертка").Run();
```

Установка формата колонки и управление сохранением фильтров/сортировок:

```EME-L
FreeBrowser = Object("FreeBrowser", objQuery, "Отчёт по остаткам");
FreeBrowser.SetColumnFormat("Цена", "%.2f");
FreeBrowser.SetFilterSaving(True);
FreeBrowser.SetSortSaving(False);
```

Сохранение сетевой настройки с диалогом ввода имени — реальный пример из класса `FreeReport`. Безаргументный конструктор используется для привязки к уже существующему в контексте отвязанному браузеру:

```EME-L
FreeBrowser = Object("FreeBrowser");
Name = is_enter_value_dialog(tr("Введите имя представления"), "");
If (~is_empty(Name))
    is_transaction(1, tr("Сохранение представления"));
    FreeBrowser.SaveNetSettings(Name, False);
    is_transaction(-1);
End If
```

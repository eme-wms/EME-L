# Класс PickToLight

Класс `PickToLight` в языке EME-L — класс для интеграции с системой PickToLight (световая подборка на гравитационных стеллажах) в системе EME.WMS. Предназначен для формирования и отправки XML-сообщений с заданиями на подборку заказов на сервер PickToLight по TCP/IP-соединению.

Класс `PickToLight` в языке EME-L поддерживает автоматическое заполнение данных из строк количеств по заказам (`QtyByOrders`) с фильтрацией по статусу заказа, весу, наличию буферных строк и незавершенным пополнениям, а также группировку по заказам и производственным линиям. После успешной отправки обновляет статусы строк подборки и помечает заказы как отправленные в PickToLight. Также обрабатывает входящие ответные сообщения от сервера, включая статусы `IN_PREPARATION` (печать этикеток отгрузочных мест) и `Forwarded` (автоматическое перемещение товара из буфера и подтверждение отгрузки строк).

Параметры подключения к серверу (IP-адрес и порт) задаются через системные параметры проекта (`Система.PickToLight.IP адрес` и `Система.PickToLight.Порт`). Работа объекта выполняется в составе фонового потока, запускаемого при старте системы при активированной настройке `Система.PickToLight`. CI-имя класса в системе EME.WMS — `CIPickToLight`.

## Создание объекта класса PickToLight

Класс `PickToLight` в языке EME-L имеет один конструктор — без аргументов.

```EME-L
'Создать объект интеграции с PickToLight'
objPickToLight = Object("PickToLight");
```

Конструктор класса `PickToLight` в языке EME-L не принимает аргументов. При создании объекта инициализируются внутренние параметры: очищаются сетевые настройки, буфер данных, массив ссылок на строки заказов и сообщение об ошибке. Объект готов к последующему вызову метода `Send` для отправки заданий на сервер PickToLight.

## Отправка заданий на подборку классом PickToLight

Методы класса `PickToLight` в языке EME-L для отправки заданий на подборку заказов на сервер PickToLight.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Send(query)` | query: Query (по ссылке) | String | Формирует и отправляет XML-сообщение с заданиями на подборку заказов на сервер PickToLight в системе EME.WMS. |

Метод `Send(query)` класса `PickToLight` в языке EME-L выполняет полный цикл отправки:

1. Проверяет корректность хоста и порта из системных параметров проекта.
2. Проверяет поля входящего запроса; если запрос не передан, создаёт внутренний запрос с нужным набором полей.
3. Если переданный запрос пуст, автоматически заполняет его данными из `QtyByOrders`, отбирая клиентские заказы со статусом «в подборке» или «частично собран», прошедшие проверку по пороговому весу, наличию строк в буфере и отсутствию незавершенных пополнений.
4. Группирует данные по номеру заказа, производственной линии и коду товара.
5. Формирует XML-сообщение (`WMS_BOX_IN` → `BOX` → `BOX_LINES` → `LINE`) и сериализует его в массив байтов.
6. Подключается к серверу PickToLight по TCP/IP и отправляет пакет данных.
7. При успешной отправке обновляет статусы отправленных строк подборки (`SetPickToLightPicked`) и помечает заказы как отправленные (`SetSentToPickToLight`).

Поле `QTY_TO_BE_PREPARED` в XML формируется с целочисленным значением количества (`%.f`).

**Возвращает**: текст сообщения об ошибки. Пустая строка при успешной отправке; описание ошибки при сбое подключения, передачи данных или отсутствии данных для отправки.

## Автоматический отбор данных для PickToLight

Когда входящий запрос не содержит строк, метод `Send` класса `PickToLight` в языке EME-L самостоятельно собирает данные для отправки. Логика отбора реализована в C++ и включает следующие проверки:

- Заказ должен находиться в статусе «в подборке» (`IsMovingToPicking`) или «частично собран» (`IsPartlyAssembled`).
- Вес заказа не должен превышать пороговое значение из параметра `Предпочтения.Кубинг.Пороговый вес клиентского заказа`.
- По заказу не должно оставаться строк подборки, которые не в буфере и не отгружены.
- Если по заказу есть незавершенные пополнения, отправка пропускается (с записью предупреждения в лог).
- В подборку попадают только товары, у которых в карточке товара установлен признак отправки в PickToLight (`IsNeedSentToPickToLight`).

## Обработка входящих сообщений от сервера PickToLight

Помимо исходящей отправки, система EME.WMS запускает фоновый поток, который слушает TCP-порт, заданный параметром `Система.PickToLight.Порт сервера`. При получении XML-сообщения (`WMS_BOX_OUT`) выполняется:

- Статус `IN_PREPARATION` — печать этикетки отгрузочного места через класс `OrdersCalculationHandlers` и метод `PrintReportShipmentPlaceLabel`.
- Статус `Forwarded` — автоматическое перемещение товара из буфера в отгрузку и подтверждение подборки.

Эта функциональность не требует явного вызова из EME-L — поток стартует автоматически при включённом параметре `Система.PickToLight`.

## Примеры работы с классом PickToLight

Создание объекта `PickToLight` и отправка подготовленного запроса в языке EME-L:

```EME-L
'Создать объект PickToLight'
objPickToLight = Object("PickToLight");

'Подготовить запрос с данными для подборки'
objQuery = Object("Query");
objQuery.Create();
objQuery.AddTEXT("VendorCode", 32);
objQuery.AddTEXT("OrderNo", 32);
objQuery.AddTEXT("Article", 32);
objQuery.AddFLOAT("Qty", "SUM");
objQuery.AddINT("QtyByOrdersRef");

Line = objQuery.AddLine();
objQuery.PutData("VendorCode", Line, "ss");
objQuery.PutData("OrderNo", Line, "666666666");
objQuery.PutData("Article", Line, "7713.50");
objQuery.PutData("Qty", Line, 10);
objQuery.PutData("QtyByOrdersRef", Line, 271);

Line = objQuery.AddLine();
objQuery.PutData("VendorCode", Line, "ss");
objQuery.PutData("OrderNo", Line, "666666666");
objQuery.PutData("Article", Line, "7692.50");
objQuery.PutData("Qty", Line, 30);
objQuery.PutData("QtyByOrdersRef", Line, 270);

'Отправить задание на PickToLight'
mess = objPickToLight.Send(objQuery);
If (mess == "")
    is_message("PickToLight", "Отправка выполнена успешно", 1, 0);
Else
    is_message("PickToLight", mess, 1, 3);
End If
```

Отправка с автоматическим заполнением данных (передать пустой запрос) в языке EME-L:

```EME-L
'Создать объект PickToLight'
objPickToLight = Object("PickToLight");

'Передать пустой запрос — данные соберутся из QtyByOrders'
objQuery = Object("Query");
objQuery.Create();
mess = objPickToLight.Send(objQuery);
If (mess != "")
    is_message("PickToLight", mess, 1, 3);
End If
```

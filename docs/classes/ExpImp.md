# Класс ExpImp

> Синонимы для поиска: CIExpImp ExpImp класс EME-L, импорт экспорт EME-L, файлы экспорта EME, ядерный экспорт EME-L, чтение exp-файла EME-L, LoadFile ExtractNextRecord GetRecordName GetExportType GetVersion GetFilialNo GetFilialStatus GetFileName GetDate GetTime GetPrevFileName GetPrevDate GetPrevTime, ExpImp64 EME-L, CExpImpFilial, экспорт/импорт системы EME, INT64_ATTRIBUTE FLAGS_ATTRIBUTE ARRAY_ATTRIBUTE ARRAY_OF_REFERENCE TEXT_STRING_ATTRIBUTE DATA_STORAGE_ATTRIBUTE IMPORT_FILE_VERSION4 SYS_EXPORT_STATUS MAKE_VERSION 52.0, коррекция порядка байт ссылок EME-L

Класс `ExpImp` в языке EME-L — это интерфейс для чтения файлов ядерного экспорта/импорта системы EME. Предоставляет методы для открытия экспортных файлов, последовательного извлечения записей с заполнением объектов `Query`, а также получения метаданных файла (версия, филиал, дата, время). Поддерживает чтение истории экспорта через доступ к информации о предыдущем файле.

Класс ExpImp в системе EME.WMS используется для программного импорта данных из файлов экспорта, созданных ядром системы или другими филиалами. Объект класса `ExpImp` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс для работы с файлами экспорта.

## Создание объекта класса ExpImp

```EME-L
objExpImp = Object("ExpImp");
objExpImp = Object("ExpImp", false);
```

Конструктор без аргументов класса ExpImp в языке EME-L создаёт объект с 64-битным диапазоном ссылок по умолчанию. Необязательный аргумент `use64BitRange` (boolean) явно задаёт разрядность ссылочных полей: `true`/`"1"` — использовать 64-битный диапазон ссылок (по умолчанию), `false`/`"0"` — использовать 32-битный диапазон ссылок. Начиная с версии ядра 52.0 (`MAKE_VERSION(52, 0)`) аргумент игнорируется — всегда используется 64-битный режим (в коде C++ соответствующие ветки обёрнуты в `#if LAST_VERSION < MAKE_VERSION(52, 0)`).

## Чтение файла и записей класса ExpImp

Методы класса ExpImp в языке EME-L для открытия файлов экспорта и последовательного извлечения записей.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `LoadFile(filePath)` | filePath: String | String | Открывает файл экспорта EME для последующего чтения записей. Если файл уже был открыт ранее, предыдущий файл закрывается автоматически. При успешном открытии возвращает пустую строку; при ошибке — описание в формате `Ошибка <код_ошибки>`. |
| `ExtractNextRecord(query)` | query: Query (по ссылке) | Boolean | Извлекает следующую запись из открытого файла экспорта и заполняет переданный запрос данными записи. Поля записи автоматически добавляются в запрос с соответствующими типами атрибутов. Поддерживает специальную обработку полей «Ключи» (64-битное целое) и «Флаги» (32-битное целое), а также коррекцию порядка байт для ссылочных полей. Массивы ссылок (`ARRAY_OF_REFERENCE`) не обрабатываются и пропускаются. `TRUE` — запись успешно извлечена и добавлена в запрос; `FALSE` — записи закончились или произошла ошибка чтения. |
| `GetRecordName()` | — | String | Возвращает имя текущей записи, извлечённой из файла экспорта методом `ExtractNextRecord`. Пустая строка, если запись ещё не была извлечена. |
| `GetExportType()` | — | String | Возвращает тип экспорта открытого файла. Тип определяется по последним двум символам имени файла для системных экспортных файлов: `"s1"` или `"s2"`. Пустая строка, если файл не является системным экспортным файлом (признак — статус филиала `SYS_EXPORT_STATUS`). |

Метод `ExtractNextRecord` класса ExpImp в языке EME-L заполняет объект `Query` построчно: при каждом успешном извлечении в запрос добавляются поля записи (с автоматически определёнными типами атрибутов) и строки данных. Правила определения типов полей при чтении в системе EME.WMS:

- Поля «Ключи» обрабатываются как 64-битное целое (`INT64_ATTRIBUTE`), «Флаги» — как 32-битное целое (`INT_ATTRIBUTE`).
- Поля с атрибутами `FLAGS_ATTRIBUTE` и `ARRAY_ATTRIBUTE` приводятся к `INT_ATTRIBUTE`.
- Ссылочные поля приводятся к `INT64_ATTRIBUTE`, для них выполняется коррекция порядка байт — обмен местами двух `long`-значений внутри 8-байтного ключа ссылки.
- Массивы ссылок (`ARRAY_OF_REFERENCE`) не поддерживаются — такие поля пропускаются.
- Текстовые поля (`TEXT_STRING_ATTRIBUTE`) без явной длины получают размер по умолчанию 256 байт.
- Поля с неподдерживаемым типом атрибута (имя атрибута не определяется ядром) получают размер по умолчанию 64 байта.
- Поля переменной длины (`VARIABLE_LENGTH_FIELD_BIT`) и хранилища данных (`DATA_STORAGE_ATTRIBUTE`, начиная с версии файла `IMPORT_FILE_VERSION4`) считывают длину из потока данных.

## Метаданные файла класса ExpImp

Методы класса ExpImp в языке EME-L для получения метаданных открытого файла экспорта. Метаданные доступны после успешного вызова `LoadFile`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetVersion()` | — | Integer | Возвращает версию формата открытого файла экспорта. |
| `GetFilialNo()` | — | Integer | Возвращает номер филиала, сформировавшего открытый файл экспорта. |
| `GetFilialStatus()` | — | Integer | Возвращает статус филиала, сформировавшего открытый файл экспорта. |
| `GetFileName()` | — | String | Возвращает имя файла экспорта, указанное в заголовке открытого файла. |
| `GetDate()` | — | Date | Возвращает дату создания открытого файла экспорта. |
| `GetTime()` | — | Time | Возвращает время создания открытого файла экспорта. |

## История экспорта класса ExpImp

Методы класса ExpImp в языке EME-L для доступа к истории экспорта — метаданным предыдущего файла, записанным в заголовке текущего. В системе EME.WMS это позволяет отслеживать цепочку экспортных файлов при многоэтапном импорте данных между филиалами.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetPrevFileName()` | — | String | Возвращает имя предыдущего файла экспорта по истории. Пустая строка, если информация о предыдущем файле отсутствует. |
| `GetPrevDate()` | — | Date | Возвращает дату предыдущего экспорта по истории. |
| `GetPrevTime()` | — | Time | Возвращает время предыдущего экспорта по истории. |

## Примеры использования класса ExpImp

Последовательное чтение записей из файла экспорта в языке EME-L (паттерн из производственного скрипта):

```EME-L
'Создать объект импорта в 32-битном режиме и открыть файл'
objExpImp = Object("ExpImp", false);
Result = objExpImp.LoadFile("D:\\Import\\export.dat");
If (Result != "")
    is_message("Импорт", "Не удалось открыть файл: " + Result, 1, 3);
    Return 0;
End If

'Подготовить запрос для приёма данных записи'
Query = Object("Query");
Query.Create();
While (objExpImp.ExtractNextRecord(Query))
    RecordName = objExpImp.GetRecordName();
    'Поля текущей записи доступны в Query'
    Query.Delete();
    Query.Create();
End While
```

Получение метаданных файла экспорта в языке EME-L:

```EME-L
objExpImp = Object("ExpImp");
Result = objExpImp.LoadFile("D:\\Import\\export.dat");
If (Result == "")
    Version = objExpImp.GetVersion();
    FilialNo = objExpImp.GetFilialNo();
    ExportType = objExpImp.GetExportType();
    PrevFile = objExpImp.GetPrevFileName();
End If
```

# Класс UserEvent

> Синонимы для поиска: CIUserEvent UserEvent класс EME-L, пользовательские события EME-L, EventsTypeMPS EventsMPS EventsTypeParams, IsActive Pulse GetRecordNo GetDialogSoftName GetDialogStartCode BeforeEventPulse AfterEventPulse

Класс `UserEvent` в языке EME-L — класс для обработки пользовательских событий, зарегистрированных в системе через запись `EventsTypeMPS`. В языке EME-L класс `UserEvent` предоставляет механизм проверки наличия обработчиков, запуска цепочки обработчиков события с передачей параметров, а также доступ к настройкам события (связанная запись, диалог, код запуска). Объект создаётся с указанием имени типа события, которое должно существовать в записи `EventsTypeMPS`.

При запуске события методом `Pulse` в языке EME-L выполняется последовательный вызов всех зарегистрированных обработчиков (записи `EventsMPS`), прошедших фильтрацию, с передачей параметров через контейнер данных. В системе EME.WMS класс `UserEvent` поддерживает сигналы `BeforeEventPulse` и `AfterEventPulse` для интеграции с внешними обработчиками.

## Создание объекта класса UserEvent

Класс `UserEvent` в языке EME-L имеет два конструктора — один без аргументов, другой с именем типа события.

```EME-L
'Конструктор без аргументов — тип события не задан'
objEvent = Object("UserEvent");
```

```EME-L
'Конструктор с указанием имени типа события'
objEvent = Object("UserEvent", "OnDocumentChanged");
```

Конструктор без аргументов в языке EME-L создаёт объект, не привязанный к типу события. Такой объект не будет функционировать до тех пор, пока не будет создан с именем типа события. В системе EME.WMS это означает, что внутренняя ссылка на тип события остаётся равной `NULL_REF` (пустая ссылка, `-1`).

## Методы класса UserEvent в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsActive()` | — | Boolean | TRUE — для события зарегистрирована хотя бы одна цепочка обработчиков (записи `EventsMPS`); FALSE — обработчики отсутствуют или тип события не найден. |
| `Pulse()` | — | Empty | Запускает выполнение всех зарегистрированных обработчиков события. Перед запуском извлекает параметры типа события из записи `EventsTypeParams` и передаёт их в контейнер данных обработчикам. Выполняет фильтрацию обработчиков и последовательно вызывает прошедшие фильтрацию. Генерирует сигналы `BeforeEventPulse` перед началом и `AfterEventPulse` после завершения обработки. Если тип события не найден, выполнение прерывается без ошибки. |
| `GetRecordNo()` | — | Integer | Возвращает номер связанной записи из параметра `"Запись"` в записи `EventsTypeParams`. Если параметр не найден или пуст, возвращает `NULL_RECORD`. |
| `GetDialogSoftName()` | — | String | Возвращает имя диалога из параметра `"Диалог"` в записи `EventsTypeParams`. Если параметр не найден, возвращает пустую строку. |
| `GetDialogStartCode()` | — | Integer | Возвращает код запуска диалога из параметра `"Код запуска диалога"` в записи `EventsTypeParams`. Если параметр не найден, возвращает `0`. |

Все методы чтения параметров (`GetRecordNo`, `GetDialogSoftName`, `GetDialogStartCode`) в языке EME-L выполняют поиск параметра по имени без учёта регистра. В системе EME.WMS это поведение наследуется от сопоставления имён параметров в записи `EventsTypeParams`.

## Примеры

Проверка и запуск события в языке EME-L:

```EME-L
objEvent = Object("UserEvent", "OnDocumentChanged");
If (objEvent.IsActive())
    objEvent.Pulse();
End If
```

Получение параметров события в языке EME-L:

```EME-L
objEvent = Object("UserEvent", "OnDocumentChanged");
RecordNo = objEvent.GetRecordNo();
DialogName = objEvent.GetDialogSoftName();
StartCode = objEvent.GetDialogStartCode();
```

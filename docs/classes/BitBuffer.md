> Синонимы для поиска: CIBitBuffer BitBuffer класс EME-L, битовый буфер EME-L, битовый массив EME-L, побитовые операции И ИЛИ НЕ Исключающее ИЛИ, фильтр записей EME-L, SetSize GetSize IsEmpty IsFull IsHewed HewItem UnHewItem HewItemGrow UnHewItemGrow HewAll Clear Invert GetNoOfHewedItems And Or Xor Not Compare FindBit SetFirstLine SetNextLine IsValidLine GetLine SetLine SetLoopMode LoadLines UnloadLines LoadBrowserSelection LoadQuerySelection LoadHewVariableSelection LoadFrom SaveTo LoadFromCache AddToCache RemoveFromCache GetContent, переменная выделения EME-L, кэш фильтров EME.WMS, выделение строк браузера EME-L, Loop по BitBuffer

Класс `BitBuffer` в языке EME-L — битовый массив (битовый буфер) для хранения и манипулирования наборами битовых флагов. Предназначен для организации произвольных наборов элементов, каждый из которых может находиться в одном из двух состояний: `'0'` (сброшен) или `'1'` (установлен). Поддерживает произвольный размер буфера, динамическое расширение, побитовые логические операции (И, ИЛИ, Исключающее ИЛИ, НЕ), инверсию, массовую установку и сброс битов, а также загрузку и выгрузку данных из источников базы данных, браузеров, запросов и диалоговых переменных выделения.

Класс BitBuffer в языке EME-L обеспечивает кэширование содержимого в системе фильтров для повторного использования, навигацию по битам в режиме текущей «строки» и поиск битов по заданному состоянию. Используется в прикладных скриптах EME-L и системе EME.WMS для реализации множественного выделения строк, фильтрации записей, побитовых масок и других операций с дискретными флагами.

## Создание объекта класса BitBuffer

Конструктор класса BitBuffer в языке EME-L имеет три перегрузки — без аргументов, с размером и с привязкой к переменной выделения диалога.

```EME-L
'Пустой буфер размером 0 бит — заполняется через SetSize или HewItemGrow'
bb = Object("BitBuffer");

'Буфер на 64 бита, все биты в '0''
bb = Object("BitBuffer", 64);

'Буфер, связанный с переменной выделения диалога (загружается из неё)'
bb = Object("BitBuffer", Dialog, "SelectedRows");
```

В языке EME-L внутренний курсор текущей «строки» класса BitBuffer инициализируется значением -1: до вызова `SetFirstLine` или `SetLine` методы `GetLine()` и `IsValidLine()` без аргумента возвращают -1 и FALSE соответственно. Режим поиска битов по умолчанию — поиск установленных в `'1'` битов (`m_loopMode = true` в теле CIBitBuffer).

## Управление размером класса BitBuffer

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetSize(размер)` | размер: Integer | Empty | Новый размер буфера в битах. При изменении размера непустого буфера текущее содержимое **очищается** (все биты сбрасываются в `'0'`). Выбрасывает исключение для буфера, связанного с переменной выделения диалога. |
| `GetSize()` | — | Integer | Текущий размер буфера в битах. |
| `IsEmpty()` | — | Boolean | TRUE — все биты сброшены в `'0'` или буфер пустой. |
| `IsFull()` | — | Boolean | TRUE — все биты установлены в `'1'` или буфер пустой. |

Не рекомендуется увеличивать размер буфера класса BitBuffer в процессе заполнения (через `HewItemGrow`, `UnHewItemGrow` или повторный `SetSize`) — это приводит к перевыделению памяти и копированию содержимого. Предпочитайте задавать размер один раз через `SetSize` или конструктор перед заполнением.

## Операции с отдельными битами в классе BitBuffer

Методы класса BitBuffer в языке EME-L для работы с отдельными битами.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsHewed(индекс)` | индекс: Integer | Boolean | TRUE — бит в `'1'`. FALSE — бит в `'0'`, индекс выходит за границы буфера или передан `NULL_REF` (без исключения). |
| `HewItem(индекс)` | индекс: Integer | Empty | Устанавливает бит в `'1'`. Если бит уже установлен — ничего не делает. Выбрасывает исключение при выходе индекса за пределы буфера. |
| `UnHewItem(индекс)` | индекс: Integer | Empty | Сбрасывает бит в `'0'`. Если бит уже сброшен — ничего не делает. Контракт индекса тот же, что у `HewItem`. |
| `HewItemGrow(индекс)` | индекс: Integer | Empty | Устанавливает бит в `'1'`; если индекс превышает текущий размер, буфер автоматически расширяется, а новые биты (до указанного индекса) заполняются `'0'`. Выбрасывает исключение для буфера, связанного с переменной выделения. |
| `UnHewItemGrow(индекс)` | индекс: Integer | Empty | Сбрасывает бит в `'0'` с автоматическим расширением буфера. Аналог `HewItemGrow` для сброса. |
| `HewAll()` | — | Empty | Устанавливает все биты в `'1'`. |
| `Clear()` | — | Empty | Сбрасывает все биты в `'0'`. |
| `Invert()` | — | Empty | Инвертирует состояние всех битов: `'1'` → `'0'`, `'0'` → `'1'`. |
| `GetNoOfHewedItems()` | — | Integer | Количество битов, установленных в `'1'`. |

Методы `IsHewed`, `HewItem` и `UnHewItem` класса BitBuffer в языке EME-L молча возвращают FALSE/Empty при передаче `NULL_REF` (например, когда `r_Rec.GetLine()` возвращает -1). Это типовой приём защиты при обходе источника данных с потенциально пустой ссылкой.

## Побитовые логические операции класса BitBuffer

Результат операции класса BitBuffer в языке EME-L перезаписывает текущий объект `BitBuffer`. Если аргумент — другой `BitBuffer`, его размер должен совпадать с размером текущего буфера, иначе выбрасывается исключение. Если аргумент — целое число, оно трактуется как 8-битная маска (`UCHAR`), и операция применяется к первым 8 битам буфера.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `And(значение)` | значение: BitBuffer (по ссылке) или Integer | Empty | Побитовое «И». Бит результата равен `'1'` только если оба соответствующих бита установлены. |
| `Or(значение)` | значение: BitBuffer (по ссылке) или Integer | Empty | Побитовое «ИЛИ». Бит результата равен `'1'` если хотя бы один из соответствующих битов установлен. |
| `Xor(значение)` | значение: BitBuffer (по ссылке) или Integer | Empty | Побитовое «Исключающее ИЛИ». Бит результата равен `'1'` если соответствующие биты различаются. |
| `Not()` | — | Empty | Побитовое «НЕ» (инверсия). Аналогичен методу `Invert`. |
| `Compare(буфер)` | буфер: BitBuffer (по ссылке) | Boolean | TRUE — размеры и содержимое буферов совпадают побайтово. |

## Поиск битов в классе BitBuffer

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FindBit(индекс, состояние)` | индекс: Integer (старт), состояние: Integer | Integer | Индекс первого бита с заданным состоянием, начиная поиск с `индекс`. Если элемент не найден — -1. В языке EME-L **любое ненулевое значение `состояние` трактуется как «искать `'1'`»** (проверка `variant2integer(args[1]) != 0` в теле метода CIBitBuffer::FindBit), `0` — «искать `'0'`». Реестр IMPLEMENT_METHOD пишет «значения больше 1 трактуются как 1», но тело авторитетно для concrete behaviour — оно проверяет `!= 0`. |

## Навигация по «строкам» в классе BitBuffer (цикл Loop)

Методы класса BitBuffer в языке EME-L для работы с текущей «строкой» (текущим битом) позволяют использовать буфер как источник данных в цикле `Loop`. Внутренний курсор инициализируется значением -1.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает курсор на индекс 0. |
| `SetNextLine()` | — | Empty | Сдвигает курсор на следующий бит согласно режиму поиска (см. `SetLoopMode`). |
| `IsValidLine()` | — | Boolean | TRUE — курсор указывает на допустимый бит. До `SetFirstLine` — FALSE. |
| `IsValidLine(номер)` | номер: Integer | Boolean | TRUE — бит с указанным номером существует в буфере. |
| `GetLine()` | — | Integer | Текущий индекс. До `SetFirstLine` — -1. |
| `SetLine(номер)` | номер: Integer | Empty | Устанавливает курсор на указанный бит. |
| `SetLoopMode(режим)` | режим: Boolean | Empty | Настраивает режим обхода: TRUE — перебор установленных в `'1'` битов (по умолчанию), FALSE — сброшенных в `'0'`. Влияет на `SetNextLine`. |

Для перебора установленных битов класса BitBuffer в языке EME-L предпочтителен цикл `Loop (bb)` — он автоматически вызывает `SetFirstLine`, `IsValidLine` и `SetNextLine` в текущем режиме поиска. Не используйте `Object("BitBuffer")` внутри циклов — создавайте буфер до итерации.

## Загрузка и сохранение в классе BitBuffer

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `LoadLines(источник_данных)` | источник_данных: dsDB или Array (по ссылке) | Empty | Загружает строки источника в буфер: для каждой строки соответствующий бит устанавливается в `'1'`. Буфер динамически расширяется. Пустой источник при непустом буфере очищает буфер. Запрещено передавать `dsDIALOG` или `dsQUERY` — выбрасывает исключение. |
| `UnloadLines(источник_данных)` | источник_данных: dsDB или Array (по ссылке) | Empty | Выгружает строки источника: для каждой строки соответствующий бит сбрасывается в `'0'`. Расширение и запрет `dsDIALOG`/`dsQUERY` аналогичны `LoadLines`. |
| `LoadBrowserSelection(браузер)` | браузер: Browser (по ссылке) | Empty | Загружает выделение из объекта браузера, перезаписывая буфер. Размер устанавливается равным количеству строк в браузере. Выбрасывает исключение для буфера, связанного с переменной выделения. |
| `LoadQuerySelection(запрос, поле)` | запрос: Query (по ссылке), поле: String | Empty | Загружает буфер из запроса: для каждой строки запроса из `поле` извлекается ссылка на строку, и соответствующий бит устанавливается в `'1'`. Выбрасывает исключение для буфера, связанного с переменной выделения, или при неверном поле. |
| `LoadHewVariableSelection(диалог, имя_переменной)` | диалог: Dialog (по ссылке), имя_переменной: String | Empty | Метод совместимости: привязывает буфер к переменной выделения диалога. Буфер не загружается, если в переменную ничего не было записано. Выбрасывает исключение при неверном типе переменной. |
| `LoadHewVariableSelection(диалог, имя_переменной, загружать_всегда)` | диалог: Dialog, имя_переменной: String, загружать_всегда: Boolean | Empty | То же с явным флагом: TRUE — загружать всегда, FALSE — не загружать при пустой переменной. |
| `LoadFrom(запись_выделения, запись, поле, строка)` | четыре Integer | Empty | Загружает буфер из поля базы данных, перезаписывая текущее содержимое. Выбрасывает исключение для буфера, связанного с переменной выделения, или при несуществующих записи/поле/строке. |
| `SaveTo(запись, поле, строка)` | три Integer | Empty | Сохраняет содержимое буфера в поле базы данных. Выбрасывает исключение, если запись пустая или запись/поле/строка не существуют. |
| `LoadFromCache(имя_фильтра)` | имя_фильтра: String | Boolean | Загружает буфер из кэша фильтров, перезаписывая текущий. TRUE — успешно загружен, FALSE — фильтр не найден. Выбрасывает исключение для буфера, связанного с переменной выделения. |
| `AddToCache(имя_фильтра)` | имя_фильтра: String | Empty | Добавляет буфер в кэш фильтров. Существующий фильтр с тем же именем перезаписывается. |
| `AddToCache(имя_фильтра, сохранять)` | имя_фильтра: String, сохранять: Boolean | Empty | То же с флагом: TRUE — оставить фильтр в кэше при разрушении объекта, FALSE — удалить. |
| `RemoveFromCache()` | — | Empty | Удаляет все фильтры, зарегистрированные данным объектом, из кэша. |
| `RemoveFromCache(имя_фильтра)` | имя_фильтра: String | Empty | Удаляет указанный фильтр из кэша. |

Кэш фильтров класса BitBuffer (`AddToCache` / `LoadFromCache` / `RemoveFromCache`) в системе EME.WMS — глобальный механизм переиспользования буферов между вызовами. Типовой паттерн: перед вызовом, зависящим от фильтра, вызвать `AddToCache("Фильтр …")`, а после — `RemoveFromCache("Фильтр …")`.

## Примеры работы с классом BitBuffer

Создание буфера класса BitBuffer по количеству строк записи и загрузка выделения из источника данных в языке EME-L:

```EME-L
bbDocs = Object("BitBuffer", is_No_of_lines("Документы"));
'Загрузить в буфер строки, отмеченные в r_Document'
bbDocs.LoadLines(r_Document);
```

Фильтрация браузера через кэш фильтров класса BitBuffer в системе EME.WMS — буфер помещается в кэш перед операцией и удаляется после:

```EME-L
bbCells = Object("BitBuffer", is_No_of_lines("Ячейки"));
'Зарегистрировать фильтр в кэше для использования внутри MakePalletPickingUp'
bbCells.AddToCache("Фильтр по ячейкам");
result = Document.MakePalletPickingUp(false);
'Снять фильтр — обязателен, иначе он «протечёт» в следующие вызовы'
bbCells.RemoveFromCache("Фильтр по ячейкам");
```

Загрузка выделения из браузера и перебор установленных битов через `Loop` в языке EME-L:

```EME-L
bb = Object("BitBuffer");
'Размер становится равным количеству строк в Browser'
bb.LoadBrowserSelection(Browser);

Loop (bb)
    idx = bb.GetLine();
    'idx — индекс выбранной строки, обработать её...'
End Loop
```

# Класс HashDB

> Синонимы для поиска: CIHashDB HashDB класс EME-L, хеш-база данных EME-L, распределённая хеш-БД EME-L, RequestAdd RequestSet RequestFind EME-L, AddKey GetKey EME-L, SetModuleNo SetKeyLifeTime SetKeyMaxLength EME-L, SaveFiles LoadFiles .hdb EME-L, SetFirstLine SetNextLine IsValidLine HashDB, IsKeyFound IsNoError IsValueExcess IsCreateKey EME-L, GetDBError SetDBError HashDB, GetKeyError HashDB, NULL_REF HashDB, CIArray ключ EME-L, хеш-БД EME.WMS, SSCC HashDB EME.WMS, модульная адресация хеш-БД, HashDBEmulator

Класс `HashDB` в языке EME-L — это класс для работы с распределённой хеш-базой данных (хеш-БД). Класс реализован на уровне C++ как `CIHashDB` (обёртка над внутренним классом `CHashDB`) и доступен в скриптах EME-L через `Object("HashDB", ...)`. Назначение класса — предоставление интерфейса для добавления, поиска, изменения и удаления ключей в хеш-БД с поддержкой модульной адресации (до 64 модулей) и контроля срока жизни ключей.

Класс `HashDB` в языке EME-L позволяет формировать ключи из строк или из массивов параметров (объект `CIArray`), устанавливать срок жизни ключей в днях с автоматическим удалением устаревших записей, ограничивать максимальные значения числовых показателей ключа и выполнять пакетные операции над набором ключей. Пакетная модель работы: ключи накапливаются во внутреннем списке объекта через повторные вызовы `AddKey`, после чего один из трёх методов запроса (`RequestAdd`, `RequestSet`, `RequestFind`) обрабатывает весь пакет за одно обращение к серверу хеш-БД. Класс также поддерживает сериализацию данных в бинарные файлы с расширением `.hdb` и последующую загрузку.

В системе EME.WMS класс `HashDB` в языке EME-L применяется для высокопроизводительного учёта грузовых мест: SSCC, короба, PDF-коды, QR-коды, дата-матрицы разделяются по функциональным модулям (приход, остатки, агрегация), каждому модулю присваивается номер от 1 до 64, а ключам — срок жизни, по истечении которого они автоматически удаляются из хеш-БД.

## Создание объекта класса HashDB в языке EME-L

Объект класса `HashDB` создаётся через `Object()`. Доступны два конструктора, различающиеся числом аргументов.

```EME-L
'Конструктор без аргументов — адрес и порт из системных настроек модулей'
objHash = Object("HashDB");
```

Конструктор без аргументов в языке EME-L создаёт объект `HashDB` с параметрами подключения по умолчанию: адрес сервера хеш-БД читается из системной настройки модулей «Система.Хеш-БД.IP адрес», порт — из «Система.Хеш-БД.Порт».

```EME-L
'Конструктор с явным адресом сервера и портом'
objHash = Object("HashDB", "192.168.1.10", 50001);
```

Второй конструктор класса `HashDB` в языке EME-L принимает два аргумента: адрес сервера хеш-БД (строка) и порт сервера хеш-БД (целое число). Параметры подключения задаются явно и не зависят от системных настроек.

## Параметры подключения и модуля класса HashDB

Методы этой группы класса `HashDB` в языке EME-L настраивают параметры подключения к хеш-БД и параметры обработки ключей. Все они вызываются до добавления ключей и выполнения запроса. Номер модуля определяет логическое разделение ключевого пространства хеш-БД (до 64 независимых модулей), срок жизни ключа задаёт автоматическое удаление устаревших записей.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetModuleNo(arg0)` | arg0: Integer | Empty | Устанавливает номер модуля для работы с хеш-БД. Допустимый диапазон: от 1 до 64 включительно. При выходе за диапазон генерируется ошибка выполнения. |
| `SetKeyLifeTime(arg0)` | arg0: Integer | Empty | Устанавливает срок жизни ключа в днях. Срок годности ключа в хеш-БД рассчитывается как текущая дата плюс срок жизни; после истечения ключ удаляется автоматически. Значение должно быть не менее 1, иначе генерируется ошибка выполнения. |
| `SetKeyMaxLength(arg0)` | arg0: Integer | Empty | Устанавливает максимальную длину ключа. Значение должно быть не менее 1, иначе генерируется ошибка выполнения. |
| `SetCreateKey(arg0)` | arg0: Boolean | Empty | Задаёт режим добавления новых ключей в хеш-БД. `TRUE` (по умолчанию) — новые ключи создаются при отсутствии в хеш-БД, `FALSE` — новые ключи игнорируются и не записываются. |
| `SetDebugMode(arg0)` | arg0: Boolean | Empty | Включает (`TRUE`) или выключает (`FALSE`) режим диагностики. При включении диагностический лог записывается в файл `CATASTRF.LOG`. |
| `GetModuleNo()` | — | Integer | Возвращает номер модуля, установленный для текущего объекта класса `HashDB`. |
| `GetKeyLifeTime()` | — | Integer | Возвращает срок жизни ключа в днях, установленный для текущего объекта. |
| `GetKeyMaxLength()` | — | Integer | Возвращает максимальную длину ключа, установленную для текущего объекта. |
| `IsCreateKey()` | — | Boolean | Возвращает текущий режим создания ключей. `TRUE` — новые ключи добавляются, `FALSE` — игнорируются. |
| `GetDBError()` | — | String | Возвращает описание ошибки хеш-БД, возникшей при выполнении последнего запроса. Пустая строка, если ошибок не было. |
| `SetDBError(arg0)` | arg0: String | Empty | Устанавливает текстовое описание ошибки хеш-БД для объекта. |

## Добавление ключей в класс HashDB

Методы `AddKey` класса `HashDB` в языке EME-L формируют внутренний список ключей для последующей пакетной обработки. Ключ может быть задан строкой или сформирован из массива параметров (объект `CIArray`, передаваемый по ссылке). При передаче пустого массива генерируется ошибка выполнения.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddKey(arg0)` | arg0: String или Array (по ссылке) | Empty | Добавляет ключ без числового значения и ограничения. arg0 — строковый ключ либо объект `CIArray` для составного ключа. |
| `AddKey(arg0, arg1, arg2)` | arg0: String/Array, arg1: Real, arg2: Real | Empty | Добавляет ключ со значением и максимальным ограничением. arg2 (максимум) не может быть меньше arg1 (значение), иначе генерируется ошибка выполнения. |
| `AddKey(arg0, arg1, arg2[, arg3])` | + arg3: Integer | Empty | То же со ссылкой на связанный объект. arg3 по умолчанию — пустая ссылка (`NULL_REF`). |
| `AddKey(arg0, arg1, arg2[, arg3[, arg4]])` | + arg4: Integer | Empty | Полный набор параметров. arg4 — код ошибки ключа, по умолчанию 0 (нет ошибки). |

Параметр arg4 (код ошибки ключа) в методе `AddKey` класса `HashDB` в языке EME-L принимает значения: `0` — ошибок нет, `1` — ключ не найден, `2` — превышено максимальное значение, `3` — запрещено добавлять новый ключ. Эти коды используются при формировании ответа сервера.

## Пакетные запросы класса HashDB

Методы `RequestAdd`, `RequestSet`, `RequestFind` класса `HashDB` в языке EME-L выполняют операцию над всеми ключами, добавленными методом `AddKey` с момента последнего вызова `Clear`. Все три метода не принимают аргументов и возвращают `Boolean` — результат выполнения запроса (`TRUE` — успех, `FALSE` — ошибка). Текст ошибки доступен через `GetDBError`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `RequestAdd()` | — | Boolean | Выполняет запрос на добавление или вычитание значений для набора ключей. Если ключ отсутствует в хеш-БД, он создаётся и инициализируется параметрами метода `AddKey`. Если ключ уже существует, изменяется только значение; ссылка, срок годности и другие параметры не затрагиваются. |
| `RequestSet()` | — | Boolean | Выполняет запрос на установку параметров набора ключей. Если ключ уже существует, все его параметры перезаписываются, включая ссылку, срок годности и максимальное значение. |
| `RequestFind()` | — | Boolean | Выполняет запрос поиска набора ключей в хеш-БД. Результат доступен через перебор списка методами `SetFirstLine`, `SetNextLine`, `IsValidLine` и методы проверок `IsKeyFound`, `IsNoError`. |

## Перебор ключей и получение данных класса HashDB

После выполнения пакетного запроса класс `HashDB` в языке EME-L предоставляет навигационный интерфейс для перебора ключей во внутреннем списке. Навигация идёт через внутренний индекс, который устанавливается методами `SetFirstLine` (начало) и `SetNextLine` (следующий элемент). Корректность индекса проверяется методом `IsValidLine`. Перебор результата обычно выполняется через `Loop`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает внутренний индекс списка ключей на первый элемент. |
| `SetNextLine()` | — | Empty | Перемещает внутренний индекс списка ключей на следующий элемент. |
| `IsValidLine()` | — | Boolean | Проверяет, что внутренний индекс указывает на корректный ключ. `TRUE` — индекс корректен, `FALSE` — выход за границы списка. |
| `GetNoOfLines()` | — | Integer | Возвращает количество ключей в текущем списке. |
| `GetKey()` | — | String или Empty | Возвращает идентификатор текущего ключа в виде строки. При невалидном индексе возвращает `Empty`. |
| `GetKey(arg0, arg1, arg2, arg3)` | arg0: Array (по ссылке), arg1–arg3: переменные по ссылке | Empty | Записывает данные текущего ключа в переданные переменные: arg0 — массив `CIArray` для составных частей ключа, arg1 — значение ключа, arg2 — максимальное значение, arg3 — ссылка на связанный объект. При невалидном индексе переменные не изменяются. |
| `GetKeyError()` | — | Integer | Возвращает код ошибки текущего ключа, возникшей при последнем запросе. `0` — ошибок нет, `1` — ключ не найден, `2` — превышено максимальное значение, `3` — запрещено добавлять новый ключ. |

## Проверки состояния ключа класса HashDB

Методы проверок класса `HashDB` в языке EME-L возвращают результат проверки состояния текущего ключа. При невалидном внутреннем индексе (если `IsValidLine` вернула `FALSE`) все методы этой группы, кроме `IsCreateKey`, возвращают `Empty`, а не `FALSE`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsNoError()` | — | Boolean/Empty | Проверяет, что при последней операции с текущим ключом не возникло ошибок. `TRUE` — ошибок нет. |
| `IsKeyFound()` | — | Boolean/Empty | Проверяет, что текущий ключ найден в хеш-БД. `TRUE` — найден, `FALSE` — не найден. |
| `IsValueExcess()` | — | Boolean/Empty | Проверяет, было ли превышение максимального значения в операции `RequestAdd`. `TRUE` — значение превышено. |
| `IsCreateKey()` | — | Boolean | Проверяет, установлен ли режим добавления новых ключей в хеш-БД. `TRUE` — режим включён. Этот метод не зависит от внутреннего индекса. |

## Сериализация класса HashDB в файлы

Методы `SaveFiles` и `LoadFiles` класса `HashDB` в языке EME-L используются для разделения клиентской и серверной частей обмена с хеш-БД. Клиент накапливает ключи, сериализует их в файлы `.hdb`, сервер загружает файлы и применяет их к хеш-БД, вызывая емельный метод `SetValue` целевого класса (по умолчанию `HashDBEmulator`).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SaveFiles(arg0)` | arg0: String (каталог) | Empty | Сохраняет ключи в бинарные файлы `.hdb`. В один файл записывается до 100 000 ключей; при превышении ключи разбиваются на несколько файлов с автоматически сгенерированными именами. В файл записываются номер модуля, срок жизни, длина ключа, сам ключ, значение и ссылка. |
| `LoadFiles(arg0)` | arg0: String (каталог) | Empty | Загружает все файлы `.hdb` из указанного каталога последовательно. После успешной загрузки каждый файл удаляется с диска. Для каждого загруженного набора вызывается емельный метод `SetValue` класса, указанного в запросе (по умолчанию `HashDBEmulator`), для записи ключей в БД. |
| `Clear()` | — | Empty | Очищает объект: удаляет все добавленные ключи и сбрасывает внутренние параметры. Вызывается перед формированием нового пакета ключей. |

## Константы класса HashDB в языке EME-L

Класс `HashDB` в языке EME-L оперирует следующими константами и пределами: диапазон номера модуля — от 1 до 64 включительно; срок жизни ключа — целое число дней, не менее 1; максимальное число ключей в одном файле `.hdb` при сериализации — 100 000; расширение файлов сериализации — `.hdb`; файл диагностического лога при включённом режиме отладки (`SetDebugMode(TRUE)`) — `CATASTRF.LOG`; коды ошибок ключа — `0` (нет ошибок), `1` (ключ не найден), `2` (превышено максимальное значение), `3` (запрещено добавлять новый ключ); пустая ссылка на связанный объект обозначается константой `NULL_REF`.

## Примеры использования класса HashDB

Установка значений набора ключей с ограничением и сроком жизни в языке EME-L — типовой паттерн из системы EME.WMS для учёта грузовых мест:

```EME-L
objHash = Object("HashDB");
objHash.SetModuleNo(1);
objHash.SetKeyLifeTime(365);
objHash.SetKeyMaxLength(64);

objHash.AddKey("SSCC-001", 1.0, 1.0);
objHash.AddKey("SSCC-002", 1.0, 1.0);

Result = objHash.RequestSet();
If (~Result)
    LastError = objHash.GetDBError();
End If
```

Поиск ключей и перебор результата в языке EME-L — после вызова `RequestFind` результат перебирается вручную: метод `SetFirstLine` устанавливает индекс на первый ключ, цикл `While` с проверкой `IsValidLine` проходит по всем ключам, а `SetNextLine` перемещает индекс вперёд. Для каждого ключа проверяется факт его наличия в хеш-БД через `IsKeyFound`. Класс `HashDB` предоставляет навигационный интерфейс (`SetFirstLine`, `SetNextLine`, `IsValidLine`), аналогичный интерфейсу записей `CEMERec`:

```EME-L
objHash.AddKey("SSCC-001");
objHash.AddKey("SSCC-002");

Result = objHash.RequestFind();
If (Result)
    objHash.SetFirstLine();
    While (objHash.IsValidLine())
        If (objHash.IsKeyFound())
            KeyId = objHash.GetKey();
        End If
        objHash.SetNextLine();
    End While
End If
```

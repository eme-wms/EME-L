# Класс CalcStockHistory

> Синонимы для поиска: CICalcStockHistory CalcStockHistory класс EME-L, остатки на дату EME-L, история остатков EME-L, обратный пересчёт остатков EME-L, GetStock GetLastDocLineRef MarkupStockDataQueyry CalcCurrentStock CalcStockAtDate EnableChangeStockData FillStockDataQuery FillChangeStockDataQuery, сток на дату EME-L, регистры EME-L счётчики N S M P Buf, запрос остатков Query EME-L, фиксация изменений остатков EME.WMS

Класс `CalcStockHistory` в языке EME-L вычисляет остатки по складским регистрам на указанную дату и время. Остатки вычисляются методом обратного пересчёта от текущих остатков по истории движений строк документов в системе EME.WMS. Класс поддерживает получение остатков на произвольную дату в прошлом и оптимизирован для многократных вызовов с убывающими датами — повторное использование одного объекта для нескольких дат быстрее, чем создание нового объекта на каждую дату.

Объект класса `CalcStockHistory` в языке EME-L создаётся через `Object("CalcStockHistory")` без аргументов. Расчёт текущих остатков выполняется лениво — при первом вызове `GetStock`, а не в конструкторе, чтобы не увеличивать время старта программы. Конструктор параметров не принимает.

## Создание объекта класса CalcStockHistory

```EME-L
'Создать объект для расчёта истории остатков'
CalcStockHistory = Object("CalcStockHistory");
```

## Метод GetStock — остатки на дату/время

Метод `GetStock(objQuery, dateTime)` класса `CalcStockHistory` в языке EME-L вычисляет остатки по регистрам на указанные дату и время и заполняет переданный объект запроса `Query`. При первом вызове выполняет вычисление текущих остатков. Для оптимизации производительности при получении остатков на несколько дат рекомендуется использовать один объект и передавать даты в обратном хронологическом порядке от последней к первой. Если запрос непустой, он предварительно очищается.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `objQuery` | Query (по ссылке) | Да | Объект запроса, в который добавляются строки с остатками по регистрам. |
| `dateTime` | DateTime | Да | Дата и время, на которые нужно получить остатки. Время не может быть больше текущего. При многократных вызовах даты передаются в обратном хронологическом порядке. |

**Возвращает**: `DateTime` — реальную дату и время, для которой вычислены остатки в языке EME-L. Это время первой операции перед запрошенной датой/временем. Если запрошены остатки на начало времён, возвращается момент первого движения в базе.

## Метод GetLastDocLineRef — ссылка на последнюю строку

Метод `GetLastDocLineRef()` возвращает ссылку на последнюю отгруженную строку документа, соответствующую остаткам, полученным последним вызовом `GetStock` в системе EME.WMS. С помощью этой ссылки можно перебрать все строки документов, чтобы учесть движения за период между двумя вызовами `GetStock`.

**Возвращает**: `Reference` — ссылку на строку документа в языке EME-L. Возвращает `NULL_REF` (-1), если метод `GetStock` не вызывался или запрашивались остатки на начало времён.

## Метод MarkupStockDataQueyry — разметка запроса

Метод `MarkupStockDataQueyry(objQuery)` размечает объект запроса `Query`, добавляя поля для хранения данных об остатках по регистрам в системе EME.WMS. Добавляет обязательные поля: `@Ячейка`, `Количество N`, `Количество S`, `Количество M`, `Количество P`, а также дополнительные поля: `Расширенное количество`, `Расширенное количество S`, `Расширенное количество M`, `Тип документа`, `@Регистр`, `Количество Buf`, `@SSCC`, `@Организация`, `@Партия товара`, `@Товарная единица`, `BBD`, `Стоковый статус`, `@Единица измерения товара`, `@Оператор`, `@Поддон`, `@ГТД`, `@Аналитика`, `@Склад`, `@Составная паллета`, `@Сертификат`, `Строки документов`, `@Регистратор`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `objQuery` | Query (по ссылке) | Да | Объект запроса, который будет размечен под данные остатков. |

**Возвращает**: `Empty`.

Название метода `MarkupStockDataQueyry` содержит опечатку оригинала (`Queyry` вместо `Query`) — сохранено для совместимости, не исправляйте при вызове из EME-L кода.

## Метод CalcCurrentStock — текущие остатки

Метод `CalcCurrentStock()` вычисляет текущие остатки по всем регистрам и сохраняет их во внутреннем массиве объекта класса `CalcStockHistory` в языке EME-L. Вызывается автоматически при первом вызове `GetStock`. Явный вызов позволяет заранее прогреть расчёт, но обычно не требуется.

**Возвращает**: `Empty`.

## Метод CalcStockAtDate — остатки на дату (нижний уровень)

Метод `CalcStockAtDate(dateTime)` вычисляет остатки на указанные дату и время методом обратного пересчёта от текущих остатков в системе EME.WMS. Перемещается по истории движений строк документов, корректируя счётчики количества. Дата следующего вызова должна быть меньше или равна дате предыдущего вызова. Метод возвращает реальную дату/время остатков.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `dateTime` | DateTime | Да | Дата и время, на которые нужно получить остатки. Время не может быть больше текущего. При многократных вызовах даты передаются в обратном хронологическом порядке. |

**Возвращает**: `DateTime` — реальную дату и время, для которой получены остатки (время первой операции перед запрошенной датой/временем).

## Метод EnableChangeStockData — фиксация изменений остатков

Метод `EnableChangeStockData(isEnabled, isSuppressTransfer)` включает или отключает режим фиксации изменений остатков при вычислении в языке EME-L. В режиме фиксации все изменения количеств, происходящие при вызове `GetStock` или `CalcStockAtDate`, сохраняются во внутреннем массиве и могут быть получены методом `FillChangeStockDataQuery`. Второй параметр позволяет подавить фиксацию изменений при перемещениях внутри регистра.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `isEnabled` | Boolean | Да | `TRUE` (=-1) — фиксация изменений включена; `FALSE` (=0) — отключена (по умолчанию). |
| `isSuppressTransfer` | Boolean | Да | `TRUE` — изменения при перемещении подавляются; `FALSE` — изменения при перемещении фиксируются (по умолчанию). |

**Возвращает**: `Empty`.

## Метод FillStockDataQuery — заполнение запроса остатками

Метод `FillStockDataQuery(objQuery)` заполняет переданный объект запроса `Query` строками с текущими остатками по регистрам в системе EME.WMS. Перед заполнением запрос очищается от старых данных. Запрос должен быть предварительно размечен методом `MarkupStockDataQueyry` или содержать совместимый набор полей.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `objQuery` | Query (по ссылке) | Да | Объект запроса, в который добавляются строки с остатками по регистрам. |

**Возвращает**: `Empty`.

## Метод FillChangeStockDataQuery — изменения остатков

Метод `FillChangeStockDataQuery(objQuery)` заполняет переданный объект запроса `Query` строками с изменениями остатков по регистрам, зафиксированными при включённом режиме фиксации изменений (`EnableChangeStockData`) в языке EME-L. Перед заполнением запрос очищается от старых данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `objQuery` | Query (по ссылке) | Да | Объект запроса, в который добавляются строки с изменениями остатков. |

**Возвращает**: `Empty`.

## Примеры использования класса CalcStockHistory

Базовый расчёт остатков на текущую дату в языке EME-L:

```EME-L
'Разметить запрос под остатки'
queryStock = Object("Query");
queryStock.Create();
queryStock.AddREF("@Ячейка", "Ячейки");
queryStock.AddFLOAT("Количество N");
queryStock.AddFLOAT("Количество S");
queryStock.AddFLOAT("Количество M");
queryStock.AddFLOAT("Количество P");
queryStock.AddINT("Тип документа");

'Рассчитать остатки на текущую дату'
CalcStockHistory = Object("CalcStockHistory");
CalcStockHistory.GetStock(queryStock, is_date() + is_time(0, 0, 0));

'Перебрать строки результата'
Loop (queryStock)
    CellRef = queryStock.Get("@Ячейка");
    QtyN = queryStock.Get("Количество N");
End Loop
```

Ежедневные остатки за период в системе EME.WMS — даты обязательно в обратном порядке, иначе `CalcStockHistory` вернёт ошибку:

```EME-L
CalcStockHistory = Object("CalcStockHistory");
LastLine = 0;
For (Date = FinalDate; Date >= StartDate; Date = Date - 1)
    queryStock.DeleteAllLines();
    CalcStockHistory.GetStock(queryStock, Date);
    For (Lines = queryStock.GetNoOfLines(); LastLine < Lines; LastLine = LastLine + 1)
        queryStock.PutData("Дата", LastLine, Date);
    End For
End For
```

Фиксация изменений остатков между двумя датами в языке EME-L:

```EME-L
CalcStockHistory = Object("CalcStockHistory");
CalcStockHistory.MarkupStockDataQueyry(queryStock);

'Включить фиксацию изменений, подавляя фиктивные перемещения'
CalcStockHistory.EnableChangeStockData(TRUE, TRUE);

'Первый расчёт — позже, второй — раньше'
CalcStockHistory.GetStock(queryStock, EndDate + is_time(0, 0, 0));
CalcStockHistory.GetStock(queryStock, StartDate + is_time(0, 0, 0));

'Получить изменения остатков за период'
queryChanges = Object("Query");
queryChanges.Create();
CalcStockHistory.MarkupStockDataQueyry(queryChanges);
CalcStockHistory.FillChangeStockDataQuery(queryChanges);
```

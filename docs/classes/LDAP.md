# Класс LDAP — подключение к Active Directory и доменная аутентификация в EME-L

Класс `LDAP` в языке EME-L — это класс для подключения к серверу LDAP (Active Directory) и выполнения операций доменной аутентификации. Полное имя C++-реализации — `CILDAP`, регистрируется в реестре классов EME-L как `LDAP`. Класс предоставляет методы для проверки доступа к домену, управления контекстом пользователя и домена, проверки членства пользователя в группах, верификации GUID доменного пользователя и получения списков пользователей групп.

Класс `LDAP` в системе EME.WMS используется для интеграции с Active Directory в сценариях доменной аутентификации и авторизации. Объект класса создаётся через `Object("LDAP")` и инициализируется подключением к домену от имени текущего пользователя операционной системы с повышением привилегий до администратора домена.

## Создание объекта класса LDAP в языке EME-L

Объект класса `LDAP` создаётся конструктором без аргументов:

```EME-L
'Конструктор без аргументов: инициализация под текущим доменным пользователем'
objLDAP = Object("LDAP");
```

Конструктор без аргументов создаёт объект и выполняет автоматическую инициализацию подключения к LDAP в языке EME-L. Под капотом C++-реализация определяет домен и имя пользователя через системный API Windows (`GetUserName`, `GetUserNameEx` с форматом `NameFullyQualifiedDN`), затем устанавливает подключение с правами администратора домена (`LDAP_SESSION_CONTEXT::InitAdmin`).

Доменные учётные данные администратора (доменное имя и пароль) читаются из записи «Список пользователей» (поля `m_DomainAdminUsername` и `m_DomainAdminPassword`). Пароль хранится в базе данных EME.WMS в зашифрованном виде (DPAPI, флаг `CRYPTPROTECT_LOCAL_MACHINE`) и кодировке Base64. Расшифровка разрешена только в режиме службы ядра (`IsNTService()`), кроме отладочной сборки (`_DEBUG`).

Если домен или доменное имя администратора не заданы в записи «Список пользователей», в языке EME-L выполняется попытка подключения от имени текущего пользователя ОС (`LDAP_SESSION_CONTEXT::Init` без учётных данных — `LDAP_AUTH_NEGOTIATE`). Протокол подключения (`LDAP` или `LDAPS`) выбирается системной настройкой протокола доменной аутентификации (`MS_DomainAuthenticationProtocol()`): для `LDAP` вызывается `ldap_init` на порту `LDAP_PORT`, для `LDAPS` — `ldap_sslinit` на порту `LDAP_SSL_PORT` с включённым SSL.

После создания объекта в языке EME-L проверьте результат через методы `IsDomainAccess` и `IsBind` — конструктор не возбуждает исключений при неудаче инициализации, объект создаётся в любом случае.

## Состояние подключения к LDAP в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsDomainAccess()` | — | Boolean | TRUE — активное подключение к серверу LDAP установлено, FALSE — отсутствует. |
| `IsBind()` | — | Boolean | TRUE — операция bind (привязка) к серверу LDAP выполнена успешно, FALSE — нет. |

Метод `IsDomainAccess` в классе `LDAP` языка EME-L проверяет, что был создан дескриптор сессии LDAP (`ldap*` в C++, инициализация через `ldap_init` / `ldap_sslinit`). Метод `IsBind` дополнительно подтверждает успешную аутентификацию через `ldap_bind_s` с флагом `LDAP_AUTH_NEGOTIATE`.

В языке EME-L подключение без bind не даёт права выполнять поисковые запросы к каталогу Active Directory — `IsBind` следует использовать как предусловие перед вызовом `UserInGroup`, `CheckDomainUserGuid` и `GetAllUsersInGroup`.

## Контекст пользователя и домена в классе LDAP

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetDomain()` | — | String | Имя домена, сохранённое в контексте объекта. Пустая строка, если домен не задан. |
| `PutDomain(arg0)` | arg0: String (домен) | Empty | Устанавливает имя домена в контексте объекта. |
| `GetUser()` | — | String | Имя пользователя, сохранённое в контексте объекта. Пустая строка, если пользователь не задан. |
| `PutUser(arg0)` | arg0: String (пользователь) | Empty | Устанавливает имя пользователя в контексте объекта. |

Контекст пользователя и домена в классе `LDAP` языка EME-L устанавливается автоматически в конструкторе на основе данных, полученных из `GetCurrentUserDomainInfo`. Контекст используется последующими методами: `UserInGroup` передаёт домен и логин в поисковый фильтр, `CheckDomainUserGuid` сверяет GUID по домену и логину пользователя.

При необходимости контекст можно переопределить через `PutDomain` и `PutUser` в языке EME-L — например, для проверки другого пользователя в рамках той же сессии LDAP без создания нового объекта. Формат домена — полное DNS-имя в виде `wms.eme.ru`; оно автоматически преобразуется в DN поиска (`DC=wms,DC=eme,DC=ru`).

## Проверка членства пользователя в группе Active Directory

### UserInGroup — проверка членства текущего пользователя в группе LDAP

В классе `LDAP` языка EME-L метод `UserInGroup` проверяет, входит ли текущий пользователь в указанную группу каталога Active Directory.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | String | Да | Имя группы в каталоге Active Directory (значение атрибута `CN`). |

**Возвращает**: `Boolean` — TRUE, если текущий пользователь входит в указанную группу; FALSE, если не входит или произошла ошибка поиска.

Метод `UserInGroup` класса `LDAP` языка EME-L выполняет поиск по фильтру `(&(objectClass=group)(cn=<имя группы>))` в поддереве домена (`LDAP_SCOPE_SUBTREE`) через `ldap_search_s`. Затем он перебирает значения атрибута `member` группы и ищет вхождение DN текущего пользователя (его атрибута `cn`). Возврат FALSE не различает «не входит в группу» и «ошибка поиска» — в обоих случаях возвращается FALSE.

Имя группы передаётся как значение `CN` (например, `"EME_Admins"`), а не как полная строка DN.

## Верификация GUID доменного пользователя в EME-L

### CheckDomainUserGuid — сверка GUID между базой EME.WMS и Active Directory

В классе `LDAP` языка EME-L метод `CheckDomainUserGuid` проверяет соответствие GUID доменного пользователя между данными в базе EME.WMS и сервером Active Directory. При расхождении обновляет домен, доменное имя и GUID в записи пользователя базы данных.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer | Да | Ссылка на строку записи «Список пользователей» в базе EME.WMS. |

**Возвращает**: `Boolean` — TRUE, если GUID совпадает и учётная запись активна; FALSE, если GUID не совпадает, пользователь не найден или учётная запись отключена либо заблокирована.

Метод `CheckDomainUserGuid` класса `LDAP` языка EME-L считывает GUID из Active Directory (бинарный атрибут `objectGUID`, 16 байт) и сравнивает его с полем GUID записи пользователя базы EME.WMS. При расхождении метод открывает серверную транзакцию (`OpenServerTransaction` / `CloseServerTransaction` с описанием «Запись домена и GUID пользователя») и обновляет поля `m_Domain`, `m_DomainUser`, `m_GUID` в записи пользователя.

После сверки GUID в классе `LDAP` языка EME-L проверяются флаги учётной записи через атрибут `userAccountControl`: если установлены `ADS_UF_ACCOUNTDISABLE` или `ADS_UF_LOCKOUT`, метод возвращает FALSE и выставляет в записи пользователя флаг «Доступ временно запрещён» (бит `USER_ACCESS_DENIED` в поле `m_Flags`). Это блокирует вход пользователя в систему EME.WMS.

> **Поведенческий нюанс C++-реализации:** в `i_LDAP.cpp` (строки 1011-1017 vs 1037-1039) выставление `USER_ACCESS_DENIED` в поле `m_Flags` выполняется **за пределами** серверной транзакции — уже после `CloseServerTransaction` либо вообще без неё, если GUID совпал и ветка обновления GUID не выполнялась. Это асимметрия: запись `m_Domain`/`m_DomainUser`/`m_GUID` строго транзакционна, а запись `m_Flags` — нет. Кроме того, признак «пустого GUID» в БД определяется по первым двум байтам массива: `userGUID[0] && userGUID[1]` — если оба нулевые, считается, что GUID в базе не задан, и метод немедленно перезапишет его значением из Active Directory.

Поиск доменного пользователя в Active Directory выполняется по доменному имени; при неудаче — по EME-логину из поля `m_Id`. Это позволяет использовать метод в языке EME-L как для пользователей с совпадающим EME-логином и доменным именем, так и для случаев, когда они различаются.

## Получение списка пользователей группы

### GetAllUsersInGroup — список логинов всех участников группы LDAP

В классе `LDAP` языка EME-L метод `GetAllUsersInGroup` возвращает список всех пользователей, входящих в указанную группу каталога Active Directory. Результат записывается в переданный массив в виде логинов пользователей (`sAMAccountName`).

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | String | Да | Имя группы в каталоге Active Directory (значение атрибута `CN`). |
| arg1 | Array | Да | Массив для записи логинов пользователей (передаётся по ссылке). |

**Возвращает**: `Boolean` — TRUE, если список получен успешно; FALSE, если группа не найдена или произошла ошибка.

Метод `GetAllUsersInGroup` класса `LDAP` языка EME-L выполняет двухпроходный поиск в Active Directory. Сначала извлекаются значения `member` группы — это DN участников в формате `CN=...,OU=...,DC=...`. Из каждого DN выделяется компонент `CN` (логическое имя учётной записи, не путать с `sAMAccountName`). Затем для каждого `CN` выполняется обратный поиск по фильтру `(&(objectClass=user)(cn=<CN>))` для получения значения `sAMAccountName` — именно оно попадает в итоговый массив.

**Требование к версии ядра EME.WMS:** метод реализован только в версии ядра 51.11 и выше (C++-проверка `LAST_VERSION >= MAKE_VERSION(51, 11)`); в более ранних версиях метод возвращает пустое значение без выполнения поиска. Тип `arg1` в языке EME-L обязан быть объектом класса `Array` — при передаче объекта другого типа метод выводит ошибку через `smart_error` и прерывает выполнение.

## Константы из тел методов класса LDAP в EME-L

Следующие числовые значения и имена встречаются в C++-реализации класса `CILDAP` и зафиксированы как часть контракта класса `LDAP` в языке EME-L.

| Константа | Значение | Назначение |
|-----------|----------|------------|
| `GUID_SIZE` | `16` | Размер бинарного представления `objectGUID` в байтах (стандарт Active Directory). |
| `LDAP_PORT` | системная константа WinLDAP (389) | Порт для протокола `LDAP` — используется при `ldap_init`. |
| `LDAP_SSL_PORT` | системная константа WinLDAP (636) | Порт для протокола `LDAPS` — используется при `ldap_sslinit`. |
| `LDAP_VERSION3` | `3` | Версия протокола LDAP — устанавливается через `LDAP_OPT_PROTOCOL_VERSION`. |
| `LDAP_AUTH_NEGOTIATE` | системная константа WinLDAP | Метод аутентификации `ldap_bind_s` — использует SSPI для выбора наиболее подходящего механизма. |
| `CRYPTPROTECT_LOCAL_MACHINE` | флаг DPAPI | Флаг шифрования пароля администратора домена — привязка к машине, а не к пользователю. |
| `USER_ACCESS_DENIED` | битовый флаг записи пользователя | Выставляется методом `CheckDomainUserGuid` при отключённой или заблокированной учётной записи. |
| `ADS_UF_ACCOUNTDISABLE` | бит `0x0002` атрибута `userAccountControl` | Флаг отключённой учётной записи в Active Directory. |
| `ADS_UF_LOCKOUT` | бит `0x0010` атрибута `userAccountControl` | Флаг заблокированной учётной записи в Active Directory. |

## Примеры использования класса LDAP в языке EME-L

Проверка подключения и членства текущего пользователя в группе в языке EME-L:

```EME-L
'Создать объект и проверить подключение к домену'
objLDAP = Object("LDAP");
If (Not objLDAP.IsDomainAccess())
    is_catastrophic_log("Подключение к LDAP не установлено");
    Return;
End If

'Проверить членство в группе EME_Admins'
If (objLDAP.UserInGroup("EME_Admins"))
    IsAdmin = True;
End If
```

Проверка GUID пользователя в базе EME.WMS и блокировка при отключённой учётной записи:

```EME-L
objLDAP = Object("LDAP");
If (objLDAP.IsBind())
    'Ссылка на строку записи Список пользователей'
    UserRef = is_get_user_ref();
    If (Not objLDAP.CheckDomainUserGuid(UserRef))
        is_message("Внимание", "GUID не совпал или учётная запись заблокирована");
    End If
End If
```

Получение списка логинов участников группы через класс `LDAP` языка EME-L:

```EME-L
'Массив заполняется по ссылке'
arrUsers = is_array();
objLDAP = Object("LDAP");
If (objLDAP.GetAllUsersInGroup("EME_Warehouse_Operators", arrUsers))
    Loop (arrUsers)
        Login = ValOf(arrUsers);
        'Обработка логина доменного пользователя'
    End Loop
End If
```

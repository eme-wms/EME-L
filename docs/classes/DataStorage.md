# Класс DataStorage

> Синонимы для поиска: CIDataStorage DataStorage класс EME-L, бинарное хранилище EME-L, CDataStorage EME-L, Put Get FromBase64 ToBase64 Zip FromZip, сжатие zlib EME-L, Base64 EME-L, поля переменной длины EME.WMS, PutOutcomeStorage PutIncomeStorage CEMERec, упаковка контекста EME-L

Класс `DataStorage` в языке EME-L — это низкоуровневое хранилище бинарных данных для работы с произвольными полями базы данных EME. Класс DataStorage в языке EME-L предоставляет методы для записи, чтения, кодирования и сжатия бинарных данных при работе с полями переменной длины. Класс является обёрткой над внутренним объектом C++ `CDataStorage` (C++-имя `CIDataStorage`).

Поддерживает инициализацию из строк и объектов `CDataStorage`, а также запись данных из строк, массивов байт и объектов `CDataStorage`. Реализует кодирование содержимого в формат Base64 и обратное декодирование для текстового представления бинарных данных. Поддерживает сжатие и распаковку данных алгоритмом zlib для экономии места при хранении в базе данных. В системе EME.WMS используется как вспомогательный контейнер при чтении и записи полей методами семейства `CEMERec` (например, `PutOutcomeStorage` / `PutIncomeStorage`).

Объект класса `DataStorage` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс-обёртка над внутренним объектом C++ `CDataStorage`.

## Создание объекта класса DataStorage

```EME-L
objDS = Object("DataStorage");
objDS = Object("DataStorage", SourceData);
```

Конструктор без аргументов класса DataStorage в языке EME-L создаёт пустое хранилище. Форма с одним аргументом инициализирует содержимое: из другой ссылки `DataStorage`/`CDataStorage` (копирует данные) или из строки (копирует строковые байты включая завершающий нулевой байт).

| Конструктор | Аргументы | Возвращает | Описание |
|-------------|-----------|------------|----------|
| `DataStorage()` | — | DataStorage | Пустой объект хранилища. |
| `DataStorage(data)` | data: CDataStorage/DataStorage (ссылка) или String | DataStorage | Копирует содержимое: из переданного объекта-хранилища или из строки (включая завершающий нулевой байт). |

## Запись и чтение данных класса DataStorage

Методы класса DataStorage в языке EME-L для записи и чтения бинарного содержимого внутреннего хранилища.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Put(data)` | data: CDataStorage/DataStorage (ссылка), массив байт или String | Empty | Записывает данные во внутреннее хранилище, заменяя текущее содержимое. Из строки копирует данные включая завершающий нулевой байт; из массива байт копирует всё содержимое массива. При отрицательном размере массива генерируется ошибка выполнения с сообщением «Некорректный массив данных». |
| `Get()` | — | CDataStorage (ссылка) | Возвращает ссылку на внутренний объект `CDataStorage`, содержащий бинарные данные хранилища. Используется для передачи данных в методы `CEMERec.PutOutcomeStorage` / `PutIncomeStorage` и аналогичные. |

Метод `Get()` класса DataStorage в языке EME-L возвращает именно ссылку на внутренний объект, а не его копию — изменения, внесённые через возвращённую ссылку принимающей стороной (например, методом записи поля `CEMERec`), отражаются на состоянии хранилища.

## Кодирование Base64 класса DataStorage

Методы класса DataStorage в системе EME.WMS для текстового представления бинарных данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FromBase64(base64String)` | base64String: String | Empty | Декодирует текстовую строку в формате Base64 и записывает полученные бинарные данные во внутреннее хранилище, заменяя предыдущее содержимое. |
| `ToBase64()` | — | String | Кодирует содержимое внутреннего хранилища в текстовую строку формата Base64. |

Пара `ToBase64()` / `FromBase64()` класса DataStorage в языке EME-L применяется для передачи бинарных данных через текстовые каналы (HTTP-параметры, JSON, XML-текст), хранения в текстовых полях базы данных EME и логирования.

## Сжатие zlib класса DataStorage

Методы класса DataStorage в языке EME-L для сжатия и распаковки бинарных данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Zip()` | — | Integer | Сжимает данные внутреннего хранилища алгоритмом zlib. Возвращает размер данных **до** сжатия в байтах — это значение необходимо сохранить отдельно для последующей распаковки. |
| `FromZip(compressedData, uncompressedSize)` | compressedData: CDataStorage/DataStorage (ссылка), uncompressedSize: Integer | Empty | Разжимает сжатые данные из переданного объекта-хранилища и записывает результат во внутреннее хранилище. `uncompressedSize` — исходный размер в байтах (значение, ранее возвращённое `Zip()`). |

Сжатие и распаковка класса DataStorage в языке EME-L — это раздельные операции над двумя хранилищами: `Zip()` сжимает содержимое в самом объекте и возвращает исходный размер; `FromZip()` читает сжатые данные из **другого** объекта и распаковывает их в текущий. Поэтому исходный размер должен сохраняться отдельно (например, в соседнем поле базы данных EME).

### Коды ошибок zlib методов Zip / FromZip класса DataStorage в языке EME-L

Методы `Zip()` / `FromZip()` класса DataStorage в языке EME-L при любом неуспешном исходе генерируют ошибку выполнения. Внутренние коды возврата библиотеки zlib, обрабатываемые реализацией:

| Код zlib | Условие | Методы класса DataStorage в языке EME-L |
|----------|---------|----------------------------------------|
| `Z_OK` | Успех — нормальный путь, ошибка не генерируется. | `Zip`, `FromZip` |
| `Z_MEM_ERROR` | Нехватка памяти при сжатии или распаковке данных. | `Zip`, `FromZip` |
| `Z_BUF_ERROR` | Недостаточный размер выходного буфера для результата. | `Zip`, `FromZip` |
| `Z_DATA_ERROR` | Повреждённые входные сжатые данные. | `FromZip` |
| `Z_STREAM_ERROR` | Некорректный параметр уровня сжатия. | `Zip` |

## Примеры использования класса DataStorage

Запись XML-строки в бинарное поле записи через `Put` + `Get` класса DataStorage в языке EME-L:

```EME-L
ds_XML = Object("DataStorage");
ds_XML.Put(XMLText);
r_EPP.PutOutcomeStorage(ds_XML.Get());
```

Чтение бинарного поля записи и декодирование ответа из строки через класс DataStorage в системе EME.WMS:

```EME-L
ds_Answer = Object("DataStorage");
ds_Answer.Put(AnswerText);
r_EPP.PutIncomeStorage(ds_Answer.Get());
```

Полный цикл сжатия контекста класса DataStorage в языке EME-L — упаковать контекст в Base64 для хранения в текстовом поле. Размер до сжатия сохраняется в отдельной переменной:

```EME-L
If (ContextSize > 10000)
    ds = Object("DataStorage");
    ds.Put(is_WIN2UTF(ContextData));
    UncompressedSize = ds.Zip();
    ContextBase64 = ds.ToBase64();
End If
```

Обратное восстановление контекста класса DataStorage в языке EME-L — из Base64 через `FromBase64` во временное хранилище, затем `FromZip` в рабочее:

```EME-L
ds_Packed = Object("DataStorage");
ds_Packed.FromBase64(ContextBase64);

ds_Context = Object("DataStorage");
ds_Context.FromZip(ds_Packed.Get(), UncompressedSize);
DataBuffer As "Dynamic" = ds_Context.Get();
ContextData = is_UTF2WIN(DataBuffer.AsString());
```

# Класс Versions (CIVersions)

Класс `Versions` в языке EME-L предназначен для интеграции с системой контроля версий (SVN) и управления версионированием объектов проекта EME DB. В системе EME.WMS класс Versions предоставляет механизм экспорта и импорта объектов проекта: EME-L классов, запросов, диалогов, браузеров, отчётов и структуры базы данных. В языке EME-L класс Versions поддерживает операции SVN: инициализацию репозитория, коммит файлов, обновление локальных копий, переименование и удаление объектов с сохранением истории.

Основные возможности класса Versions в языке EME-L:

- Экспорт отдельных объектов (классы, запросы, диалоги, браузеры, отчёты) в SVN или в файловую систему.
- Полный экспорт проекта в указанную директорию с сохранением структуры папок.
- Импорт диалогов, браузеров и отчётов из файлов (доступен с версии ядра 51.9).
- Экспорт объектов в строку (получение текстового представления без записи в файл).
- Получение информации об авторстве и ревизии отдельных строк кода через SVN blame.
- Управление жизненным циклом объектов в репозитории: переименование и удаление.
- Автоматическое определение оптимального пути подключения к репозиторию (внутренний/внешний).

Объект класса `Versions` в языке EME-L создаётся без параметров. Автоматическая инициализация подключения к SVN выполняется при старте системы. В системе EME.WMS при необходимости можно выполнить ручную инициализацию методом `Init`. Для проверки состояния подключения используется метод `IsConnected`. Класс зарегистрирован под именем `Versions`, C++-класс — `CIVersions` (файл `i_Versions.cpp`). Доступен с версии ядра 28.1.

## Создание объекта класса Versions

```EME-L
'Создать объект для работы с версионированием в EME-L'
versions = Object("Versions");
```

Конструктор без аргументов класса Versions в языке EME-L создаёт объект в неподключённом состоянии. Автоматическая инициализация подключения к SVN выполняется при старте системы EME.WMS. Если инициализация не была выполнена (например, при загрузке проекта из монопольного режима), вызовите `Init()`.

## Проверка и инициализация подключения в EME-L

Методы класса Versions в языке EME-L для управления подключением к SVN.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `IsConnected()` | — | `Integer` (`1` — подключено, `0` — нет) | Проверяет, установлено ли подключение к SVN-репозиторию и готова ли система EME.WMS к операциям версионирования. |
| `Init()` | — | `Empty` | Выполняет ручную инициализацию подключения к SVN: устанавливает локальную рабочую директорию, автором указывает текущего оператора, определяет тип подключения и при необходимости выполняет checkout. |

## Экспорт объектов проекта в EME-L

Методы экспорта класса Versions в языке EME-L записывают текстовое представление объектов в локальную рабочую копию репозитория. В системе EME.WMS, если подключение к SVN активно, после записи выполняется cleanup, update и commit. Если подключение отсутствует, методы `ExportClass`, `ExportQuery` и `ExportStructure` возвращают строку `"Экспорт в каталог выполнен, но СВН отключен"`; метод `Export` возвращает `"Ok"` при успешном завершении.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Export(папка_выгрузки)` | `папка_выгрузки`: `String` | `String` | Выполняет полный экспорт всех объектов проекта в указанную директорию. Создаёт подпапки `EMEL`, `QUERY`, `DIALOG`, `BROWSER`, `REPORT` и файл `db_structure.txt`. Перед выгрузкой содержимое целевой директории очищается. Возвращает `"Ok"` или описание ошибки. |
| `ExportClass(класс)` | `класс`: ссылка на `EMELClass` | `String` | Экспортирует EME-L класс в SVN. Сохраняет имя, родительский класс, тестовый диалог, конструктор и методы. Обновляет дату/время модификации в записи класса. Возвращает пустую строку при успехе или описание ошибки. |
| `ExportQuery(запрос)` | `запрос`: ссылка на `Queries` | `String` | Экспортирует запрос в SVN. Сохраняет имя, родительский запрос, диалог, класс и текст запроса. |
| `ExportStructure()` | — | `String` | Экспортирует структуру базы данных в файл `db_structure.txt` в локальной рабочей копии SVN. |
| `ExportDialog(номер_строки, папка_выгрузки)` | `номер_строки`: `Integer`, `папка_выгрузки`: `String` | `String` | Выгружает шаблон диалога в файл `.dlg` в указанную директорию. Имя файла формируется по софт-имени диалога. |
| `ExportDialogToString(номер_строки, строка_выгрузки)` | `номер_строки`: `Integer`, `строка_выгрузки`: `String` (по ссылке) | `String` | Формирует текстовое представление шаблона диалога и записывает его в переданную строковую переменную. Возвращает пустую строку при успехе или описание ошибки. Второй аргумент должен передаваться по ссылке (`RefOf`). |
| `ExportBrowser(номер_строки, папка_выгрузки)` | `номер_строки`: `Integer`, `папка_выгрузки`: `String` | `String` | Выгружает шаблон браузера в файл `.brw` в указанную директорию. |
| `ExportBrowserToString(номер_строки, строка_выгрузки)` | `номер_строки`: `Integer`, `строка_выгрузки`: `String` (по ссылке) | `String` | Формирует текстовое представление шаблона браузера и записывает его в переданную строковую переменную. |
| `ExportReport(номер_строки, папка_выгрузки)` | `номер_строки`: `Integer`, `папка_выгрузки`: `String` | `String` | Выгружает шаблон отчёта в файл `.rep` в указанную директорию. |
| `ExportReportToString(номер_строки, строка_выгрузки)` | `номер_строки`: `Integer`, `строка_выгрузки`: `String` (по ссылке) | `String` | Формирует текстовое представление шаблона отчёта и записывает его в переданную строковую переменную. |

## Управление жизненным циклом объектов в SVN

Методы переименования и удаления класса Versions в языке EME-L требуют активного подключения к SVN. В системе EME.WMS при ошибке возвращают описание ошибки; при успехе — пустое значение.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `rename_class(класс, новое_имя)` | `класс`: ссылка на `EMELClass` или `String`; `новое_имя`: `String` | `Empty` или `String` | Переименовывает EME-L класс в SVN-репозитории с сохранением истории. |
| `rename_query(запрос, новое_имя)` | `запрос`: ссылка на `Queries` или `String`; `новое_имя`: `String` | `Empty` или `String` | Переименовывает запрос в SVN-репозитории с сохранением истории. |
| `delete_class(класс)` | `класс`: ссылка на `EMELClass` | `Empty` или `String` | Удаляет EME-L класс из SVN-репозитория с сохранением истории. |
| `delete_query(запрос)` | `запрос`: ссылка на `Queries` | `Empty` или `String` | Удаляет запрос из SVN-репозитория с сохранением истории. |

## Информация об авторстве (SVN blame) в EME-L

Методы blame класса Versions в языке EME-L выполняют SVN-команду `blame` для получения информации о ревизии и авторе указанной строки. В системе EME.WMS используют локальную рабочую копию репозитория. Результат зависит от того, что файл был предварительно сохранён в SVN; в противном случае строка может не быть найдена.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Blame_ClassMethod(имя_класса, номер_строки)` | `имя_класса`: `String`, `номер_строки`: `Integer` | `String` | Возвращает информацию о ревизии (автор, дата, сообщение) для указанной строки в секции методов класса. Если строка не найдена, возвращает сообщение об отсутствии данных. |
| `Blame_ClassConstructor(имя_класса, номер_строки)` | `имя_класса`: `String`, `номер_строки`: `Integer` | `String` | Возвращает информацию о ревизии для указанной строки в конструкторе класса. |
| `Blame_Query(имя_запроса, номер_строки)` | `имя_запроса`: `String`, `номер_строки`: `Integer` | `String` | Возвращает информацию о ревизии для указанной строки в тексте запроса. |

## Примеры использования класса Versions в EME-L

### Проверка подключения и ручная инициализация

```EME-L
'Проверить подключение к SVN и при необходимости инициализировать'
versions = Object("Versions");

If (versions.IsConnected() == 0)
    versions.Init();
End If

If (versions.IsConnected() == 0)
    is_message(tr("Ошибка"), tr("Подключение к SVN не установлено"), 1);
    Return;
End If
```

### Полный экспорт проекта в папку

```EME-L
'Выгрузить все объекты проекта в директорию для заливки в SVN'
objVersions = Object("Versions");
strFolder = Object("System").BrowseForFolder("d:\\db.wms\\Dat63-0\\Repositor\\");

If (strFolder == "")
    Return;
End If

Result = objVersions.Export(strFolder);
is_message(tr("Экспорт"), Result, 1);
```

### Экспорт отчёта в строку и в файл

```EME-L
'Получить текстовое представление отчёта и сохранить его в файл'
Versions = Object("Versions");
r_Reports = Object("dsDB", "ReportTemplates");
r_Reports.SetLine(0);

stDir = Object("System").GetDataDir() + "Тест выгрузки_1\\";
Object("System").CreateDir(stDir);

str = "";
fname = is_correct_filename(r_Reports.GetName() + ".rep");
res = Versions.ExportReportToString(r_Reports.GetLine(), RefOf str);

objFile = Object("File", stDir + fname);
objFile.PutData(str);

'Также можно выгрузить сразу в файл другой папкой'
stDir2 = Object("System").GetDataDir() + "Тест выгрузки_2\\";
Object("System").CreateDir(stDir2);
res = Versions.ExportReport(r_Reports.GetLine(), stDir2);
```

### Импорт диалога из файла

```EME-L
'Импортировать шаблон диалога ранее выгруженного методом ExportDialog'
versions = Object("Versions");
res = versions.ImportDialog("d:\\db.wms\\Repositor\\DIALOG\\MyDialog.dlg");

If (res != "")
    is_message(tr("Ошибка импорта"), res, 1);
End If
```

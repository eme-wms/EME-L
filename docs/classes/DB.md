# Класс DB

> Синонимы для поиска: CIDB DB класс EME-L, структура базы данных EME-L, модификация схемы EME-L, OpenSchema CloseSchema SetRecord CheckSchema GetSchema SchemaToFile, AddDouble AddLong AddShort AddDate AddTime AddMoney AddVVR AddOwner AddLineId AddDivisionNo, AddUnknown AddText AddFlags AddArray, AddRef AddFirstRef AddBackRef AddRefArray AddSort AddChainSort AddChain, SetExport ClrExport SetGlobal ClrGlobal SetRecordFlags SetFieldFlags, OpenMenu LoadUserMenu CopyMenuFrom CloseMenu AddDialog AddModule MenuToFile DeleteUserMenu, пользовательское меню EME-L, флаги записи EME-L SYSTEM_RECORD_BIT TAKE_PART_IN_VVR_BIT

Класс `DB` в языке EME-L — это инструментарий для программной модификации структуры базы данных и управления пользовательским меню системы EME. Класс DB в языке EME-L предоставляет методы для создания и изменения записей, добавления полей различных типов (числа, строки, даты, ссылки, массивы), настройки сортировочных путей и цепочек между записями. Поддерживает управление флагами записей и полей (экспорт, глобальность, разделение), а также получение программного представления схемы записей и выгрузку структуры в файл.

Класс DB в системе EME.WMS включает средства редактирования пользовательского меню: добавление диалогов и модулей, копирование меню между операторами, загрузку и сохранение меню в файлы. Используется в скриптах установки и обновления конфигурации системы EME для автоматизации изменений метаданных БД и настройки пользовательского интерфейса.

Объект класса `DB` в языке EME-L создаётся через `Object("DB")` без аргументов. Все операции со структурой БД выполняются в контексте открытой схемы (`OpenSchema` … `CloseSchema`) и текущей записи (`SetRecord`). Операции с меню требуют предварительного вызова `OpenMenu` или `LoadUserMenu`.

## Создание объекта класса DB

```EME-L
db = Object("DB");
```

Конструктор без аргументов создаёт объект инструментария DB в языке EME-L. Все параметры задаются последующими вызовами методов.

## Управление схемой БД класса DB

Открытие и закрытие схемы обрамляют блок модификации структуры записей и полей. Между `OpenSchema` и `CloseSchema` класса DB в языке EME-L выполняются вызовы `SetRecord`, методов добавления полей и настройки флагов. Метод `CheckSchema` проверяет, соответствуют ли накопленные временные изменения фактической структуре БД.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `OpenSchema()` | — | Empty | Открывает схему БД для редактирования структуры. Инициализирует внутренние карты записей и полей, подготавливает объект к операциям добавления/модификации. |
| `CloseSchema()` | — | Empty | Закрывает схему и сохраняет все накопленные изменения структуры в БД. |
| `CheckSchema()` | — | Integer | Проверяет соответствие накопленных временных изменений фактической структуре БД по имени, атрибуту и размеру. 1 — всё соответствует, 0 — обнаружено несоответствие. |
| `GetSchema(arg0)` | arg0: String (имя записи) | String | Возвращает программное представление структуры указанной записи в виде кода на EME-L (вызовы `SetRecord`, `AddLong`, `AddText` и др.). Пустая строка, если запись не найдена. |
| `SchemaToFile(arg0)` | arg0: String (имя файла) | Empty | Выгружает структуру всех записей БД в текстовый файл в виде EME-L кода. |
| `SchemaToFile(arg0, arg1)` | arg0: String (файл), arg1: String (запись) | Empty | Выгружает структуру указанной записи в файл. |

## Установка текущей записи класса DB

После `SetRecord` класса DB в языке EME-L все последующие методы добавления полей (`AddLong`, `AddText`, `AddRef` и т. д.) применяются к выбранной записи. Если запись не существует, она создаётся.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetRecord(arg0, arg1)` | arg0: String (имя записи), arg1: String (класс записи) | Empty | Устанавливает текущую запись. Если запись отсутствует — создаёт новую. Существующие флаги записи сохраняются. |
| `SetRecord(arg0, arg1, arg2)` | arg0: String (имя), arg1: String (класс), arg2: Integer (флаги) | Empty | То же с явной битовой маской флагов записи. Если запись существует, обновляет флаги. |
| `SetRecordFlags(arg0, arg1)` | arg0: String (имя записи), arg1: Integer (флаги) | Empty | Устанавливает флаги существующей записи без изменения текущей записи объекта. |
| `SetFieldFlags(arg0, arg1[, arg2...])` | arg0: String (запись), arg1: Integer (флаги), arg2+: String (имена полей) | Empty | Устанавливает флаги для одного или нескольких полей указанной записи. Текущая запись объекта не изменяется. Несуществующие поля пропускаются. |

### Флаги записей класса DB

Битовая маска `arg2` метода `SetRecord` / `SetRecordFlags` класса DB в языке EME-L:

| Константа | Значение | Описание |
|-----------|----------|----------|
| `SYSTEM_RECORD_BIT` | 0x00000001 | Системная запись, участвующая в системном экспорте. |
| `TAKE_PART_IN_VVR_BIT` | 0x00000010 | На запись могут ссылаться VVR-поля. |
| `RECORD_NOT_IN_USE_BIT` | 0x00000020 | Запись не используется. |
| `ALLOW_DELETE_AFTER_EXPORT_BIT` | 0x00000200 | Разрешает удалять после экспорта. |
| `ALLOW_REANIMATE_LINES_BIT` | 0x00000800 | Позволяет реанимировать удалённые строки вместо создания новых. |

### Флаги полей класса DB

Битовая маска `arg1` метода `SetFieldFlags` класса DB в языке EME-L:

| Константа | Описание |
|-----------|----------|
| `FIELD_NOT_IN_USE_BIT` | Поле не используется. |
| `SECURED_BY_CRC_BIT` | Защита поля контрольной суммой. |
| `DONT_CYPHER_THIS_FIELD_BIT` | Не шифровать поле. |
| `DONT_IMPORT_BIT` | Не импортировать поле. |
| `DONT_EXPORT_BIT` | Не экспортировать поле. |
| `DONT_CHECK_OWNERSHIP_BIT` | Не проверять владельца поля. |
| `DONT_COPY_TO_SERVER_BIT` | Не копировать поле на сервер. |
| `VARIABLE_LENGTH_FIELD_BIT` | Поле переменной длины. |

## Режимы добавления полей класса DB

Методы `SetExport`/`ClrExport` и `SetGlobal`/`ClrGlobal` класса DB в языке EME-L управляют флагами, которые будут применены ко всем последующим добавляемым полям. Режим сохраняется до следующего вызова соответствующего метода.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetExport()` | — | Empty | Сбрасывает `DONT_IMPORT_BIT`, `DONT_EXPORT_BIT`, `DONT_CHECK_OWNERSHIP_BIT` — поля добавляются как экспортируемые. |
| `ClrExport()` | — | Empty | Устанавливает те же флаги — поля добавляются как неэкспортируемые (глобальные, но не участвующие в обмене). |
| `SetGlobal()` | — | Empty | Сбрасывает `DONT_COPY_TO_SERVER_BIT` — поля копируются на сервер. |
| `ClrGlobal()` | — | Empty | Устанавливает `DONT_COPY_TO_SERVER_BIT` — поля локальные, не копируются на сервер. |

Типичная последовательность при работе с классом DB в языке EME-L: экспортируемые поля добавляются первыми, затем `ClrExport()` переводит объект в режим глобальных полей, а `ClrGlobal()` — в режим локальных.

## Добавление типизированных полей класса DB

Методы класса DB в языке EME-L добавляют поле заданного типа в текущую запись. Работают только между `OpenSchema` и `CloseSchema` после вызова `SetRecord`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddDouble(arg0)` | arg0: String (имя поля) | Empty | Поле числа с плавающей точкой (double). |
| `AddLong(arg0)` | arg0: String (имя поля) | Empty | Поле целого числа (4 байта, long). |
| `AddShort(arg0)` | arg0: String (имя поля) | Empty | Поле короткого целого (2 байта, short). |
| `AddDate(arg0)` | arg0: String (имя поля) | Empty | Поле даты. |
| `AddTime(arg0)` | arg0: String (имя поля) | Empty | Поле времени. |
| `AddMoney(arg0)` | arg0: String (имя поля) | Empty | Поле денежного типа. |
| `AddVVR(arg0)` | arg0: String (имя поля) | Empty | Поле ссылки с переменным вектором — ссылка на записи разных типов без жёсткой привязки к конкретной записи. |
| `AddOwner(arg0)` | arg0: String (имя поля) | Empty | Поле маркера собственника (владельца строки). Версия ядра ниже 50.0. |
| `AddOwner()` | — | Empty | Служебное поле маркера собственника с фиксированным именем. Версия ядра 50.0 и выше. |
| `AddLineId()` | — | Empty | Служебное поле идентификатора строки — позволяет экспортировать строку записи в другой раздел при репликации. |
| `AddDivisionNo()` | — | Empty | Служебное поле номера раздела — позволяет изменить номер раздела-владельца строки в записях мастер-данных. |

## Сортировка и цепочки класса DB

Сортировочный путь класса DB в языке EME-L — строка с полями, разделёнными `#`. Для ссылки на поле другой записи используется формат `ИмяЗаписи$ИмяПоля`. Специальные значения: `<999>` — поле 999, `<998>` — поле 998, `<997>` — поле 997 текущей записи.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddSort(arg0, arg1)` | arg0: String (имя поля), arg1: String (сортировочный путь) | Empty | Поле с атрибутом сортировки по указанному пути. |
| `AddChainSort(arg0, arg1)` | arg0: String (имя поля), arg1: String (сортировочный путь) | Empty | Поле с атрибутом сортировки цепочки. Управляет порядком строк в цепочках между родительской и дочерней записями. |
| `AddChain(arg0, arg1, arg2, arg3, arg4, arg5)` | arg0: String (дочерняя запись), arg1: String (класс дочерней записи), arg2: String (поле первой ссылки), arg3: String (поле обратной ссылки), arg4: String (поле сортировки цепочки), arg5: String (дополнительный путь) | Empty | Создаёт цепочку между текущей (родительской) записью и указанной дочерней записью. Автоматически добавляет необходимые ссылочные поля и сортировку цепочки в обе записи. |

## Управление пользовательским меню класса DB

Методы класса DB в системе EME.WMS для управления пользовательским меню:

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `OpenMenu()` | — | Empty | Открывает меню администратора (пользователь 0). |
| `OpenMenu(arg0)` | arg0: Integer (ссылка на оператора) | Empty | Открывает меню указанного оператора. |
| `LoadUserMenu()` | — | Empty | Загружает меню оператора, заданного предыдущим `OpenMenu`, для добавления новых пунктов без удаления существующих. |
| `LoadUserMenu(arg0)` | arg0: Integer (ссылка на оператора) | Empty | То же для указанного оператора. |
| `CopyMenuFrom(arg0)` | arg0: Integer (ссылка на оператора) | Empty | Копирует меню указанного оператора в меню текущего пользователя. Перед копированием удаляет существующее меню текущего пользователя. |
| `CloseMenu()` | — | Empty | Завершает редактирование меню и сохраняет изменения в БД. |
| `AddDialog(arg0, arg1[, arg2[, arg3]])` | arg0: String (пункт меню), arg1: String (имя диалога), arg2: Integer (код запуска, по умолчанию 0), arg3: Integer (флаг запуска, по умолчанию 0) | Empty | Добавляет диалог в конфигурацию системы и пункт меню текущего пользователя. |
| `AddModule(arg0, arg1, arg2, arg3)` | arg0: String (пункт меню), arg1: String (имя модуля без .dll), arg2: Integer (код запуска), arg3: Integer (флаг запуска) | Empty | Добавляет модуль (DLL) в конфигурацию системы и пункт меню. |
| `MenuToFile(arg0, arg1)` | arg0: Integer (ссылка на оператора), arg1: String (имя файла) | Empty | Выгружает меню указанного оператора в текстовый файл в виде EME-L кода. |
| `DeleteUserMenu()` | — | Integer | Удаляет меню текущего пользователя из БД. Возвращает количество удалённых пунктов. |

## Примеры

Добавление записи с полями различных типов и цепочкой методом AddChain класса DB в языке EME-L:

```EME-L
db = Object("DB");
db.OpenSchema();

db.SetRecord("Классификатор транзакций", "TransactionClassifier");
db.AddUnknown("Ключи", 8);
db.AddUnknown("Флаги", 4);
db.AddRef("@Оператор", "Список пользователей");
db.AddDate("Дата");
db.AddTime("Время");
db.AddText("Наименование", 0);
db.AddLong("Код транзакции");
db.AddRef("Рекурсия", "Классификатор транзакций");
db.AddSort("Сортировка-рек", "Рекурсия#Код транзакции");
db.AddFirstRef("@Операции по транзакции", "Операции по транзакции", "@Заголовок");
db.ClrExport();
db.AddFlags("Параметры поиска", 4);
db.ClrGlobal();

db.CloseSchema();
```

Установка флага записи `TAKE_PART_IN_VVR_BIT` (значение 16) у существующей записи через SetRecordFlags класса DB в языке EME-L без изменения её полей:

```EME-L
DB = Object("DB");
DB.OpenSchema();
If (is_try_find_record(RecordName) != NULL_RECORD)
    DB.SetRecordFlags(RecordName, is_or(is_get_record_flags(RecordName), 16));
End If
DB.CloseSchema();
```

Выгрузка всей схемы БД в файл методом SchemaToFile класса DB в языке EME-L:

```EME-L
db = Object("DB");
db.OpenSchema();
db.SchemaToFile("d:\\EME.txt");
db.CloseSchema();
```

Добавление диалога в меню оператора методами OpenMenu и AddDialog класса DB в языке EME-L:

```EME-L
dbMenuEditor = Object("DB");
dbMenuEditor.OpenMenu(UserRef);
dbMenuEditor.LoadUserMenu();
dbMenuEditor.AddDialog("Склад.Приход", "ReceiptDialog", 0);
dbMenuEditor.CloseMenu();
```

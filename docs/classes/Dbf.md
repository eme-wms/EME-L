# Класс Dbf

> Синонимы для поиска: CIDbf Dbf класс EME-L, dBASE III DBF EME-L, работа с DBF файлами EME-L, Open Close CreateFile CreateField CreateFileFromQuery AppendLine Put Get DeleteLine RestoreLine SetLine SetFirstLine SetLastLine SetNextLine SetPrevLine GetLine IsValidLine IsDeleted IsOpen IsExists GetNoOfLines GetNoOfFields GetFieldName GetFieldPos GetFieldType GetFieldLength GetFieldDec SetViewDeleted SetCodePage SetTruncateMode GetLastModified, Query экспорт DBF EME-L, кодовая страница OEM ANSI DBF EME-L

Класс `Dbf` в языке EME-L инкапсулирует чтение и запись файлов формата dBASE III (DBF) без memo-полей. Класс Dbf в системе EME.WMS предоставляет методы для создания файла с заданной структурой полей, открытия существующего файла, навигации по записям, чтения и записи значений полей, пометки строк на удаление, управления кодовой страницей и экспорта данных из объекта `Query`.

Объект класса `Dbf` в языке EME-L создаётся через `Object("Dbf")` без аргументов и не привязан к контексту диалога или браузера — это самостоятельный файловый класс для работы с DBF. При уничтожении объекта открытый файл автоматически закрывается с сохранением изменений.

## Создание объекта класса Dbf

```EME-L
'Создать объект для работы с DBF-файлом в языке EME-L'
dbf = Object("Dbf");
```

Конструктор без аргументов создаёт объект класса Dbf в языке EME-L в закрытом состоянии. Для начала работы вызовите метод `Open` (открыть существующий файл) или методы `CreateField` + `CreateFile` (создать новый файл).

## Открытие и закрытие файла

Методы класса Dbf в языке EME-L для открытия существующих DBF-файлов и управления жизненным циклом файлового дескриптора.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Open(arg0)` | arg0: String (имя файла) | Integer | Открывает существующий DBF-файл. 0 — успех, 1 — файл не найден или нет доступа, 2 — неверная структура файла. |
| `Close()` | — | Empty | Закрывает файл с автоматическим сохранением изменений. После закрытия объект можно использовать для другого файла. |
| `IsOpen()` | — | Boolean | TRUE — файл открыт, FALSE — закрыт. |
| `IsExists(arg0)` | arg0: String (имя файла) | Boolean | Статический метод: TRUE — файл существует в файловой системе, FALSE — не найден. |

## Создание нового файла

Структура полей файла класса Dbf в языке EME-L задаётся методами `CreateField` до вызова `CreateFile`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CreateField(arg0, arg1, arg2, arg3)` | arg0: String (имя, ≤ 10 символов), arg1: String (тип: `"C"`, `"N"`, `"D"`, `"L"`), arg2: Integer (длина, 1–255), arg3: Integer (знаки после запятой) | Empty | Добавляет поле в структуру создаваемого файла. Для типа `"L"` длина всегда 1, для `"D"` — всегда 8. Нельзя вызывать при открытом файле. |
| `CreateFile(arg0)` | arg0: String (имя файла) | Integer | Создаёт новый DBF-файл с ранее заданной структурой полей и открывает его. 0 — успех. |
| `CreateFileFromQuery(arg0, arg1)` | arg0: String (имя файла), arg1: Query (по ссылке) | Integer | Создаёт DBF-файл из структуры и данных объекта `Query`, записывает все строки и открывает файл. Поля с национальными символами в имени заменяются на `FIELD0000`, `FIELD0001` и т.д. Поля запроса с неопределённой длиной получают длину 20 символов. Числовые поля (FLOAT/MONEY) создаются как `N(20,8)`, целочисленные и ссылочные (INT/REFERENCE) — как `N(10,0)`. При экспорте truncate mode отключается принудительно, поэтому данные, превышающие длину поля, обрезаются без ошибки. 0 — успех, 1 — некорректный запрос. |

Типы полей DBF в языке EME-L:

| Код | Тип EME-L | Описание |
|-----|-----------|----------|
| `"C"` | String | Символьное поле. Длина 1–255. |
| `"N"` | Real / Integer | Числовое поле. Разрядность не может превышать длину минус 2. |
| `"D"` | Date | Поле даты. Длина всегда 8. |
| `"L"` | Boolean | Логическое поле в языке EME-L. Длина всегда 1. При чтении значения `Y`/`y`/`T`/`t` трактуются как TRUE; при записи сохраняется `T` (TRUE) или `F` (FALSE). |

## Навигация по записям

Методы навигации класса Dbf в языке EME-L перемещают указатель текущей строки. Удалённые строки по умолчанию пропускаются, если не включён режим `SetViewDeleted`. Перед чтением данных всегда проверяйте `IsValidLine`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFirstLine()` | — | Empty | Устанавливает текущую строку на первую неудалённую запись. |
| `SetLastLine()` | — | Empty | Устанавливает текущую строку на последнюю неудалённую запись. |
| `SetNextLine()` | — | Empty | Переходит к следующей неудалённой записи. Если такой нет — позиция сбрасывается. Используется с оператором `Loop`. |
| `SetPrevLine()` | — | Empty | Переходит к предыдущей неудалённой записи. |
| `SetLine(arg0)` | arg0: Integer (номер строки, с 0) | Empty | Устанавливает указатель на заданную строку. Удалённая строка сбрасывает позицию при скрытом режиме. |
| `GetLine()` | — | Integer | Номер текущей строки (с 0) или `NULL_REF` (-1), если строка не установлена. |
| `IsValidLine()` | — | Boolean | TRUE — текущая строка корректна и не является маркером новой строки. |
| `IsDeleted()` | — | Boolean | TRUE — текущая строка помечена на удаление. |
| `GetNoOfLines()` | — | Integer | Общее количество строк в файле, включая удалённые. |
| `GetNoOfFields()` | — | Integer | Количество полей в структуре файла. |

## Чтение и запись значений

Методы класса Dbf в языке EME-L для доступа к данным текущей строки. Физическая запись в файл выполняется при переходе к другой строке или при закрытии файла.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Get(arg0)` | arg0: String (имя поля) или Integer (номер поля) | String / Real / Date / Boolean | Значение поля текущей строки. Тип результата зависит от типа поля. |
| `Put(arg0, arg1)` | arg0: String/Integer (поле), arg1: значение (тип зависит от поля) | Empty | Записывает значение в поле текущей строки. Физическая запись выполняется при переходе к другой строке или при закрытии. |
| `AppendLine()` | — | Empty | Добавляет новую пустую строку в конец файла. Данные заносятся методом `Put`. |
| `DeleteLine()` | — | Empty | Помечает текущую строку на удаление (символ `*`). Физического удаления не происходит. |
| `RestoreLine()` | — | Empty | Снимает пометку удаления с текущей строки. |

## Метаданные полей

Методы класса Dbf в языке EME-L возвращают информацию о структуре полей открытого DBF-файла.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetFieldName(arg0)` | arg0: Integer (номер поля, с 0) | String | Имя поля (до 10 символов). |
| `GetFieldPos(arg0)` | arg0: String (имя поля) | Integer | Порядковый номер поля (с 0). |
| `GetFieldType(arg0)` | arg0: String/Integer (поле) | String | Тип поля: `"C"`, `"N"`, `"D"` или `"L"`. |
| `GetFieldLength(arg0)` | arg0: String/Integer (поле) | Integer | Длина поля в байтах (максимум 255). |
| `GetFieldDec(arg0)` | arg0: String/Integer (поле) | Integer | Количество знаков после запятой (0 для нечисловых полей). |

## Настройки и метаданные файла

Методы класса Dbf в языке EME-L управляют режимами кодировки, отображения удалённых строк и поведения при превышении длины строкового поля.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetViewDeleted(arg0)` | arg0: Boolean | Boolean | Включает (`TRUE`) или отключает (`FALSE`) режим показа удалённых строк. Возвращает предыдущее значение режима. |
| `SetCodePage(arg0)` | arg0: String (`"OEM"`, `"ANSI"` или `""`) | Empty | Устанавливает преобразование кодовой страницы для строковых полей. `""` — без преобразования (по умолчанию). |
| `SetTruncateMode(arg0, arg1)` | arg0: String/Integer (поле), arg1: Boolean | Empty | Включает (`TRUE`, по умолчанию) генерацию ошибки при превышении длины строкового поля или отключает (`FALSE`) для автоматического обрезания. |
| `GetLastModified()` | — | Date | Дата последней модификации из заголовка файла. При отсутствии — 01.01.1900. |

## Примеры

Создание нового DBF-файла со структурой полей и заполнение строк — метод `CreateField` класса Dbf в языке EME-L задаёт структуру, `AppendLine` добавляет строку, `Put` записывает значение:

```EME-L
'Создать объект и задать структуру полей файла'
dbf = Object("Dbf");
dbf.SetCodePage("ANSI");
dbf.CreateField("Order",   "C", 50, 0);
dbf.CreateField("Comment", "C", 50, 0);
dbf.CreateFile("D:\\Export.dbf");

'Добавить строки с данными'
dbf.AppendLine();
dbf.Put("Order",   "0001");
dbf.Put("Comment", "Первая поставка");

dbf.AppendLine();
dbf.Put("Order",   "0002");
dbf.Put("Comment", "Вторая поставка");

dbf.Close();
```

Чтение существующего DBF-файла с обходом строк методом `SetFirstLine` / `SetNextLine` класса Dbf в языке EME-L:

```EME-L
'Открыть файл и настроить кодовую страницу'
dbf = Object("Dbf");
dbf.Open("D:\\Export.dbf");
dbf.SetCodePage("ANSI");

'Обойти все неудалённые строки'
For (dbf.SetFirstLine(); dbf.IsValidLine(); dbf.SetNextLine())
    Order   = dbf.Get("Order");
    Comment = dbf.Get("Comment");
End For

dbf.Close();
```

Экспорт данных из объекта `Query` в DBF-файл одним вызовом `CreateFileFromQuery` класса Dbf в языке EME-L:

```EME-L
'Создать запрос и выполнить экспорт в DBF'
q = Object("Query");
q.AddString("Name");
q.AddString("Code");
'... заполнение запроса данными ...'

dbf = Object("Dbf");
dbf.CreateFileFromQuery("D:\\QueryExport.dbf", q);
dbf.Close();
```

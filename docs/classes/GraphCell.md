### Класс GraphCell — графическая ячейка отчёта в языке EME-L

Класс `GraphCell` (C++ класс `CIGraphCell`) в языке EME-L предназначен для программного создания и размещения графических элементов в графической ячейке отчёта. Класс предоставляет механизм формирования визуального содержимого графических ячеек отчётов путём создания примитивов (прямоугольники, многоугольники, прямые диагональные кресты), диаграмм, временных рядов, столбчатых диаграмм, а также размещения изображений из файлов или бинарных данных. Поддерживает заполнение графика данными из запроса базы данных и управление режимом масштабирования.

В системе EME.WMS объект `GraphCell` создаётся в контексте выполнения отчёта и привязывается к текущей графической ячейке. Класс используется в прикладных скриптах EME-L для программного формирования графического содержимого отчётных форм — как правило, внутри обработчика `GraphCell_OnFillGraphWindow`.

Основные возможности класса `GraphCell` в языке EME-L:

- Создание базовых графических примитивов: прямоугольники, многоугольники, прямые диагональные кресты.
- Создание диаграмм с возвратом объекта `Diagram` для детальной настройки осей, легенды и внешнего вида.
- Создание временных рядов, столбчатых диаграмм и треугольных диаграмм.
- Размещение изображений из файлов или объектов `CIDataStorage` с поддержкой масштабирования и ограничений размеров.
- Заполнение графика данными на основе запроса к базе данных.
- Управление режимом растягивания изображения по всей ячейке независимо от пропорций.

### Создание объекта GraphCell в языке EME-L

Класс `GraphCell` имеет один конструктор — без аргументов. Объект создаётся в контексте графической ячейки отчёта и не принимает параметров.

```EME-L
'Создание объекта, привязанного к текущей графической ячейке отчёта'
GC = Object("GraphCell");
```

Конструктор извлекает графическое окно ячейки из контекста выполнения отчёта. Если контекст не задан (например, объект создан вне обработчика отчёта), генерируется ошибка.

### Графические примитивы в языке EME-L

Методы создают базовые геометрические элементы в ячейке и возвращают ссылку на созданный объект. Все координаты и размеры задаются в пикселях целыми числами.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CreateRectangle(arg0, arg1, arg2, arg3)` | arg0: Integer (x), arg1: Integer (y), arg2: Integer (cx), arg3: Integer (cy) | Object | Прямоугольник в графической ячейке отчёта. `x`, `y` — координаты левого верхнего угла; `cx`, `cy` — ширина и высота в пикселях. |
| `CreatePolygon(arg0, arg1)` | arg0: Integer (x), arg1: Integer (y) | Object | Многоугольник в графической ячейке отчёта. `x`, `y` — координаты опорной точки. |
| `CreateCross(arg0, arg1, arg2, arg3)` | arg0: Integer (x), arg1: Integer (y), arg2: Integer (cx), arg3: Integer (cy) | Object | Прямой диагональный крест в графической ячейке отчёта. `x`, `y` — координаты; `cx`, `cy` — ширина и высота обрамляющего прямоугольника в пикселях. |

### Диаграммы и графики в системе EME.WMS

Методы создают специализированные графические объекты для визуализации данных в графической ячейке отчёта. Доступность методов зависит от версии графического окна (`GRAPH_WINDOW_VERSION`).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CreateDiagram(arg0, arg1, arg2, arg3)` | arg0: Integer (x), arg1: Integer (y), arg2: Integer (cx), arg3: Integer (cy) | Object (Diagram) | Объект диаграммы в графической ячейке отчёта. Возвращает объект `Diagram` для детальной настройки осей, легенды и внешнего вида. `x`, `y` — координаты; `cx`, `cy` — ширина и высота обрамляющего прямоугольника в пикселях. |
| `CreateTimeTable()` | — | Object | Объект временного ряда (график с временными рядами) в графической ячейке отчёта. |
| `Create3Diagram()` | — | Object | Объект треугольной диаграммы в графической ячейке отчёта. Доступен при `GRAPH_WINDOW_VERSION > 26`. |
| `CreateHistogramD()` | — | Object | Объект столбчатой диаграммы в графической ячейке отчёта. Доступен при `GRAPH_WINDOW_VERSION > 24`. |
| `CreateHistogramKrostD()` | — | Object | Объект столбчатой диаграммы типа Krost в графической ячейке отчёта. Доступен при `GRAPH_WINDOW_VERSION >= 31`. |
| `FillGraphQuery(arg0, arg1, arg2, arg3, arg4)` | arg0: Object/String (запрос), arg1–arg4: String/Integer (фильтры) | Object или Empty | Заполняет график данными на основе запроса к базе данных в системе EME.WMS. `arg0` — объект запроса или имя запроса для выборки данных графика; `arg1`–`arg4` — параметры фильтрации и конфигурации графика. Возвращает объект графика с заполненными данными или `Empty` при ошибке. |

### Размещение изображений методом CreatePicture в языке EME-L

Метод `CreatePicture` создаёт прямоугольный графический элемент и заполняет его изображением из файла или бинарных данных. Поддерживает масштабирование и ограничение максимальных размеров изображения. Первый аргумент обязателен, остальные необязательны.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CreatePicture(arg0)` | arg0: String (путь к файлу) или Object (CIDataStorage/CDataStorage) | Object или NULL | Прямоугольник с изображением в графической ячейке отчёта. Минимум один параметр — источник изображения. |
| `CreatePicture(arg0, arg1)` | + arg1: Integer (x) | Object или NULL | То же с координатой X. По умолчанию 0. |
| `CreatePicture(arg0, arg1, arg2)` | + arg2: Integer (y) | Object или NULL | То же с координатой Y. По умолчанию 0. |
| `CreatePicture(arg0, arg1, arg2, arg3)` | + arg3: Real (scale) | Object или NULL | То же с коэффициентом масштабирования. `1.0` — оригинальный размер. По умолчанию `1.0`. |
| `CreatePicture(arg0, arg1, arg2, arg3, arg4)` | + arg4: Integer (maxwidth) | Object или NULL | То же с ограничением максимальной ширины. При превышении изображение пропорционально уменьшается. По умолчанию 0 (без ограничения). |
| `CreatePicture(arg0, arg1, arg2, arg3, arg4, arg5)` | + arg5: Integer (maxheight) | Object или NULL | То же с ограничением максимальной высоты. При превышении изображение пропорционально уменьшается. По умолчанию 0 (без ограничения). |
| `CreatePicture(arg0, arg1, arg2, arg3, arg4, arg5, arg6)` | + arg6: String (format) | Object или NULL | То же с явным форматом изображения при загрузке из бинарных данных. Поддерживаемые значения: `"png"`, `"jpg"`, `"bmp"`, `"gif"`, `"ico"`. Используется только при передаче объекта `CIDataStorage`/`CDataStorage`. По умолчанию определяется автоматически. |

При передаче имени файла метод проверяет существование файла и загружает изображение. При передаче бинарных данных (объект `CIDataStorage` или `CDataStorage`) метод декодирует их; параметр `format` позволяет явно указать формат декодирования. При ошибке загрузки возвращается `NULL`.

### Настройка отображения в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetAnisotropicMode()` | — | Object (GraphCell) | Устанавливает режим растягивания изображения по всей графической ячейке отчёта, независимо от соотношения сторон исходного содержимого. Возвращает текущий объект `GraphCell` — допускает цепочку вызовов. |

### Примеры использования GraphCell в языке EME-L

Размещение прямоугольников с логотипами в ячейке отчёта (паттерн из реальных прикладных скриптов):

```EME-L
GraphCell_OnFillGraphWindow()
{
    'Создание объекта графической ячейки'
    GC = Object("GraphCell");

    'Размещение прямоугольника с координатами и размером'
    Rect = GC.CreateRectangle(48, 0, 96, 96);
    Rect.SetLineColor(is_RGB(255, 255, 255));
}
```

Создание диаграммы с настройкой данных через запрос в системе EME.WMS:

```EME-L
GraphCell_OnFillGraphWindow()
{
    GC = Object("GraphCell");

    'Диаграмма с координатами и размером'
    Diagram = GC.CreateDiagram(0, 0, 400, 300);

    'Заполнение графика данными из запроса'
    GC.FillGraphQuery("my_report_query", "2024", "", "", "");
}
```

# Класс RobotStockWMS

Класс `RobotStockWMS` в языке EME-L — вспомогательный класс для тестирования складских модулей WMS с использованием робота. В системе EME.WMS он представляет собой промежуточное хранилище количественных данных о товарах, которое позволяет сравнивать остатки, полученные роботом, с данными базы данных.

В языке EME-L класс хранит количества в двух представлениях: в конкретной единице измерения товара (по ссылке GIMU) и в базовой единице измерения. Каждая запись во внутреннем хранилище однозначно идентифицируется набором параметров: склад (ссылка), собственник (ссылка), товар (ссылка), стоковый статус (строка), партия (ссылка), срок годности ББД в формате `ДД.ММ.ГГГГ` и (только для конкретной единицы) ссылка на единицу измерения товара GIMU.

В системе EME.WMS класс `RobotStockWMS` (CI-имя `CIRobotStockWMS`) предназначен для автоматизированных проверок: робот выполняет складские операции, а класс накапливает ожидаемые остатки, после чего сверяет их с регистрами БД.

Основные возможности класса `RobotStockWMS` в языке EME-L:

- Привязка к экземпляру робота для логирования результатов сравнения.
- Загрузка остатков из регистров БД в двух вариантах: как «эталон БД» (`qtyDB`) и как «данные робота из БД» (`qtyRobot`).
- Установка, получение, увеличение и уменьшение количества товара по заданному набору параметров.
- Сравнение стока робота с данными БД с детальным логированием расхождений (склад, собственник, товар, партия, единица измерения, количества).

Объект класса `RobotStockWMS` создаётся через оператор `Object()`. Данные во внутренних структурах накапливаются по вызовам `PutQtyRobot` / `PlusQtyRobot` / `MinusQtyRobot` (сторона робота) и `FillQtyFromDB` (сторона БД). Сравнение выполняется методом `CompareStockWithDB`, который возвращает текстовый отчёт и при наличии привязанного робота дублирует его в лог робота.

## Создание объекта

В языке EME-L класс `RobotStockWMS` имеет два конструктора — без аргументов и с указанием робота.

```EME-L
'Конструктор без аргументов: робот не привязан'
objStock = Object("RobotStockWMS");
```

```EME-L
'Конструктор с привязкой робота для логирования'
objStock = Object("RobotStockWMS", objRobot);
```

При создании без робота операции сравнения (`CompareStockWithDB`) в языке EME-L по-прежнему работают и возвращают текстовый отчёт, но не дублируют его в лог робота. Привязать робота позже можно методом `SetRobot` — это необходимо, когда объект робота создаётся в другом методе EME-L и не может быть передан в конструктор (объект, созданный через `Object()` внутри метода, уничтожается при выходе из него, поэтому ссылку нужно передавать через `SetRobot` из внешнего метода).

## Привязка робота

В языке EME-L методы этой группы управляют привязкой экземпляра робота к объекту `RobotStockWMS`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetRobot(arg0)` | arg0: Robot (по ссылке) | Empty | Устанавливает экземпляр робота в языке EME-L. Последующие вызовы `CompareStockWithDB` будут дублировать отчёт в лог робота. Без вызова `SetRobot` отчёт возвращается только как строка. |

## Управление количеством робота

Методы этой группы работают с полем `qtyRobot` — стороной робота во внутреннем хранилище класса `RobotStockWMS` в языке EME-L. Количество может задаваться как в конкретной единице измерения товара (GIMU), так и пересчитываться в базовую единицу автоматически.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `PutQtyRobot(arg0, arg1, arg2, arg3, arg4, arg5, arg6, arg7)` | arg0: Integer (склад), arg1: Integer (собственник), arg2: Integer (товар), arg3: String (статус), arg4: Integer (партия), arg5: String (ББД `ДД.ММ.ГГГГ`), arg6: Integer (GIMU), arg7: Real (количество) | Integer | Записывает (прибавляет) количество в структуру робота в языке EME-L. При отсутствии товара создаёт новую запись. Если указан несуществующий товар и привязан робот, выводит сообщение в лог робота. Автоматически пересчитывает и накапливает количество в базовой единице. Возвращает 0. |
| `PlusQtyRobot(arg0, arg1, arg2, arg3, arg4, arg5, arg6, arg7)` | те же, что `PutQtyRobot`; arg7 — прибавляемое количество (Real) | Real | Увеличивает текущее количество робота на `arg7` в языке EME-L. Получает значение через `GetQtyRobot`, прибавляет и записывает через `PutQtyRobot`. Возвращает 0 (Real). |
| `MinusQtyRobot(arg0, arg1, arg2, arg3, arg4, arg5, arg6, arg7)` | те же, что `PutQtyRobot`; arg7 — вычитаемое количество (Real) | Real | Уменьшает текущее количество робота на `arg7` в языке EME-L. Получает значение через `GetQtyRobot`, вычитает и записывает через `PutQtyRobot`. Возвращает 0 (Real). |
| `GetQtyRobot(arg0, arg1, arg2, arg3, arg4, arg5, arg6)` | arg0: Integer (склад), arg1: Integer (собственник), arg2: Integer (товар), arg3: String (статус), arg4: Integer (партия), arg5: String (ББД), arg6: Integer (GIMU) | Real | Возвращает количество товара в структуре робота в языке EME-L для указанной единицы измерения. |
| `GetQtyRobotBaseMU(arg0, arg1, arg2, arg3, arg4, arg5)` | arg0–arg5: склад, собственник, товар, статус, партия, ББД (типы как выше, без GIMU) | Real | Возвращает общее количество робота в базовой единице в языке EME-L для всех упаковок указанного статуса, партии и ББД. |

## Управление количеством из БД

В языке EME-L методы этой группы работают с полем `qtyDB` — стороной базы данных во внутреннем хранилище класса `RobotStockWMS`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `FillQtyFromDB()` | — | Integer | Заполняет `qtyDB` количественными данными из регистров базы данных в языке EME-L. Проходит по всем записям регистров, суммирует физическое количество (`TotalQtyPhis`) и одновременно проверяет заполненность всех полей. Количество сохраняется как в конкретной единице, так и в базовой. Возвращает 0. |
| `FillQtyRobotFromDB()` | — | Integer | Заполняет `qtyRobot` данными из регистров БД (имитирует данные робота на основе БД) в языке EME-L. Алгоритм аналогичен `FillQtyFromDB`, но накапливает значения в поле `qtyRobot`. Возвращает 0. |
| `GetQtyDB(arg0, arg1, arg2, arg3, arg4, arg5, arg6)` | arg0: Integer (склад), arg1: Integer (собственник), arg2: Integer (товар), arg3: String (статус), arg4: Integer (партия), arg5: String (ББД), arg6: Integer (GIMU) | Real | Возвращает количество товара из базы данных в языке EME-L для указанной единицы измерения. |
| `GetQtyDBBaseMU(arg0, arg1, arg2, arg3, arg4, arg5)` | arg0–arg5: склад, собственник, товар, статус, партия, ББД (типы как выше, без GIMU) | Real | Возвращает общее количество из базы данных в базовой единице в языке EME-L для всех упаковок указанного статуса, партии и ББД. |

## Сравнение стока

В языке EME-L методы этой группы выполняют сравнение данных робота с данными базы данных в классе `RobotStockWMS`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `CompareStockWithDB()` | — | String | Сравнивает данные робота (`qtyRobot`) с данными базы (`qtyDB`) по всем товарным позициям во внутреннем хранилище класса `RobotStockWMS` в языке EME-L. Проверяет остатки как в конкретной единице измерения, так и в базовой. При обнаружении расхождений формирует детализированный лог с наименованиями склада, собственника, товара, партии, единицы измерения и количествами. Если к объекту привязан робот (`SetRobot`), лог дополнительно выводится через систему логирования робота. Возвращает текстовый отчёт о сравнении. |

## Примеры

В языке EME-L создание объекта `RobotStockWMS`, заполнение данными робота и сравнение с базой:

```EME-L
'Создать хранилище стока с привязкой робота'
objStock = Object("RobotStockWMS", objRobot);

'Заполнить эталонные количества из БД'
objStock.FillQtyFromDB();

'Добавить количество, измеренное роботом'
objStock.PutQtyRobot(1, 1, 100, "0", 1, "31.12.2026", 1, 10.0);

'Сравнить и получить отчёт'
strReport = objStock.CompareStockWithDB();
```

Увеличение и уменьшение количества робота в языке EME-L:

```EME-L
'Увеличить остаток товара на 5 единиц'
objStock.PlusQtyRobot(1, 1, 100, "0", 1, "31.12.2026", 1, 5.0);

'Уменьшить остаток товара на 3 единицы'
objStock.MinusQtyRobot(1, 1, 100, "0", 1, "31.12.2026", 1, 3.0);

'Получить текущее количество робота'
qtyRobot = objStock.GetQtyRobot(1, 1, 100, "0", 1, "31.12.2026", 1);
```

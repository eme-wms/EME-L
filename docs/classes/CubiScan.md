# Класс CubiScan

> Синонимы для поиска: CICubiScan CubiScan класс EME-L, 3D-сканер габаритов CubiScan EME-L, измеритель ВГХ CubiScan EME-L, VolumeOne CubiScan EME-L, сканер веса и габаритов EME-L, TCP/IP сканер ВГХ, Measure Ping Calibration RunCommand GetLength GetWidth GetHeight GetWeight GetVolumeWeight GetSizeMeasureUnit GetWeightMeasureUnit GetStatus GetStatusDescription GetError GetDeviceId GetResultString SetDebug, msOk msReady msBusu msSTAT_UNDER msSTAT_UNSTABLE msSTAT_OVER msWrongResult msTimeoutError msSocketError msHardwareError msCornerSensorError msZeroError msMeasureError, дюймы сантиметры фунты килограммы CubiScan, калибровка CubiScan

Класс `CubiScan` в языке EME-L управляет 3D-сканером габаритов и веса CubiScan (аппаратная модель VolumeOne) через TCP/IP-соединение. Класс CubiScan в системе EME.WMS предоставляет методы для измерения длины, ширины, высоты, веса и объемного веса объекта, проверки доступности устройства, калибровки, отправки произвольных команд и получения текстового описания ошибок и статусов. CubiScan — это аппаратный бренд 3D-сканеров ВГХ (весогабаритных характеристик); класс EME-L инкапсулирует их сетевой протокол поверх TCP/IP.

Объект класса `CubiScan` в языке EME-L создаётся с указанием IP-адреса устройства (строка) и номера TCP-порта (целое число). Соединение открывается и закрывается на каждый командный запрос (`Ping`, `Measure`, `Calibration`, `RunCommand`) — отдельного метода подключения не существует. После создания объекта измеренные значения сброшены в ноль, код ошибки — `msOk`, статус устройства — «Готово».

## Создание объекта класса CubiScan

Конструктор класса CubiScan в языке EME-L требует ровно два аргумента и не имеет перегрузок.

```EME-L
'Подключение к сканеру CubiScan по адресу 10.24.2.251, порт 443'
objCubi = Object("CubiScan", "10.24.2.251", 443);
```

Конструктор класса CubiScan в языке EME-L возвращает ссылку на объект CICubiScan. Объект автоматически освобождает ресурсы при выходе из области видимости (новых соединений в деструкторе не разрывается, так как каждый командный метод сам открывает и закрывает TCP-сокет).

## Измерение и опрос устройства класса CubiScan

Методы класса CubiScan в языке EME-L для измерения, проверки доступности, калибровки и отправки произвольных команд. Каждый метод сам открывает TCP-соединение, отправляет команду, читает ответ и закрывает сокет — отдельный вызов `Connect` не требуется. Текст последнего ответа доступен через `GetResultString`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Measure()` | — | Integer | Запускает измерение (команда `M`). 0 — успех (`msOk`), 200 — устройство готово (`msReady`), прочие — код ошибки (тайм-аут, сокет, оборудование, угловой датчик, нуль, измерение, неверный ответ). |
| `Ping()` | — | Integer | Проверяет доступность (команда `T`) и получает идентификатор устройства. 0 — устройство доступно (`msOk`), прочие — код ошибки. |
| `Calibration()` | — | Integer | Запускает калибровку (команда `D`). Метод следует вызывать повторно до получения статуса `msReady` (200) или кода ошибки. Промежуточные статусы — этапы калибровки (`msStartCalibrate`, `msProcCalibrate1`–`4`, `msEndCalibrate`). |
| `RunCommand(arg0)` | arg0: String | Integer | Отправляет произвольную команду по протоколу CubiScan. 0 — успех (`msOk`), прочие — код ошибки. |

## Результаты измерения класса CubiScan

Методы класса CubiScan в языке EME-L для получения результатов последнего измерения. Значения обновляются только после успешного вызова `Measure`. При отсутствии объекта, нестабильности или превышении размерности возвращается отрицательный код ошибки: `-1` (`msSTAT_UNDER` — объект отсутствует), `-2` (`msSTAT_UNSTABLE` — объект нестабилен), `-3` (`msSTAT_OVER` — превышение размерности).

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetLength()` | — | Real | Измеренная длина объекта в единицах устройства (дюймы или сантиметры). |
| `GetWidth()` | — | Real | Измеренная ширина объекта в единицах устройства. |
| `GetHeight()` | — | Real | Измеренная высота объекта в единицах устройства. |
| `GetWeight()` | — | Real | Измеренный вес объекта в единицах устройства (фунты или килограммы). |
| `GetVolumeWeight()` | — | Real | Измеренный объемный вес объекта в единицах устройства. |
| `GetSizeMeasureUnit()` | — | String | Единица измерения габаритов: `"in"` (дюймы), `"cm"` (сантиметры) или `""` (измерение не проводилось). |
| `GetWeightMeasureUnit()` | — | String | Единица измерения веса: `"lb"` (фунты), `"kg"` (килограммы) или `""`. |

## Статус, ошибки и диагностика класса CubiScan

Методы класса CubiScan в языке EME-L для получения статуса устройства, описаний ошибок и отладочной информации. Статус изменяется в процессе выполнения операций (измерение, калибровка, пинг) и отражает состояние устройства.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetStatus()` | — | Integer | Текущий статус устройства: 0 (`msOk`), 200 (`msReady`), промежуточные значения — этапы калибровки (`msStartCalibrate`, `msProcCalibrate1`–`4`, `msEndCalibrate`), прочие — коды ошибок или состояние занятости (`msBusu`). |
| `GetStatusDescription()` | — | String | Текстовое описание текущего статуса устройства на русском языке: «Устройство готово», «Удалите с платформы все объекты», «Калибровка завершена», «Устройство занято» и т. д. |
| `GetError()` | — | String | Текстовое описание последней ошибки объекта CubiScan на русском языке. При отсутствии ошибок — «Операция завершена успешно.» |
| `GetDeviceId()` | — | Integer | Идентификационный номер устройства CubiScan, полученный при последнем успешном вызове `Ping`. 0, если `Ping` не вызывался или устройство не ответило. |
| `GetResultString()` | — | String | Необработанная строка ответа, полученная от устройства при выполнении последней команды (`Ping`, `Measure`, `Calibration`, `RunCommand`). Пустая строка, если команда не выполнялась. Используется для расширенной диагностики. |
| `SetDebug(arg0)` | arg0: Boolean | Empty | Включает (`TRUE`) или отключает (`FALSE`) отладочный режим объекта CubiScan. В отладочном режиме все отправляемые и принимаемые команды записываются в системный лог в шестнадцатеричном виде вместе с сообщениями об ошибках. Вызов без аргументов не изменяет режим. |

## Примеры работы с классом CubiScan

Полный цикл измерения ВГХ (весогабаритных характеристик) объекта в языке EME-L:

```EME-L
'Создать объект сканера CubiScan'
objCubi = Object("CubiScan", "10.24.2.251", 443);

'Включить отладочный режим для диагностики TCP-обмена'
objCubi.SetDebug(True);

'Проверить доступность устройства'
nResult = objCubi.Ping();
If (nResult == 0)
    'Измерить габариты и вес объекта'
    nResult = objCubi.Measure();
    If (nResult == 0)
        strMsg = "Length = " + objCubi.GetLength() + ", Width = " + objCubi.GetWidth() + ", Height = " + objCubi.GetHeight() + ", Weight = " + objCubi.GetWeight();
        is_message("Cubi ВГХ", strMsg, 1, 1);
    Else
        'Вывести описание ошибки измерения'
        is_message("Cubi Ошибка", objCubi.GetError(), 1, 3);
    End If
Else
    is_message("Cubi Ошибка", objCubi.GetError(), 1, 3);
End If
```

Проверка системы единиц измерения устройства CubiScan в языке EME-L:

```EME-L
'Определить систему единиц, используемую устройством'
strSizeUnit = objCubi.GetSizeMeasureUnit();
strWeightUnit = objCubi.GetWeightMeasureUnit();
If (strSizeUnit == "cm" && strWeightUnit == "kg")
    is_message("Cubi", "Метрическая система", 1, 0);
Else
    is_message("Cubi", "Империческая система", 1, 0);
End If
```

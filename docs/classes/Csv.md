# Класс Csv

> Синонимы для поиска: CICsv Csv класс EME-L, чтение CSV EME-L, парсер CSV EME-L, ReadFirstLine ReadNextLine SetFirstLine SetNextLine IsValidLine GetNoOfFields GetFieldValue Open Close GetNoOfLines UseTextWrap GetContent TextWrap CSV, импорт CSV EME-L, разделители CSV EME-L, TSV EME-L, разбить строку по разделителю EME-L, Loop по CSV EME-L

Класс `Csv` в языке EME-L выполняет чтение и разбор текстовых файлов формата CSV (Comma Separated Values). Класс Csv в языке EME-L предоставляет механизм последовательного чтения строк CSV-файла с разбором полей по заданным разделителям. Поддерживает настраиваемые символы разделителей полей и конца строки, а также символ кавычек для обрамления текстовых полей.

Класс Csv в системе EME.WMS используется в прикладных скриптах EME-L для импорта данных из внешних CSV-файлов — например, загрузки справочников товаров, кодов стран, заказов клиентов.

Объект класса `Csv` в языке EME-L создаётся через `Object()` с разделителями по умолчанию (`;\n\0`) и кавычками (`"`) либо с явным указанием строки разделителей и символа кавычек. Это самостоятельный класс, не привязанный к контексту диалога или браузера.

## Создание объекта класса Csv

```EME-L
objCsv = Object("Csv");
objCsv = Object("Csv", ";\n");
objCsv = Object("Csv", ";\n", "\"");
```

Конструктор без аргументов класса Csv в языке EME-L создаёт парсер с разделителями по умолчанию: символ разделителя полей `;`, символ конца строки `\n`, нулевой символ `\0`; символ кавычек — двойная кавычка.

## Конструкторы класса Csv

| Конструктор | Аргументы | Описание |
|-------------|-----------|----------|
| `Csv()` | — | Разделители по умолчанию (`;`, `\n`, `\0`), кавычки — двойная кавычка. |
| `Csv(разделители)` | разделители: String (3 символа) | `[0]` — разделитель полей, `[1]` — конец строки, `[2]` — нулевой символ-завершитель. Кавычки — двойная кавычка. |
| `Csv(разделители, кавычки)` | разделители: String (3 символа), кавычки: String | Дополнительно задаётся символ обрамления текстовых полей. Используется первый символ строки; пустая строка отключает кавычки. |

## Управление файлом

Методы класса Csv в языке EME-L для открытия и закрытия CSV-файла.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Open(имя_файла)` | имя_файла: String | Integer | Открывает CSV-файл для чтения в бинарном режиме и читает первую строку. 0 — успех, 1 — ошибка открытия. Поддерживает UTF-16 LE с BOM. Если файл уже открыт, предыдущий закрывается. |
| `Close()` | — | Empty | Закрывает файл и освобождает ресурсы. Сбрасывает буфер, массив полей и позицию. Безопасен для неоткрытого файла. |

## Чтение строк

Методы `ReadFirstLine` / `ReadNextLine` выполняют прямое последовательное чтение. Пара `SetFirstLine` / `SetNextLine` идентична по поведению и предназначена для использования в операторе `Loop` в качестве методов установки/перехода строки. Методы класса Csv в языке EME-L для последовательного чтения строк CSV.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `ReadFirstLine()` | — | Empty | Сбрасывает позицию в начало файла и читает первую строку. Поля разбираются во внутренний массив. |
| `ReadNextLine()` | — | Empty | Читает следующую строку. |
| `SetFirstLine()` | — | Empty | То же, что `ReadFirstLine`; для оператора `Loop`. |
| `SetNextLine()` | — | Empty | То же, что `ReadNextLine`; для оператора `Loop`. |
| `IsValidLine()` | — | Boolean | TRUE — текущая строка содержит хотя бы одно поле. Используется для определения конца данных при последовательном чтении. |
| `GetNoOfLines()` | — | Integer | Подсчитывает общее количество строк по числу вхождений символа конца строки. При вызове для неоткрытого файла генерируется ошибка. |

## Доступ к полям

Методы класса Csv в языке EME-L для доступа к полям текущей разобранной строки.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetNoOfFields()` | — | Integer | Количество полей в текущей строке. Для пустой строки — 0. |
| `GetFieldValue(номер_поля)` | номер_поля: Integer | String | Значение поля по индексу (с 0). При выходе за пределы строки генерируется исключение. Поля, обрамлённые кавычками, возвращаются без них; сдвоенные кавычки заменяются на одинарные. |

## Режим TextWrap

Метод `UseTextWrap` класса Csv в языке EME-L включает режим чтения текстовых полей, заключённых в кавычки, с поддержкой переноса строки внутри поля.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `UseTextWrap()` | — | Empty | Включает режим TextWrap. Поля, обрамлённые символом кавычек, могут содержать символы конца строки и разделители внутри значения; конец поля определяется парой кавычка-разделитель или кавычка-конец-строки. Двойные кавычки внутри поля трактуются как экранирование одинарной. |

## Отладка

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetContent()` | — | String | Содержимое внутреннего буфера чтения. Используется для диагностики проблем с чтением и разбором CSV-данных. |

## Порядок вызова методов

Типичный сценарий использования класса Csv в языке EME-L: `Open(...)` → `ReadFirstLine()` / `SetFirstLine()` → цикл `ReadNextLine()` / `SetNextLine()` с проверкой `IsValidLine()` → `GetNoOfFields()` и `GetFieldValue(i)` для каждой строки → `Close()`.

## Примеры

Последовательное чтение строк через `For` (стиль `ReadFirstLine` / `ReadNextLine`):

```EME-L
'Импорт данных из CSV с разделителем по умолчанию'
csv = Object("Csv");
Result = csv.Open("D:\\Data\\import.csv");
If (Result != 0)
    Return;
End If

csv.ReadFirstLine();
For (csv.ReadNextLine(); csv.IsValidLine(); csv.ReadNextLine())
    Code = is_trim_all(csv.GetFieldValue(0));
    Name = csv.GetFieldValue(1);
    'Code и Name — значения первого и второго поля текущей строки'
End For
```

Чтение через оператор `Loop` с парой `SetFirstLine` / `SetNextLine` и табом в качестве разделителя:

```EME-L
'Чтение TSV-файла через Loop; разделитель полей — таб, конец строки — \n'
File = Object("Csv", "\t\n");
File.Open(FileName);
Loop(File)
    Code = is_trim_all(File.GetFieldValue(0));
    If (~r_Goods.SetLineByCode(Code))
        Continue;
    End If
    'Обработка строки'
End Loop
```

# Класс Collection

> Синонимы для поиска: CICollection Collection класс EME-L, коллекция объектов EME-L, контейнер объектов EME-L, именованный список EME-L, Collection Put Get GetData PutData Clear ClearData GetSize GetName GetIndex GetContent GetArray GetString GetMap, реестр объектов EME-L, Object("Collection"), коллекция EME.WMS

Класс `Collection` в языке EME-L — это универсальный контейнер для хранения именованных или индексированных элементов: объектов EME-L классов и произвольных данных (строк, чисел, дат, массивов). Класс Collection в языке EME-L совмещает поведение списка и ассоциативного массива: каждый элемент может быть получен как по текстовому ключу, так и по порядковому индексу, а элементы без явного имени получают автоматический числовой ключ.

Объект класса `Collection` в языке EME-L создаётся через `Object()`. Коллекция класса Collection в системе EME.WMS может передавать свой контекст выполнения в конструкторы объектов, создаваемых методом `Put`, — это полезно, когда вложенные объекты должны разделять с вызывающим скриптом состояние транзакций, соединений с БД и т. п.

> **Примечание:** класс `Collection` в языке EME-L не имеет записи `REGISTRY_EMEL_CLASS` в `CEMEPrj.cpp` — описание черпается из блока `BEGIN_IMPLEMENT` файла `i_Collection.cpp`. Класс зарегистрирован как встроенный, имя скрипта `Collection` подтверждается реальными вызовами `Object("Collection")` в кодовой базе EME.WMS.

## Создание объекта класса Collection

```EME-L
objCol = Object("Collection");
objCol = Object("Collection", Context);
```

Конструктор без аргументов создаёт пустую коллекцию с текущим контекстом выполнения. Конструктор с аргументом `context` явно задаёт контекст, который будет передан в конструкторы объектов, создаваемых методом `Put` класса Collection.

## Чтение элементов

Методы семейства `Get` класса Collection в языке EME-L возвращают элемент по имени или индексу. Перегрузки различаются поведением при отсутствии элемента в коллекции.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Get(arg0)` | arg0: String или Integer | Object / произвольный тип | Элемент по имени или индексу. Если элемент не найден — генерирует ошибку выполнения. |
| `Get(arg0, arg1)` | arg0: String или Integer, arg1: Boolean | Object / произвольный тип / Empty | Элемент по имени или индексу. arg1=TRUE — при отсутствии возвращает `Empty`; arg1=FALSE (по умолчанию) — генерирует ошибку. |
| `Get(arg0, arg1, arg2)` | arg0: String или Integer, arg1: Boolean, arg2: произвольный тип | Object / произвольный тип | Элемент по имени или индексу. При отсутствии и arg1=TRUE возвращает значение `defaultValue` (arg2). |
| `GetData(arg0)` | arg0: String или Integer | произвольный тип / Empty | Элемент по имени или индексу. При отсутствии возвращает `Empty` без ошибки. Не создаёт объекты — работает только с хранимыми данными. |

Методы `Get` и `GetData` класса Collection в языке EME-L различаются обработкой отсутствующего элемента: `Get` по умолчанию генерирует ошибку (можно переопределить вторым/третьим аргументом), а `GetData` молча возвращает `Empty`. Для типизированного доступа к вложенным объектам `Array`, `String`, `Map` используйте методы `GetArray`, `GetString`, `GetMap` класса Collection (см. ниже) — они проверяют тип и облегчают статическую линковку.

## Запись элементов

Методы семейства `Put` класса Collection в языке EME-L добавляют или замещают элементы. Метод `Put` создаёт новые объекты EME-L классов или сохраняет произвольные данные; метод `PutData` — упрощённая обёртка только для данных.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Put(arg0, arg1, [arg2], [arg3], ...)` | arg0: String или Integer (имя/индекс), arg1: String (имя класса), [arg2, arg3, ...]: аргументы конструктора класса | Object | Если arg1 непустое имя класса — создаёт объект этого класса (передавая остальные аргументы в его конструктор), сохраняет в коллекцию и возвращает. Созданный объект помечается для автоматического удаления при уничтожении коллекции. |
| `Put(arg0, arg1, [arg2], [arg3])` | arg0: String или Integer, arg1: `""` (пустая строка), arg2: произвольный тип (данные), arg3: Boolean (флаг владения) | произвольный тип | При пустом arg1 сохраняет данные arg2. arg3=TRUE — если arg2 содержит объект, он будет удалён вместе с коллекцией; arg3=FALSE (по умолчанию) — ответственность за удаление лежит на вызывающем коде. |
| `Put(, arg1, [arg2], ...)` | arg1: String (имя класса) или `""`, далее аргументы как выше | Object / произвольный тип | Если arg0 опущен (пустой), элемент добавляется в конец коллекции с автоматическим числовым ключом. |
| `PutData(arg0, arg1)` | arg0: String (имя), arg1: произвольный тип (данные) | произвольный тип | Сохраняет произвольные данные под именем. Не создаёт объектов. Эквивалент `Put(arg0, "", arg1)`. |
| `PutData(arg0, arg1, arg2)` | arg0: String (имя), arg1: произвольный тип (данные), arg2: Boolean (флаг владения) | произвольный тип | То же с флагом владения объектом (актуально, если arg1 — объект). Эквивалент `Put(arg0, "", arg1, arg2)`. |

При замещении существующего элемента класса Collection в языке EME-L старый объект (если он был помечен для удаления) сначала уничтожается, затем записывается новый. Это позволяет обновлять содержимое коллекции без утечек памяти в системе EME.WMS.

## Удаление элементов

Методы класса Collection в языке EME-L для удаления элементов из коллекции.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Clear()` | — | Empty | Полностью очищает коллекцию. Все объекты, помеченные для удаления, уничтожаются. |
| `Clear(arg0)` | arg0: String или Integer | Empty | Удаляет один элемент по имени или индексу. Если элемент — объект с флагом владения, объект уничтожается. Оставшиеся элементы перенумеруются. |
| `ClearData(arg0)` | arg0: String или Integer | Empty | Удаляет один элемент по имени или индексу, но **не** уничтожает объект, даже если он помечен для удаления. Оставшиеся элементы перенумеруются. |

Методы `Clear` (с аргументом) и `ClearData` класса Collection в языке EME-L различаются только обработкой удаляемого объекта: `Clear` honourит флаг владения и удаляет объект, `ClearData` всегда оставляет объект в памяти. Полный `Clear()` без аргументов honourит флаги владения для всех элементов коллекции.

## Информация о коллекции

Методы класса Collection в языке EME-L для получения метаданных о коллекции и её элементах.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetSize()` | — | Integer | Количество элементов в коллекции. |
| `GetName(arg0)` | arg0: Integer | String / Empty | Имя элемента по порядковому индексу. `Empty`, если индекс некорректен. |
| `GetIndex(arg0)` | arg0: String | Integer | Порядковый индекс элемента по имени. `-1`, если элемент не найден. |
| `GetContent()` | — | String | Строковое представление содержимого для отладки: для каждого элемента выводится имя, тип и (при наличии) содержимое через `GetContent` или `AsString` самого объекта. |

## Типизированный доступ к вложенным объектам

Методы `GetArray`, `GetString`, `GetMap` класса Collection в языке EME-L проверяют, что элемент по имени/индексу является объектом ожидаемого класса; при несовпадении генерируют ошибку выполнения. Полезны при статической линковке, когда компилятору нужно знать тип объекта.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetArray(arg0)` | arg0: String или Integer | Array | Объект `Array`, хранящийся в коллекции. |
| `GetString(arg0)` | arg0: String или Integer | String | Объект `String`, хранящийся в коллекции. |
| `GetMap(arg0)` | arg0: String или Integer | Map | Объект `Map`, хранящийся в коллекции. |

## Примеры использования класса Collection

Коллекция класса Collection в языке EME-L как реестр объектов класса `Menu` (по имени + владение коллекцией):

```EME-L
'Создать коллекцию для хранения пунктов меню'
m_MenuCollection = Object("Collection");

'Добавить объект Menu под именем "root#" — коллекция владеет объектом'
SubMenu = m_MenuCollection.Put("root#", "Menu", TRUE);

'Получить объект обратно по имени'
RootMenu As "Menu" = m_MenuCollection.Get("root#");
```

Коллекция класса Collection в языке EME-L как структура данных «ключ → значение»:

```EME-L
'Создать коллекцию для настроек'
Config = Object("Collection");

'Сохранить произвольные данные под именованными ключами'
Config.PutData("Server", "sql-prod");
Config.PutData("Port", 1433);
Config.PutData("Timeout", 30);

'Получить значение по ключу; при отсутствии — значение по умолчанию'
Timeout = Config.Get("Timeout", TRUE, 10);
Missing = Config.Get("Missing", TRUE, EMPTY);
```

Проверка наличия и удаление элемента класса Collection в языке EME-L:

```EME-L
'Получить индекс элемента по имени; -1 — элемент не найден'
Index = Config.GetIndex("Server");
If (Index >= 0)
    'Удалить элемент и его данные'
    Config.Clear(Index);
End If
```

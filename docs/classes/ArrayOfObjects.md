# Класс ArrayOfObjects

> Синонимы для поиска: CIArrayOfObjects ArrayOfObjects класс EME-L, массив объектов EME-L, динамический массив объектов EME-L, SmartPointer массив EME-L, умный указатель массив EME-L, GetAt SetAt SetSize GetSize Add InsertAt Remove RemoveAt RemoveAll GetContent Lookup, коллекция объектов EME-L, массив умных указателей EME-L, CIArrayOfObjects методы

Класс `ArrayOfObjects` в языке EME-L — динамический массив указателей на объекты с автоматическим управлением памятью через механизм `SmartPointer`. Каждый элемент массива в языке EME-L представляет собой умный указатель (SmartPointer) на объект, что обеспечивает корректное время жизни хранящихся объектов: они освобождаются автоматически при удалении из массива или при уничтожении самого массива.

Класс ArrayOfObjects в системе EME.WMS предназначен для хранения и манипулирования наборами объектов внутри системы EME. Поддерживает операции добавления, вставки, удаления и доступа к элементам по индексу. Индексация элементов начинается с 0.

Размер массива в языке EME-L может быть **только уменьшен**. Попытка увеличить размер через `SetSize` или вставить элемент за пределами текущего размера приводит к ошибке. Это ограничение класса ArrayOfObjects в языке EME-L отличает его от произвольных растущих коллекций — массив рассчитан на пакетную обработку заранее известного набора объектов.

Объект класса `ArrayOfObjects` в языке EME-L создаётся через `Object()` и не привязан к контексту диалога или браузера — это самостоятельный класс для организации коллекций объектов в прикладных скриптах EME-L.

## Создание объекта класса ArrayOfObjects

```EME-L
arr = Object("ArrayOfObjects");
```

Конструктор без аргументов класса ArrayOfObjects в языке EME-L создаёт пустой массив. Элементы добавляются методами `Add` или `InsertAt`. Класс не предоставляет конструктора с предустановленным размером — для заполнения массива фиксированной длины сначала вызывается `SetSize`-эквивалент через последовательные `Add`, либо внешний объект сохраняется по конкретному индексу методом `SetAt` после создания пустых слотов.

В языке EME-L префикс `arr` по соглашению используется для объектов класса ArrayOfObjects и других Array-подобных классов (`Array`, `MultiArray`). Следуйте этому правилу для читаемости кода в скриптах EME-L.

## Размер массива

Методы класса ArrayOfObjects в языке EME-L для управления размером массива. Размер можно только уменьшить — увеличение генерирует ошибку.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetSize()` | — | Integer | Текущее количество элементов в массиве ArrayOfObjects в языке EME-L. |
| `SetSize(arg0)` | arg0: Integer | Empty | Уменьшает размер массива. `arg0` должен быть меньше или равен текущему размеру; попытка увеличить размер генерирует ошибку. |

## Доступ к элементам

Методы класса ArrayOfObjects в языке EME-L для чтения элементов по индексу. `GetAt` генерирует ошибку при неверном индексе; `Lookup` возвращает `NULL` или значение по умолчанию — выбирайте метод по контексту проверки.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetAt(arg0)` | arg0: Integer | Reference | Возвращает элемент по индексу в массиве ArrayOfObjects. Индексация с 0. Выход за пределы генерирует ошибку. |
| `Lookup(arg0)` | arg0: Integer | Reference | Возвращает элемент по индексу. Если индекс выходит за пределы массива, возвращает `NULL`. |
| `Lookup(arg0, arg1)` | arg0: Integer, arg1: Reference | Reference | То же, но при выходе индекса за пределы возвращает `arg1` как значение по умолчанию. `arg1` должен быть объектом. |
| `GetContent()` | — | String | Строковое представление содержимого класса ArrayOfObjects в языке EME-L для отладки: имена классов объектов (или `<nullptr>` для пустых указателей), разделённые `"; "`. |

`GetAt` и `Lookup` в языке EME-L различаются обработкой неверного индекса: `GetAt` генерирует ошибку и прерывает выполнение, `Lookup` безопасно возвращает `NULL` или значение по умолчанию. Используйте `Lookup` в проверочном коде и `GetAt` когда корректность индекса гарантирована инвариантом вызывающего кода.

## Добавление и вставка

Методы `Add` и `InsertAt` класса ArrayOfObjects в языке EME-L имеют по две перегрузки: одна принимает готовый объект `SmartPointer`, другая принимает параметры и конструирует новый объект внутри массива. Вторая форма удобна когда объект создаётся специально для помещения в массив и нигде больше не используется.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Add(arg0)` | arg0: Reference (SmartPointer) | Integer | Добавляет существующий объект `SmartPointer` в конец массива ArrayOfObjects. Возвращает индекс добавленного элемента. |
| `Add(...)` | параметры конструктора SmartPointer (≥1) | Integer | Создаёт новый `SmartPointer` из переданных параметров и добавляет в конец. Вызов без аргументов генерирует ошибку. |
| `InsertAt(arg0, arg1)` | arg0: Integer, arg1: Reference (SmartPointer) | Empty | Вставляет существующий объект по индексу. `arg0` должен быть меньше текущего размера; иначе генерируется ошибка. |
| `InsertAt(arg0, ...)` | arg0: Integer, далее параметры конструктора SmartPointer (≥1) | Empty | Создаёт новый `SmartPointer` из параметров и вставляет по индексу. Вызов с одним аргументом или без аргументов генерирует ошибку. |

## Удаление элементов

Методы класса ArrayOfObjects в языке EME-L для удаления элементов. `Remove` снимает последний элемент молча (пустой массив игнорируется), `RemoveAt` требует корректного индекса, `RemoveAll` очищает массив полностью.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Remove()` | — | Empty | Удаляет последний элемент массива ArrayOfObjects. Если массив пуст, операция игнорируется. |
| `RemoveAt(arg0)` | arg0: Integer | Empty | Удаляет элемент по индексу. Индекс больше или равный размеру генерирует ошибку. |
| `RemoveAll()` | — | Empty | Очищает массив, удаляя все элементы. |

## Присвоение существующему индексу

Метод `SetAt` класса ArrayOfObjects в языке EME-L заменяет элемент по индексу. Как и `Add`/`InsertAt`, имеет две перегрузки — для готового объекта и для параметров конструктора.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetAt(arg0, arg1)` | arg0: Integer, arg1: Reference (SmartPointer) | Empty | Присваивает элементу по индексу существующий объект `SmartPointer`. |
| `SetAt(arg0, ...)` | arg0: Integer, далее параметры конструктора SmartPointer (≥1) | Empty | Присваивает элементу по индексу новый объект, созданный из переданных параметров. Вызов с одним аргументом или без аргументов генерирует ошибку. |

## Примеры

Создание массива, добавление готовых объектов и обход класса ArrayOfObjects в языке EME-L:

```EME-L
arr = Object("ArrayOfObjects");
arr.Add(objItem1);
arr.Add(objItem2);
'arr.GetSize() возвращает количество элементов'

Loop (arr)
    'Тело цикла работает с каждым элементом массива'
    item = arr.GetAt(arr.GetSize() - 1);
End Loop
```

Безопасный доступ через `Lookup` класса ArrayOfObjects в языке EME-L со значением по умолчанию:

```EME-L
objDefault = Object("File");
objFound = arr.Lookup(idx, objDefault);
'objFound = элемент массива, либо objDefault если idx за пределами'
```

Отладка содержимого массива ArrayOfObjects в системе EME.WMS:

```EME-L
arr.Add(objItem);
Content = arr.GetContent();
'Content содержит имена классов объектов через "; "'
```

Уменьшение размера массива в языке EME-L:

```EME-L
arr.SetSize(arr.GetSize() - 1);
'Последний элемент удалён, размер уменьшен'
```

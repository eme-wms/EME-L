### IrrAnimatedMesh — класс анимированной полигональной сетки движка Irrlicht (язык EME-L)

Класс `IrrAnimatedMesh` в языке EME-L — это класс анимированной полигональной сетки движка Irrlicht. Представляет трёхмерный объект с поддержкой ключевой анимации (key-frame animation), загружаемой из mesh-файла. Позволяет управлять диапазоном и скоростью воспроизведения анимации, окрашивать вершины в RGBA-цвет, задавать прозрачность (альфа-канал) вершин, а также добавлять и удалять объёмные тени (shadow volume).

Класс `IrrAnimatedMesh` в системе EME.WMS наследует все методы базового класса `IrrNode` — управление положением, поворотом, масштабом, видимостью, именем, материалом и текстурой узла. Используется в EME-L-скриптах для отображения анимированных 3D-моделей в сцене Irrlicht: движущиеся персонажи, механизмы, циклические анимации. В отличие от статического узла `IrrMesh`, `IrrAnimatedMesh` хранит набор ключевых кадров и может проигрывать их в цикле.

### Создание объекта IrrAnimatedMesh

В блоке регистрации `BEGIN_IMPLEMENT(CIIrrAnimatedMesh)` в C++ исходнике нет зарегистрированных конструкторов (`IMPLEMENT_CTOR` отсутствует), поэтому объект `IrrAnimatedMesh` в языке EME-L не создаётся через `Object("IrrAnimatedMesh", ...)` напрямую. Канонический способ получить анимированную полигональную сетку — вызвать метод `AddAnimatedMesh(filePath)` у родительского узла `IrrNode`. Этот метод создаёт дочерний узел, загружает mesh-файл и возвращает ссылку на новый объект `IrrAnimatedMesh`.

```EME-L
'Создать устройство Irrlicht и корневой узел сцены'
objDevice = Object("IrrDevice");
objRoot   = objDevice.GetRootNode();

'Добавить дочерний узел анимированной сетки из mesh-файла'
objAnim = objRoot.AddAnimatedMesh("D:\\Models\\worker.mesh");
```

Метод `AddAnimatedMesh` в системе EME.WMS принимает путь к mesh-файлу (строка). После создания к узлу сразу применяются масштаб `(1, 1, 1)` и пустое имя; в дальнейшем имя задаётся методом `SetName` базового класса `IrrNode`. Если mesh-файл не удалось загрузить, ядро EME-L выводит ошибку через `smart_error` с указанием пути. Глобальный масштаб узла учитывается так же, как и для остальных узлов Irrlicht.

### Метод SetMesh класса IrrAnimatedMesh

Класс `IrrAnimatedMesh` в языке EME-L предоставляет метод `SetMesh` для динамической замены полигональной сетки. Метод перестраивает внутренний указатель Irrlicht, не меняя остальных свойств узла (имя, положение, текстура, настройки анимации). Возвращаемая ссылка на `IrrAnimatedMesh` позволяет объединять настройку в цепочку вызовов (method chaining), например `objAnim.SetMesh("model.mesh").SetClickable(TRUE)`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetMesh(arg0)` | arg0: String (путь к mesh-файлу) | IrrAnimatedMesh (ссылка на текущий объект) | Загружает новый mesh-файл и заменяет текущую полигональную сетку объекта на загруженную модель. Возвращает ссылку на текущий объект `IrrAnimatedMesh` для цепочки вызовов. |

В отличие от одноимённого метода класса `IrrOctree` или `IrrMesh`, метод `SetMesh` класса `IrrAnimatedMesh` в языке EME-L принимает только путь к mesh-файлу (строка); объект `IrrMeshCreator` не поддерживается, поскольку источник должен содержать ключевую анимацию.

### Методы цвета и прозрачности вершин класса IrrAnimatedMesh

Цветовые методы класса `IrrAnimatedMesh` в системе EME.WMS работают с вершинами полигональной сетки напрямую через `IMeshManipulator` Irrlicht — они не меняют материал узла. Цвет задаётся как 32-битное значение в формате RGBA (`0xAARRGGBB`), где старший байт — альфа-канал, затем красный, зелёный и синий.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetVertexColors(arg0)` | arg0: Integer (RGBA, `0xAARRGGBB`) | Empty | Окрашивает все вершины текущей полигональной сетки в указанный RGBA-цвет. 32-битное значение цвета, где старший байт — альфа-канал. |
| `SetVertexColorAlpha(arg0)` | arg0: Integer (0–255) | Empty | Задаёт прозрачность (альфа-канал) вершин текущей полигональной сетки. Диапазон: `0` — полностью прозрачно, `255` — непрозрачно. |

Для полной замены цвета в языке EME-L используйте `SetVertexColors` с непрозрачным значением (`255` в альфа-канале, например `0xFFFFCC00`), а для управления прозрачностью существующих вершин — `SetVertexColorAlpha`. Значение `alpha`, равное `0`, делает вершины невидимыми; `255` — полностью непрозрачными.

### Методы анимации класса IrrAnimatedMesh

Анимация в языке EME-L для класса `IrrAnimatedMesh` управляется двумя методами: `SetFrameLoop` выбирает поддиапазон ключевых кадров, который будет проигрываться циклически, а `SetAnimationSpeed` задаёт темп воспроизведения в кадрах в секунду. Скорость внутри приводится к вещественному типу Irrlicht `f32`, но в EME-L передаётся как целое число.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SetFrameLoop(arg0, arg1)` | arg0: Integer (начальный кадр), arg1: Integer (конечный кадр) | Empty | Устанавливает диапазон кадров для циклического воспроизведения анимации. |
| `SetAnimationSpeed(arg0)` | arg0: Integer (кадров в секунду) | Empty | Задаёт скорость воспроизведения анимации в кадрах в секунду. |

Диапазон кадров в системе EME.WMS определяется содержимым mesh-файла: ключевые кадры нумеруются от `0` до значения, зависящего от модели. Передача диапазона `(0, 0)` фактически фиксирует первый кадр (стоп-кадр). Для плавной циклической анимации выбирайте начальный и конечный кадры так, чтобы поза совпадала (зацикливание без рывка).

### Методы теней класса IrrAnimatedMesh

Тени в языке EME-L для класса `IrrAnimatedMesh` реализуются как узлы объёмной тени (shadow volume scene node) Irrlicht. Оба метода идемпотентны: повторный вызов `AddShadow` не накапливает тени, а `RemoveShadow` безопасно вызывает на узле без тени.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddShadow()` | — | Empty | Добавляет к объекту объёмную тень (shadow volume). Если тень уже добавлена, повторного добавления не происходит. |
| `RemoveShadow()` | — | Empty | Удаляет ранее добавленную объёмную тень из сцены. Если тень не была добавлена, метод ничего не делает. |

Для отображения теней в сцене Irrlicht в системе EME.WMS требуется, чтобы устройство `IrrDevice` имело включённый режим теней (shadow mode). Объёмная тень следует за анимацией полигональной сетки: при воспроизведении ключевых кадров тень обновляется в соответствии с текущей позой модели.

### Примеры использования IrrAnimatedMesh в языке EME-L

Создание анимированной модели и настройка воспроизведения:

```EME-L
'Корневой узел сцены Irrlicht'
objDevice = Object("IrrDevice");
objRoot   = objDevice.GetRootNode();

'Загрузить анимированную модель и настроить анимацию'
objAnim = objRoot.AddAnimatedMesh("D:\\Models\\worker.mesh");
objAnim.SetPosition(10.0, 0.0, 5.0);
objAnim.SetAnimationSpeed(30);
objAnim.SetFrameLoop(0, 100);
objAnim.SetName("Worker#A1");
```

Подсветка вершин и добавление тени:

```EME-L
'Окрасить вершины в жёлто-зелёный (непрозрачный, 0xFF в альфа-канале)'
objAnim.SetVertexColors(0xFFCCFF00);

'Задать полупрозрачность вершин (128 из 255)'
objAnim.SetVertexColorAlpha(128);

'Добавить объёмную тень (идемпотентно — повторный вызов безопасен)'
objAnim.AddShadow();
```

Динамическая замена сетки с цепочкой вызовов:

```EME-L
'Заменить модель в существующем узле IrrAnimatedMesh'
objAnim.SetMesh("D:\\Models\\worker_v2.mesh").SetClickable(TRUE);
```

### KeyResolver — класс определителя ключей строк записи

Класс `KeyResolver` (C++ `CIKeyResolver`) в языке EME-L определяет соответствие между ключами строк и номерами строк записей базы данных EME. Ключ строки — стабильный идентификатор строки в записи, не зависящий от физического порядка в файле; псевдоключ — производное значение, построенное из номера записи и номера строки (или из внутреннего идентификатора строки для Worker Data), применяемое при импорте и экспорте данных.

Класс используется в системе EME.WMS при импорте данных для сопоставления ссылок на строки между разными базами и филиалами, где прямые номера строк могут различаться, а ключи обеспечивают стабильную идентификацию. Экземпляр создаётся через `Object("KeyResolver")`, ключи загружаются вызовами `LoadKey` (по одной строке) или `ResolvePseudoKeys` (пакетно из объекта Query), после чего поиск выполняется методами `GetLine` и `GetPseudoKeyLine`.

В языке EME-L объект `KeyResolver` не привязан к контексту диалога или браузера: это самостоятельный класс для работы с внутренними структурами ключей ядра. Перед использованием методов поиска необходимо явно загрузить ключи — пустой объект не содержит данных.

**Ограничение версии ядра.** Методы `GetLine` и `LoadKey` работают только в ядре версии ниже 52.0. В версии 52.0 и выше они вызывают внутреннюю ошибку (`internal_error`). Методы `ResolvePseudoKeys`, `GetPseudoKeyLine` и `GetLinePseudoKey` продолжают работать во всех версиях ядра; в ядре ≥ 52.0 они используют единое поле ключа (`KEY_FIELD_ID`) вместо раздельной логики Worker Data / обычных записей.

#### Создание объекта

Класс `KeyResolver` имеет один конструктор — без аргументов. Создаваемый объект пуст; инициализация ключей выполняется отдельными вызовами методов.

| Конструктор | Аргументы | Возвращает | Описание |
|-------------|-----------|------------|----------|
| `KeyResolver()` | (нет) | ссылка на KeyResolver | Создаёт пустой объект определителя ключей. Перед использованием методов поиска (`GetLine`, `GetPseudoKeyLine`) необходимо загрузить ключи через `LoadKey` или `ResolvePseudoKeys`. |

```EME-L
'Создание пустого объекта определителя ключей в языке EME-L'
objResolver = Object("KeyResolver");
```

#### LoadKey — загрузка ключа строки во внутреннее хранилище

Загружает ключ указанной строки записи во внутреннее хранилище определителя. При первой загрузке ключей для записи инициализирует хранилище записью целиком (вызывает `CKeysArray::Init`) и возвращает `1` (все ключи строки добавлены автоматически). При повторной загрузке для уже инициализированной записи добавляет ключ конкретной строки (через `LoadDbLine`) и возвращает его значение.

В языке EME-L метод `LoadKey` работает только в ядре версии ниже 52.0. В версии 52.0 и выше вызывается `internal_error()`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer | Да | Номер записи. При недопустимом номере записи генерируется ошибка. |
| arg1 | Integer | Да | Номер строки в записи. |

**Возвращает**: `Integer` — при первой инициализации возвращает `1`; при повторных вызовах возвращает ключ указанной строки (целое число).

```EME-L
objResolver = Object("KeyResolver");
nRec = 15;
'Первая загрузка для записи инициализирует хранилище и возвращает 1'
nResult = objResolver.LoadKey(nRec, 0);
If (nResult = 1)
    'Хранилище ключей записи готово к поиску через GetLine'
    nLine = objResolver.GetLine(nRec, 123456);
End If
```

#### GetLine — поиск строки по ключу

Возвращает номер строки, соответствующий заданному ключу в указанной записи. Поиск выполняется во внутреннем хранилище, заполненном методом `LoadKey`. В версиях ядра ≥ 46.0 ключ читается как 64-битное значение (`variant2int64`), в более ранних — как 32-битное (`variant2integer`).

В языке EME-L метод `GetLine` работает только в ядре версии ниже 52.0. В версии 52.0 и выше вызывается `internal_error()`.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer | Да | Номер записи. При недопустимом номере генерируется ошибка. |
| arg1 | Integer | Да | Ключ строки (в версиях ядра ≥ 46.0 — 64-битное значение). |

**Возвращает**: `Integer` — номер строки; `NULL_REF`, если строка с указанным ключом не найдена.

```EME-L
'Поиск строки по ключу в системе EME.WMS после LoadKey'
nLine = objResolver.GetLine(nRec, nKey);
If (nLine = NULL_REF)
    Message("Строка не найдена");
End If
```

Типовой паттерн из модуля синхронизации остатков с ERP (класс ERPRemainsThread): ключ хранится во внешней БД как HEX-строка и преобразуется в целое перед поиском.

```EME-L
'#DEV_INITIALS 27.06.2025 EME company
Синхронизация остатков: поиск строки документа по ключу из внешней БД'
KeyResolver = Object("KeyResolver");
r_DocLines = Object("dsDB", "DocLines");

For (Line = 0; Line < TotalCount; Line = Line + 1)
    'Ключ хранится в первых 8 символах поля id как HEX-строка'
    HexLineKey = is_left(RemainsQuery.GetData("id", Line), 8);
    IntLineKey = HexToInt(HexLineKey);
    r_DocLines.SetLine(KeyResolver.GetLine(r_DocLines.GetRecord(), IntLineKey));
    If (r_DocLines.IsValidLine())
        'Строка найдена — обработать данные'
    End If
End For
```

#### ResolvePseudoKeys — пакетное заполнение карты псевдоключей

Заполняет внутреннюю карту соответствия псевдоключей номерам строк для указанной записи. Извлекает значения ключей из заданного поля запроса и выполняет пакетный поиск соответствующих строк в базе данных с использованием блочного чтения: размер пакета — 1000 строк (`BATCH_SIZE`), за один проход читается блок через `get_elementary_data_block`.

Для Worker Data (записи с битом `WORKER_DATA_BIT`) поиск выполняется по внутренним идентификаторам строк (`LINE_ID.KEY`) — поле «Идентификатор строки». Для обычных записей псевдоключ формируется из номера записи и номера строки (`line + 1`) — прибавление единицы необходимо, потому что в SQL-представлении ноль используется как `NULL` для ключа и недопустим как значение ссылки. В ядре ≥ 52.0 используется единое поле ключа `KEY_FIELD_ID` для всех записей.

Метод работает во всех версиях ядра EME-L. Перед вызовом карта очищается (`m_PseudoKeys.clear()`), поэтому повторный вызов заменяет предыдущие данные.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer / String | Да | Номер записи или имя записи. При передаче строки имя ищется через `smart_find_record`. |
| arg1 | Query (по ссылке) | Да | Объект Query, содержащий значения псевдоключей. |
| arg2 | String / Integer | Да | Имя поля или индекс поля в запросе, содержащего псевдоключи. При передаче строки имя поля ищется через `FindField`. |

**Возвращает**: `Empty`.

```EME-L
'Заполнить карту псевдоключей из результата запроса'
objResolver = Object("KeyResolver");
objResolver.ResolvePseudoKeys("Товары", objQuery, "ИдентификаторСтроки");
'Карта готова — вызвать GetPseudoKeyLine для каждого значения'
```

Передача поля по индексу (0) вместо имени — паттерн из класса ERPTests:

```EME-L
objResolver.ResolvePseudoKeys(RecordNo, Query, 0);
```

#### GetPseudoKeyLine — номер строки по псевдоключу

Возвращает номер строки, соответствующий заданному псевдоключу, ранее загруженному методом `ResolvePseudoKeys`. Поиск выполняется по внутренней карте; ключ интерпретируется как 64-битное значение (`variant2int64`).

В языке EME-L если псевдоключ не найден в карте, генерируется ошибка через `smart_error` с шестнадцатеричным представлением ключа. Метод работает во всех версиях ядра.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer (64-битное) | Да | Псевдоключ. |

**Возвращает**: `Integer` — номер строки. Если псевдоключ не найден в карте, генерируется ошибка.

```EME-L
'Получить номер строки по псевдоключу после ResolvePseudoKeys'
nLine = objResolver.GetPseudoKeyLine(nPseudoKey);
```

#### GetLinePseudoKey — псевдоключ для строки записи

Возвращает псевдоключ для указанной строки записи. Для Worker Data (записи с битом `WORKER_DATA_BIT`) возвращает внутренний идентификатор строки (`LINE_ID.KEY`). Для обычных записей формирует псевдоключ на основе номера записи и номера строки: используется значение `line + 1`, так как ноль зарезервирован как SQL `NULL`. В ядре ≥ 52.0 просто читает `KEY_FIELD_ID` для строки.

Метод работает во всех версиях ядра EME-L.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| arg0 | Integer / String | Да | Номер записи или имя записи. При передаче строки имя ищется через `smart_find_record`. |
| arg1 | Integer | Да | Номер строки в записи. |

**Возвращает**: `Integer` — 64-битное значение псевдоключа.

```EME-L
'Получить псевдоключ для строки записи в языке EME-L'
nPseudoKey = objResolver.GetLinePseudoKey("Товары", nLine);
```

Передача KeyResolver по ссылке в метод-обёртку — паттерн из класса ERPReplication (в системе EME.WMS):

```EME-L
/* #DEV_INITIALS 27.06.2025 EME company
Возвращает ключ строки в реплике */
GetReplicaLineKey(KeyResolver As "KeyResolver", r_Record As "CEMERec")
{
    Return KeyResolver.GetLinePseudoKey(r_Record.GetRecord(), r_Record.GetLine());
}
```

#### Константы

| Имя | Значение | Назначение |
|------|----------|------------|
| `BATCH_SIZE` | `1000` | Размер пакета при блочном поиске псевдоключей в `ResolvePseudoKeys`. Определяет количество строк, читаемых за один вызов `get_elementary_data_block`. |
| `WORKER_DATA_BIT` | битовый флаг записи | Признак Worker Data. Записи с этим битом используют внутренние идентификаторы строк (`LINE_ID.KEY`) вместо адресной пары (запись, строка) при построении псевдоключей. |
| `KEY_FIELD_ID` | идентификатор поля ключа | Единое поле ключа, используемое в ядре ≥ 52.0 вместо раздельной логики Worker Data / обычных записей. |

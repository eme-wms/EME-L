### Timer — класс измерения временных интервалов в языке EME-L

Класс `Timer` (C++ `CITimer`) в языке EME-L предназначен для измерения временных интервалов выполнения действий в миллисекундах с записью результатов в базу данных. В системе EME.WMS класс `Timer` используется для профилирования производительности операций, особенно при работе с роботом автоматизированного тестирования.

Класс `Timer` доступен в версиях ядра EME.WMS 28.6 и выше (условие компиляции `LAST_VERSION >= MAKE_VERSION(28, 6)`). Объект класса `Timer` в EME-L создаётся через `Object()` без параметров и не привязан к контексту диалога или браузера.

### Создание объекта Timer в EME-L

Класс `Timer` в языке EME-L имеет один конструктор — без аргументов.

```EME-L
'Создать таймер для измерения времени выполнения операций'
timer = Object("Timer");
```

Конструктор `Object("Timer")` в EME-L не принимает параметров. При попытке передать аргументы генерируется ошибка `too many`. По умолчанию после создания объекта `Timer` включён режим подробного логирования (`isUseLog = true`).

### Методы класса Timer в языке EME-L

В языке EME-L класс `Timer` предоставляет методы для запуска и остановки таймеров, а также для управления режимом логирования.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Start()` | — | Empty | Запускает таймер для действия без названия в EME-L (эквивалентно `Start("БЕЗ НАЗВАНИЯ")`). |
| `Start(name)` | name: String | Empty | Запускает таймер в EME-L для действия с указанным именем. Если имя не передано, используется `"БЕЗ НАЗВАНИЯ"`. |
| `Stop()` | — | Integer | Останавливает таймер в EME-L для действия без названия и возвращает время выполнения в миллисекундах. Если действие не найдено, возвращает 0. |
| `Stop(name)` | name: String | Integer | Останавливает таймер в EME-L для действия с указанным именем и возвращает время выполнения в миллисекундах. Если имя не передано, используется `"БЕЗ НАЗВАНИЯ"`. Если действие не найдено, возвращает 0. |
| `SetUseLog(useLog)` | useLog: Boolean | Empty | Устанавливает режим подробного логирования в EME-L: `true` — включить (по умолчанию), `false` — отключить для точного профилирования. |

### Поведение таймеров в системе EME.WMS

В системе EME.WMS класс `Timer` поддерживает одновременный запуск нескольких таймеров с различными именами действий. В режиме подробного логирования (`isUseLog = true`) каждый запуск и остановка таймера записываются в лог робота сразу. В режиме без логирования (`isUseLog = false`) записи в лог выполняются только при запуске первого таймера и после остановки всех таймеров, чтобы не влиять на точность измерения.

После остановки последнего работающего таймера в EME-L данные всех таймеров автоматически сохраняются в базу данных в запись `RobotFullTest`. Время фиксируется по системному времени с точностью до миллисекунд. Имена действий создаются или находятся в справочнике `RobotQualifierActions`. Если запись для имени отсутствует, она создаётся автоматически при первом запуске таймера с этим именем.

Несколько таймеров с одинаковым именем в EME-L обрабатываются по принципу стека: останавливается последнее начатое действие с таким именем. Если найдено несколько незавершённых действий с одним именем, в лог записывается предупреждение.

### Примеры использования класса Timer в EME-L

Пример измерения времени выполнения операции робота в EME-L:

```EME-L
'Создаём таймер без параметров'
timer = Object("Timer");

'Засечь время выполнения участка кода'
timer.Start();
'... выполняемые действия ...'
timeMs = timer.Stop();
Robot.Log("Операция выполнена за %d мс", timeMs);
```

Пример измерения времени поименованных действий в EME-L:

```EME-L
'Создаём таймер с подробным логированием по умолчанию'
timer = Object("Timer");

'Засечь время открытия диалога'
timer.Start("открытие диалога группы клиентов");
Robot.SelectMenuDialog("ClientsClassifier.Dialog");
timer.Stop("открытие диалога группы клиентов");

'Засечь время создания группы клиентов'
timer.Start("создание группы клиентов");
Robot.ButtonClick("ClientsClassifier.IntegratorAddBtn");
'... остальные действия ...'
timer.Stop("создание группы клиентов");
```

Пример точного профилирования с отключенным логированием в EME-L:

```EME-L
'Отключить логирование для точных измерений'
timer = Object("Timer");
timer.SetUseLog(false);

timer.Start("быстрая операция");
'... измеряемый код ...'
duration = timer.Stop("быстрая операция");
```

# Класс IrrAtlas

> Синонимы для поиска: CIIrrAtlas IrrAtlas класс EME-L, атлас текстур Irrlicht EME-L, текстурный атлас EME-L, бин-пакинг текстур EME-L, AddTexture Pack GetRect GetImage SaveXml LoadXml, IrrImage EME-L, Rect EME-L, Xml EME-L, текстуры Irrlicht EME-L, 3D-графика EME.WMS Irrlicht

Класс `IrrAtlas` в языке EME-L — класс атласа текстур движка Irrlicht. Предназначен для объединения нескольких отдельных текстур в одно большое изображение — атлас. В системе EME.WMS и в языке EME-L класс `IrrAtlas` позволяет добавлять текстуры по имени из файла или из объекта `IrrImage`, упаковывать их в заданный прямоугольный атлас алгоритмом бин-пакинга, получать координаты размещения каждой текстуры в виде объекта `Rect`, а также сохранять и восстанавливать описание размещения текстур в XML.

Основные возможности класса `IrrAtlas` в языке EME-L:

- добавление текстур по имени с загрузкой из файла или копированием из `IrrImage`;
- упаковка текстур в атлас заданных размеров с контролем успешности;
- получение итогового изображения атласа как объекта `IrrImage`;
- получение прямоугольника размещения отдельной текстуры как объекта `Rect`;
- сохранение и загрузка метаданных размещения текстур через `Xml`.

Класс `IrrAtlas` в языке EME-L используется в прикладных скриптах для подготовки текстурных атласов при работе с двумерной и трёхмерной графикой Irrlicht — например, для сборки атласов стеллажей в 3D-визуализации WMS.

Объект класса `IrrAtlas` создаётся через `Object()` без привязки к контексту диалога или браузера. Это самостоятельный класс: вы добавляете текстуры одну за другой, затем вызываете `Pack`, после чего готовое изображение и координаты размещения можно прочитать или сериализовать в XML. Класс `IrrAtlas` в языке EME-L доступен только при сборке ядра с включённой поддержкой Irrlicht (макрос компиляции `IRRLICHT_IS_AVAILABLE`).

## Создание объекта класса IrrAtlas

Класс `IrrAtlas` в языке EME-L имеет единственный конструктор без аргументов — пустой атлас, в который текстуры добавляются методом `AddTexture`.

```EME-L
'Создать пустой атлас текстур Irrlicht'
objAtlas = Object("IrrAtlas");
```

Конструктор не имеет перегрузок и не принимает аргументов. Все текстуры добавляются после создания через `AddTexture`, а упаковка выполняется отдельным вызовом `Pack`.

## Добавление текстур в языке EME-L

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `AddTexture(arg0, arg1)` | arg0: String (имя), arg1: String (путь к файлу) **или** IrrImage (объект) | Empty | Добавляет текстуру в атлас под уникальным именем. При совпадении имени генерирует ошибку выполнения. Если arg1 — строка, текстура загружается из файла; неудача загрузки также генерирует ошибку. Если arg1 — объект `IrrImage`, изображение копируется из этого объекта. |

Имя текстуры в языке EME-L должно быть уникальным внутри атласа — повтор имени приводит к ошибке выполнения. Порядок добавления сохраняется: метод `Pack` размещает текстуры именно в порядке их добавления.

Внутри метод `AddTexture` класса `IrrAtlas` различает тип второго аргумента: строка трактуется как путь к файлу текстуры (загрузка через `IrrImage.LoadFile`), а объект `IrrImage` — как готовое изображение для копирования. При неудачной загрузке файла генерируется ошибка выполнения.

## Упаковка текстур в атлас

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `Pack(arg0, arg1)` | arg0: Integer (ширина, px), arg1: Integer (высота, px) | String | Упаковывает все добавленные текстуры в изображение заданных размеров. Фон атласа заполняется белым цветом. Возвращает пустую строку, если все текстуры размещены успешно; иначе — имя первой текстуры, которая не поместилась. |

Текстуры размещаются алгоритмом бин-пакинга в порядке их добавления. Если хотя бы одна текстура не уместилась, метод `Pack` класса `IrrAtlas` в языке EME-L немедленно прекращает упаковку и возвращает имя этой текстуры — оставшиеся текстуры не обрабатываются. Координаты размещения каждой упакованной текстуры доступны через `GetRect`.

Перед упаковкой метод `Pack` очищает предыдущее изображение атласа и создаёт новое размером `width × height` пикселей, заполненное белым цветом (`Qt::GlobalColor::white`). После успешной упаковки итоговое изображение хранится внутри объекта и доступно через `GetImage`.

## Чтение результата упаковки

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `GetRect(arg0)` | arg0: String (имя текстуры) | Rect | Прямоугольник размещения текстуры в атласе (координаты в пикселях от левого верхнего угла). Если текстура с заданным именем отсутствует, генерирует ошибку выполнения. |
| `GetImage()` | — | IrrImage | Итоговое изображение атласа, сформированное методом `Pack`. Если упаковка ещё не выполнялась — пустое изображение. |

Метод `GetRect` класса `IrrAtlas` в языке EME-L возвращает объект `Rect` с полями `left`, `top`, `right`, `bottom` — координатами прямоугольника в пикселях относительно левого верхнего угла атласа. Метод `GetImage` возвращает объект `IrrImage`, который можно сохранить в файл методом `SaveFile` или использовать для создания текстуры Irrlicht.

## Сериализация размещения текстур в XML

Методы `SaveXml` и `LoadXml` класса `IrrAtlas` в языке EME-L работают с объектом `Xml`. Для каждой текстуры создаётся узел `Item` с атрибутами `name`, `left`, `top`, `right`, `bottom`; узлы размещаются внутри корневого элемента `AtlasItems`.

| Метод | Аргументы | Возвращает | Описание |
|-------|-----------|------------|----------|
| `SaveXml(arg0)` | arg0: Xml (по ссылке) | Empty | Сохраняет метаданные размещения текстур в XML-объект. |
| `LoadXml(arg0)` | arg0: Xml (по ссылке) | Empty | Загружает метаданные размещения из XML-объекта. Перед загрузкой очищает текущий атлас и все ранее добавленные текстуры. Ожидает корневой узел `AtlasItems`; при его отсутствии генерирует ошибку выполнения. |

Формат XML, создаваемый методом `SaveXml` класса `IrrAtlas` в языке EME-L:

```XML
<AtlasItems>
    <Item name="upright_fr" left="0" top="0" right="128" bottom="128"/>
    <Item name="beam_fr"    left="0" top="128" right="256" bottom="256"/>
</AtlasItems>
```

Метод `LoadXml` в языке EME-L выполняет обратную операцию: читает узлы `Item` внутри `AtlasItems`, создаёт текстуры с указанными именами и восстанавливает их прямоугольники размещения. При этом сами пиксельные данные текстур не сохраняются — `LoadXml` восстанавливает только метаданные размещения (координаты). Перед загрузкой метод очищает текущее содержимое атласа (все текстуры и их координаты), что позволяет использовать один и тот же объект для повторной загрузки разных наборов метаданных.

## Примеры использования IrrAtlas в языке EME-L

Сборка атласа из набора PNG-файлов с сохранением итогового изображения и описания размещения в XML — реальный паттерн из класса `Wms3DDoctors`:

```EME-L
'Подготовить путь к каталогу текстур стеллажа'
Path = is_ensure_last_slash("D:\\EME\\WMS\\3D\\Rack\\");

Atlas = Object("IrrAtlas");
Atlas.AddTexture("upright_fr", Path + "upright_fr.png");
Atlas.AddTexture("upright_sd", Path + "upright_sd.png");
Atlas.AddTexture("beam_fr", Path + "beam_fr.png");
Atlas.AddTexture("box_up", Path + "box_up.png");

'Упаковать все текстуры в атлас 500×1000 px'
Result = Atlas.Pack(500, 1000);
If (Result != "")
    is_message("Атлас", is_format(tr("Не удалось упаковать текстуру %s"), ""+Result), "OK", "STOP");
    Return;
End If

'Сохранить изображение и метаданные размещения'
Atlas.GetImage().SaveFile(Path + "rack.png");
Xml = Object("Xml");
Atlas.SaveXml(Xml);
Xml.SaveFile(Path + "rack.xml");
```

Добавление текстуры из объекта `IrrImage` вместо файла — однотонная текстура-заполнитель 32×32, добавляемая в тот же атлас:

```EME-L
'Создать однотонную текстуру 32×32 и добавить её в атлас'
Strip = Object("IrrImage", 32, 32);
Strip.Fill(is_RGBA(0xFD, 0xD8, 0x2F, 0xFF));
Atlas.AddTexture("strip", Strip);
```

В этом примере второй аргумент `AddTexture` — объект `IrrImage`, а не строка пути. Класс `IrrAtlas` в языке EME-L различает эти два случая по типу аргумента: строка загружает текстуру из файла, объект `IrrImage` копирует готовое изображение.

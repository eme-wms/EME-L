# Классы Query, BitBuffer, Array, MultiArray, Map — структуры данных и доступ к базе данных в EME-L

> Синонимы для поиска: Query класс запрос, BitBuffer битовый буфер, Array массив EME-L, MultiArray двумерный массив, Map хэш-таблица, структуры данных EME-L, классы данных EME БД, таблица в памяти EME-L, CEMEQuery, CIMap, CIArray, CIMultiArray, CIBitBuffer, данные в EME.WMS

## Обзор классов для работы с данными в языке EME-L

В языке EME-L для работы с данными и обеспечения доступа к базе данных используются пять основных классов: `Query`, `BitBuffer`, `Array`, `MultiArray` и `Map`. Каждый из этих классов реализует определённую структуру данных и предоставляет набор методов для манипулирования информацией в памяти программы. в системе EME.WMS эти классы активно применяются при построении SQL-запросов, фильтрации данных, формировании отчётов и обмене между компонентами системы.

## Класс Query — табличное хранилище данных

`Query` — класс запроса, позволяющий хранить данные в виде таблицы. Имеет встроенные средства добавления, сортировки, поиска, агрегирования и группировки полей. в языке EME-L запрос Query является центральной структурой для работы с результатами SQL-запросов и промежуточными вычислениями.

В запросе `Query` отсутствует возможность удаления строк — это необходимо учитывать при проектировании программ. Однако можно сформировать поле с признаком, по которому нужно игнорировать строку. Также имеется группа методов, позволяющих сравнивать запросы друг с другом. В качестве отладочного функционала можно выводить данные запроса в дамп.

### Методы загрузки и выгрузки строк в Query

Для загрузки и выгрузки строк в запрос используются три пары методов в EME-L:

- `CEMEQuery::LoadAllLines` / `UnloadAllLines` — загрузка/выгрузка абсолютно всех строк записи. Метод не обладает никакими параметрами.
- `CEMEQuery::LoadLine` / `UnloadLine` — загрузка/выгрузка одной строки записи. Параметры метода: номер строки или объект `CEMERec`.
- `CEMEQuery::LoadLines` / `UnloadLines` — загрузка/выгрузка множества строк записи. Параметры метода: объект `CEMERec`, либо битовый буфер, либо массив ссылок.

## Класс BitBuffer — битовый буфер

`BitBuffer` — класс битового буфера: непрерывный массив битов с прямым доступом по индексу, побитовыми логическими операциями (И, ИЛИ, Исключающее ИЛИ, НЕ), навигацией по «строкам» и кэшем фильтров. в языке EME-L битовый буфер широко применяется для фильтрации записей и отметки выбранных строк. Полное описание — на отдельной странице: [Класс BitBuffer](./classes/BitBuffer.md). **Не рекомендуется** увеличивать размер буфера параллельно с его заполнением.

## Класс Array — одномерный массив

`Array` — класс динамического массива произвольных типов в языке EME-L: непрерывный массив в памяти с прямым доступом по индексу (с 0), методами добавления, удаления, поиска, сортировки и итерации через `Loop`. Полное описание — на отдельной странице: [Класс Array](./classes/Array.md). **Не рекомендуется** увеличивать размер массива параллельно с его заполнением.

## Класс MultiArray — двумерный массив

`MultiArray` — класс двумерного массива. Рекомендуется использовать в случае, когда по одному индексу нужно хранить несколько значений. в языке EME-L колонкам второго измерения можно присвоить имена и получать по ним значения.

## Класс Map — хэш-таблица

`Map` — класс хэш-таблицы. Представляет собой список элементов, состоящих из пары данных «ключ — значение». Доступ к значению осуществляется по ключу. в системе EME.WMS хэш-таблицы применяются для быстрого поиска данных по ключу.

Перед использованием класса `Map` нужно задать размер хэш-таблицы. Рекомендованный размер — 1,3 × {предполагаемое количество элементов}. С ростом числа данных из-за возникающих коллизий появляется нелинейное замедление. Неудачный выбор размера хэш-таблицы также может сильно замедлить выполнение программы.

Пример работы с Map в EME-L:

```EME-L
mapLotRef = Object("Map");
mapLotRef.SetSize(1000);
mapLotRef.SetAt(line, array);
result = mapLotRef.Lookup(line);
```

## Рекомендации по использованию структур данных в EME-L

При работе с классами данных в языке EME-L следует соблюдать следующие правила для избежания утечек памяти и медленной работы:

1. Никогда не использовать `Object("ClassName")` внутри цикла, даже как параметр метода.
2. Никогда не использовать простые имена для переменных-членов класса — всегда добавлять префикс `m_`.
3. Стараться не инициализировать переменные-члены класса типа `Object` вне конструктора.
4. Использовать оператор `Const` для создания объектов-членов класса.
